## 2. 机器人端部署

### 2.1 systemd 服务

```ini
# /etc/systemd/system/qooremote-onboard.service
[Unit]
Description=QooRobot Remote Teleop Service
After=network.target qoobody.service qoobrain.service
Requires=qoobody.service

[Service]
Type=simple
User=qoo
ExecStart=/opt/qoo/bin/qooremote-onboard --config /etc/qoo/qooremote.yaml
Restart=on-failure
RestartSec=5
LimitNOFILE=65536

[Install]
WantedBy=multi-user.target
```

### 2.2 配置文件

```yaml
# /etc/qoo/qooremote.yaml
teleop:
  max_sessions: 2
  takeover_timeout: 30s
  heartbeat_interval: 100ms
  heartbeat_timeout: 500ms

video:
  encoder: nvh264enc  # or vah264enc, x264enc
  resolutions: [1920x1080, 1280x720, 640x480]
  framerate: 30
  bitrate_max: 20000000  # 20Mbps

webrtc:
  stun_servers: ["stun:stun.qoobot.ai:3478"]
  turn_servers:
    - url: "turn:turn.qoobot.ai:3478?transport=udp"
      username: "${TURN_USER}"
      credential: "${TURN_PASS}"

safety:
  joint_limits_path: /etc/qoo/joint_limits.yaml
  emergency_stop_pin: GPIO17
```

## 3. 云端部署

### 3.1 TURN 服务器

```yaml
# coturn 配置
apiVersion: apps/v1
kind: Deployment
metadata:
  name: coturn
spec:
  replicas: 2
  template:
    spec:
      containers:
      - name: coturn
        image: coturn/coturn:4.6-alpine
        ports:
        - containerPort: 3478
          protocol: UDP
        - containerPort: 5349
          protocol: TCP
        env:
        - name: TURN_REALM
          value: "qoobot.ai"
```

### 3.2 Session Manager

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: qooremote-session
spec:
  replicas: 2
  template:
    spec:
      containers:
      - name: session-manager
        image: qoobot/qooremote-session:latest
        ports:
        - containerPort: 8080
        - containerPort: 8443
        env:
        - name: REDIS_HOST
          value: "redis-cluster"
        - name: QOOAUTH_URL
          value: "https://auth.qoobot.ai"
```

## 5. 运维手册

### 5.1 日常检查

- 检查 TURN 服务器健康状态
- 检查 WebRTC 连接质量指标
- 检查机器人端服务运行状态
- 清理过期会话和录制文件

### 5.2 故障处理

| 故障 | 排查步骤 |
|:-----|---------|
| 无法建立 WebRTC | STUN/TURN 服务器可达性、防火墙规则、ICE candidate |
| 视频卡顿 | 带宽检测、码率自适应是否生效、编码器负载 |
| 控制延迟高 | 网络 RTT、安全看门狗是否误触发 |
| 断线频繁 | WiFi 信号、TURN 中继质量、心跳配置 |
| 紧急停止误触发 | SafetyWatch 日志、心跳超时配置 |
