# qooremote 架构设计

## 1. 系统定位

qooremote 是 QOS 两大核心子系统之一（与 qoobrain 并列），负责远程人工遥控。当自主大脑无法应对长尾场景时，人类操作员可远程接管机器人的感知、决策与控制。

**对标 Android**：无障碍服务 (Accessibility Service) + 设备管理 (Device Policy) + 远程桌面

## 3. 核心组件

### 3.1 操作员端 — qooremote Client SDK

| 组件 | 说明 | 实现位置 |
|------|------|----------|
| MotionCaptureDriver | 动捕设备数据采集与标准化 | `qoodev/teleop/` |
| VRDriver | VR 头显位姿/手柄输入获取 | `qoodev/teleop/` |
| GamepadDriver | 手柄/键盘输入映射 | `qoodev/teleop/` |
| WebRTCPeer | WebRTC 端到端连接管理 | `qoosvc/` 复用 |
| CommandEncoder | 控制指令编码与序列化 | `qoobot-cloud/proto/teleop/` |
| StreamDecoder | 视频/音频流解码与渲染 | `qooweb/` 前端 |

### 3.2 云端 — qoocloud-teleop 微服务

| 组件 | 说明 | 端口 |
|------|------|------|
| SignalRelayService | WebRTC 信令中继，SDP/ICE 交换 | :8208 |
| SessionManager | 遥控会话生命周期管理 | :8208 |
| StreamForwarder | 视频/音频流转发 (SFU) | :8208 |
| ControlRelayService | 控制指令低延迟转发 | :8208 |
| TeachingRecorder | 示教数据采集与存储 | :8208 |

### 3.3 机器人端 — qooremote Agent

| 组件 | 说明 | 实现位置 |
|------|------|----------|
| VideoStreamer | 多路视频采集+编码+推流 | `qoosvc/src/svc_video/` |
| AudioStreamer | 音频采集+编码+推流 | `qoosvc/src/svc_voice/` 扩展 |
| CommandExecutor | 接收控制指令→HAL执行 | `qoosvc/src/svc_remote/` |
| TakeoverHandler | 自主↔遥控模式切换 | `qoobrain/` 安全监护 |
| TeachingCollector | 示教数据采集 | `qoosvc/src/svc_remote/` |

## 4. 数据流

### 4.1 控制下行 (Operator → Robot)

```
动捕/VR/手柄 → Client SDK → CommandEncoder(Protobuf) → WebRTC DataChannel
    → qoocloud-teleop(SFU) → WebRTC DataChannel → CommandExecutor → qoobody HAL
```

延迟目标：<50ms (LAN), <150ms (WAN 4G/5G)

### 4.2 视频上行 (Robot → Operator)

```
Camera → qoobody HAL → VideoStreamer → H.264/H.265 HW Encoder → WebRTC VideoTrack
    → qoocloud-teleop(SFU) → WebRTC VideoTrack → StreamDecoder → 渲染
```

延迟目标：<100ms (LAN), <300ms (WAN)

### 4.3 音频双向

```
MicArray → AudioStreamer → Opus Encoder → WebRTC AudioTrack → SFU
    → Operator Speaker

Operator Mic → Opus Encoder → WebRTC AudioTrack → SFU
    → AudioStreamer → Robot Speaker
```

## 5. 自主↔遥控 切换机制

```
                    ┌──────────────────────┐
                    │    QOS 运行模式        │
                    └──────────┬───────────┘
                               │
                   ┌───────────┼───────────┐
                   ▼           ▼           ▼
            ┌──────────┐ ┌──────────┐ ┌──────────┐
            │ 完全自主  │ │ 混合模式  │ │ 完全遥控  │
            │ (AUTO)   │ │ (HYBRID) │ │ (TELEOP) │
            └──────────┘ └──────────┘ └──────────┘
```

**切换触发条件**：
| 触发类型 | 条件 | 动作 |
|----------|------|------|
| 自动→遥控 | 碰撞风险/未知物体/置信度<阈值/任务超时 | 自动请求人工接管 |
| 遥控→自动 | 操作员交还控制权 | 平滑过渡，保持运动连续性 |
| 安全切换 | 急停/通信丢失/看门狗超时 | 立即 STOP + 通知操作员 |

## 6. 协议设计

### 6.1 控制指令协议 (Protobuf)

```protobuf
message TeleopCommand {
  uint64 timestamp = 1;           // 时间戳 (ns)
  uint64 sequence = 2;            // 序列号 (丢包检测)
  oneof command {
    FullbodyCommand fullbody = 3; // 全身运动指令
    JointCommand joint = 4;       // 单关节指令
    GripperCommand gripper = 5;   // 末端执行器
    HeadCommand head = 6;         // 头部/云台
    EmergencyStop stop = 7;       // 紧急停止
  }
}

message TeleopState {
  uint64 timestamp = 1;
  FullbodyState fullbody = 2;
  repeated JointState joints = 3;
  SafetyStatus safety = 4;
  BatteryStatus battery = 5;
  repeated DiagnosticEvent diagnostics = 6;
}
```

### 6.2 会话管理协议

```protobuf
message SessionRequest {
  string robot_id = 1;
  string operator_id = 2;
  ControlMode requested_mode = 3;
  repeated MediaType media = 4;   // video/audio/data
}

message SessionResponse {
  string session_id = 1;
  SessionStatus status = 2;
  string sdp_offer = 3;
  repeated IceServer ice_servers = 4;
}
```

## 7. 安全设计

| 层级 | 机制 | 说明 |
|------|------|------|
| 传输安全 | DTLS-SRTP | WebRTC 标准加密 |
| 身份认证 | JWT + QooIDaaS | 操作员身份验证 |
| 权限控制 | RBAC | 不同操作员不同控制权限 |
| 指令校验 | 范围/速率限制 | 防止恶意指令 |
| 通信安全 | 心跳+超时 | 通信丢失自动 STOP |
| 审计日志 | 全指令记录 | 合规审计追溯 |
