# qooremote — 应用架构设计

> 版本：v0.1 | 最后更新：2026-06-29 | 状态：Draft | 子项目：qooremote（远程遥控）
> 对标：无障碍服务 + 设备管理 + 远程桌面

---

## 1. 架构总览

qooremote 采用**端-云-控 三层架构**，机器人端负责采集与执行，云端负责信令中继与会话管理，操控端负责呈现与控制。

```
┌────────────────────────────────────────────────────────────┐
│                    qooremote 三层架构                        │
├────────────────────────────────────────────────────────────┤
│                                                             │
│  🖥️ 操控端 (Client)                                         │
│  ┌──────────────────────────────────────────────────┐     │
│  │ Stream Renderer │ Control Input │ Haptic Feedback │     │
│  │ WebRTC Client   │ State Machine │ UX Framework    │     │
│  └───────────────────────┬──────────────────────────┘     │
│                          │                                  │
│                          │ WebRTC (P2P) + WebSocket (Sig)   │
│                          │                                  │
│  ☁️ 信令中继 (Cloud)      │                                  │
│  ┌───────────────────────▼──────────────────────────┐     │
│  │ Session Manager │ Auth Service  │ TURN Relay     │     │
│  │ Room Manager    │ Recording Svc │ Monitor Svc    │     │
│  └───────────────────────┬──────────────────────────┘     │
│                          │                                  │
│                          │ WebRTC (P2P) + DataChannel       │
│                          │                                  │
│  🤖 机器人端 (Robot)      │                                  │
│  ┌───────────────────────▼──────────────────────────┐     │
│  │ Stream Encoder  │ Control Router │ Safety Watch  │     │
│  │ Sensor Bridge   │ Data Recorder │ Teleop Core    │     │
│  └──────────────────────────────────────────────────┘     │
│                                                             │
└────────────────────────────────────────────────────────────┘
```

## 2. 模块职责

### 2.1 机器人端 (onboard_core, C++17)

| 模块 | 职责 |
|:-----|------|
| TeleopCore | 遥控核心：接管/释放/紧急停止状态机、指令仲裁 |
| StreamEncoder | 多路视频编码 (HW: NVENC/VAAPI/QSV)、自适应码率 |
| ControlRouter | 指令路由：遥控指令→qoobody 执行器、安全校验 |
| SafetyWatch | 安全监控：连接心跳、延迟检测、异常制动 |
| SensorBridge | 传感器桥接：多模态传感器数据→WebRTC DataChannel |
| DataRecorder | 示教录制：关节/视频/力觉多模态同步录制 |

### 2.2 云端 (cloud_service, Java/Spring Boot)

| 模块 | 职责 |
|:-----|------|
| SessionManager | 遥控会话 CRUD、生命周期管理、超时释放 |
| AuthService | 操作员认证（qooauth 集成）、Token 签发 |
| RoomManager | WebRTC Room 管理、操作员配对 |
| TURNRelay | TURN 中继：NAT 穿透失败时的流量转发 |
| RecordingSvc | 录制数据接收、存储、转发给训练管道 |

### 2.3 操控端 (web_portal, TypeScript/Vue3)

| 模块 | 职责 |
|:-----|------|
| WebRTCClient | WebRTC 连接管理、ICE 协商 |
| StreamRenderer | 视频解码渲染、多路画中画 |
| ControlInput | 手柄/键盘/触屏输入映射 |
| HapticFeedback | 力反馈/振动渲染（如设备支持） |
| TeleopUX | 操控界面：视频+传感器面板+控制栏 |

## 3. 控制数据流

```
操作员操控指令
    │
    ▼
WebRTC DataChannel (可靠有序)
    │
    ▼
机器人端 ControlRouter
    │
    ├── 安全检查 (力矩限幅/位置限位/速度限幅)
    │
    ▼
指令仲裁: 遥控指令 vs 自主指令 vs 安全指令
    │
    ▼
qoobody 执行器
    │
    ▼
传感器反馈 → 编码 → DataChannel → 操控端
```

## 4. 安全架构

### 4.1 多层安全停止

```
层次      延迟     触发条件
────────────────────────────────────
L0 硬件  <1ms    急停按钮、碰撞传感器
L1 固件  <10ms   力矩超限、速度超限
L2 通信  <500ms  心跳超时、连接断开
L3 应用  <100ms  操作员点击急停
L4 云端  <2s     远程强制断开会话
```

### 4.2 指令安全检查

```
每条操控指令到达:
    1. 速率检查：|v| < v_max?
    2. 加速度检查：|a| < a_max?
    3. 力矩检查：|τ| < τ_max?
    4. 位置检查：关节限位内?
    5. 碰撞检查：安全区域内有障碍?
    6. 全部通过 → 执行
       任一失败 → 拒绝 + 报警
```
