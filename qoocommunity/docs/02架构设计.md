### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **五端统一** | 同一套后端 API 支撑 Web/App/小程序/H5，仅前端实现不同 |
| **开放优先** | 核心数据 API 公开，文档开源，决策透明 |
| **社区驱动** | 贡献者等级、投票治理、社区维护生态 |
| **知识沉淀** | 论坛/问答/教程/案例形成可检索知识库 |
| **国际化** | 中/英/日/韩/德 多语言，社区翻译贡献 |

### 1.3 外部依赖

| 依赖 | 用途 |
|------|------|
| qooauth | 统一认证、OAuth 2.0 登录 |
| qoodev | 文档自动生成、SDK API 参考 |
| qoostore | 技能市场数据（案例展示、排行） |
| qoocloud | 云服务基础设施 |
| GitHub API | 开源仓库贡献数据同步 |

## 二、核心子系统

### 2.1 📚 文档站点子系统 (Docs Site)

```
用户访问文档站点
        │
        ▼
┌──────────────────────┐
│  Static Site Gen      │  ← 从 Git 仓库拉取 Markdown
│  (VitePress / Docus)  │
├──────────────────────┤
│ · Markdown → HTML     │
│ · 多语言路由          │
│ · 版本化路由          │
│ · API 自动生成        │  ← 从 qoodev SDK 源码生成
│ · 交互式 Playground   │  ← WebContainer / CodeSandbox
│ · Algolia 全文搜索    │
│ · 反馈/编辑链接       │  → "Edit this page on GitHub"
└──────────────────────┘
        │
        ▼
   CDN 分发 (全球加速)
```

**技术选型**：VitePress 静态站点生成 + Algolia DocSearch + WebContainer Playground

## 三、数据流

### 3.1 文档站点发布流程

```
开发者修改文档 (GitHub PR)
        │
        ▼
PR 合并到 main 分支
        │
        ▼
GitHub Actions 触发构建
        │
        ├──→ VitePress 构建静态站点
        ├──→ API 文档自动生成 (从 qoodev SDK 源码)
        ├──→ Algolia 索引更新
        └──→ CDN 部署
                │
                ▼
          全球用户访问
```

### 3.2 贡献者等级晋升流程

```
开发者提交 PR
        │
        ▼
CI 自动检查 (lint/test/build)
        │
        ▼
Code Review (Maintainer)
        │
        ▼
合并 PR → 贡献记录入库
        │
        ▼
自动化等级评估
  ├── PR 数量 ≥ 10 + 活跃 ≥ 3月 → 推荐 Maintainer
  ├── 社区投票 2/3 + TSC 审批 → 晋升 Committer
  └── 年度选举 → TSC Member
        │
        ▼
权限/徽章自动更新
```

### 3.3 活动生命周期流程

```
活动创建(Event Service)
        │
        ▼
活动发布 → 通知推送(邮件/App/小程序)
        │
        ▼
用户报名 → 票务确认
        │
        ▼
活动进行中 → 直播推流 → 实时互动
        │
        ▼
活动结束 → 录播归档 → 资料上传 → 反馈收集
        │
        ▼
数据分析 → 参会报告
```

## 四、技术选型

| 层次 | 技术 | 版本 | 说明 |
|------|------|------|------|
| **后端框架** | Spring Boot | 3.2.x | 微服务基础 |
| **微服务治理** | Spring Cloud + Nacos | 2023.x + 2.3.x | 服务发现/配置中心 |
| **API 网关** | Spring Cloud Gateway | 4.x | 统一入口 |
| **数据库** | PostgreSQL | 16 | 主数据存储 |
| **缓存** | Redis | 7.x | 会话/热点数据 |
| **搜索** | Elasticsearch | 8.x | 论坛/问答全文搜索 |
| **消息队列** | RocketMQ | 5.x | 异步通知/事件 |
| **对象存储** | MinIO / 阿里云OSS | - | 图片/视频/附件 |
| **文档站点** | VitePress | 1.x | 静态站点生成 |
| **前端 Web** | Vue 3 + Element Plus | 3.x + 2.x | Web Portal |
| **移动 App** | UniApp + Vue 3 | 3.x | iOS/Android |
| **小程序** | 微信原生 + uni-app | - | 微信小程序 |
| **容器化** | Docker + Docker Compose | - | 本地开发 |
| **CI/CD** | GitHub Actions | - | 自动化构建部署 |
| **监控** | Prometheus + Grafana | - | 服务监控 |

## 五、安全架构

| 层级 | 措施 |
|------|------|
| **传输层** | TLS 1.3 全站 HTTPS |
| **认证层** | qooauth JWT/OAuth 2.0 统一认证 |
| **授权层** | RBAC (Member/Moderator/Admin/TSC) |
| **内容安全** | 敏感词过滤、垃圾检测、XSS 防护 |
| **数据安全** | 用户数据加密存储、GDPR 合规 |
| **API 安全** | 速率限制 (Rate Limiting)、CSRF Token |
