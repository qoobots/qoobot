# qoocommunity — 开源社区 功能清单与开发进度

> 最后更新：2026-06-29

---

## 定位

qoocommunity 是 QooBot **开发者与用户社区**。它是生态繁荣的根基——文档站点、贡献指南、论坛、开发者大会、高校合作等共同构成开发者飞轮：越多开发者→越多技能→越多用户→更多开发者加入。

| 维度 | 说明 |
|------|------|
| **定位** | 开发者与用户社区：文档站点、贡献指南、开发者大会——生态繁荣的根基 |
| **对标** | GitHub 开源社区 + Stack Overflow + Apple WWDC |
| **依赖** | qoodev（文档/工具）、qoostore（技能市场）、qoocloud |
| **被依赖** | 无（社区是生态的"消费者"也是"贡献者"） |

---

## 设计理念

1. **开放优先**：核心项目开源、文档公开、决策透明
2. **低门槛贡献**：从文档修正到核心代码，多层级贡献路径
3. **知识沉淀**：问答、教程、案例形成可检索的知识库
4. **社区驱动**：社区投票决定路线图优先级、社区维护核心生态项目

### 当前实现说明

**☁️ 云后端 (qoocommunity-cloud)**: 约 80% 完成
- 9 个微服务模块全部实现，Entity + Repository + Service + Controller 四层完整
  - `qoocommunity-forum` — 论坛（Topic/Category/Reply/Like/Bookmark）
  - `qoocommunity-qa` — 问答（Question/Answer/Vote）
  - `qoocommunity-event` — 活动（Event/Registration/Agenda/Material）
  - `qoocommunity-academy` — 学院（Course/Lesson/Enrollment/Certification）
  - `qoocommunity-contributor` — 贡献者（Contributor/CLA 签署/等级晋升）
  - `qoocommunity-governance` — 治理（TSC/SIG/RFC 投票流程）
  - `qoocommunity-content` — 内容（Blog/Showcase）
  - `qoocommunity-gateway` — API 网关（Spring Cloud Gateway + Nacos 服务发现）
  - `qoocommunity-common` — 公共模块（ApiResponse/PageResponse/异常处理/枚举）
- 基础设施完备：Docker Compose 6 服务 (PostgreSQL/Redis/ES/MinIO/Nacos/RocketMQ)
- 3 个 Flyway 迁移文件覆盖 32 张数据表
- API Gateway 统一路由 + Nacos 注册中心 + Flyway 数据库版本管理
- 待完善：DTO 类（当前 Controller 写操作使用 Map）、全文搜索集成、测试

**🌐 前端 (qoocommunity-web)**: 约 35% 完成
- 技术栈：Vue 3 + TypeScript + Element Plus + Vite + Vue Router
- 25 个 Vue 文件，其中：
  - **8 个页面完整对接后端 API**：论坛首页/分类/帖子详情、问答首页/问题详情、活动列表/活动详情、学院首页
  - **14 个页面为"建设中"占位**：课程详情、认证中心、博客列表/详情、案例展示、贡献者墙/个人主页、社区宪章、RFC 列表、路线图、我的课程、个人资料
  - **1 个首页 (HomeView)**：静态展示，硬编码数据
  - **3 个通用组件**：AppHeader/AppFooter/MarkdownViewer
- 已对接的 API 模块：forumApi、qaApi、eventApi、academyApi
- 路由结构完整，覆盖全部 7 个子系统
