# qooremote — 安全架构设计

> 版本：v0.1 | 最后更新：2026-06-29 | 状态：Draft | 子项目：qooremote（远程遥控）

---

## 1. 安全设计原则

1. **安全优先于便利**：任何安全风险优先考虑，即使牺牲操控体验
2. **纵深防御**：硬件→固件→通信→应用多层安全
3. **Fail-Safe**：任何故障都导向安全状态（停止运动）
4. **最小权限**：操作员仅获得完成任务所需的最小控制权限

## 3. 身份认证

### 3.1 操作员认证流程

```
操作员
  │
  ▼
OAuth2 + PKCE (qooauth)
  │
  ▼
双因素认证 (可选: TOTP/FIDO2)
  │
  ▼
签发 JWT Token (短期, 15min)
  │
  ▼
WebSocket 连接携带 Token
  │
  ▼
服务器验证 Token + 权限 scope
  │
  ▼
建立遥控会话
```

### 3.2 权限分级

| 级别 | 权限 | 认证要求 |
|:----:|------|---------|
| L0 观察者 | 仅观看视频流、传感器 | 密码 |
| L1 基础操控 | 移动+基础操作 | 密码 + 2FA |
| L2 高级操控 | 全关节控制+录制 | 密码 + 2FA + 审批 |
| L3 管理员 | 会话管理+权限分配 | 密码 + 硬件密钥 (FIDO2) |

## 4. 通信安全

### 4.1 WebRTC 安全

| 层面 | 措施 |
|:-----|------|
| 信令通道 | WSS (WebSocket over TLS 1.3) + JWT 认证 |
| 媒体通道 | DTLS-SRTP，AES-128-GCM 加密 |
| 数据通道 | SCTP over DTLS，端到端加密 |
| ICE | STUN/TURN 服务器认证、TURN 长期凭证 |

## 5. 机器人端安全

### 5.1 安全看门狗

```cpp
class SafetyWatch {
    // 心跳超时 → 安全停止
    void onHeartbeatTimeout() {
        emergencyStop();
        switchToAutonomousSafe();
    }
    
    // 指令速率异常 → 拒绝
    bool validateCommandRate(int cmds_per_second) {
        return cmds_per_second <= MAX_CMD_RATE;
    }
    
    // 位置/速度/力矩限位检查
    bool validateJointCommand(const JointCmd& cmd) {
        return cmd.position >= joint.lower_limit &&
               cmd.position <= joint.upper_limit &&
               abs(cmd.velocity) <= joint.velocity_limit &&
               abs(cmd.torque) <= joint.torque_limit;
    }
};
```

### 5.2 多层限位

```
指令 → 软件限位 (qooremote) → 固件限位 (qoobody) → 
       硬件限位 (物理限位开关) → 执行器
       
任一限位触发 → 拒绝执行 + 减速停止
```

## 6. 隐私保护

| 数据 | 保护措施 |
|:-----|---------|
| 视频流 | 端到端加密、可选人脸模糊、不云端存储 |
| 音频流 | 端到端加密、可选静音检测 |
| 操控会话 | 会话结束后 24h 自动删除 |
| 示教录制 | 选择是否上传、上传数据脱敏 |
| 操作员身份 | 匿名化操作日志 |

## 7. 合规

| 法规 | 相关要求 | 措施 |
|:-----|---------|------|
| GDPR | 视频包含个人信息 | 端到端加密、不存储、人脸可选模糊 |
| ISO 3691-4 | 远程操控安全 | 多层安全停止、心跳超时 |
| ISO 10218 | 工业机器人遥控 | 权限分级、安全限位 |
| PIPL | 中国个人信息保护 | 数据本地化、明示同意 |
