# qooweb — 交互设计

> **版本**：v1.0  
> **最后更新**：2026-06-29  
> **设计语言**：Apple 风格 — 极简、流畅、沉浸式

---

## 一、设计理念

qooweb 遵循 **Apple Human Interface Guidelines** 的设计哲学：

1. **清晰（Clarity）**：大量留白、清晰的排版层级、聚焦内容本身
2. **顺从（Deference）**：UI 退居幕后，让内容成为主角
3. **深度（Depth）**：通过毛玻璃、阴影、动画营造空间层次感

## 二、全局交互模式

### 2.1 导航栏交互

```
┌─────────────────────────────────────────────────────────────┐
│  [Logo]  首页  产品▾  生态▾  愿景  关于  GitHub    EN  🌙  ☰ │
└─────────────────────────────────────────────────────────────┘
```

| 交互元素 | 行为 | 动画 |
|----------|------|------|
| 导航栏 | 固定顶部，始终可见 | 毛玻璃模糊背景 `backdrop-filter: saturate(180%) blur(20px)` |
| 产品下拉 | Hover 展开 Mega 菜单 | 下拉箭头旋转 180°，菜单 opacity 0→1 + translateY |
| 生态下拉 | Hover 展开 Mega 菜单（四列分组） | 同上 |
| 语言切换 | 点击跳转 `en/` 目录 | 无动画 |
| 主题切换 | 点击切换 🌙↔☀️ | 图标即时切换，页面颜色平滑过渡 |
| 移动端汉堡 | 点击展开/收起导航 | 三条横线变叉号（CSS transform） |
| 导航链接 | 点击跳转 | 当前页 `.active` 高亮 |
| 外部链接 | 新标签页打开 | `target="_blank" rel="noopener"` |

### 2.2 Mega 下拉菜单交互

**桌面端**：
- 触发：鼠标 Hover `.nav-dropdown-trigger`
- 产品菜单：2 列（机器人平台 + 开源图纸）
- 生态菜单：4 列（核心层 + 平台层 + 保障层 + 生态层），全宽展开
- 每个子项目带描述文字（`.nav-dropdown-desc`）

**移动端**：
- 菜单变为静态内联展开
- 无全宽效果，垂直堆叠

### 2.3 主题切换交互

```
用户点击 🌙/☀️
    ↓
js/main.js → applyTheme(dark)
    ↓
document.documentElement.setAttribute("data-theme", "dark")
    ↓
CSS 变量全部切换到 [data-theme="dark"] 值
    ↓
所有元素平滑过渡（transition: background 0.4s, color 0.4s）
    ↓
localStorage 持久化偏好
```

**默认行为**：首次访问默认深色模式（`saved !== "light"` 时为 dark）

| 统计数字 | 静态展示 | HTML 硬编码 |
| CTA 按钮 | Hover 缩放 + 箭头平移 | CSS transform + transition |
| 滚动指示器 | 脉冲动画 | CSS @keyframes scrollLine (2s) |

### 3.2 滚动揭示动画

所有带 `.fade-up` 类的元素在进入视口时触发：

```
元素在视口外
    ↓
IntersectionObserver 检测进入 (threshold: 0.15)
    ↓
添加 .visible 类
    ↓
CSS transition: opacity 0→1, translateY(30px)→0
    ↓
移除 Observer（只触发一次）
```

## 七、响应式交互适配

### 7.1 桌面端（> 1024px）
- 完整 Mega 下拉菜单
- 多列网格布局
- Canvas 粒子动画全效

### 7.2 平板端（768px - 1024px）
- 图纸展示 2 列变 4→2 列
- 部分网格简化

### 7.3 移动端（< 768px）
- 导航折叠为汉堡菜单
- Mega 菜单变为垂直内联展开
- 产品卡片双列变单列
- Hero 全屏高度（`min-height: 100vh`）
- Canvas 粒子保持但降低性能开销
- 路线图时间线变为垂直布局
- 页脚垂直堆叠
- 星空网格 5 列变 2 列

### 7.4 小屏手机（< 480px）
- 星空网格 2 列变 1 列

## 八、动画规范

| 动画类型 | 缓动函数 | 时长 | 用途 |
|----------|----------|------|------|
| 微交互 | `cubic-bezier(0.25, 0.1, 0.25, 1)` | 0.2s | 按钮 Hover、链接 |
| 平滑过渡 | `cubic-bezier(0.25, 0.1, 0.25, 1)` | 0.4s | 主题切换、卡片 Hover |
| 入场动画 | `cubic-bezier(0.25, 0.1, 0.25, 1)` | 0.8s | 滚动揭示 |
| 粒子动画 | — | 60fps 循环 | Canvas requestAnimationFrame |
| 光晕呼吸 | `ease-in-out` | 6-8s | Hero 渐变光晕 |
| 星空闪烁 | `ease-in-out` | 2-6s 随机 | 愿景星空 |

## 九、可访问性

| 措施 | 实现 |
|------|------|
| 语义化 HTML | `<nav>` `<section>` `<footer>` `<h1>`-`<h4>` |
| ARIA 标签 | `aria-label` 用于主题切换、汉堡菜单 |
| 键盘导航 | 原生 `<a>` 和 `<button>` 支持 Tab 键 |
| 屏幕阅读器 | `.sr-only` 类用于隐藏辅助文本 |
| 颜色对比 | 深色/浅色主题均保证 WCAG AA 级对比度 |
| 减少动画 | 未实现 `prefers-reduced-motion`（后续可加） |

## 十、性能优化

| 策略 | 实现 |
|------|------|
| 零 JS 框架 | 原生 JS，无运行时开销 |
| 单一 CSS 文件 | 一次加载，全局复用 |
| CSS 动画优先 | 用 CSS animation 替代 JS 动画 |
| Canvas 条件加载 | 仅在 `#hero-particles` 存在时初始化 |
| 视频属性 | `preload="auto"` + `playsinline` |
| 图片格式 | SVG 矢量图（小文件）+ JPG 压缩 |
| 无外部字体请求 | 使用系统字体栈（SF Pro / Segoe UI） |
