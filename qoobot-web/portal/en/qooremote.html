qooremote — Remote Human Teleoperation System

One of QOS's two core subsystems (alongside qoobrain). When the autonomous brain cannot handle long-tail scenarios, human operators can remotely take over the robot's perception, decision-making, and control.

🖥️ onboard_core ☁️ cloud_service 🖱️ desktop_console 🌐 web_portal 📱 mobile_app

System Architecture

🕹️ Operator Side

Supports motion capture devices (OptiTrack/XSens), VR headsets (Meta Quest), gamepads (Xbox/DualSense), keyboard & mouse, and other input devices. The operator SDK provides unified input abstraction and standardized control command output.

☁️ qoocloud-teleop (:8208)

WebRTC signaling relay (SDP/ICE exchange), low-latency control command forwarding (<50ms LAN), video stream forwarding (SFU, up to 4 streams), teleoperation session lifecycle management, and teaching data storage.

🤖 Robot-Side Agent

Multi-stream video capture with H.264/H.265 hardware encoding, control command reception and HAL execution, smooth autonomous ↔ teleop switching, safety validation (range/rate/heartbeat timeout), and teaching data collection.
