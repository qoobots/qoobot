# 02_系统功能设计（DDD 战略设计）

> 基于 DDD 领域驱动设计方法论，对 Brain OS 进行限界上下文划分、统一语言定义和领域模型设计。

---

## 1. DDD 战略设计总览

### 1.1 核心域、支撑域、通用域

| 域类型 | 域名称 | 说明 |
|--------|--------|------|
| **核心域** | 认知域、决策域 | Brain OS 的核心差异化竞争力 |
| **支撑域** | 感知域、控制域、交互域 | 支撑核心域运行的必要能力 |
| **通用域** | 知识域、安全域 | 可复用基础设施 |

### 1.3 上下文映射关系

```
                     U (上游)   D (下游)
                     ────────▶───────

  │                     │              │

  └──U──▶ 知识 ◀──D── 决策             │
           ▲                           │
           │                           │
交互 ──U──▶┘          安全 ◀──D── 控制  │
  │                     ▲              │
  └─────────U───────────┘              │
                                       │
           决策 ──U──▶ 交互             │
```

**上下文关系说明**：

| 关系类型 | 上下文对 | 说明 |
|----------|---------|------|
| **上游-下游 (U-D)** | 感知 → 认知 | 认知消费感知的场景理解结果 |
| **上游-下游 (U-D)** | 认知 → 决策 | 决策消费认知的任务分解结果 |
| **上游-下游 (U-D)** | 决策 → 控制 | 控制消费决策的行为树和轨迹 |
| **上游-下游 (U-D)** | 决策 → 交互 | 交互消费决策的多轨迹方案 |
| **上游-下游 (U-D)** | 交互 → 知识 | 知识存储交互历史和用户偏好 |
| **防腐层 (ACL)** | 控制 → ROS 2 | 控制通过 ACL 与 ROS 2 中间件隔离 |
| **防腐层 (ACL)** | 认知 → 基础模型 | 认知通过 ACL 调用 LLM/VLA 模型 |
| **共享内核** | 所有上下文 | 统一的坐标系统、时间戳格式、机器人本体描述（URDF） |

## 2. 统一语言 (Ubiquitous Language)

### 2.1 核心术语表

| 术语 | 英文 | 定义 | 所属上下文 |
|------|------|------|-----------|
| **场景快照** | Scene Snapshot | 某一时刻的环境感知结果，包含 3D 点云、检测框、占用网格 | 感知 |
| **语义地图** | Semantic Map | 带语义标签的 3D 场景表示，区分物体类别与实例 | 感知 |
| **任务意图** | Task Intent | 从自然语言指令中提取的结构化任务描述 | 认知 |
| **技能** | Skill | 一个封装的、可复用的原子能力单元（如"抓取"、"导航"） | 决策 |
| **行为树** | Behavior Tree | 技能的层次化组合与执行编排，用树形结构描述任务 | 决策 |
| **轨迹方案** | Trajectory Plan | 一个完整的动作序列候选，包含时间轴、关节角度、末端位姿 | 决策 |
| **轨迹策略类型** | Strategy Type | 最优/保守/激进/探索/逆向 五种策略分类 | 决策 |
| **人在回路选择** | HITL Selection | 人类监督者对多个轨迹方案的评估与选择 | 交互 |
| **协作模式** | Collaboration Mode | 自主/建议/确认 三种人机协作模式 | 交互 |
| **执行命令** | Execution Command | 下发到控制上下文的最终动作指令，含安全约束 | 控制 |
| **经验片段** | Experience Episode | 一次完整任务交互的记录，用于训练和改进 | 知识 |
| **幽灵轨迹** | Ghost Trail | 在 3D 场景中半透明渲染的未来预测轨迹 | 交互 |

## 3. 各限界上下文详细功能设计

### 3.1 感知上下文 (Perception Context)

#### 3.1.1 核心职责
将原始传感器数据转化为结构化的环境理解。

#### 3.1.2 功能模块

| 模块 | 功能点 | 输入 | 输出 | 关键技术 |
|------|--------|------|------|----------|
| **SLAM 定位** | 实时 6DoF 位姿估计 | RGB-D/双目图像流 | 位姿 (x,y,z,qx,qy,qz,qw)、稀疏地图 | ORB-SLAM3 |
| **SLAM 定位** | 回环检测与全局优化 | 关键帧序列 | 全局一致轨迹 | ORB-SLAM3 |
| **SLAM 定位** | 多传感器融合定位 | IMU + 里程计 + 视觉 | 融合位姿 | EKF/UKF |
| **3D 场景重建** | 稠密点云重建 | 关键帧 + 深度图 | 稠密点云 | 3D Gaussian Splatting |
| **3D 场景重建** | 实时渲染输出 | 3DGS 模型 | RGB 渲染帧 (>30fps) | 3DGS 光栅化 |
| **3D 场景重建** | 动态物体更新 | 新观测帧 | 增量更新的 GS 模型 | 增量 3DGS |
| **占用网络** | 3D 体素占用预测 | 3DGS 点云 + 检测结果 | 占用网格 (free/occupied/unknown) | 自研基于 GS |
| **占用网络** | 动态障碍物预测 | 历史占用序列 | 未来占用预测 | Transformer |
| **目标检测** | 实时 2D/3D 检测 | RGB 图像 / 点云 | 检测框 + 类别 + 置信度 | YOLOv11 |
| **目标检测** | 精细实例分割 | ROI 区域 | 像素级掩码 | SAM 2 |
| **目标检测** | 多目标跟踪 (MOT) | 连续帧检测结果 | 跟踪 ID + 轨迹 | ByteTrack |
| **6D 位姿估计** | 物体 6D 位姿 | RGB-D 图像 + 物体模型 | 位置 + 四元数姿态 | FoundationPose |
| **6D 位姿估计** | 未见物体位姿 | RGB-D 图像 | 位姿估计 | FoundationPose 零样本 |

#### 3.1.3 领域事件

| 事件 | 触发条件 | 携带数据 |
|------|----------|----------|
| `PoseUpdated` | 位姿更新 | timestamp, pose(x,y,z,qx,qy,qz,qw), confidence |
| `SceneReconstructed` | 新一轮重建完成 | scene_id, gs_model_ref, timestamp |
| `ObjectDetected` | 新目标检测到 | timestamp, bbox, class, confidence, track_id |
| `ObjectLost` | 目标丢失 | track_id, last_known_position |
| `OccupancyUpdated` | 占用网格更新 | timestamp, occupancy_grid_ref |
| `PoseEstimated` | 抓取目标位姿更新 | object_id, pose, confidence |

### 3.2 认知上下文 (Cognition Context)

#### 3.2.1 核心职责
理解任务意图，推理环境语义，预测未来状态。

#### 3.2.2 功能模块

| 模块 | 功能点 | 输入 | 输出 | 关键技术 |
|------|--------|------|------|----------|
| **意图理解** | 自然语言指令解析 | 用户语音/文本 | TaskIntent（动作、目标、约束） | Qwen2.5-7B |
| **意图理解** | 模糊指令消歧 | 歧义指令 + 场景上下文 | 明确 TaskIntent | Qwen2.5-7B + 对话 |
| **意图理解** | 多轮对话澄清 | 对话历史 | 补充约束后的 TaskIntent | Qwen2.5-7B |
| **任务分解** | 顶层任务→子任务序列 | TaskIntent | SubtaskList（有序技能序列） | Qwen2.5-7B function calling |
| **任务分解** | 约束传播与推理 | SubtaskList + 物理约束 | 带约束的 SubtaskList | DeepSeek-V3（复杂场景） |
| **常识推理** | 物体属性推理 | 检测结果 | 物体物理属性（重量、材质、易碎性） | Qwen2.5-VL-7B |
| **常识推理** | 场景语义理解 | 语义地图 | 场景语义图（功能区域、可操作性） | Qwen2.5-VL-7B |
| **世界模型** | 动作后果预测 | 当前状态 + 候选动作 | 预测未来 N 帧状态 | Diffusion Policy |
| **世界模型** | 物理合规验证 | 预测状态 | 碰撞/稳定性/可行性评分 | 物理引擎 (Bullet/PhysX) |
| **多模态融合** | 视觉-语言对齐 | 图像 + 文本 | 多模态嵌入 | CLIP / Qwen2.5-VL |
| **多模态融合** | 跨模态推理 | 多模态嵌入序列 | 融合理解结果 | Cross-Attention |

#### 3.2.3 领域事件

| 事件 | 触发条件 | 携带数据 |
|------|----------|----------|
| `IntentRecognized` | 意图解析完成 | TaskIntent (action, target, constraints) |
| `TaskDecomposed` | 任务分解完成 | SubtaskList, dependencies |
| `SceneUnderstood` | 场景语义理解完成 | SemanticGraph |
| `WorldStatePredicted` | 世界模型预测完成 | predicted_states[], scores[] |
| `AmbiguityDetected` | 检测到模糊指令 | ambiguity_type, candidates[] |

### 3.3 决策上下文 (Decision Context)  [核心域]

#### 3.3.1 核心职责
将认知结果转化为可执行的行为树和多条候选轨迹方案，支撑人在回路选择。

#### 3.3.2 功能模块

| 模块 | 功能点 | 输入 | 输出 | 关键技术 |
|------|--------|------|------|----------|
| **行为树生成** | LLM → 行为树 JSON | SubtaskList + 语义场景 | BehaviorTree JSON | Qwen2.5-7B + BehaviorTree.CPP |
| **行为树生成** | 预定义技能库匹配 | SubtaskList | 带技能节点的行为树 | Skill Registry |
| **行为树生成** | 动态参数绑定 | 行为树骨架 + 场景实体 | 参数化行为树 | BehaviorTree.CPP |
| **多轨迹规划** | 最优策略轨迹 | 场景状态 + 目标任务 | TrajectoryPlan (best) | MoveIt 2 + STOMP |
| **多轨迹规划** | 保守策略轨迹 | 场景状态 + 安全边际 | TrajectoryPlan (safe) | MoveIt 2 + 约束规划 |
| **多轨迹规划** | 激进策略轨迹 | 场景状态 + 时间最优 | TrajectoryPlan (fast) | MoveIt 2 + 时间优化 |
| **多轨迹规划** | 探索策略轨迹 | 场景状态 + 随机噪声 | TrajectoryPlan (explore) | 随机化 + 约束采样 |
| **策略评估** | 轨迹方案评分 | TrajectoryPlan[] | ScoredTrajectory[] | 价值函数 (安全性/效率/平滑度) |
| **策略评估** | 碰撞安全性评分 | 轨迹 + 占用网格 | collision_score | FCL 碰撞检测 |
| **策略评估** | 物理可行性评分 | 关节轨迹 | feasibility_score | 动力学约束检查 |
| **人在回路** | 轨迹方案推送 | ScoredTrajectory[] | 可视化指令 | gRPC → 交互上下文 |
| **人在回路** | 用户选择接收 | 用户选择 / 超时 | 最终执行轨迹 | gRPC |
| **人在回路** | 自主模式执行 | N/A | 直接执行最优轨迹 | 无需等待 |
| **执行调度** | 行为树逐节点执行 | BehaviorTree | 执行中状态流 | BehaviorTree.CPP runtime |
| **执行调度** | 异常中断与恢复 | 异常事件 | 回退/重试/重新规划 | BehaviorTree.CPP |

#### 3.3.3 行为树节点类型

```
行为树节点分类：
├── Control Nodes（控制节点）
│   ├── Sequence（顺序执行）
│   ├── Fallback（选择执行，故障转移）
│   ├── Parallel（并行执行）
│   ├── ReactiveSequence（响应式顺序）
│   └── IfThenElse（条件分支）
├── Decorator Nodes（装饰节点）
│   ├── RetryUntilSuccessful（重试直到成功）
│   ├── Timeout（超时控制）
│   ├── KeepRunningUntilFailure（持续运行）
│   └── Subtree（子树引入）
├── Action Nodes（动作节点）
│   ├── NavigateTo(target)       # 导航
│   ├── DetectObject(class)      # 检测物体
│   ├── PickObject(id)           # 抓取物体
│   ├── PlaceObject(id, target)  # 放置物体
│   ├── AvoidObstacle(entity)    # 避障
│   ├── Wait(duration)           # 等待
│   ├── Speak(text)              # 语音输出
│   ├── Observe(scene)           # 主动感知
│   └── HITLConfirm(trajectories)# 人在回路确认
└── Condition Nodes（条件节点）
    ├── IsObjectVisible(id)
    ├── IsPathClear()
    ├── IsGraspStable()
    └── IsHumanPresent()
```

#### 3.3.4 领域事件

| 事件 | 触发条件 | 携带数据 |
|------|----------|----------|
| `BehaviorTreeGenerated` | 行为树创建 | bt_id, bt_json, task_intent_id |
| `TrajectoriesPlanned` | 多轨迹生成完毕 | trajectories[]（含类型和评分） |
| `TrajectorySelected` | 轨迹被选中（人工/自动） | trajectory_id, reason (auto/human/override) |
| `ExecutionStarted` | 行为树开始执行 | bt_id, start_time |
| `NodeCompleted` | 行为树节点执行完成 | node_id, status (SUCCESS/FAILURE), duration |
| `ExecutionInterrupted` | 执行被中断 | node_id, interrupt_reason |
| `HITLTimeout` | 人在回路超时 | waiting_duration |

### 3.4 控制上下文 (Control Context)

#### 3.4.1 核心职责
将决策上下文的执行命令转化为实际的机器人运动控制和 ROS 2 指令。

#### 3.4.2 功能模块

| 模块 | 功能点 | 输入 | 输出 | 关键技术 |
|------|--------|------|------|----------|
| **运动规划** | 笛卡尔路径规划 | 起止位姿 + 场景约束 | 末端路径点序列 | MoveIt 2 + OMPL |
| **运动规划** | 关节空间规划 | 起止关节角 | 关节轨迹 | MoveIt 2 + STOMP |
| **逆运动学** | 末端位姿→关节角 | 目标位姿 | 关节角度解 | TRAC-IK |
| **逆运动学** | 冗余自由度优化 | 多 IK 解 | 最优关节配置 | TRAC-IK + 代价函数 |
| **轨迹优化** | 时间最优轨迹 | 路径点 | 平滑速度/加速度曲线 | STOMP / TOWR |
| **轨迹执行** | 轨迹跟踪控制 | 关节轨迹 | 关节力矩/速度指令 | PID / 计算力矩 |
| **轨迹执行** | 实时路径修正 | 传感器反馈 | 修正后的控制指令 | 视觉伺服 |
| **ROS 2 桥接** | Topic/Service/Action 封装 | 内部指令 | ROS 2 消息 | ROS 2 rclcpp |
| **ROS 2 桥接** | 生命周期管理 | 节点状态变更 | ROS 2 Lifecycle Node | ROS 2 |
| **力控/阻抗** | 力控模式切换 | 模式指令 | 力/位置混合控制 | 导纳/阻抗控制 |
| **力控/阻抗** | 接触力监控 | 关节力矩传感器 | 接触状态 | 力阈值检测 |

#### 3.4.3 与 ROS 2 的防腐层

```
控制上下文内部      |  ACL (防腐层)     |  ROS 2 外部
───────────────────┼──────────────────┼───────────────────
ExecutionCommand    │  → msg转换       │  JointTrajectory
JointState          │  ← msg转换       │  sensor_msgs/JointState
PoseCommand         │  → msg转换       │  geometry_msgs/PoseStamped
SafetyStop          │  → msg转换       │  std_msgs/Empty
```

#### 3.4.4 领域事件

| 事件 | 触发条件 | 携带数据 |
|------|----------|----------|
| `MotionStarted` | 运动开始执行 | plan_id, target_pose, estimated_duration |
| `MotionCompleted` | 运动执行完毕 | plan_id, actual_duration, final_pose |
| `MotionAborted` | 运动被中止 | plan_id, abort_reason |
| `CollisionDetected` | 碰撞发生 | contact_point, contact_force, involved_links |
| `JointLimitReached` | 关节接近极限 | joint_id, current_angle, limit_angle |

### 3.5 知识上下文 (Knowledge Context)

#### 3.5.1 核心职责
管理和检索机器人积累的经验、知识和数据。

#### 3.5.2 功能模块

| 模块 | 功能点 | 输入 | 输出 | 存储 |
|------|--------|------|------|------|
| **工作记忆** | 当前任务上下文存储 | 任务状态、场景快照 | 快速上下文检索 | Python dict → Redis |
| **工作记忆** | 最近 N 条观测缓存 | 感知事件流 | 短期历史 | 环形缓冲区 |
| **长期知识库** | 物体知识图谱 | 类别+属性+操作方式 | 物体操作知识 | PostgreSQL + pgvector |
| **长期知识库** | 场景模板库 | 典型场景配置 | 场景匹配结果 | PostgreSQL + pgvector |
| **长期知识库** | 语义向量检索 | 查询文本/图像嵌入 | 相似经验 Top-K | pgvector |
| **时序日志** | 所有传感器数据记录 | 原始传感器消息 | 回放文件 | ROS 2 bag + SQLite |

#### 3.7.1 核心职责

|------|--------|------|------|----------|
| **碰撞检测** | 实时碰撞预测 | 当前状态 + 未来轨迹 | 碰撞风险等级 | FCL / 占用网络 |
| **碰撞检测** | 安全距离监控 | 位置 + 速度 | 距离告警 | 几何计算 |
| **安全约束** | 关节限位监控 | 关节角度 + 速度 | 限位告警 | 阈值比较 |
| **安全约束** | 力矩/力限制 | 关节力矩 | 力超限告警 | 阈值比较 |

| **紧急停止** | 软件急停 | 任意安全事件 | 刹车指令 | 最高优先级 |

#### 3.7.3 安全事件等级

| 等级 | 名称 | 触发条件 | 系统响应 | 恢复方式 |
|------|------|----------|----------|----------|
| **S0** | 致命 | 碰撞已发生、结构损坏 | 立即断电急停 | 人工检修 |
| **S1** | 严重 | 即将碰撞 (<50ms)、力严重超限 | 立即软停止 | 人工确认 |
| **S2** | 警告 | 距离过近、速度过快、接近限位 | 减速+告警 | 自动恢复 |
| **S3** | 提示 | 效率低下、路径不佳 | 记录日志 | 静默 |

## 4. 跨上下文流程

### 4.1 核心业务流程：自然语言任务执行

```
用户："把红色杯子拿过来"
  │
  ├─→ [交互] 语音→文本 → UserUtterance
  │
  ├─→ [认知] 意图理解 → TaskIntent{action: "fetch", target: "red_cup"}
  │       │
  │       └─→ [认知] 任务分解 → SubtaskList[navigate→detect→pick→return]
  │
  ├─→ [感知] 场景理解
  │       ├── SLAM 定位："我在 (1.2, 0.8, 0.0)"
  │       ├── 3D 重建 → 场景底图
  │       ├── 目标检测 → "检测到红色杯子 @ (2.5, 0.3, 1.0)"
  │       └── 位姿估计 → "杯子 6D 位姿已获取"
  │
  ├─→ [决策] 行为树生成
  │       └── LLM→JSON → BehaviorTree (navigate→detect→pick→return)
  │
  ├─→ [决策] 多轨迹生成
  │       ├── 最优轨迹 (评分 0.92)
  │       ├── 保守轨迹 (评分 0.78)
  │       └── 激进轨迹 (评分 0.85)
  │
  ├─→ [安全] 轨迹安全验证 → 全部通过
  │
  ├─→ [交互] 可视化渲染 → 3 条 Ghost Trail 推送到 Dashboard
  │       │
  │       ├── (建议模式) 倒计时 3 秒
  │       ├── 人类未选择 → 超时自动选择最优
  │       └─→ TrajectorySelected(trajectory_id=optimal, reason=timeout)
  │
  ├─→ [控制] 执行最优轨迹
  │       ├── MoveIt 2 轨迹跟踪
  │       ├── ROS 2 指令下发
  │       └── 实时状态反馈
  │
  ├─→ [知识] 经验记录 → ExperienceEpisode{success: true, ...}
  │
  └─→ [交互] 任务完成通知
```
