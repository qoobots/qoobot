## 2. 关键技术决策

### 2.1 为什么选择 CAN-FD 作为主通信总线

| 对比 | CAN-FD | RS485 | USB | Ethernet |
|:-----|:-----:|:----:|:---:|:------:|
| 实时性 | ✅ 确定延迟 | 🟡 | ❌ | 🟡 |
| 可靠性 | ✅ 差分信号 | ✅ | 🟡 | 🟡 |
| 带宽 | 8Mbps | 10Mbps | 480Mbps+ | 1Gbps+ |
| 拓扑 | 多主总线 | 主从 | 点对点 | 星型 |
| 机器人标配 | ✅ | 🟡 | ❌ | 🟡 |
| 适合配件 | ✅ | 🟡 | ✅(高速) | ✅(大数据) |

**决策**：CAN-FD 作为配件默认总线（低延迟+多主），USB/Ethernet 用于高速/大数据量配件。

### 2.2 配件认证签名体系

```
Root CA (qoobot)
    │
    ▼
MFQ Intermediate CA
    │
    ├── 配件证书 (每个认证配件型号)
    └── 配件实例签名 (每件产品出厂时签发)
    
验证链: 配件实例签名 → MFQ 证书 → Root CA
```

### 2.3 热插拔检测

| 接口类型 | 检测方式 | 检测时间 |
|:---------|---------|:------:|
| CAN-FD | 总线节点心跳超时/出现 | <100ms |
| USB | VBUS 电压检测 + USB 枚举 | <500ms |
| GPIO 检测 | 插入检测引脚 (ID pin) | <10ms |
| 机械快换 | 接近开关 + 锁紧确认 | <50ms |

## 4. 安全设计

### 4.1 电气安全

| 保护 | 实现 |
|:-----|------|
| 过流保护 | 自恢复保险丝 (PTC) + 电子保险丝 (eFuse) |
| 过压保护 | TVS 二极管 + OVP IC |
| 反接保护 | MOSFET 理想二极管 |
| 热插拔保护 | 浪涌电流限制 + ESD 保护 |
| 隔离 | 数字隔离器 (CAN/RS485) + 光耦 (IO) |

### 4.2 软件安全

| 机制 | 说明 |
|:-----|------|
| MFQ 证书验证 | 配件插入时验证证书有效性 |
| 权限边界 | 配件仅能访问声明的功能范围 |
| 故障隔离 | 配件故障不影响机器人核心系统 |
| 安全停止 | 配件异常时自动切断电源/控制 |
