## 1. 用户角色

| 角色 | 说明 | 交互界面 |
|------|------|----------|
| 🔧 配件开发者 | 第三方厂商，提交认证申请、下载 SDK | Web 开发者中心 |
| ✅ MFQ 审核员 | QooBot 内部，审核认证申请、质量审查 | Web 管理后台 |
| 🧪 实验室工程师 | 授权实验室，执行兼容性测试、提交报告 | Web 实验室门户 |

## 4. SDK CLI 命令

```bash
# 配件开发 SDK 命令行工具
qoogear init <project-name>       # 初始化配件项目
qoogear build                     # 构建配件驱动包
qoogear test                      # 运行认证自检套件
qoogear simulate <peripheral>     # 启动配件模拟器
qoogear submit                    # 提交认证申请
qoogear status <application-id>   # 查询认证状态
qoogear docs <standard>           # 查看接口规范
qoogear download <reference-id>   # 下载参考设计
```

## 5. 认证申请表单结构

```yaml
application:
  applicant:
    company_name: string
    contact_name: string
    contact_email: string
    website: string
  product:
    name: string
    category: enum[gripper, sensor, wearable, power, mobility, tool]
    description: text
    model_number: string
  certification:
    level: enum[basic, premium, pro]
    standards: [string]  # 适用的接口标准ID列表
  technical:
    mechanical_spec: file  # 机械图纸/3D 模型
    electrical_spec: file  # 电气原理图
    communication_protocol: file  # 通信协议文档
    firmware_source: file  # 固件源码（可选，Premium+）
    test_samples: int  # 送测样品数量
  attachments:
    product_photos: [file]
    user_manual: file
    safety_datasheet: file
```

## 6. 通知体系

| 事件 | 通知方式 | 接收者 |
|------|----------|--------|
| 申请提交成功 | 邮件 + 站内信 | 开发者 |
| 申请审核通过/驳回 | 邮件 + 站内信 | 开发者 |
| 分配测试实验室 | 邮件 + 站内信 | 开发者 + 实验室 |
| 测试报告提交 | 站内信 | 审核员 |
| 证书颁发 | 邮件 + 站内信 + 公开 | 开发者 |
| 证书即将过期 | 邮件（30/14/7 天前） | 开发者 |
| 标准新版本发布 | 邮件 + 站内信 | 所有开发者 |
| 安全漏洞通知 | 邮件（紧急） | 受影响配件开发者 |
