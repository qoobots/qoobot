## 1. 产品定位

qoosvc 是 QooBot 机器人的**系统能力基座**。它不是面向最终用户的"App"，而是所有上层技能共同依赖的共享服务层——类比 iOS 为 App 提供的 Core Motion、Speech、MapKit 等框架服务。

```

│              🎯 技能生态 (qoostore)                    │

├─────────────────────────────────────────────────────┤
│              📦 qoosvc — 系统服务层                    │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐     │
│  │🎙️语音│ │🗺️导航│ │🏠空间│ │🩺诊断│ │🤝交互│     │
```

## 4. 核心功能流程

### 4.1 语音对话流程

```
用户说话
  │
  ▼
[唤醒词检测] ──未检测到──→ 继续监听
  │ 检测到
  ▼
[ASR 流式识别] ──────────────────────────┐
  │                                      │
  ▼                                      ▼
[NLU 意图识别] ──置信度低──→ [对话澄清/追问]
  │ 置信度高
  ▼
[对话管理 → 任务执行]
  │
  ▼
[TTS 语音回应]
  │
  ▼
回到监听状态
```

### 4.2 导航流程

```
导航指令 "带我去厨房"
  │
  ▼
[NLU 提取目的地 + 偏好]
  │
  ▼
[全局路径规划] ──无地图──→ [SLAM探索/建图]
  │ 有地图
  ▼
[局部路径规划 ←→ 传感器数据(20Hz)]
  │
  ▼
[运动控制 → qoobody 执行]
  │
  ├── 到达 ✓
  └── 受阻 ──→ [重规划/恢复策略]
```
