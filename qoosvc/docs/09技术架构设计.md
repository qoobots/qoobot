## 2. 关键技术决策

### 2.1 为什么选择 ROS2 而非自定义 IPC

```
对比项          │ ROS2 (DDS)            │ 自定义 IPC (ZeroMQ/gRPC)
────────────────┼───────────────────────┼─────────────────────────
生态系统        │ 机器人领域标准 ✅       │ 需要自建轮子
传感器驱动      │ 大量 ROS2 驱动可用     │ 需要自行适配
可视化工具      │ RViz2/Foxglove 等      │ 需要自建
社区支持        │ 活跃的机器人社区        │ 无
与 qoobrain 对接│ 原生 ROS2 通信 ✅       │ 需要桥接层
实时性          │ DDS QoS 可配置          │ 自定义可控
```

**决策**：选择 ROS2，与 qoobrain 统一的通信范式，复用机器人生态。

### 2.2 为什么 Python 优先 + C++ 性能路径

```
       Python 3.11                    C++17
┌─────────────────────────┐   ┌─────────────────────┐
│ · ASR/NLU/TTS 流程编排   │   │ · 音频特征提取       │
│ · 对话管理逻辑           │   │ · 信号处理 (DSP)     │
│ · 导航高层决策           │   │ · 规划算法 (A*/TEB)  │
│ · 健康报告生成           │   │ · SLAM 后端优化      │
│ · 技能 SDK               │   │ · 点云处理           │
└─────────────────────────┘   └─────────────────────┘
   快速开发 · 灵活迭代          极致性能 · 低延迟
```

**决策**：Python 用于高频迭代的业务逻辑，C++ 用于计算密集型路径。通过 pybind11 绑定。

### 2.3 端侧 ASR 选型：Whisper vs Sherpa-ONNX

| 指标 | Whisper (small) | Sherpa-ONNX (Zipformer) |
|:-----|:--------------:|:----------------------:|
| 识别准确率 (中文) | ~90% | ~92% |
| 实时率 (ARM) | ~0.3 | ~0.15 |
| 内存占用 | ~800MB | ~200MB |
| 支持流式 | 否（需适配） | 原生支持 ✅ |
| 多语言 | 99种语言 | 中/英/日等主要语言 |

**决策**：选择 Sherpa-ONNX，更低延迟和内存，原生流式支持更适合机器人场景。

## 3. 性能设计

### 3.1 语音管线性能

```
音频采集 (16kHz) ──[5ms]──▶ 特征提取 ──[10ms]──▶ ASR 推理 ──[50ms]──▶ 文本
                                                                    │
                                                              [20ms] ▼
                                                                 NLU 推理
                                                                    │
                                                              [30ms] ▼
                                                              Intent+Slots

总管线延迟：~115ms（端侧全链路）
```

### 3.2 导航管线性能

```
传感器更新 ──[10ms]──▶ 障碍物检测 ──[5ms]──▶ 局部规划 ──[20ms]──▶ 速度指令
                                                             │
                                                      频率: 50Hz
                                                      单帧延迟: ~35ms
```

## 4. 可扩展性设计

### 4.1 插件化服务架构

```
ServiceRegistry
  │
  ├── SpeechEngine (必需)
  │     ├── ASRProvider (插件: SherpaONNX / Whisper / Cloud)
  │     ├── NLUProvider (插件: Rasa / Snips / LLM)
  │     └── TTSProvider (插件: VITS / Piper / Azure)
  │
  ├── NavEngine (必需)
  │     ├── GlobalPlannerProvider (插件: AStar / Dijkstra / RRT)
  │     └── LocalPlannerProvider (插件: TEB / DWA / MPC)
  │
  └── CustomService (可选)
        └── 第三方服务插件
```

### 4.2 技能服务扩展

```python
# 技能开发者通过 SDK 注册新的服务能力
from qoosvc import ServiceRegistry, SpeechCapability

@SpeechCapability.register("custom_greeting")
class CustomGreetingCapability(SpeechCapability):
    def on_intent(self, intent, slots):
        if intent == "greet":
            return "你好呀！今天想做什么？"
```
