# qoosvc — 人形机器人系统服务

> **运行位置**：机器人端（机器人计算平台），运行在 qoobrain 之上，**非云端服务**
> **技术栈**：C++17 + Python 3.11 + CMake，通过 DDS/iceoryx2 与 qoobrain 通信
>
> 机器人的"Siri + MapKit + ARKit + HomeKit + HealthKit"：
> 语音助手、导航服务、空间理解、多机互联、自诊断。

## 定位

qoosvc 是 QooBot 生态的基础系统服务层，提供机器人场景下的通用能力，
让每个机器人都具备语音交互、自主导航、空间感知等开箱即用的能力。

## 模块

| 模块 | 说明 | 状态 |
|------|------|------|
| `voice/` | 语音助手服务（唤醒、识别、合成、对话） | 📋 规划中 |
| `nav/` | 导航服务（SLAM、路径规划、定位） | 📋 规划中 |
| `space/` | 空间理解（场景重建、语义地图、物体关系） | 📋 规划中 |
| `connect/` | 多机器人互联（发现、组网、协同） | 📋 规划中 |
| `health/` | 机器人自诊断（硬件健康、软件状态、预测性维护） | 📋 规划中 |
| `vision/` | 视觉基础服务（检测、分割、跟踪、OCR） | 📋 规划中 |
| `manipulation/` | 操作基础服务（抓取、放置、力控） | 📋 规划中 |
| `human/` | 人机交互服务（手势、表情、意图识别） | 📋 规划中 |
| `knowledge/` | 常识知识服务（物体属性、操作知识、场景常识） | 📋 规划中 |

## 服务能力矩阵

| 服务 | 输入 | 输出 | 典型延迟 |
|------|------|------|----------|
| 语音指令 | 语音流 | 意图+实体 | <500ms |
| 导航 | 目标点 | 路径+速度指令 | <100ms |
| 空间理解 | RGBD | 语义地图 | <200ms |
| 多机协同 | 任务 | 分配方案 | <1s |
| 自诊断 | 传感器数据 | 健康报告 | 实时 |
| 抓取 | 目标物体 | 抓取姿态 | <100ms |
| 人脸识别 | 图像 | 身份+表情 | <100ms |

## iPhone 类比

| Apple 服务 | qoosvc 对应 |
|------------|-------------|
| Siri | voice（语音助手） |
| MapKit | nav（导航服务） |
| ARKit | space（空间理解） |
| HomeKit | connect（多机互联） |
| HealthKit | health（自诊断） |
| Vision | vision（视觉服务） |
| Core ML | 委托给 qoocore |

## 与 qoobrain 的关系

```
qoobrain (大脑OS) ──调用系统服务──→ qoosvc
     │                                  │
     ├── 规划结果 ──→ 调用 nav/space    │
     ├── 交互意图 ──→ 调用 voice/human  │
     └── 执行监控 ──→ 调用 health       │
```
