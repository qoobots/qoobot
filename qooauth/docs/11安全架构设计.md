## 2. 认证流程设计

### 2.1 多因素认证流程

```
┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
│  用户    │     │  客户端  │     │  Auth    │     │   2FA    │
│          │     │          │     │  Server  │     │  Provider│
└────┬─────┘     └────┬─────┘     └────┬─────┘     └────┬─────┘
     │               │               │                   │
     │ 1. 输入凭证   │               │                   │
     │──────────────>│               │                   │
     │               │               │                   │
     │               │ 2. POST /auth/login               │
     │               │──────────────>│                   │
     │               │               │                   │
     │               │               │ 3. 验证密码       │
     │               │               │ (argon2id)        │
     │               │               │                   │
     │               │               │ 4. 检测风险       │
     │               │               │ (设备/IP/行为)    │
     │               │               │                   │
     │               │ 5. 需要 2FA   │                   │
     │               │<──────────────│                   │
     │               │               │                   │
     │ 6. 输入 TOTP  │               │                   │
     │──────────────>│               │                   │
     │               │               │                   │
     │               │ 7. POST /auth/2fa/verify          │
     │               │──────────────>│                   │
     │               │               │                   │
     │               │               │ 8. 验证 TOTP      │
     │               │               │ (RFC 6238)        │
     │               │               │                   │
     │               │               │ 9. 签发 Token     │
     │               │               │                   │
     │               │ 10. Access + Refresh Token        │
     │               │<──────────────│                   │
```

### 2.2 异常检测与自适应认证

| 风险信号 | 检测方法 | 响应措施 |
|:---------|:---------|:---------|
| **新设备登录** | 设备指纹不匹配 | 强制 2FA + 邮件通知 |
| **异地登录** | GeoIP 距离 >500km | 强制 2FA |
| **异常时间** | 历史活跃时间偏离 | 提升风险评分 |
| **暴力破解** | 连续失败 >5 次 | 账户锁定 30min |
| **撞库攻击** | 多账户同 IP 失败 | IP 限流 + CAPTCHA |

6 维风险评分：设备指纹(25%) + IP信誉(20%) + 地理位置(15%) + 行为模式(20%) + 时间模式(10%) + 网络环境(10%)

## 3. 加密架构

### 3.1 端到端加密 (E2EE)

```
┌─────────────────────────────────────────────────────────────────┐
│                    X3DH + Double Ratchet                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  密钥协商 (X3DH):                                               │
│  ┌────────────────────────────────────────────────────────┐     │
│  │ Alice                         Bob                      │     │
│  │  IK_A (Identity Key)          IK_B                    │     │
│  │  EK_A (Ephemeral Key)         SPK_B (Signed PreKey)   │     │
│  │                               OPK_B (One-Time PreKey) │     │
│  │                                                       │     │
│  │  DH1 = DH(IK_A, SPK_B)                                │     │
│  │  DH2 = DH(EK_A, IK_B)                                 │     │
│  │  DH3 = DH(EK_A, SPK_B)                                │     │
│  │  DH4 = DH(EK_A, OPK_B)                                │     │
│  │                                                       │     │
│  │  SK = KDF(DH1 || DH2 || DH3 || DH4)                   │     │
│  └────────────────────────────────────────────────────────┘     │
│                                                                 │
│  消息加密 (Double Ratchet):                                     │
│  ┌────────────────────────────────────────────────────────┐     │
│  │  Root Key ──→ Chain Key ──→ Message Key                │     │
│  │       (HKDF)      (HKDF)      (AES-256-GCM)            │     │
│  │                                                       │     │
│  │  每次发送: DH Ratchet → 新 Root Key                    │     │
│  │  每条消息: Symmetric Ratchet → 新 Message Key          │     │
│  └────────────────────────────────────────────────────────┘     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 密钥管理

| 密钥类型 | 生成 | 存储 | 轮换周期 | 备份 |
|:---------|:-----|:-----|:---------|:-----|
| **根 CA 私钥** | HSM 内部生成 | HSM 不可导出 | 5 年 | HSM 备份卡 |
| **JWT 签名密钥** | Ed25519 KeyGen | HSM 封装 | 90 天 | 加密导出 |
| **用户密码哈希** | argon2id 派生 | PostgreSQL | — | DB 备份 |
| **会话密钥** | 随机 256-bit | Redis (加密) | 24h | — |
| **API Key** | 随机 256-bit | DB (SHA-256 哈希) | 用户触发 | — |
| **E2EE 预密钥** | X25519 KeyGen | DB (加密存储) | 用尽补充 | — |

## 4. 威胁模型与缓解

| 威胁 | 攻击向量 | 缓解措施 | 严重度 |
|:-----|:---------|:---------|:------:|
| **凭证填充** | 泄露密码库批量尝试 | argon2id + 速率限制 + 异常检测 | 高 |
| **MITM 中间人** | 网络拦截 | TLS 1.3 + 证书固定 + HSTS | 高 |
| **Token 窃取** | XSS/恶意软件 | HttpOnly Cookie + 短 TTL + 设备绑定 | 高 |
| **重放攻击** | 捕获并重放请求 | JWT jti + nonce + 时间窗口 | 中 |
| **提权攻击** | RBAC 绕过 | 服务端强制鉴权 + API Gateway 校验 | 高 |
| **密钥泄露** | 密钥文件泄露 | HSM 硬件保护 + 自动轮换 | 严重 |
| **DDoS** | 流量洪水 | Cloudflare WAF + Token Bucket + CDN | 中 |
| **SQL 注入** | 恶意输入 | MyBatis 参数化查询 + 输入校验 | 高 |
| **侧信道** | 时序攻击 | 恒定时间比较 + 随机延迟 | 低 |
