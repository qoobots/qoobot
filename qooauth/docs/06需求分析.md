# qooauth — 需求分析

> 版本：v0.1 | 最后更新：2026-06-29 | 状态：Draft
>
> **子项目**：qooauth（账号与安全）| **对标**：Apple ID / OIDC Provider
>
> **定位**：统一身份基础设施。为所有云端/设备端/用户端服务提供认证、授权、加密能力。

---

## 1. 需求概述

qooauth 是 QooBot 生态的身份基石。它提供从用户注册到设备激活、从开发者接入到企业管理的完整身份基础设施，对标 Apple ID 在 Apple 生态中的角色。

## 2. 功能性需求 (FR)

|:----------|:-----|:-----|:-------|

| FR-102 | QooBot ID 登录 | 用户名密码登录，会话 Token 签发，设备绑定 | P0 |

| FR-104 | 账户恢复 | 恢复密钥、备用邮箱、信任设备恢复流程 | P1 |
| FR-105 | 双因素认证 (2FA) | TOTP RFC 6238、FIDO2/WebAuthn 硬件密钥、恢复码备用 | P0 |
| FR-106 | 账户安全设置 | 密码修改、信任设备管理、登录历史、会话管理 | P0 |
| FR-107 | 个人资料管理 | 昵称、头像、语言偏好、时区设置 | P1 |

## 3. 非功能性需求 (NFR)

### NFR-100: 安全性

| 编号 | 需求 | 目标值 | 测量方法 |
|:-----|:-----|:-------|:---------|
| NFR-101 | 身份认证强度 | NIST SP 800-63B AAL2+ | 审计检查清单 |
| NFR-102 | 加密算法强度 | AES-256-GCM, Ed25519, HKDF-SHA512 | 密码学审查 |
| NFR-103 | 密钥轮换周期 | 主密钥 90d，会话密钥 24h | 自动化监控 |
| NFR-104 | 暴力破解防护 | 5 次失败锁定 30min | 渗透测试 |
| NFR-105 | Token 有效期 | Access 15min, Refresh 7d | 配置验证 |

| 编号 | 需求 | 目标值 | 测量方法 |

### NFR-300: 可扩展性

### NFR-400: 合规性

| 编号 | 需求 | 目标 |

## 4. 用户角色与用例

| 角色 | 描述 | 核心用例 |
|:-----|:-----|:---------|
| **普通用户** | QooBot 机器人机主 | 注册/登录、设备绑定、家庭共享、账户安全 |
| **企业管理员** | 管理团队设备的企业 IT | MDM 设备配置、组织管理、合规审计 |
| **家长** | 管理子女设备 | 家长控制、屏幕时间、购买审批 |
| **开发者** | 第三方技能/应用开发者 | OAuth 应用注册、API Key 管理、技能签名 |
| **系统管理员** | qooauth 运维人员 | 用户管理、证书管理、安全监控 |

## 5. 约束条件

| 约束 | 描述 |
|:-----|:-----|
| **技术栈** | Java 17 + Spring Cloud + PostgreSQL + Redis + gRPC |
| **部署环境** | Kubernetes + Docker，多云（AWS/Azure/阿里云） |
| **依赖关系** | qooauth 为基础服务，无上游依赖，被所有其他子项目依赖 |
| **国际化** | 支持中/英/日/韩/德多语言 |
| **向后兼容** | API 版本化，至少保持 2 个大版本的向后兼容 |
