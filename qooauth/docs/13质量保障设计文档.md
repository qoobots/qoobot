## 2. 测试分层

### 2.1 单元测试

| 维度 | 要求 |
|:-----|:-----|
| **框架** | JUnit 5 + Mockito + AssertJ |
| **覆盖率目标** | ≥ 80% 行覆盖率, ≥ 70% 分支覆盖率 |
| **重点覆盖** | 认证逻辑、Token 签发/验证、密码哈希、RBAC 权限校验 |
| **运行频率** | 每次 `git push`，每个 PR |

```java
// 示例：JWT Token 验证单元测试
@Test
void shouldRejectExpiredToken() {
    String token = jwtService.generateToken(user, Duration.ofMinutes(-1));
    assertThrows(TokenExpiredException.class, () -> jwtService.validateToken(token));
}

@Test
void shouldRejectTamperedToken() {
    String token = jwtService.generateToken(user, Duration.ofMinutes(15));
    String tampered = token.substring(0, token.length() - 5) + "xxxxx";
    assertThrows(InvalidTokenException.class, () -> jwtService.validateToken(tampered));
}
```

### 2.2 集成测试

| 维度 | 要求 |
|:-----|:-----|
| **框架** | Spring Boot Test + Testcontainers |
| **覆盖场景** | 完整的 OAuth2 授权流程、设备激活全链路、2FA 注册验证 |
| **数据隔离** | Testcontainers PostgreSQL + Redis |
| **运行频率** | 每个 PR + 合并到 main |

```java
// 示例：OAuth2 Authorization Code 流程集成测试
@SpringBootTest(webEnvironment = WebEnvironment.RANDOM_PORT)
@Testcontainers
class OAuth2FlowIntegrationTest {
    
    @Test
    void shouldCompleteAuthorizationCodeFlow() {
        // 1. 注册 OAuth 客户端
        // 2. 发起授权请求
        // 3. 用户登录确认
        // 4. 获取 authorization code
        // 5. 用 code + PKCE verifier 换 token
        // 6. 用 access token 访问 userinfo
    }
}
```

## 5. 安全测试矩阵

### 5.1 安全测试类型

| 测试类型 | 工具 | 频率 | 覆盖范围 |
|:---------|:-----|:-----|:---------|
| **SAST** | SonarQube + SpotBugs | 每次 PR | Java 代码 |
| **DAST** | OWASP ZAP | 每版本 | 运行中的应用 |
| **SCA** | OWASP Dependency Check | 每次构建 | 第三方依赖 |
| **容器扫描** | Trivy | 每次镜像构建 | Docker 镜像 |
| **密钥扫描** | TruffleHog / Gitleaks | 每次 PR | 代码仓库 |
| **渗透测试** | 第三方团队 | 每季度 | 全栈 |
| **模糊测试** | Jazzer (JVM Fuzzer) | 每版本 | 输入解析 |

### 5.2 OWASP Top 10 覆盖

| OWASP 风险 | 防护措施 | 验证方式 |
|:-----------|:---------|:---------|
| A01: 访问控制失效 | RBAC + API Gateway 鉴权 | 集成测试 + DAST |
| A02: 加密失败 | TLS 1.3 + AES-256-GCM + argon2id | 配置审计 |
| A03: 注入 | MyBatis 参数化 + 输入校验 | SAST + DAST |
| A04: 不安全设计 | 威胁建模 + 安全评审 | 设计审查 |
| A05: 安全配置错误 | 安全基线 + K8s Pod Security | IaC 扫描 |
| A06: 脆弱组件 | Dependency Check + Renovate | 每次构建 |
| A07: 认证失败 | 2FA + 速率限制 + 异常检测 | 渗透测试 |
| A08: 软件与数据完整性 | Ed25519 签名 + 校验和 | 单元测试 |
| A09: 日志监控失败 | ELK + SIEM + 告警规则 | 运维审查 |
| A10: SSRF | 输入校验 + 网络策略 | DAST |
