## 2. 模块职责

| 目录 | 职责 | 依赖 |
|------|------|------|
| `cmd/` | 程序入口，初始化依赖注入 | `internal/` |
| `internal/config/` | 环境变量/配置文件加载与验证 | 无 |
| `internal/server/` | HTTP/gRPC 服务器生命周期 | `internal/service/` |
| `internal/server/middleware/` | 认证/限流/CORS/日志中间件 | `internal/crypto/` |
| `internal/service/` | 业务逻辑，编排 repository + crypto | `internal/repository/`, `internal/crypto/` |
| `internal/repository/` | 数据访问层，SQL 查询封装 | `internal/model/`, PostgreSQL |
| `internal/crypto/` | 加密原语，JWT/Hash/AES/证书 | libsodium, Go crypto |
| `internal/model/` | 领域模型，与数据库表对应 | 无 |
| `pkg/` | 公共库，可被其他项目引用 | 无 |
| `api/` | API 规范定义（OpenAPI/Protobuf） | 无 |
| `web/` | Web 控制台前端 | `api/openapi/` |
| `tests/` | 集成测试与 E2E 测试 | `internal/` |
| `deploy/` | K8s/Terraform 部署配置 | 无 |

## 3. 代码规范

### 3.1 Go 代码规范

- 遵循 [Effective Go](https://go.dev/doc/effective_go) 和 [Uber Go Style Guide](https://github.com/uber-go/guide)
- 使用 `golangci-lint` 进行静态分析
- 包名使用小写单词，不使用下划线
- 导出函数/类型必须有 doc comment
- 错误处理使用 `fmt.Errorf("context: %w", err)` 包装
- 单元测试覆盖率 > 80%

### 3.2 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 文件名 | snake_case | `auth_service.go` |
| 包名 | 小写单词 | `package auth` |
| 接口名 | 单方法 + er 后缀 | `type TokenSigner interface` |
| 结构体 | PascalCase | `type AuthService struct` |
| 函数 | PascalCase (导出) / camelCase (私有) | `func NewAuthService()` / `func validateEmail()` |
| 常量 | PascalCase | `const MaxLoginAttempts = 5` |
| 变量 | camelCase | `var accessToken string` |
| 数据库表 | snake_case 复数 | `api_keys`, `audit_logs` |
| API 端点 | kebab-case | `/api/v1/auth/refresh-token` |
