# 02 — 架构设计

> 版本：v0.1 | 最后更新：2026-06-28 | 状态：Draft
> 子项目：qooauth（账号与安全）| 架构：Java + Spring Cloud 微服务
> 对标：Apple ID / Google Account 统一身份基础设施

---

## 1. 设计目标与原则

### 1.1 核心设计目标

| 目标 | P0 指标 | P1 目标 | 说明 |
|------|---------|---------|------|
| 认证延迟 | < 100ms (P99) | < 50ms | Token 签发/验证端到端延迟 |
| 服务可用性 | 99.99% | 99.999% | 身份服务必须高可用 |
| 并发认证 | 10000 TPS | 50000 TPS | 峰值认证请求处理能力 |
| 密钥安全 | HSM 存储 | HSM + Enclave | 根密钥保护级别 |
| Token 安全 | Ed25519 | Ed25519 + RS256 | 签名算法支持 |
| 合规认证 | SOC2 Type II | ISO27001 | 安全合规标准 |
| 全球部署 | 3 区域 | 6+ 区域 | 多区域就近服务 |

### 1.2 设计原则

- **零信任架构**：每次请求都需认证，不信任网络边界
- **纵深防御**：多层安全防护（网络→应用→数据→密钥）
- **隐私优先**：数据最小化采集、端到端加密、用户可控
- **标准兼容**：完全遵循 OAuth 2.0 / OIDC / FIDO2 / WebAuthn 国际标准
- **可扩展性**：水平扩展无状态服务，状态外置到 Redis/PostgreSQL
- **多租户隔离**：严格租户数据隔离，支持 SaaS 部署
- **优雅降级**：依赖服务故障时核心认证链路不受影响

## 5. 安全架构

### 5.1 纵深防御层次

```
第一层：网络边界
  ├── WAF (Web Application Firewall)
  ├── DDoS 防护
  └── IP 白名单/黑名单

第二层：API 网关
  ├── TLS 1.3 终结
  ├── 速率限制 (Token Bucket)
  ├── 请求验证 (大小/格式)
  └── 认证过滤器

第三层：应用安全
  ├── Spring Security 过滤器链
  ├── JWT 验证 + 黑名单检查
  ├── CSRF 保护
  ├── XSS 过滤
  └── 输入校验 (Bean Validation)

第四层：数据安全
  ├── 数据库加密 (TDE)
  ├── 列级加密 (敏感字段)
  ├── 传输加密 (TLS)
  └── 备份加密

第五层：密钥安全
  ├── HSM 硬件安全模块
  ├── 密钥分级 (根密钥→派生密钥)
  ├── 密钥轮换策略
  └── 访问审计
```

### 5.2 密钥管理策略

| 密钥类型 | 存储位置 | 轮换周期 | 备份策略 |
|----------|----------|----------|----------|
| 根密钥 (Root CA) | HSM | 每年 | 离线多份 |
| JWT 签名密钥 | HSM (派生) | 每 90 天 | 不备份（可派生） |
| 设备 CA 证书 | HSM | 每 2 年 | 离线备份 |
| API 密钥 | PostgreSQL (加密) | 按需 | 数据库备份 |
| 用户密码哈希 | PostgreSQL (argon2id) | N/A | 数据库备份 |
| Session 密钥 | Redis (仅内存) | 每 24h | 不持久化 |

### 6.2 审计日志

所有操作记录以下字段：
- `timestamp`：事件时间 (UTC)
- `actor`：操作主体 (用户/设备/服务)
- `action`：操作类型 (LOGIN/REGISTER/TOKEN_ISSUE/...)
- `resource`：操作资源
- `result`：成功/失败
- `context`：IP/User-Agent/地理位置
- `trace_id`：分布式追踪 ID
