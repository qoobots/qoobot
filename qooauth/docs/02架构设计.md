## 1. 设计目标与原则

### 1.1 核心设计目标

| 目标 | P0 指标 | P1 目标 | 说明 |
|------|---------|---------|------|
| 登录可用性 | 99.99% | 99.999% | 认证服务 SLA，影响所有上层服务 |
| 登录延迟 | < 500ms | < 200ms | 端到端认证时间 |
| Token 签发 QPS | > 10,000 | > 50,000 | JWT/Token 签发吞吐量 |
| 密钥安全 | FIPS 140-3 | 硬件 HSM | 密钥存储与操作安全等级 |
| 多设备并发 | 1,000,000 | 10,000,000 | 同时在线设备数 |
| 数据加密 | AES-256-GCM | 后量子密码 | 传输与存储加密算法 |

### 1.2 设计原则

- **零信任架构**：不信任任何网络/设备/用户，每次请求均需认证与授权
- **隐私优先**：默认最小化数据采集，本地处理优先，云端同步需显式授权
- **密钥分层**：根密钥（HSM）→ 服务密钥 → 用户密钥 → 会话密钥，逐层派生
- **防御纵深**：网络层（TLS/mTLS）→ 应用层（JWT/签名）→ 数据层（E2E 加密）
- **无缝体验**：单点登录（SSO）、设备间信任传递、Passkey 无密码认证
- **可审计性**：全量操作日志、不可篡改审计链、合规报告自动生成

### 2.2 子系统职责

| 子系统 | 职责 | 关键模块 |
|--------|------|---------|
| 用户账户 | 注册/登录/恢复/注销，用户资料管理 | UserSvc, SessionSvc |
| 认证基础设施 | JWT/OAuth/SSO/API Key 签发与验证 | AuthSvc, OAuthProvider |
| 设备身份 | X.509 证书、安全启动、设备绑定/迁移 | DeviceSvc, KeySvc |
| 家庭与团队 | 共享、RBAC、企业 MDM | AuthzSvc, OrgSvc |
| 安全与隐私 | E2E 加密、隐私标签、权限管理 | KeySvc, PrivacySvc |
| 威胁防护 | 异常检测、设备指纹、完整性校验 | ThreatDetect |
| 机器人间信任 | mTLS、协作授权、群组管理 | DeviceSvc, AuthzSvc |
| 审计与合规 | 操作审计、不可篡改日志、合规报告 | AuditSvc |
| 开发者认证 | 开发者证书、技能签名、沙箱 | DevCertSvc |

## 3. 技术选型

### 3.1 语言与框架

| 组件 | 技术选型 | 理由 |
|------|---------|------|
| 后端服务 | Go (Gin/Echo) | 高并发、低延迟、成熟中间件生态 |
| 加密库 | Go crypto + libsodium | 标准库完善 + 社区验证的安全原语 |
| 数据库 | PostgreSQL 15+ | 成熟的关系型数据库，支持 JSONB |
| 缓存/会话 | Redis Cluster | 低延迟会话存储、速率限制 |
| 消息队列 | NATS/Kafka | 异步事件（登录通知、审计日志） |
| 审计日志 | ClickHouse | 海量日志存储、OLAP 查询 |
| HSM 集成 | PKCS#11 / SoftHSM | 硬件安全模块标准接口 |
| 身份协议 | OAuth 2.1 / OIDC / WebAuthn | 行业标准认证协议 |
| API 协议 | REST (OpenAPI 3.0) + gRPC | 对外 REST + 内部 gRPC |

### 3.2 安全标准合规

| 标准 | 适用范围 | 关键要求 |
|------|---------|---------|
| FIPS 140-3 | 加密模块 | 经认证的加密算法实现 |
| ISO 27001 | 信息安全管理 | 安全策略、风险评估、持续改进 |
| SOC 2 Type II | 服务组织控制 | 安全性、可用性、机密性 |
| GDPR/CCPA/PIPL | 隐私保护 | 数据最小化、用户权利、跨境传输 |
| WebAuthn Level 2 | 无密码认证 | FIDO2/Passkey 标准 |
| NIST SP 800-63B | 数字身份指南 | 认证器保证级别 (AAL) |

## 4. 核心数据流

### 4.1 用户登录流程

```
用户 → 接入层 → 认证服务 → 用户数据库 → 密钥服务 → 会话服务 → JWT
  │       │         │           │           │          │          │
  │  POST │  验证    │  查询     │  验证     │  签名    │  存储    │  返回
  │ /login│  凭证   │  用户     │  密码     │  JWT    │  Redis   │  Token
  ▼       ▼         ▼           ▼           ▼          ▼          ▼
[客户端] → [API Gateway] → [AuthSvc] → [UserDB] → [KeySvc] → [Session] → [响应]
                                │                       │
                                ▼                       ▼
                          [ThreatDetect]          [AuditSvc]
                          风险评估/限流             操作审计
```

### 4.2 设备激活流程

```
新设备 → 扫码/蓝牙 → 设备身份服务 → 证书签发 → 账户绑定 → 安全启动链
  │         │            │             │          │           │
  │ 出厂    │  配对     │  验证       │  签发    │  绑定     │  验证
  │ 证书   │  码       │  X.509     │  设备证书│  账户     │  固件签名
  ▼         ▼            ▼             ▼          ▼           ▼
[设备] → [配对流] → [DeviceSvc] → [KeySvc] → [UserDB] → [Boot验证]
```

## 5. 部署架构

### 5.1 多区域部署

```
┌──────────────┐  ┌──────────────┐  ┌──────────────┐

│  数据本地化   │  │  GDPR 合规   │  │  标准部署    │
└──────────────┘  └──────────────┘  └──────────────┘
         │                │                │
         └────────────────┼────────────────┘
                          │
                  ┌──────────────┐
                  │ Global Sync  │
                  │ (异步复制)    │
                  └──────────────┘
```

### 5.2 高可用设计

- **认证服务**：无状态，水平扩展，K8s HPA 自动伸缩
- **数据库**：PostgreSQL 主从复制 + 自动故障切换
- **Redis**：Cluster 模式，跨 AZ 部署
- **HSM**：双活部署，自动切换
- **灾难恢复**：跨区域异步复制，RPO < 5min，RTO < 30min
