## 1. 数据模型概览

qooregs 的数据存储以文件系统为主（Markdown + YAML），辅以 SQLite 管理认证追踪和审查记录。核心数据模型包括：

```
┌─────────────────────────────────────────────────────────────┐
│                      数据模型关系                            │
│                                                             │
│  ┌──────────┐     ┌──────────────┐     ┌──────────────┐    │
│  │ Product  │────<│  Checklist   │     │ Certification│    │
│  │          │     │              │     │              │    │
│  │ name     │     │ product_id   │     │ product_id   │    │
│  │ model    │     │ market       │     │ market       │    │
│  │ version  │     │ items[]      │     │ type         │    │
│  └──────────┘     │ status       │     │ status       │    │
│                   └──────────────┘     │ expiry_date  │    │
│                                        └──────┬───────┘    │
│                                               │             │
│  ┌──────────┐     ┌──────────────┐            │             │
│  │Regulation│────<│  CheckItem   │            │             │
│  │          │     │              │            │             │
│  │ id       │     │ regulation_id│            │             │
│  │ title    │     │ title        │            │             │
│  │ market   │     │ priority     │            │             │
│  │ category │     │ method       │            │             │
│  └──────────┘     └──────────────┘            │             │
│                                               │             │
│  ┌──────────┐     ┌──────────────┐     ┌──────┴───────┐    │
│  │ GapReport│     │  AuditRecord │     │  RegChange   │    │
│  │          │     │              │     │              │    │
│  │ product  │     │ action       │     │ regulation   │    │
│  │ gaps[]   │     │ user         │     │ change_type  │    │
│  │ severity │     │ timestamp    │     │ impact       │    │
│  └──────────┘     └──────────────┘     │ notified     │    │
│                                        └──────────────┘    │
└─────────────────────────────────────────────────────────────┘
```

## 2. 知识库数据格式

### 2.1 法规条目 (knowledge/*.yaml)

```yaml
# 每个法规文件结构
regulation:
  id: string             # 唯一标识符 (e.g., "ISO_13482_2014")
  title: string          # 法规全称
  short_name: string     # 简称
  version: string        # 版本/年份
  category: enum         # safety|wireless|privacy|ai_ethics|consumer|export|environmental
  markets: [string]      # 适用市场: global|eu|cn|us|jp|kr
  
  authority:             # 监管机构
    name: string
    url: string
    
  summary: string        # 法规概述
  full_text_url: string  # 法规全文链接
  
  check_items:           # 检查项列表
    - id: string
      title: string
      description: string
      priority: enum     # P0|P1|P2|P3
      verification_method: enum  # document_review|lab_test|self_declaration|inspection
      applicable_products: [string]  # 适用产品类型
      sub_items: [string]            # 子检查项
      references: [string]           # 参考标准
```

### 2.2 产品定义 (product.yaml)

```yaml
# 产品合规配置
product:

      - type: "RGB-D Camera"

```

### 2.3 检查清单 (checklists/*.yaml)

```yaml
checklist:
  id: string             # "checklist_{product}_{market}_{timestamp}"
  product_id: string
  product_version: string
  market: string
  generated_at: datetime
  updated_at: datetime
  
  sections:
    - category: string
      label: string
      items:
        - id: string          # 对应 regulation.check_items.id
          regulation_id: string
          title: string
          status: enum        # passed|needs_verification|not_checked|not_applicable|failed
          priority: enum
          evidence: string    # 证据文件路径
          notes: string
          reviewer: string
          reviewed_at: datetime
```

## 3. 数据库设计 (SQLite)

### 3.1 认证追踪表

```sql
CREATE TABLE certifications (
    id              TEXT PRIMARY KEY,          -- "cert_{product}_{market}_{type}"
    product_id      TEXT NOT NULL,
    product_version TEXT NOT NULL,
    market          TEXT NOT NULL,             -- cn/eu/us/jp
    cert_type       TEXT NOT NULL,             -- SRRC/FCC/CE_RED/...
    cert_number     TEXT,                      -- 证书编号
    status          TEXT NOT NULL DEFAULT 'not_applied',
    applied_at      TEXT,
    approved_at     TEXT,
    expires_at      TEXT,
    renewed_at      TEXT,
    lab_name        TEXT,                      -- 测试实验室
    report_url      TEXT,                      -- 测试报告链接
    notes           TEXT,
    created_at      TEXT NOT NULL DEFAULT (datetime('now')),
    updated_at      TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE INDEX idx_cert_product ON certifications(product_id);
CREATE INDEX idx_cert_status ON certifications(status);
CREATE INDEX idx_cert_expiry ON certifications(expires_at);
```

### 3.2 差距分析记录表

```sql
CREATE TABLE gap_reports (
    id              TEXT PRIMARY KEY,          -- "gap_{product}_{market}_{timestamp}"
    product_id      TEXT NOT NULL,
    product_version TEXT NOT NULL,
    market          TEXT NOT NULL,
    severity        TEXT NOT NULL,             -- critical/high/medium/low
    gap_items_json  TEXT NOT NULL,             -- JSON: 差距项列表
    recommendations_json TEXT,                 -- JSON: 整改建议
    generated_at    TEXT NOT NULL DEFAULT (datetime('now')),
    resolved_at     TEXT
);

CREATE INDEX idx_gap_product ON gap_reports(product_id, market);
```

### 3.3 法规变更记录表

```sql
CREATE TABLE regulation_changes (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    regulation_id   TEXT NOT NULL,
    change_type     TEXT NOT NULL,             -- new_version/amendment/repeal/new
    title           TEXT NOT NULL,
    description     TEXT,
    source_url      TEXT,
    effective_date  TEXT,
    impact_level    TEXT,                      -- critical/high/medium/low
    affected_products TEXT,                    -- 受影响产品 (JSON array)
    notified        INTEGER NOT NULL DEFAULT 0,
    created_at      TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE INDEX idx_regchange_reg ON regulation_changes(regulation_id);
CREATE INDEX idx_regchange_notified ON regulation_changes(notified);
```

### 3.4 审计记录表

```sql
CREATE TABLE audit_records (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    action          TEXT NOT NULL,             -- checklist_update/cert_apply/gap_resolve/...
    product_id      TEXT,
    market          TEXT,
    user_id         TEXT NOT NULL,
    details_json    TEXT,
    created_at      TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE INDEX idx_audit_product ON audit_records(product_id);
CREATE INDEX idx_audit_created ON audit_records(created_at DESC);
```

### 3.5 风险评估记录表

```sql
CREATE TABLE risk_assessments (
    id              TEXT PRIMARY KEY,          -- "risk_{product}_{method}_{timestamp}"
    product_id      TEXT NOT NULL,
    product_version TEXT NOT NULL,
    method          TEXT NOT NULL,             -- fmea/hazop
    items_json      TEXT NOT NULL,             -- JSON: 风险项列表
    summary_json    TEXT,                      -- JSON: 汇总统计
    generated_at    TEXT NOT NULL DEFAULT (datetime('now')),
    reviewed_by     TEXT,
    reviewed_at     TEXT
);

CREATE INDEX idx_risk_product ON risk_assessments(product_id);
```

## 4. 文档模板变量

### 4.1 CE 技术文件变量

```yaml
# 模板变量上下文
product:
  name: string
  model: string
  version: string
  description: string
  safety_features: [{name, description, standard}]

manufacturer:
  name: string
  address: string
  authorized_rep: {name, address}  # 欧盟授权代表

applicable_directives:
  - id: string           # 2006/42/EC
    title: string
    conformity_route: string  # self_declaration/notified_body
    notified_body: string

risk_assessment:
  major_hazards: [{source, level, mitigation, residual}]

tests:
  - standard: string
    lab: string
    report_id: string
    date: string
    result: string       # passed/failed/conditional
```

### 4.2 DPIA 报告变量

```yaml
# 数据保护影响评估变量
dpia:
  project_name: string
  data_controller: {name, contact}
  data_processor: {name, contact}
  
  data_processing:
    - data_type: string         # camera/microphone/location/biometric
      purpose: string
      legal_basis: string       # consent/legitimate_interest/legal_obligation
      retention_period: string
      third_party_access: boolean
      cross_border: boolean
      
  risk_assessment:
    - risk: string
      likelihood: enum          # low/medium/high
      impact: enum
      mitigation: string
      
  stakeholder_consultation: string
  dpo_review: {name, date, recommendation}
```

## 5. 文件存储结构

```
qooregs/
├── knowledge/                      # 知识库 — YAML 文件
│   ├── safety/*.yaml               # 每个标准一个文件
│   ├── wireless/*.yaml
│   ├── privacy/*.yaml
│   ├── ai_ethics/*.yaml
│   ├── consumer/*.yaml
│   ├── export/*.yaml
│   └── environmental/*.yaml
│
├── products/                       # 产品定义
│   └── {product_id}/
│       ├── product.yaml            # 产品合规配置
│       └── evidence/               # 合规证据文件
│           ├── test_reports/
│           ├── risk_assessments/
│           └── certificates/
│
├── checklists/                     # 检查清单 (自动生成)
│   └── {product_id}/
│       └── {product}_{market}.yaml
│
├── reports/                        # 生成的报告
│   └── {product_id}/
│       ├── ce_technical_file.md
│       ├── fcc_test_report.md
│       ├── dpia_report.md
│       ├── gap_analysis_{market}.md
│       └── compliance_declaration_{market}.md
│
├── data/                           # SQLite 数据库
│   └── qooregs.db                  # 认证/差距/变更/审计记录
│
└── templates/                      # Jinja2 模板
    ├── ce_technical_file.md.j2
    ├── fcc_test_report.md.j2
    ├── doc_template.md.j2
    ├── dpia_report.md.j2
    └── risk_assessment_*.md.j2
```

## 6. 数据安全

| 数据类别 | 敏感级别 | 访问控制 | 加密 |
|---------|---------|---------|------|
| 法规知识库 | 公开 | 全员可读 | 否 |
| 产品技术规格 | 机密 | 项目团队 | Git 权限控制 |
| 认证证书 | 机密 | 合规团队 | 文件加密存储 |
| 测试报告 | 机密 | 项目+合规团队 | 文件加密存储 |
| 差距分析报告 | 内部 | 项目+管理层 | Git 权限控制 |
| 审计记录 | 内部 | 合规团队 | 数据库加密 |
