# 09 — 技术架构设计

> 版本：v0.1.0 | 最后更新：2026-06-30 | 状态：Draft
> 子项目：qooremote（远程机器人监控遥控工具）

---

## 1. 技术栈总览

| 层级 | 桌面端 (console) | Web 端 (web) | 移动端 (mobile) |
|------|------------------|-------------|-----------------|
| 语言 | Python 3.11+ | TypeScript 5.3+ | TypeScript 5.3+ |
| UI 框架 | PySide6 6.5+ | Vue 3.4+ | Vue 3 / UniApp |
| UI 组件库 | Qt Widgets + QSS | Element Plus 2.5+ | uView Plus |
| 状态管理 | Qt Signals/Slots + 自定义 VM | Pinia 2.1+ | Pinia 2.1+ |
| 通信 | websockets + aiortc | 原生 WebSocket + RTCPeerConnection | uni.connectSocket |
| 3D 渲染 | PyOpenGL / Qt 3D | Three.js 0.160+ | — |
| 图表 | pyqtgraph / matplotlib | ECharts 5.5+ | uCharts |
| 数据存储 | HDF5 (h5py) + SQLite | — | uni.storage |
| 音频 | PyAudio + python-opus | Web Audio API | uni.createInnerAudioContext |
| 输入设备 | SDL2 / evdev / pywinusb | Gamepad API | — |
| 包管理 | pip (setuptools) | npm (Vite) | npm (HBuilderX) |
| 测试 | pytest + pytest-asyncio | Vitest | — |
| 代码检查 | mypy + ruff | ESLint + Prettier | ESLint + Prettier |

## 2. 桌面端技术架构

### 2.1 运行时架构

```
                    ┌─────────────────────┐
                    │   main.py           │
                    │   QApplication       │
                    └──────────┬──────────┘
                               │
                    ┌──────────▼──────────┐
                    │   MainWindow         │
                    │   (QMainWindow)      │
                    └──────────┬──────────┘
                               │
          ┌────────────────────┼────────────────────┐
          │                    │                    │
  ┌───────▼───────┐   ┌───────▼───────┐   ┌───────▼───────┐
  │  QSplitter    │   │  QDockWidget  │   │  QDockWidget  │
  │  视频面板      │   │  仪表盘面板    │   │  操控面板      │
  └───────┬───────┘   └───────┬───────┘   └───────┬───────┘
          │                   │                   │
  ┌───────▼───────────────────▼───────────────────▼───────┐
  │                    ViewModel Layer                     │
  │  (QObject-based, Signal/Slot to UI)                   │
  └───────────────────────────┬───────────────────────────┘
                              │
  ┌───────────────────────────▼───────────────────────────┐
  │                    Service Layer                        │
  │  (asyncio event loop on QThread)                       │
  └────────────────────────────────────────────────────────┘
```

### 2.2 线程模型

```
┌─────────────────────────────────────────────────────┐
│                    Main Thread (Qt)                  │
│  ┌─────────────────────────────────────────────────┐│
│  │  UI Rendering (QWidget::paintEvent)             ││
│  │  User Input (Mouse/Keyboard Events)             ││
│  │  QTimer (30Hz UI refresh)                       ││
│  │  Signal/Slot Dispatch                           ││
│  └─────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│               Network Thread (asyncio)               │
│  ┌─────────────────────────────────────────────────┐│
│  │  WebSocket Client (heartbeat + message rx/tx)   ││
│  │  WebRTC Manager (ICE/DTLS/SRTP)                 ││
│  │  State Parser (JSON → RobotState)               ││
│  │  Command Serializer (TeleopCmd → JSON)           ││
│  └─────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│                 I/O Thread (Blocking)                │
│  ┌─────────────────────────────────────────────────┐│
│  │  Audio Capture/Playback (PyAudio)               ││
│  │  Gamepad Polling (SDL2)                         ││
│  │  Recording I/O (HDF5 write)                     ││
│  └─────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────┐
│              Render Thread (Optional)                │
│  ┌─────────────────────────────────────────────────┐│
│  │  OpenGL 3D Rendering (QOpenGLWidget)            ││
│  │  Video Decode + Texture Upload                  ││
│  └─────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────┘
```

### 3.3 状态管理架构 (Pinia)

```
stores/
├── robotStore.ts        # 机器人列表 + 选择状态
│   ├── robots: Robot[]
│   ├── selectedId: string | null
│   └── actions: fetchRobots, selectRobot
│
├── stateStore.ts        # 当前选中机器人实时状态
│   ├── status: RobotStatus
│   ├── joints: JointState[]
│   ├── sensors: SensorData
│   └── actions: updateFromMessage, clearState
│
├── connectionStore.ts   # WebSocket/WebRTC 连接状态
│   ├── wsStatus: ConnectionStatus
│   ├── rtcStatus: ConnectionStatus
│   ├── latency: number
│   └── actions: connect, disconnect, reconnect
│
├── alertStore.ts        # 告警管理
│   ├── activeAlerts: Alert[]
│   ├── alertHistory: Alert[]
│   └── actions: addAlert, acknowledge, dismiss
│
└── settingsStore.ts     # 用户设置
    ├── theme: 'dark' | 'light'
    ├── language: string
    └── actions: updateSettings
```

### 4.2 条件编译策略

```
#ifdef APP-PLUS (原生 App)
  - 使用 plus.video 进行硬件解码
  - 使用 plus.push 进行厂商推送
  - 使用 plus.audio 进行低延迟音频
#endif

#ifdef H5 (浏览器)
  - 使用原生 WebRTC API
  - 使用 Web Audio API
  - 使用 Service Worker 离线缓存
#endif

#ifdef MP-WEIXIN (微信小程序)
  - 使用 live-player 组件
  - 使用 live-pusher 组件
  - 使用 wx.setInnerAudioOption 音频优化
#endif
```

| ruff | Lint + Format | `pyproject.toml [tool.ruff]` |
| mypy | 类型检查 | `pyproject.toml [tool.mypy]` |

```toml
# pyproject.toml (技术规范部分)
[tool.ruff]
line-length = 100
target-version = "py311"

[tool.ruff.lint]

strict = true
warn_return_any = true
```
