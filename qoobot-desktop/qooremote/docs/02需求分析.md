# 02 — 需求分析

> 版本：v0.1.0 | 最后更新：2026-06-30 | 状态：Draft
> 子项目：qooremote（远程机器人监控遥控工具）

---

## 1. 需求全景

qooremote 作为仿生人远程遥控系统，承载五大需求域：

```
                    ┌─────────────────────────────────┐
                    │   实时性 (Real-time)              │
                    │   视频 < 200ms / 控制 < 50ms      │
                    │   语音 < 150ms 端到端延迟          │
                    └───────────────┬─────────────────┘
                                    │
    ┌───────────────────┐  ┌───────┴────────┐  ┌───────────────────┐
    │ 可用性 (Usability) │  │ 可靠性 (Reliability) │ 安全性 (Safety)  │
    │ 直观的操控界面     │  │ 99.5% 连接可用性  │  │ 紧急制动 < 100ms │
    │ 多平台一致体验     │  │ 自动重连/降级     │  │ 权限与审计       │
    └─────────┬─────────┘  └───────┬──────────┘  └────────┬──────────┘
              │                   │                       │
              └───────────────────┼───────────────────────┘
                                  │
               ┌──────────────────┴──────────────────┐
               │                                     │
    ┌──────────┴──────────┐              ┌───────────┴───────────┐
    │ 可扩展 (Extensible)  │              │ 跨平台 (Cross-Platform)│
    │ 插件化操控模式       │              │ desktop + web + mobile │
    │ 多机器人管理         │              │ 统一 API + 适配 UI     │
    └─────────────────────┘              └───────────────────────┘
```

## 2. 功能需求

### 2.1 实时仪表盘需求 (Dashboard)

| 编号 | 需求 | 描述 | 优先级 |
|:----:|------|------|:------:|
| FR-DASH-01 | 机器人状态总览 | 实时展示机器人 CPU/内存/磁盘/温度等系统状态 | P0 |
| FR-DASH-02 | 电池与电源监控 | 显示电量百分比、充放电状态、预估剩余续航时间 | P0 |
| FR-DASH-03 | 传感器数据面板 | IMU（加速度/陀螺仪/磁力计）、力/扭矩传感器实时数据 | P0 |
| FR-DASH-04 | 关节状态面板 | 所有自由度关节的角度/角速度/力矩/温度实时数值 | P0 |
| FR-DASH-05 | 自定义仪表盘布局 | 用户可拖拽/添加/删除面板组件 | P1 |
| FR-DASH-06 | 数据趋势图 | 关键指标历史曲线（1min/5min/30min） | P1 |

### 2.2 视频回传需求 (Video)

| 编号 | 需求 | 描述 | 优先级 |
|:----:|------|------|:------:|
| FR-VID-01 | 多路视频回传 | 同时显示 4+ 路摄像头视频流，支持切换布局 | P0 |
| FR-VID-02 | 低延迟传输 | 端到端延迟 < 200ms（局域网），< 500ms（广域网） | P0 |
| FR-VID-03 | 分辨率自适应 | 支持 480p/720p/1080p 多分辨率切换 | P1 |
| FR-VID-04 | 带宽自适应 | 根据网络带宽自动调整码率与帧率 | P1 |
| FR-VID-05 | 视频录制 | 支持单路/多路视频同步录制到本地 | P1 |
| FR-VID-06 | 截图与标注 | 视频画面截图并在图上标注/画圈 | P2 |

### 2.3 遥操作控制需求 (Teleoperation)

| 编号 | 需求 | 描述 | 优先级 |
|:----:|------|------|:------:|
| FR-TEL-01 | 手柄操控支持 | Xbox/PS 标准游戏手柄映射到机器人运动控制 | P0 |
| FR-TEL-02 | 键盘操控支持 | WASD + 鼠标控制移动/转向/手臂等基本动作 | P0 |
| FR-TEL-03 | 末端执行器位姿控制 | 6 DOF 空间位姿（xyz+RPY）精确控制 | P0 |
| FR-TEL-04 | 关节空间控制 | 独立控制每个关节的位置/速度/力矩 | P0 |
| FR-TEL-05 | 控制指令平滑 | 操作输入平滑滤波，避免指令突变导致机器人抖动 | P1 |
| FR-TEL-06 | 力反馈 | 操作手柄接收机器人力/力矩反馈（需硬件支持） | P2 |
| FR-TEL-07 | 动作捕捉集成 | 支持 OptiTrack/Vicon/Xsens 等动捕系统 | P1 |
| FR-TEL-08 | VR 头显集成 | 支持 Meta Quest/HTC Vive 等 VR 设备第一人称操控 | P2 |

### 2.4 语音对讲需求 (Voice)

| 编号 | 需求 | 描述 | 优先级 |
|:----:|------|------|:------:|
| FR-VOX-01 | 双向实时语音 | 操作端↔机器人端双向通话，延迟 < 150ms | P1 |
| FR-VOX-02 | PTT 一键对讲 | 按住说话、松开监听模式 | P1 |
| FR-VOX-03 | 免提全双工 | 免按键持续双向通话 | P2 |
| FR-VOX-04 | 音频编解码 | Opus 编解码，支持 8-48kHz 采样率 | P1 |
| FR-VOX-05 | 回声消除与降噪 | AEC + ANS 音频前处理 | P2 |

### 2.5 远程接管需求 (Takeover)

| 编号 | 需求 | 描述 | 优先级 |
|:----:|------|------|:------:|
| FR-TAK-01 | 模式无缝切换 | 自主运行 → 半自主（人工建议）→ 全手动控制无缝切换 | P0 |
| FR-TAK-02 | 紧急接管 | 一键紧急按钮，< 100ms 内停止机器人运动并切换至手动 | P0 |
| FR-TAK-03 | 控制权限管理 | 多操作员时的控制权分配与优先级机制 | P1 |
| FR-TAK-04 | 接管前状态同步 | 接管时自动同步当前机器人全状态（姿态/任务/传感器） | P1 |
| FR-TAK-05 | 接管操作审计 | 完整记录接管时间/操作员/控制指令/机器人响应 | P2 |
| FR-TAK-06 | 安全围栏 | 手动操控时的虚拟安全边界，越界自动制动 | P1 |

### 2.6 示教录制需求 (Teach)

| 编号 | 需求 | 描述 | 优先级 |
|:----:|------|------|:------:|
| FR-TCH-01 | 操作数据采集 | 同步录制关节轨迹 + 传感器数据 + 摄像头视频 | P1 |
| FR-TCH-02 | 多通道同步 | 所有数据通道时间戳精确对齐（误差 < 5ms） | P1 |
| FR-TCH-03 | 操作回放 | 录制数据在仿真或真实机器人上精确回放 | P1 |
| FR-TCH-04 | 训练数据导出 | 导出为 qoodev/.qoodata 格式，供模型训练使用 | P1 |
| FR-TCH-05 | 动作分段标注 | 在录制过程中或事后对动作片段添加标签 | P2 |
| FR-TCH-06 | 录制管理 | 录制列表/搜索/标签/删除/导出管理 | P2 |

### 2.7 告警通知需求 (Alerting)

| 编号 | 需求 | 描述 | 优先级 |
|:----:|------|------|:------:|
| FR-ALT-01 | 实时异常推送 | 碰撞/过温/低电量/通信中断等事件实时弹窗+声音通知 | P0 |
| FR-ALT-02 | 告警分级 | 紧急/严重/警告/信息四级告警分类 | P0 |
| FR-ALT-03 | 告警规则配置 | 用户自定义阈值（温度上限/电量下限等） | P1 |
| FR-ALT-04 | 告警静默与确认 | 操作员确认/静默/升级告警 | P1 |
| FR-ALT-05 | 告警历史查询 | 按时间/类型/等级筛选与导出 | P2 |

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 视频端到端延迟(局域网) | < 200ms | WebRTC stats API |
| 视频端到端延迟(4G/5G) | < 500ms | WebRTC stats API |
| 控制指令延迟(局域网) | < 50ms | RTT 测量 |
| 控制指令延迟(4G/5G) | < 100ms | RTT 测量 |
| 语音端到端延迟 | < 150ms | 环回测试 |
| 仪表盘数据刷新率 | > 30 Hz | 帧率统计 |
| 桌面端 CPU 占用(空闲) | < 10% | 任务管理器 |
| 桌面端内存占用 | < 1GB | 进程内存统计 |
| 同时连接机器人数 | > 5 台 | 压力测试 |
| 视频录制磁盘写入 | > 50 MB/s | 顺序写入测试 |

### 3.2 可靠性需求

| 指标 | 目标值 |
|------|--------|
| WebSocket 连接可用性 | > 99.5% |
| 断线自动重连成功率 | > 99% |
| 重连恢复时间 | < 5s |
| 24h 连续运行稳定性 | 无内存泄漏/无崩溃 |
| 视频流帧率稳定性 | 抖动 < ±5% |

### 3.3 安全性需求

| 需求 | 描述 | 优先级 |
|------|------|:------:|
| 通信加密 | WebSocket/WebRTC 全链路 TLS 1.3 加密 | P0 |
| 身份认证 | 操作员 JWT Token 认证 + 设备证书双向认证 | P0 |
| 控制指令签名 | 遥操作指令 HMAC 签名防篡改 | P0 |
| 紧急制动独立信道 | 紧急制动通过独立 CAN FD/GPIO 硬件通道 | P0 |
| 操作权限分级 | 管理员/操作员/观察者三级权限 | P1 |
| 操作审计日志 | 所有控制操作完整记录，不可篡改 | P1 |

### 3.4 跨平台需求

| 平台 | 最低要求 | 推荐配置 |
|------|----------|----------|

## 4. 用户角色与场景

### 4.1 用户角色

| 角色 | 描述 | 核心需求 |
|------|------|----------|
| 机器人操作员 | 远程操控机器人的一线人员 | 实时视频回传、低延迟控制、紧急接管 |
| 机器人管理员 | 管理多台机器人运行状态 | 多机仪表盘、告警监控、会话管理 |
| 示教工程师 | 录制操作数据用于模型训练 | 操作录制、数据标注、回放与分析 |
| 安全监督员 | 监控操作安全性 | 紧急制动权限、操作审计、安全围栏配置 |

### 4.2 典型使用场景

| 场景 | 描述 | 涉及模块 |
|------|------|----------|
| 远程巡检 | 操作员远程控制机器人进入危险区域（化工厂/核设施）进行视觉巡检 | 视频 + 遥操作 + 语音 |
| 远程操作 | 当自主 AI 遇到长尾场景（如复杂抓取/开门）时，人工接管完成操作 | 远程接管 + 遥操作 + 视频 |
| 示教采集 | 操作员操纵机器人完成一遍任务，系统采集数据用于后续模仿学习训练 | 示教录制 + 遥操作 + 视频 |
| 多机编队监控 | 管理员同时监控 3-5 台机器人的运行状态，接收异常告警 | 仪表盘 + 告警 + 多机管理 |
| 移动巡检 | 操作员在手机上查看机器人实时画面，接收推送告警 | 视频 + 仪表盘 + 告警 |

## 5. 依赖与约束

### 5.1 上游依赖

| 依赖子系统 | 提供能力 | 影响范围 |
|------------|----------|----------|
| qoobrain | 接收控制指令、状态上报 | 遥操作控制、远程接管 |
| qoosvc | 系统状态服务（传感器/关节/电量） | 仪表盘、告警 |
| qoocloud | 信令中继、WebSocket/WebRTC 转发 | 通信连接、视频/语音回传 |

### 5.2 技术约束

| 约束 | 说明 |
|------|------|
| Python 3.11+ | 桌面控制台运行时最低版本要求 |
| PySide6 >= 6.5 | Qt for Python 的最新 LTS 版本 |
| WebRTC 支持 | 依赖浏览器/平台原生 WebRTC 实现 |
| Internet 连接 | 远程操作需要 qoocloud 作为信令服务器 |
