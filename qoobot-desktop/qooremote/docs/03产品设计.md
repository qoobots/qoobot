# 03 — 产品设计

> 版本：v0.1.0 | 最后更新：2026-06-30 | 状态：Draft

---

## 1. 产品定义

### 1.1 产品定位

qooremote 是 QooBot 仿生人的 **"远程神经桥接器"**——它在人类操作员与仿生机器人之间建立高带宽、低延迟的感知-控制闭环，使人类能够在任何地点像操控自己的延伸身体一样操控远程机器人。

**一句话定位**：面向仿生人操作员的专业远程控制台，Tesla Optimus 遥控示教 + Boston Dynamics 远程操作台的开源替代。

### 1.2 目标用户

| 用户角色 | 画像 | 使用方式 | 频率 |
|----------|------|----------|:----:|
| 机器人操作员 | 经过培训的操控人员，熟悉机器人运动学与控制 | 桌面控制台（手柄/键鼠操控） | 日常 |
| 运维工程师 | 负责机器人日常运行监控的技术人员 | Web 仪表盘 + 移动端告警 | 全天候 |
| 示教工程师 | AI/机器人工程师，采集训练数据 | 桌面控制台（示教录制模式） | 周期性 |
| 安全管理员 | 具有紧急制动权限的管理人员 | Web/桌面端监控面板 | 按需 |
| 移动巡检员 | 需要移动中查看机器人状态的人员 | 手机 App | 按需 |

### 1.3 竞品对比

| 维度 | qooremote | Foxglove Studio | ROS 2 rqt/rviz | Tesla Optimus Teleop | Boston Dynamics Spot |
|------|-----------|-----------------|----------------|----------------------|---------------------|
| 视频回传 | 多路 WebRTC | Rosbridge 视频 | 需配置 | 专有协议 | 专有协议 |
| 遥操作 | 动捕+VR+手柄三种模式 | 无 | 需自开发 | 动捕专有 | 手柄控制 |
| 语音对讲 | 双向 Opus | 无 | 无 | 无 | 双向对讲 |
| 示教录制 | 多通道同步录制 | 数据录制 | Rosbag | 内置 | 无 |
| 3D 数字孪生 | Three.js 实时渲染 | 3D 面板 | RViz | 专有 | 专有 |
| 告警通知 | 自定义规则+推送 | 无 | 无 | 无 | 基础 |
| 跨平台 | desktop+web+mobile | desktop+web | desktop | desktop | desktop+mobile |
| 授权 | Apache 2.0 开源 | MPL 2.0 开源 | BSD 开源 | 闭源 | 闭源 |

## 3. 用户体验设计理念

### 3.1 设计原则

| 原则 | 说明 | 实现方式 |
|------|------|----------|
| 眼见即所得 | 操作员看到的视频应为实时画面 | 低延迟 WebRTC + 时间戳水印 |
| 操控即直觉 | 操控方式与人类自然操作习惯一致 | 手柄映射+动捕映射+VR 空间映射 |
| 安全第一 | 任何异常都应导向安全状态 | 失联自动制动+硬件急停冗余 |
| 信息分层 | 重要信息突出、次要信息可折叠 | 三级信息架构（关键/重要/补充） |
| 跨端一致 | 桌面/Web/移动端核心体验一致 | 统一设计组件库+品牌规范 |

### 3.2 信息架构

```
qooremote 信息架构
│
├── [关键级] 始终可见，不可隐藏
│   ├── 连接状态指示器 (🟢在线 🟡延迟 🔴离线)
│   ├── 紧急制动按钮 (红色大按钮，始终可用)
│   ├── 连接延迟数值 (ms)
│   └── 活跃告警计数 (🔔×N)
│
├── [重要级] 默认可见，可折叠
│   ├── 视频画面
│   ├── 电池电量/续航
│   ├── 机器人姿态状态
│   └── 最新告警条目
│
└── [补充级] 可折叠/切换查看
    ├── 详细传感器数值
    ├── 关节力矩/温度
    ├── 网络带宽统计
    └── 系统资源使用
```

## 5. 商业模式（开源生态）

### 5.1 开源策略

qooremote 采用 Apache 2.0 协议开源，社区免费使用全部功能。

### 5.2 生态价值

| 维度 | 说明 |
|------|------|
| 降低门槛 | 开源遥控工具使研究机构/中小企业无需自研遥控系统 |
| 数据飞轮 | 社区操作员产生的遥操作数据反哺 AI 模型训练 |
| 硬件生态 | 社区贡献更多手柄/动捕/VR 设备驱动适配 |
| 标准化 | 推动机器人遥操作协议的行业标准 |
