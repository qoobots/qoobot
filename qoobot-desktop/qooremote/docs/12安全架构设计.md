# 12 — 安全架构设计

> 版本：v0.1.0 | 最后更新：2026-06-30 | 状态：Draft
> 子项目：qooremote（远程机器人监控遥控工具）

---

## 1. 安全设计原则

| 原则 | 说明 | 实现 |
|------|------|------|
| 纵深防御 | 多层安全机制，不依赖单一防护 | 传输加密 + 身份认证 + 指令签名 + 硬件安全 |
| 最小权限 | 操作员只拥有完成任务的最小控制权限 | 三级角色权限 + 操作范围限制 |
| 安全默认 | 出厂默认启用所有安全功能 | 默认启用 TLS/认证/签名 |
| 故障安全 | 任何通信故障导致机器人进入安全状态 | 失联自动制动 + 硬件急停冗余 |
| 可审计 | 所有遥控操作完整记录不可篡改 | 操作审计日志 + 视频同步录制 |
| 隐私保护 | 视频/音频数据加密传输，本地处理优先 | 端到端 SRTP 加密 |

```
┌─────────────────────────────────────────────────────────┐
│                    应用安全层                             │
│  权限控制 (RBAC) | 操作审计 | 敏感操作确认                 │

│                    传输安全层                             │
│  TLS 1.3 (WSS) | SRTP (WebRTC) | HMAC 指令签名          │

├─────────────────────────────────────────────────────────┤
│                    物理安全层 (机器人端)                   │
│  紧急制动独立回路 | 安全 MCU | 碰撞检测 | 看门狗           │
└─────────────────────────────────────────────────────────┘
```

## 3. 认证与授权

### 3.1 操作员认证

```
认证流程:
1. 操作员在 qooremote 中输入 Token（或通过 OAuth 2.0 设备授权码流程获取）
2. qooremote → qoocloud: auth 消息携带 JWT Token
3. qoocloud 验证 Token 签名、有效期、权限
4. 返回 auth_ok 并附 operator_id + roles
5. Token 本地存储于操作系统密钥链（Windows Credential Manager / macOS Keychain / Linux Secret Service）
```

### 3.2 机器人认证（mTLS）

| 证书 | 颁发者 | 存储位置 | 用途 |
|------|--------|----------|------|
| Root CA | QooBot CA | 操作端信任库 | 验证机器人证书 |
| Robot 证书 | QooBot CA | 机器人 TPM/安全存储 | 机器人身份证明 |
| Operator 证书 | QooBot CA (可选) | 操作系统密钥链 | 高安全场景双向认证 |

### 3.3 三级角色权限

| 角色 | 权限 | 典型用户 |
|------|------|----------|
| 🔴 管理员 (Admin) | 全权限：操控 + 配置 + 管理操作员 | 技术负责人 |
| 🟠 操作员 (Operator) | 操控机器人 + 查看状态 + 录制 | 一线操控员 |
| 🔵 观察者 (Observer) | 仅查看：视频 + 状态（无控制权限） | 监督/培训人员 |

```yaml
# 权限矩阵
permissions:
  admin:
    - robot.control            # 控制机器人
    - robot.mode.switch        # 切换运行模式
    - robot.emergency_stop     # 紧急制动
    - session.manage           # 管理会话
    - recording.manage         # 录制管理
    - alert.config             # 告警规则配置
    - operator.manage          # 管理操作员
    - settings.write           # 修改系统设置

  operator:
    - robot.control
    - robot.mode.switch
    - robot.emergency_stop
    - session.manage
    - recording.manage

  observer:
    - robot.view               # 查看机器人状态
    - video.view               # 查看视频
    - alert.view               # 查看告警
```

## 4. 通信安全

### 4.1 传输加密

| 信道 | 协议 | 加密 | 密钥管理 |
|------|------|:----:|----------|
| 信令 (WSS) | WebSocket Secure | TLS 1.3 | 标准 TLS 证书 |
| 媒体 (WebRTC) | SRTP | DTLS-SRTP | 自动协商密钥 |
| 数据 (DataChannel) | SCTP over DTLS | DTLS 1.2+ | 自动协商密钥 |
| 控制指令 | DataChannel + HMAC | 应用层 HMAC | 预共享/会话协商 |

### 4.2 控制指令签名

```
HMAC 签名流程:

1. 建立会话时，通过安全信道交换 HMAC 密钥 (或使用 DTLS 导出密钥)
2. 每条控制指令计算:
   HMAC = SHA-256(payload || timestamp || sequence || shared_secret)
3. 机器人端验证 HMAC，不匹配则拒绝指令并告警
4. 防止: 重放攻击(sequence)、篡改(HMAC)、伪造(shared_secret)
```

```json
{
  "type": "teleop.pose",
  "seq": 12345,
  "timestamp": 1719700000100,
  "pose": {...},
  "hmac": "sha256:a1b2c3d4e5f6..."
}
```

### 4.3 防重放攻击

| 机制 | 说明 |
|------|------|
| 单调递增序列号 | 每条指令 sequence 严格递增，机器人拒绝重复/倒退的序列号 |
| 时间窗口验证 | 指令 timestamp 与机器人当前时间差 > 5s 则拒绝 |
| 会话绑定 | HMAC 密钥绑定到会话，新会话生成新密钥 |

## 5. 紧急制动安全

### 5.1 双通道制动架构

```
操作端                          机器人端
──────                          ──────
        ┌─ 软件通道 (DataChannel) ─┐
        │  emergency_stop 指令     │
        │  (可能有延迟)            │
qooremote ───────────────────────► qoobrain → 安全 MCU → 制动
        │                          │
        │  [未来硬件] 独立硬件通道  │
        └─ CAN FD / GPIO 直连 ────► 安全 MCU → 制动
           (耳机插孔适配器 / USB)    (< 100ms 硬件级响应)
```

### 5.2 失联自动制动

```
连接超时 → 自动制动流程:

1. 机器人端持续监控 DataChannel 心跳
2. 超过 1s 未收到心跳 → 进入 DEGRADED 状态
3. 超过 3s 未收到心跳 → 自动触发紧急制动
4. 制动后保持安全姿态，等待操作员重连
5. 操作员重连后需手动确认"解除制动"
```

## 6. 数据安全

### 6.1 敏感数据处理

| 数据类型 | 传输 | 本地存储 | 云端存储 |
|----------|:----:|:------:|:------:|
| 认证 Token | TLS 加密 | 操作系统密钥链 | ❌ 不存储 |
| 控制指令 | TLS + HMAC | ❌ 不存储 (实时) | 审计日志 (可选) |
| 操作审计日志 | TLS | 加密存储 (AES-256) | 可选云同步 |
| 视频流 | SRTP | 录制文件可选加密 | ❌ 不存储 |
| 音频流 | SRTP | 录制文件可选加密 | ❌ 不存储 |
| 录制数据 | — | AES-256-GCM 可选加密 | ❌ 不存储 |
| 用户偏好 | — | 文件系统权限 | ❌ 不存储 |

### 6.2 录制数据加密

```python
# 录制数据可选加密
class EncryptedRecorder:
    def __init__(self, password: str):
        self.key = hashlib.pbkdf2_hmac(
            'sha256', password.encode(), salt, 100000
        )
        self.cipher = AES.new(self.key, AES.MODE_GCM)
```

## 7. 安全审计

### 7.1 审计日志格式

```json
{
  "event_id": "audit-20260630-001",
  "timestamp": "2026-06-30T14:30:22.123Z",
  "operator_id": "op-42",
  "robot_id": "qoobot-01",
  "session_id": "sess-xyz789",
  "event_type": "robot.control",
  "event_detail": {
    "command_type": "teleop.pose",
    "frame": "left_hand",
    "sequence": 12345
  },
  "result": "success",
  "client_ip": "192.168.1.100",
  "client_version": "0.1.0"
}
```

### 7.2 必须审计的事件

| 事件类型 | 说明 |
|----------|------|
| `session.start` | 遥控会话建立 |
| `session.end` | 遥控会话结束 |
| `mode.switch` | 运行模式切换 |
| `robot.control` | 所有控制指令 |
| `safety.emergency_stop` | 紧急制动触发/解除 |
| `auth.login` | 操作员登录 |
| `auth.login_failed` | 登录失败 |
| `config.change` | 安全相关配置变更 |
