## 1. 部署模式总览

qooremote 作为操作端软件，支持三种部署模式：

| 模式 | 平台 | 适用场景 | 部署方式 |
|------|------|----------|----------|
| 桌面安装 | Windows/macOS/Linux | 专业操作员站 | pip install / 安装包 |
| Web 托管 | Nginx / CDN | 浏览器快速访问 | 静态文件部署 |
| 移动发布 | iOS / Android | 移动巡检/对讲 | App Store / 应用市场 |

## 2. 桌面端部署

### 2.1 系统要求

| 操作系统 | 最低版本 | 推荐版本 |
|----------|----------|----------|
| Windows | Windows 10 (64-bit) | Windows 11 |
| macOS | macOS 13 Ventura | macOS 14 Sonoma |
| Linux | Ubuntu 22.04 | Ubuntu 24.04 |
| Python | 3.11 | 3.12 |

### 2.2 pip 安装

```bash
# 从 PyPI 安装 (未来)
pip install qooremote

# 从源码安装
git clone https://github.com/qoobot/qoobot.git
cd qoobot/qoobot-desktop/qooremote
pip install -e .

# 安装包含可选依赖（手柄/动捕/VR 支持）
pip install -e ".[gamepad,mocap,vr]"
```

### 2.3 桌面安装包构建

```bash
# Windows .exe 构建
pip install pyinstaller
pyinstaller --name qooremote --windowed --icon=resources/icons/app.ico console/main.py

# macOS .app 构建
pyinstaller --name qooremote --windowed --icon=resources/icons/app.icns console/main.py
cd dist && zip -r qooremote-macos.zip qooremote.app

# Linux AppImage
python setup.py bdist_appimage
```

### 2.4 配置文件生成

```bash
# 首次启动自动生成默认配置
qooremote-desktop
# → 创建 $HOME/.qooremote/ 目录及默认配置

# 或手动初始化
qooremote init --token YOUR_JWT_TOKEN --cloud wss://cloud.qoobot.dev/signaling
```

### 2.5 桌面端部署结构


## 3. Web 端部署

### 3.1 构建产物

```bash
cd qooremote/web
npm ci
npm run build
# → dist/ 目录包含静态文件
```

### 3.2 Nginx 部署

```nginx
# /etc/nginx/sites-available/qooremote-web
server {
    listen 443 ssl http2;
    server_name remote.qoobot.dev;

    ssl_certificate     /etc/ssl/certs/qoobot.pem;
    ssl_certificate_key /etc/ssl/private/qoobot.key;
    ssl_protocols       TLSv1.3;

    root /var/www/qooremote-web;
    index index.html;

    # SPA 路由回退
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存
    location /assets/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # 安全头
    add_header X-Frame-Options "DENY";
    add_header X-Content-Type-Options "nosniff";
    add_header Referrer-Policy "strict-origin-when-cross-origin";
}
```

### 3.3 Docker 部署

```dockerfile
# Dockerfile
FROM node:20-alpine AS build
WORKDIR /app
COPY web/package*.json ./
RUN npm ci
COPY web/ .
RUN npm run build

FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
COPY web/nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

```yaml
# docker-compose.yml
version: '3.8'
services:
  qooremote-web:
    build: .
    ports:
      - "8080:80"
    restart: unless-stopped
```

## 5. 运维管理

### 5.1 日志管理

```yaml
# qooremote 日志配置
logging:
  level: "INFO"                    # DEBUG | INFO | WARNING | ERROR
  format: "%(asctime)s [%(levelname)s] %(name)s: %(message)s"
  handlers:
    console:
      enabled: true
      level: "INFO"
    file:
      enabled: true
      path: "$HOME/.qooremote/logs/qooremote.log"
      max_size_mb: 50
      backup_count: 5
      level: "DEBUG"
```

### 5.2 健康检查


### 5.3 升级策略

| 模式 | 说明 |
|------|------|
| 桌面端 | `pip install --upgrade qooremote` 或下载新安装包 |
| Web 端 | 部署新构建产物，刷新即更新 |
| 移动端 | App Store / 应用市场自动更新 |
| 强更策略 | 后端 API 返回最低客户端版本，低于则弹窗强制更新 |

## 6. 监控与告警

### 6.1 应用指标

| 指标 | 采集方式 | 告警阈值 |
|------|----------|:------:|
| 连接延迟 | 信令心跳 RTT | > 500ms |
| 视频帧率 | WebRTC stats API | < 20 fps |
| 视频丢包率 | WebRTC stats API | > 5% |
| 桌面端 CPU | psutil | > 80% |
| 桌面端内存 | psutil | > 1.5 GB |
| 崩溃率 | 崩溃报告 | > 1% |

### 6.2 错误上报

```python
# 崩溃自动上报（需用户同意）
class CrashReporter:
    def collect_crash_info(self, exc_info) -> dict:
        return {
            "version": "0.1.0",
            "platform": platform.platform(),
            "python_version": sys.version,
            "traceback": traceback.format_exc(),
            "last_action": self.last_user_action
        }
```

## 7. 灾难恢复

### 7.1 故障场景与恢复

| 场景 | 影响 | 恢复措施 |
|------|------|----------|
| 操作端断电 | 失去控制 | 机器人端失联自动制动 |
| 网络中断 | 失去控制 | 自动重连 + 视频降质 |
| 桌面应用崩溃 | 控制中断 | 重启后自动重连 |
| Token 过期 | 无法连接 | 引导重新登录获取 Token |
| 磁盘空间满 | 录制失败 | 提前警告 + 自动清理旧录制 |
