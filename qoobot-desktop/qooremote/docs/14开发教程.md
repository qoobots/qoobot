## 2. 桌面端开发指南

### 2.1 项目架构理解

qooremote 采用 **MVVM 分层架构**：

```
UI 层 (PySide6 Widgets)
    ↕ Qt Signals/Slots
ViewModel 层 (QObject + 数据绑定)
    ↕ 异步调用
Service 层 (asyncio + WebSocket/WebRTC)
```

**关键原则**：
- UI 层不直接访问网络，通过 ViewModel 获取数据
- Service 层运行在独立线程 (asyncio event loop)，不阻塞 UI
- 数据模型 (models/) 是纯数据类，不包含业务逻辑

### 2.2 创建第一个面板

```python
# console/ui/panels/hello_panel.py
from PySide6.QtWidgets import QWidget, QVBoxLayout, QLabel
from PySide6.QtCore import Signal

class HelloPanel(QWidget):
    """示例面板：显示机器人名称和状态"""

    robot_name_changed = Signal(str)

    def __init__(self, robot_vm, parent=None):
        super().__init__(parent)
        self._robot_vm = robot_vm
        self._setup_ui()
        self._connect_signals()

    def _setup_ui(self):
        layout = QVBoxLayout(self)
        self._name_label = QLabel("未连接")
        self._status_label = QLabel("--")
        layout.addWidget(QLabel("机器人名称:"))
        layout.addWidget(self._name_label)
        layout.addWidget(QLabel("状态:"))
        layout.addWidget(self._status_label)

    def _connect_signals(self):
        # 监听 ViewModel 数据变化
        self._robot_vm.robot_name_changed.connect(
            self._name_label.setText
        )
        self._robot_vm.robot_status_changed.connect(
            self._update_status
        )

    def _update_status(self, status: dict):
        mode = status.get("mode", "unknown")
        battery = status.get("battery_percent", 0)
        self._status_label.setText(
            f"模式: {mode} | 电量: {battery}%"
        )
```

### 2.3 创建 ViewModel

```python
# console/ui/viewmodels/robot_vm.py
from PySide6.QtCore import QObject, Signal
from qooremote.console.core.models.robot_state import RobotState

class RobotViewModel(QObject):
    """机器人状态 ViewModel"""

    robot_name_changed = Signal(str)
    robot_status_changed = Signal(dict)
    connection_state_changed = Signal(str)

    def __init__(self, signaling_client, parent=None):
        super().__init__(parent)
        self._signaling = signaling_client
        self._state: RobotState | None = None

    def on_state_received(self, state_json: dict):
        """Service 层收到状态数据后调用此方法"""
        self._state = RobotState.from_json(state_json)
        # 通知 UI 更新
        self.robot_status_changed.emit({
            "mode": self._state.mode,
            "battery_percent": self._state.battery_percent
        })
```

import asyncio
from qooremote.console.core.signaling.client import SignalingClient

async def connect_example():
    client = SignalingClient()

    # 连接
    connected = await client.connect(
        "wss://cloud.qoobot.dev/signaling",
        "YOUR_JWT_TOKEN"
    )

    if connected:
        # 获取机器人列表
        robots = await client.list_robots()
        for robot in robots:
            print(f"发现机器人: {robot.name} ({robot.status})")

        # 建立遥控会话
        session = await client.request_session(
            "qoobot-01",
            mode="monitor"
        )
        print(f"会话已建立: {session.session_id}")

        # ... WebRTC 连接建立后，通过 DataChannel 收发数据 ...

        # 结束会话
        await client.end_session(session.session_id)

    await client.disconnect()

<script setup lang="ts">
import { computed } from 'vue'

const props = defineProps<{
  percentage: number
  charging: boolean
  estimatedRuntime: number
}>()

const batteryColor = computed(() => {
  if (props.percentage > 50) return '#22C55E'
  if (props.percentage > 20) return '#EAB308'
  return '#EF4444'
})
</script>

### 3.3 使用 WebSocket Composable

```typescript
// web/src/composables/useWebSocket.ts
import { ref, onMounted, onUnmounted } from 'vue'

export function useWebSocket(url: string, token: string) {
  const status = ref<'connecting' | 'connected' | 'disconnected'>('disconnected')
  const latency = ref(0)
  let ws: WebSocket | null = null

  function connect() {
    ws = new WebSocket(url)
    status.value = 'connecting'

    ws.onopen = () => {
      status.value = 'connected'
      // 发送认证消息
      ws!.send(JSON.stringify({ type: 'auth', token }))
    }

    ws.onmessage = (event) => {
      const msg = JSON.parse(event.data)
      // 处理不同类型的消息
      handleMessage(msg)
    }

    ws.onclose = () => {
      status.value = 'disconnected'
      // 自动重连
      setTimeout(connect, 3000)
    }
  }

  function send(data: object) {
    if (ws?.readyState === WebSocket.OPEN) {
      ws.send(JSON.stringify(data))
    }
  }

  function disconnect() {
    ws?.close()
  }

  onMounted(() => connect())
  onUnmounted(() => disconnect())

  return { status, latency, send, disconnect }
}
```

### 5.3 常见问题

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| 无法连接 qoocloud | Token 过期 / 网络不通 | 检查 Token 有效期，ping cloud 地址 |
| 视频黑屏 | WebRTC ICE 协商失败 | 检查 STUN/TURN 配置，查看 webrtc-internals |
| 手柄无响应 | 驱动未安装 / 权限不足 | 检查设备管理器，Linux 需 udev 规则 |
| 录制文件损坏 | 磁盘空间不足 / 未正常关闭 | 检查磁盘空间，确保调用 stop() |

## 6. 贡献指南

### 6.1 代码提交规范

```
提交信息格式: <type>(<scope>): <description>

类型:
  feat:     新功能
  fix:      修复 Bug
  docs:     文档变更
  style:    代码格式（不影响功能）
  refactor: 重构
  test:     测试变更
  chore:    构建/工具变更

示例:
  feat(teleop): 添加 Xbox 手柄力反馈支持
  fix(video): 修复高延迟下视频花屏问题
  docs(readme): 更新安装说明
```

### 6.2 Pull Request 流程

1. Fork 仓库，创建功能分支 `feat/my-feature`
2. 编写代码 + 测试
3. 运行 `pytest` 确保测试通过
4. 运行 `ruff check . && mypy console/` 确保静态检查通过
5. 提交 PR，描述变更内容
6. 等待 Code Review
