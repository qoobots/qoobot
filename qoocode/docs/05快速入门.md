## 2. 创建第一个技能

### 初始化项目

```bash
qoo init my-first-skill
cd my-first-skill
```

这会生成如下项目结构：

```
my-first-skill/
├── qoo.toml              # 项目配置
├── README.md
├── .gitignore
├── __init__.py
├── src/
│   ├── __init__.py
│   └── skill.py          # 技能主代码
├── tests/
│   ├── __init__.py
│   └── test_skill.py     # 单元测试
├── config/
│   └── default.yaml       # 配置文件
└── data/                  # 数据目录
```

### 编写技能代码

编辑 `src/skill.py`：

```python
"""my-first-skill - A simple QooBot skill."""

from qoobot_sdk import QooSkill, SkillContext


class MyFirstSkill(QooSkill):
    """A skill that makes the robot wave."""

    def __init__(self):
        super().__init__(
            name="my-first-skill",
            version="0.1.0",
        )
        self._counter = 0

    async def on_start(self, ctx: SkillContext) -> None:
        ctx.logger.info("👋 Starting my-first-skill!")
        # Initialize joints to home position
        await ctx.control.move_joints({
            "right_shoulder_pitch": 0.0,
            "right_elbow": 0.0,
        })

    async def on_tick(self, ctx: SkillContext) -> None:
        self._counter += 1
        # Wave every 2 seconds
        if self._counter % 200 == 0:
            ctx.logger.info(f"Waving! Cycle: {self._counter}")

    async def on_stop(self, ctx: SkillContext) -> None:
        ctx.logger.info("Skill stopped gracefully")
```

### 运行测试

```bash
qoo test
```

### 在仿真中运行

```bash
qoo run --sim mujoco --scene home
```

## 3. SDK 核心概念

### 技能生命周期

```
CREATED → STARTING → RUNNING → STOPPING → STOPPED
                      ↕
                   PAUSED
```

| 方法 | 触发时机 | 用途 |
|------|---------|------|
| `on_start()` | 技能启动 | 初始化、连接服务 |
| `on_tick()` | 每个控制周期 (~100Hz) | 主逻辑循环 |
| `on_pause()` | 技能暂停 | 保存状态 |
| `on_resume()` | 技能恢复 | 恢复状态 |
| `on_stop()` | 技能停止 | 清理资源 |

### SkillContext 可用数据

```python
async def on_tick(self, ctx: SkillContext):
    # 传感器数据
    rgb = ctx.camera_rgb        # RGB 图像
    depth = ctx.camera_depth    # 深度图
    points = ctx.lidar_points   # LiDAR 点云
    imu = ctx.imu_data          # IMU 读数
    joints = ctx.joint_states   # 关节状态

    # 日志
    ctx.logger.info(f"Frame: {ctx.tick_count}")
```

### 控制机器人

```python
from qoobot_sdk.control import (
    JointGroupCommand, EndEffectorCommand, Pose
)

# 关节控制
cmd = JointGroupCommand.position_control({
    "right_shoulder_pitch": 0.5,
    "right_elbow": -0.3,
})

# 末端执行器
ee_cmd = EndEffectorCommand(
    name="right_hand",
    pose=Pose(position=(0.5, 0.2, 0.8)),
)
```

## 4. 项目类型

### 技能项目 (`qoo init -t skill`)

适用于：感知任务、操作任务、交互行为

### 服务项目 (`qoo init -t service`)

适用于：导航引擎、语音服务、诊断服务

### 模型项目 (`qoo init -t model`)

适用于：视觉模型、策略模型、感知模型

## 5. CLI 命令速查

| 命令 | 说明 |
|------|------|
| `qoo init <name>` | 创建新项目 |
| `qoo init <name> -t service` | 创建服务项目 |
| `qoo init <name> -t model` | 创建模型项目 |
| `qoo build` | 编译项目 |
| `qoo build --release` | Release 模式编译 |
| `qoo run` | 仿真运行 |
| `qoo run --sim isaac` | 指定仿真器 |
| `qoo run --headless` | 无界面模式 |
| `qoo test` | 运行测试 |
| `qoo test -v` | 详细输出 |
| `qoo test -c` | 覆盖率报告 |
| `qoo doctor` | 环境诊断 |
