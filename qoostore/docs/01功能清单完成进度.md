## 定位

qoostore 是 QooBot 的**机器人 App Store**，采用**端云协同架构**，连接开发者与用户，为机器人技能提供发现、分发、商业化的完整闭环。

| 维度 | 说明 |
|------|------|
| **定位** | 机器人 App Store：连接开发者与用户，技能发现、分发、商业化 |
| **架构** | 端云协同：`qoostore-cloud`（云端商店后端）+ `qoostore-edge`（机器人端运行环境） |
| **对标** | Apple App Store（云端商店 + iOS 端 App Store 客户端） |
| **依赖** | qoodev（SDK）、qoocloud（分发/计费）、qooauth（认证/权限） |
| **被依赖** | qoocommunity |

### 端云分工

| 组件 | 运行位置 | 技术栈 | 职责 |
|------|---------|--------|------|
| qoostore-cloud | 云端 | Java 21 + Spring Cloud | 技能商店后端、发布审核、计费、分发、运营分析 |
| qoostore-edge | 机器人端 | C++17 + Python 3.11 | 技能安装管理、沙箱隔离、权限管控、运行时监控 |

---

## 设计理念

1. **开发者优先**：从 SDK 到审核到变现，全链路降低开发者摩擦
2. **信任与安全**：严格审核 + 沙箱运行，用户下载即用、无需担心安全
3. **智能推荐**：基于机器人型号、使用场景、用户偏好的个性化推荐
4. **公平变现**：透明分成、灵活定价、开发者收益可预期

### 🖥️ qoostore-edge 端侧运行环境 (C++17)

| 模块 | 功能 |
|------|------|
| `skill_manager` | 技能安装管理：安装/卸载/更新/启用/停用/版本管理/仓库管理 |
| `sandbox` | 沙箱隔离：cgroups v2 资源配额/OverlayFS 文件系统隔离/Network Namespace 网络隔离 |
| `permission` | 权限管控：声明式权限模型(NORMAL/DANGEROUS/SIGNATURE)/授权/撤销/持久化 |
| `runtime_monitor` | 运行时监控：CPU/内存/网络统计/异常检测/崩溃收集/定期上报(5s) |
| `skill_ipc` | 技能间通信：Intent 广播/事件订阅/数据共享(ContentProvider)/安全IPC(Binder) |
| `skill_downloader` | 下载器：断点续传/增量更新/包校验(SHA-256)/签名验证 |

**总计：38 个 API 端点 + 6 个端侧模块**
