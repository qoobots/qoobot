## 2. CDN 分发架构

```
用户下载技能
    │
    ▼
DNS 智能解析 (就近接入)
    │
    ├── 中国 → 阿里云 CDN
    ├── 北美 → CloudFront
    ├── 欧洲 → CloudFront / CloudFlare
    └── 其他 → CloudFlare
        │
        ▼
    边缘节点缓存命中?
    ├── 是 → 直接返回
    └── 否 → 回源 MinIO → 缓存 → 返回
```

## 3. CI/CD

```
Git Push
    │
    ▼
├── qoostore-cloud: Maven build → Docker → K8s deploy
└── qoostore-edge: CMake build → cross-compile → .deb → OTA channel
```

### 3.1 发布策略

| 组件 | 策略 | 验证 |
|:-----|------|------|
| 云端微服务 | 滚动更新 (maxUnavailable: 1) | 健康检查 + 金丝雀 |
| 端侧组件 | OTA 灰度 (5%→20%→100%) | 崩溃率监控 |
| 技能审核模型 | 在线热更新 (配置中心) | A/B 测试 |

## 6. 运维手册

### 6.1 日常检查

- 审核队列是否积压
- 支付成功率是否正常
- CDN 带宽/命中率
- 技能包存储用量
- 端侧崩溃率趋势

### 6.2 故障处理

| 故障 | 处理 |
|:-----|------|
| 商店首页加载慢 | 检查 DB 慢查询、Redis 命中率、CDN 状态 |
| 支付回调失败 | 人工补单、检查签名密钥、对账 |
| 技能审核积压 | 增加自动审核比例、临时增加人工审核 |
| CDN 回源率高 | 预热热门技能包、检查缓存策略 |
| 端侧安装失败率高 | 检查签名证书、CDN 可达性、磁盘空间 |
