## 2. 模块职责

### 2.1 qoostore-cloud

| 微服务 | 职责 |
|:-------|------|
| Store Service | 技能 CRUD、搜索、分类、评价管理 |
| Publish Service | 技能上传、版本管理、审核流程 |
| Payment Service | 支付集成、订单管理、结算 |
| Review Service | 自动扫描、人工审核、举报处理 |
| Recommend Service | 个性化推荐、编辑精选、热搜 |
| Developer Service | 开发者门户、API 密钥、数据分析 |
| Analytics Service | 下载/收入/评分/崩溃统计 |
| CDN Service | 技能包 CDN 分发管理 |

### 2.2 qoostore-edge

| 模块 | 职责 |
|:-----|------|
| Installer | 技能包下载、验签、解压、注册 |
| Sandbox | gVisor/seccomp 沙箱运行环境 |
| Permission Manager | 声明式权限检查、动态授权 |
| Updater | 自动更新检查、增量下载 |
| Monitor | 运行时监控、异常终止、用量统计 |

## 3. 技能包结构

```
my_skill.qooskill (ZIP 压缩包)
├── manifest.yaml          # 技能元数据
│   ├── name, version, author
│   ├── permissions (所需权限列表)
│   ├── dependencies (依赖的库/技能)
│   ├── min_robot_version
│   └── supported_models (QooBot S/L/Pro)
├── skill.py               # 技能主入口
├── models/                # 可选 AI 模型
├── assets/                # 资源文件 (图标/音频/UI)
├── config.yaml            # 默认配置
└── signature.sig          # Ed25519 签名
```

## 4. 数据流

### 4.1 技能安装流程

```
用户点击安装
    │
    ▼
qoostore-edge: 从 CDN 下载 .qooskill
    │
    ▼
验签 (Ed25519): 签名有效?
    ├── 否 → 拒绝安装 + 报告异常
    └── 是 → 解压 → 解析 manifest
              │
              ▼
        权限检查: 用户授权必需权限?
              │
              ▼
        依赖解析: 依赖满足?
              │
              ▼
        注册到 qoobrain Skill Manager
              │
              ▼
        安装完成 ✓
```

### 4.2 技能运行沙箱

```
技能启动请求
    │
    ▼
Permission Manager: 检查权限声明
    │
    ▼
Sandbox: 创建隔离环境
  · CPU 限额 (cgroups)
  · 内存限额 (cgroups)
  · 网络隔离 (netns)
  · 文件系统隔离 (chroot/overlayfs)
  · 系统调用过滤 (seccomp)
    │
    ▼
技能运行 → Monitor 持续监控
    │
    ├── 正常 → 技能完成
    └── 异常 (超资源/恶意行为) → 强制终止 + 报告
```
