## 3. 关键测试场景

### 3.1 技能审核管道测试

```java
@Test
void testFullReviewPipeline() {
    // 1. 开发者上传技能
    var skill = publishService.upload(validSkillPackage);
    assertThat(skill.getStatus()).isEqualTo(PENDING_REVIEW);
    
    // 2. 自动扫描
    var scanResult = reviewService.autoScan(skill.getId());
    assertThat(scanResult.getVulnerabilities()).isEmpty();
    assertThat(scanResult.getPassed()).isTrue();
    
    // 3. 人工审核通过
    reviewService.approve(skill.getId(), "reviewer_001", "审核通过");
    assertThat(skill.getStatus()).isEqualTo(PUBLISHED);
}
```

### 3.2 沙箱隔离测试

```cpp
TEST(Sandbox, SkillCantAccessSystemFiles) {
    Sandbox sandbox;
    sandbox.configure({
        .cpu_limit = 1.0,
        .mem_limit = 512MB,
        .filesystem_readonly = {"/opt/qoo/system/"},
        .filesystem_writable = {"/data/skills/test_skill/"},
        .seccomp_profile = "skill_default"
    });
    
    auto result = sandbox.execute([]() {
        // 尝试读取系统文件
        return std::filesystem::exists("/opt/qoo/system/secret.key");
    });
    
    // 应该返回 false（被 seccomp/namespace 拦截）
    EXPECT_FALSE(result);
}
```

## 7. 混沌工程

| 场景 | 注入方式 | 预期 |
|:-----|---------|------|
| 商店服务宕机 | Kill Pod | 自动恢复、CDN 缓存兜底 |
| 支付服务故障 | 网络延迟 | 订单不丢失、自动对账 |
| CDN 故障 | 禁用 CDN | 回源 MinIO 直连 |
| Redis 缓存故障 | Kill Redis | DB 兜底 |
| 数据库主库故障 | 主从切换 | 服务降级、只读模式 |
