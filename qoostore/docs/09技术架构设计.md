## 2. 关键技术决策

### 2.1 为什么选择 gVisor 沙箱

| 对比 | gVisor | Docker | seccomp alone |
|:-----|:-----:|:-----:|:------------:|
| 安全隔离 | ✅ 强 (用户态内核) | 🟡 中 | 🟡 中 |
| 性能开销 | ~10% | ~0% | ~0% |
| 资源限制 | ✅ cgroup 集成 | ✅ | ✅ |
| 启动速度 | <100ms | ~500ms | <10ms |
| 适合场景 | 不可信代码执行 | 可信代码 | 已知程序 |

**决策**：gVisor 作为默认沙箱（高安全场景），seccomp+cgroup 作为轻量级选项（高性能场景）。

### 2.2 技能签名体系

```
开发者本地签名:
  Ed25519 私钥 → 签名技能包 → signature.sig

云端验证:
  公钥(注册时上传) → 验证 signature.sig → 云端二次签名(平台证书)

端侧验证链:
  .qooskill → 平台证书验证 → 开发者签名验证 → 安装
```

## 4. 数据模型

### 4.1 技能核心表

| 表 | 关键字段 | 说明 |
|:---|---------|------|
| skills | id, name, developer_id, category, status, price_model | 技能主表 |
| skill_versions | id, skill_id, version, changelog, package_url, signature | 版本管理 |
| skill_reviews | id, skill_id, user_id, rating, content, status | 用户评价 |
| developer_apps | id, developer_id, name, api_key, webhook_url | 开发者应用 |
| orders | id, user_id, skill_id, amount, status, payment_method | 订单 |

### 4.2 端侧数据

| 存储 | 内容 |
|:-----|------|
| SQLite | 已安装技能列表、版本、配置 |
| 文件系统 | 技能包、沙箱数据目录 |
| 内存 | 运行时权限状态 |
