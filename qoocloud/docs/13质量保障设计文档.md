## 5. 混沌工程

### 5.1 故障注入场景

| 故障 | 注入方式 | 预期行为 |
|:-----|---------|---------|
| 单 Pod 杀死 | `kubectl delete pod` | 自动重新调度，无请求失败 |
| GPU 节点宕机 | 节点关机 | 推理请求路由到其他 GPU 节点 |
| MySQL 主库故障 | MGR 主切换 | <30s 切换完成，服务降级 |
| Redis 节点故障 | 哨兵切换 | 缓存 miss，DB 兜底 |
| Kafka Broker 故障 | Broker 下线 | 消息不丢失，消费者 rebalance |
| 网络延迟 | Istio fault injection | 超时重试、熔断生效 |
| 区域故障 | DNS 切换 | 流量迁移到备 Region |

### 5.2 混沌实验定义

## 6. 代码规范

### 6.1 Java 规范

- Google Java Style Guide + Checkstyle 强制执行
- 禁止使用 `System.out.println`（使用 SLF4J）
- 所有公共 API 必须有 JavaDoc
- Optional 替代 null 返回
- Lombok 减少样板代码（但禁止 @Data 在 JPA Entity）

### 6.2 架构约束 (ArchUnit)

```java
@Test
void services_should_not_depend_on_controllers() {
    noClasses()
        .that().resideInAPackage("..service..")
        .should().dependOnClassesThat()
        .resideInAPackage("..controller..")
        .check(classes);
}
```
