# qoocloud — 安全架构设计

> 版本：v0.1 | 最后更新：2026-06-29 | 状态：Draft | 子项目：qoocloud（云端服务）

---

## 1. 安全设计原则

1. **零信任架构**：不信任任何网络边界，所有通信都需认证与加密
2. **纵深防御**：多层安全控制——网络→应用→数据→基础设施
3. **最小权限**：每个服务、每个用户仅获得最小必要权限
4. **安全左移**：安全审查在 CI 阶段自动执行

## 2. 威胁模型

```
┌──────────────────────────────────────────────────────────┐
│                      威胁表面                              │
├──────────────────┬───────────────────────────────────────┤
│ 🌐 公网 API       │ DDoS、凭证泄露、注入攻击、API 滥用       │
│ 📡 MQTT 通道      │ 设备仿冒、消息窃听/篡改、重放攻击        │
│ 🧠 推理 API       │ Prompt 注入、模型窃取、推理数据泄露      │
│ 🗄️ 数据存储       │ 数据泄露、未授权访问、SQL 注入            │
│ ☁️ 云基础设施      │ 容器逃逸、供应链攻击、密钥泄露           │
│ 🔧 运维通道       │ SSH 未授权、K8s API Server 暴露         │
└──────────────────┴───────────────────────────────────────┘
```

### 2.1 关键威胁与缓解

| 威胁 | 严重性 | 缓解措施 |
|:-----|:-----:|---------|
| 设备仿冒 | 🔴 高 | X.509 证书 + mTLS + 设备指纹 |
| Prompt 注入攻击 | 🔴 高 | 输入过滤、角色限定、输出审查 |
| 模型窃取 | 🔴 高 | API 限流、水印、访问审计 |
| 推理数据泄露 | 🔴 高 | 传输加密、不存储推理数据、租户隔离 |
| DDoS 攻击 | 🟠 中 | CDN + WAF + 自适应限流 |
| 容器逃逸 | 🟠 中 | 非 root 运行、seccomp、AppArmor |
| SQL 注入 | 🟡 低 | ORM 参数化查询、WAF 规则 |

## 3. 身份认证体系

### 3.1 设备认证

```
设备首次开机
    │
    ▼
本地生成 Ed25519 密钥对
    │
    ▼
向 qooauth 请求设备证书 (CSR)
    │
    ▼
qooauth 签发 X.509 证书 (有效期 2 年)
    │
    ▼
后续通信使用 mTLS
    │
    ▼
定期自动续签证书
```

### 3.2 用户/开发者认证

```
用户/开发者
    │
    ▼
OAuth2 Authorization Code Flow + PKCE
    │
    ▼
JWT Access Token (有效期 1h)
    │
    ▼
Refresh Token (有效期 30d, 可轮换)
    │
    ▼
API 调用时 Bearer Token
```

### 3.3 服务间认证

```
微服务 A                      微服务 B
    │                            │
    │──── mTLS (Istio 自动) ────▶│
    │                            │
    │──── JWT Service Token ────▶│
    │     (短期, 自动轮换)        │
```

## 4. 数据安全

### 4.1 数据分类

| 级别 | 类型 | 存储策略 | 加密策略 |
|:----:|------|---------|---------|
| L0 (绝密) | 用户密码、密钥材料 | HSM/Vault | AES-256-GCM + HSM |
| L1 (机密) | 个人身份信息、设备证书 | 加密数据库 | AES-256-GCM |
| L2 (敏感) | 设备遥测、推理日志 | 加密存储 | AES-256-CTR |
| L3 (内部) | 系统日志、性能指标 | 普通存储 | 传输加密 |
| L4 (公开) | 公开模型、文档 | CDN/对象存储 | 无 |

### 4.2 推理数据保护

- 推理输入数据不持久化存储（内存处理，处理完即销毁）
- 推理日志脱敏（去除 PII、仅保留匿名化特征）
- 用户可选择端到端加密模式（加密 payload，云端不可解密）
