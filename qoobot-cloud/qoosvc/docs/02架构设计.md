# 02 — 架构设计

> 版本：v0.1 | 最后更新：2026-06-27 | 状态：Draft
> 子项目：qoosvc（系统服务）| 架构：模块化 C++/Python 混合运行时
> 对标：iOS Core Services / Android System Services 层

---

## 1. 设计目标与原则

### 1.1 核心设计目标

| 目标 | P0 指标 | P1 目标 | 说明 |
|------|---------|---------|------|
| 语音响应延迟 | < 300ms (ASR) | < 150ms | 唤醒词→理解→响应端到端延迟 |
| 导航成功率 | > 99% | > 99.9% | 室内自主导航任务完成率 |
| SLAM 建图精度 | < 5cm | < 2cm | 全局地图一致性误差 |
| 自诊断覆盖 | > 90% 硬件 | > 95% | POST 自检覆盖的硬件组件比例 |
| 多机器人协同 | 5 台 | 50 台 | 同时协作的机器人数量 |
| 服务可用性 | 99.9% | 99.99% | 系统服务层可用性 |

### 1.2 设计原则

- **模块化隔离**：每个子系统独立编译为共享库(.so/.dylib)，通过 qoobrain IPC 总线通信
- **渐进式能力**：基础能力（语音唤醒/导航）即使离线也必须可用，高级能力（多语言翻译/云增强）按需加载
- **资源感知**：根据当前 CPU/内存/电量动态降级服务等级，保障核心功能
- **安全优先**：语音/视觉数据本地处理优先，云端传输需用户显式授权
- **实时性保证**：控制相关服务（导航/避障）运行在 RT 优先级，交互相关服务运行在普通优先级
- **服务可组合**：上层技能可通过统一 API 组合调用多个系统服务

## 3. 技术选型

### 3.1 语言与框架

| 组件 | 技术选型 | 理由 |
|------|---------|------|
| 核心语言 | C++17 (导航/控制/实时服务) + Python 3.11 (AI/诊断/交互) | 实时性要求高的用 C++，AI/工具链用 Python |
| 构建系统 | CMake 3.22+ + setuptools | 与 qoobrain/qoobody 统一构建体系 |
| 通信框架 | DDS (Fast DDS / Cyclone DDS) | qoobrain 标准 IPC，发布/订阅模式 |
| 共享内存 | iceoryx2 | 零拷贝传感器数据传输 |
| SLAM 框架 | GTSAM + g2o | 因子图优化 + 图优化后端 |
| 路径规划 | OMPL + 自研 | 通用运动规划库 + 定制化导航算法 |
| ASR 引擎 | Whisper.cpp (端侧) + 云端模型 | 轻量端侧推理 + 高精度云端增强 |
| NLU 引擎 | ONNX Runtime + 自研意图槽位模型 | 跨平台推理 + 定制化对话理解 |
| TTS 引擎 | Piper TTS (端侧) + VITS (云端) | 轻量离线合成 + 高品质在线合成 |
| 人脸识别 | InsightFace (ONNX) | 端侧推理，无需云端 |
| 服务配置 | YAML | 人类可读，易于调试 |
| 服务监控 | Prometheus + Grafana | 与 qoobrain 观测体系统一 |
| 日志 | spdlog (C++) + structlog (Python) | 高性能结构化日志 |

### 3.2 与 qoobrain 的关系

| 维度 | qoobrain | qoosvc |
|------|---------|--------|
| 定位 | 机器人操作系统内核 | 系统级服务/能力 |
| 层级 | 底层（感知/认知/控制） | 中间层（面向任务的能力） |
| 抽象级别 | 传感器数据 / 控制指令 | 语义命令 / 任务描述 |
| 用户可见 | 否（开发者 API） | 是（"Hey QooBot, 去厨房"） |
| 示例 | 发布 `/camera/rgb` 图像 | 提供 `FaceRecognition.identify()` |

## 4. 核心数据流

### 4.1 语音交互全链路

```
[麦克风阵列] → qoobody HAL → qoobrain 音频感知
                                    │
                                    ▼  PCM 音频流
┌───────────────────────────────────────────────────────┐
│ svc_voice                                             │
│                                                       │
│  WakeWord Detector ──→ "Hey QooBot" 检测               │
│         │ 唤醒后 5 秒窗口                              │
│         ▼                                             │
│  ASR Engine (Whisper.cpp) ──→ "去厨房拿一瓶水"          │
│         │ 文本                                        │
│         ▼                                             │
│  NLU Engine ──→ intent: navigate_to                   │
│                  slots: {location: "厨房"}             │
│         │                                             │
│         ├──→ svc_navigation: navigate_to("厨房")      │
│         │                                             │
│         ▼                                             │
│  TTS Engine (Piper) ──→ "好的，正在前往厨房"            │
│         │                                             │
└─────────┼─────────────────────────────────────────────┘
          ▼ 音频输出
   [扬声器] ← qoobody HAL
```

### 4.2 自主导航流程

```
[用户指令/任务] → svc_navigation
                       │
                       ▼
              ┌─────────────────┐
              │  Global Planner  │ ← svc_spatial 提供地图
              │  (A*/Dijkstra)  │
              └────────┬────────┘
                       │ 全局路径
                       ▼
              ┌─────────────────┐
              │  Local Planner   │ ← qoobrain 实时定位
              │  (TEB/DWA)      │ ← qoobrain 障碍物检测
              └────────┬────────┘
                       │ 速度指令 (v, ω)
                       ▼
              ┌─────────────────┐
              │  Collision Check │ ← qoobrain 安全监护
              └────────┬────────┘
                       │ 安全速度指令
                       ▼
              qoobody 电机控制 → 轮毂电机
```

### 4.3 自诊断流程

```
[开机/定时触发] → svc_diagnostics
                       │
                       ▼
              ┌─────────────────┐
              │   POST 自检      │
              │  · 传感器连通性  │ ← qoobody HAL
              │  · 电机编码器    │
              │  · 电池电压      │
              │  · IMU 校准      │
              │  · 通信链路      │
              └────────┬────────┘
                       │ 诊断结果
                       ▼
              ┌─────────────────┐
              │  健康报告生成    │
              │  · 评分 0-100   │
              │  · 异常列表     │
              │  · 维护建议     │
              └────────┬────────┘
                       │
          ┌────────────┼────────────┐
          ▼            ▼            ▼
     [本地日志]   [LED 警告]   [云端上报]
```
