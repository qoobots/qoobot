## 1. 设计目标与原则

### 1.1 核心设计目标

| 目标 | P0 指标 | P1 目标 | 说明 |
|------|---------|---------|------|
| 登录可用性 | 99.99% | 99.999% | 认证服务 SLA，影响所有上层服务 |
| 登录延迟 | < 500ms | < 200ms | 端到端认证时间 |
| Token 签发 QPS | > 10,000 | > 50,000 | JWT/Token 签发吞吐量 |
| 密钥安全 | FIPS 140-3 | 硬件 HSM | 密钥存储与操作安全等级 |
| 多设备并发 | 1,000,000 | 10,000,000 | 同时在线设备数 |
| 数据加密 | AES-256-GCM | 后量子密码 | 传输与存储加密算法 |

### 1.2 设计原则

- **零信任架构**：不信任任何网络/设备/用户，每次请求均需认证与授权
- **隐私优先**：默认最小化数据采集，本地处理优先，云端同步需显式授权
- **密钥分层**：根密钥（HSM）→ 服务密钥 → 用户密钥 → 会话密钥，逐层派生
- **防御纵深**：网络层（TLS/mTLS）→ 应用层（JWT/签名）→ 数据层（E2E 加密）
- **无缝体验**：单点登录（SSO）、设备间信任传递、Passkey 无密码认证
- **可审计性**：全量操作日志、不可篡改审计链、合规报告自动生成

### 2.2 Spring Cloud 微服务职责

| 微服务 | Spring Boot 模块 | 职责 | 关键依赖 |
|--------|-----------------|------|---------|
| auth-service | qooauth-auth | 注册/登录/登出、JWT 签发/验证、密码策略 | Spring Security, jjwt, Nimbus JOSE |
| authz-service | qooauth-authz | RBAC/ABAC、OAuth2 授权服务器、权限校验 | Spring Security OAuth2 AS |
| user-service | qooauth-user | 用户 CRUD、资料管理、账户恢复/注销 | Spring Data JPA, MyBatis-Plus |
| device-service | qooauth-device | 设备注册/绑定/证书/远程锁死/定位 | Bouncy Castle, Spring Data JPA |
| session-service | qooauth-session | 会话生命周期、Redis 共享会话、SSO | Spring Session Data Redis |
| key-service | qooauth-key | 密钥派生(HKDF)、证书签发(X.509)、HSM 集成 | Bouncy Castle, SunPKCS11 |
| audit-service | qooauth-audit | 审计日志采集/查询、合规报告生成 | Spring Data JPA, MyBatis-Plus |
| threat-detection | qooauth-threat | 异常登录检测、设备指纹、撞库防护 | Sentinel, Redis |
| gateway | qooauth-gateway | API 网关：限流/路由/JWT 预验签/日志 | Spring Cloud Gateway |

## 4. 核心数据流

### 4.1 用户登录流程

```
用户 → 接入层 → auth-service → user-service → key-service → session-service → JWT
  │       │         │              │             │              │             │
  │ POST  │ 验证    │  查询        │  验证        │  签名         │  存储       │  返回
  │/login │ 凭证    │  Feign 调用  │  密码        │  JWT          │  Redis      │  Token
  ▼       ▼         ▼              ▼             ▼              ▼             ▼
[客户端] → [Gateway] → [auth-service] → [user-service] → [key-service] → [session] → [响应]
                │            │                                    │
                │            ▼                                    ▼
                │      [threat-detection]                   [audit-service]
                │       Sentinel 风险评估/限流               RocketMQ 异步审计
```

### 4.2 设备激活流程

```
新设备 → 扫码/蓝牙 → device-service → key-service → user-service → 安全启动链
  │         │            │               │              │             │
  │ 出厂    │  配对      │  验证          │  签发         │  绑定        │  验证
  │ 证书   │  码        │  X.509        │  设备证书     │  账户        │  固件签名
  ▼         ▼            ▼               ▼              ▼             ▼
[设备] → [配对流] → [device-service] → [key-service] → [user-service] → [Boot验证]
```

### 5.2 高可用设计

- **Spring Cloud 微服务**：无状态，水平扩展，K8s HPA 自动伸缩
- **Nacos 集群**：3 节点部署，AP 模式
- **Sentinel 控制台**：独立部署，规则持久化至 Nacos
- **数据库**：PostgreSQL 主从复制 + 自动故障切换
- **Redis**：Cluster 模式，跨 AZ 部署
- **HSM**：双活部署，自动切换
- **灾难恢复**：跨区域异步复制，RPO < 5min，RTO < 30min

### 5.3 容器化

```dockerfile
# 基础镜像
FROM eclipse-temurin:21-jdk-alpine

# Spring Boot Fat JAR
COPY target/qooauth-*.jar /app/app.jar

# 启动参数：虚拟线程 + G1GC
ENTRYPOINT ["java", \
    "--enable-preview", \
    "-XX:+UseG1GC", \
    "-XX:MaxGCPauseMillis=200", \
    "-XX:+UseStringDeduplication", \
    "-jar", "/app/app.jar"]
```
