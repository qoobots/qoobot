# 02 — 架构设计

> 版本：v0.3 | 最后更新：2026-06-27 | 状态：Draft
> 子项目：qooauth（账号与安全）| 架构：单一 Spring Boot 微服务
> 对标：Apple ID + Keychain + MDM 统一身份基础设施

---

## 1. 设计目标与原则

### 1.1 核心设计目标

| 目标 | P0 指标 | P1 目标 | 说明 |
|------|---------|---------|------|
| 登录可用性 | 99.99% | 99.999% | 认证服务 SLA，影响所有上层服务 |
| 登录延迟 | < 500ms | < 200ms | 端到端认证时间 |
| Token 签发 QPS | > 10,000 | > 50,000 | JWT/Token 签发吞吐量 |
| 密钥安全 | FIPS 140-3 | 硬件 HSM | 密钥存储与操作安全等级 |
| 多设备并发 | 1,000,000 | 10,000,000 | 同时在线设备数 |
| 数据加密 | AES-256-GCM | 后量子密码 | 传输与存储加密算法 |

### 1.2 设计原则

- **零信任架构**：不信任任何网络/设备/用户，每次请求均需认证与授权
- **隐私优先**：默认最小化数据采集，本地处理优先，云端同步需显式授权
- **密钥分层**：根密钥（HSM）→ 服务密钥 → 用户密钥 → 会话密钥，逐层派生
- **防御纵深**：网络层（TLS/mTLS）→ 应用层（JWT/签名）→ 数据层（E2E 加密）
- **无缝体验**：单点登录（SSO）、设备间信任传递、Passkey 无密码认证
- **可审计性**：全量操作日志、不可篡改审计链、合规报告自动生成
- **单体优先**：单一微服务部署，内部 package 分层，避免过早分布式化。当单服务 QPS 超过 50,000 或团队规模超过 50 人时再考虑拆分

### 2.3 各 Service 职责

| Service | 职责 | 关键依赖 |
|---------|------|---------|
| AuthService | 注册/登录/登出、JWT 签发/验证、TOTP/Passkey、密码策略 | JwtTokenProvider, PasswordEncoder |
| UserService | 用户 CRUD、资料管理、账户恢复/注销 | UserRepository |
| DeviceService | 设备注册/绑定/证书/远程锁死/定位 | Bouncy Castle, DeviceRepository |
| OAuthService | OAuth 2.0 / OIDC Provider、客户端管理 | Spring Security OAuth2 AS |
| SessionService | 会话生命周期、Redis 共享会话、SSO | Spring Session Data Redis |
| ApiKeyService | API Key 生成/验证/吊销 | ApiKeyRepository |
| KeyService | 密钥派生(HKDF)、证书签发(X.509)、HSM 集成 | Bouncy Castle, SunPKCS11 |
| AuditService | 审计日志采集/查询、合规报告 | AuditLogRepository, RocketMQ |
| ThreatDetectionService | 异常登录检测、设备指纹、撞库防护 | Sentinel, Redis |
| FamilyService | 家庭创建/成员管理/角色 | FamilyRepository |
| OrganizationService | 组织管理、企业 MDM、RBAC | OrganizationRepository |
| DeveloperService | 开发者注册/认证、证书管理、沙箱 | DeveloperCertRepository |

## 3. 技术选型

### 3.1 语言与框架

| 组件 | 技术选型 | 理由 |
|------|---------|------|
| 语言 | Java 21 LTS | 虚拟线程(Virtual Threads)、模式匹配、密封类 |
| Web 框架 | Spring Boot 3.3 + Spring MVC | 同步编程模型，虚拟线程提供高并发 |
| 安全框架 | Spring Security 6.x | 认证/授权标准框架 |
| OAuth2/OIDC | Spring Security OAuth2 Authorization Server | 官方 OAuth2 授权服务器实现 |
| JWT | jjwt + Nimbus JOSE + JWT | 高性能 JWT 签发/验证 |
| ORM | MyBatis-Plus + Spring Data JPA | MP 用于复杂查询，JPA 用于标准 CRUD |
| 数据库迁移 | Flyway | Java 生态标准数据库版本管理 |
| 加密库 | Bouncy Castle + Java Cryptography Architecture (JCA) | 业界最完整的 Java 加密库 + JDK 内置 |
| 数据库 | PostgreSQL 16 | 成熟的关系型数据库，支持 JSONB |
| 缓存/会话 | Redis 7 + Spring Session Data Redis | 低延迟会话存储、分布式会话共享 |
| 消息队列 | RocketMQ | 异步事件（登录通知、审计日志） |
| 服务注册/配置 | Nacos 2.x | 注册发现 + 配置中心 |
| 熔断限流 | Sentinel | 实时监控、Dashboard、多种限流策略 |
| API 文档 | SpringDoc OpenAPI 3.0 | 自动生成 REST API 文档 |
| API 协议 | REST (OpenAPI 3.0) + gRPC (内部) | 对外 REST + 内部 gRPC |
| HSM 集成 | SunPKCS11 + Bouncy Castle PKCS#11 | JDK 内置 HSM 标准接口 |
| 身份协议 | OAuth 2.1 / OIDC / WebAuthn | 行业标准认证协议 |
| 构建工具 | Maven | 作为 qoobot-cloud 的子模块 |

### 3.2 为什么不拆分成多个微服务

| 考量 | 结论 |
|------|------|
| 团队规模 | 初期 < 10 人，单体微服务足够 |
| 数据一致性 | 用户/设备/会话/审计数据强关联，单体事务更简单 |
| 运维复杂度 | 单一 JAR 部署，避免 9 个微服务的运维负担 |
| 性能 | 进程内调用远快于 RPC/HTTP 跨服务调用 |
| 演进路线 | 当单服务 QPS > 50K 或团队 > 50 人时，可按子系统逐步拆分 |

### 3.3 安全标准合规

| 标准 | 适用范围 | 关键要求 |
|------|---------|---------|
| FIPS 140-3 | 加密模块 | 经认证的加密算法实现（Bouncy Castle FIPS） |
| ISO 27001 | 信息安全管理 | 安全策略、风险评估、持续改进 |
| SOC 2 Type II | 服务组织控制 | 安全性、可用性、机密性 |
| GDPR/CCPA/PIPL | 隐私保护 | 数据最小化、用户权利、跨境传输 |
| WebAuthn Level 2 | 无密码认证 | FIDO2/Passkey 标准 |
| NIST SP 800-63B | 数字身份指南 | 认证器保证级别 (AAL) |

## 4. 核心数据流

### 4.1 用户登录流程

```
[客户端] → [Gateway] → [qooauth AuthController]
                            │
                            ├─ AuthService.authenticate()
                            │   ├─ UserRepository.findByEmail()
                            │   ├─ PasswordEncoder.matches()
                            │   ├─ ThreatDetectionService.evaluate()
                            │   └─ JwtTokenProvider.generate()
                            │
                            ├─ SessionService.create()
                            │   └─ Redis (Spring Session)
                            │
                            ├─ AuditService.record() → RocketMQ
                            │
                            └─ 返回 JWT Access Token + Refresh Token
```

### 4.2 设备激活流程

```
[新设备] → [Gateway] → [qooauth DeviceController]
                            │
                            ├─ DeviceService.activate()
                            │   ├─ 验证出厂证书
                            │   ├─ KeyService.issueDeviceCert()  → X.509 签发
                            │   ├─ DeviceRepository.save()
                            │   └─ UserRepository (绑定账户)
                            │
                            └─ AuditService.record() → RocketMQ
```

### 4.3 微服务间通信

```
┌──────────────┐   REST/OpenFeign    ┌──────────────┐
│  qoocloud    │ ◄─────────────────► │   qooauth    │
│  (云端服务)   │   用户认证/鉴权      │  (账号安全)   │
└──────────────┘                     └──────┬───────┘
                                            │
┌──────────────┐   REST/OpenFeign           │ RocketMQ (异步)
│  qoostore      │ ◄─────────────────►        ▼
│  (技能市场)   │   开发者认证/鉴权    ┌──────────────┐
└──────────────┘                     │  RocketMQ    │
                                     │  审计/通知    │
                                     └──────────────┘
```

## 5. 部署架构

### 5.1 单服务多实例

```
                    ┌──────────────────────┐
                    │   qoobot-cloud-gateway│
                    │   (负载均衡 + 路由)    │
                    └──────────┬───────────┘
                               │
               ┌───────────────┼───────────────┐
               ▼               ▼               ▼
        ┌──────────┐   ┌──────────┐   ┌──────────┐
        │ qooauth  │   │ qooauth  │   │ qooauth  │
        │ 实例 1   │   │ 实例 2   │   │ 实例 3   │
        └────┬─────┘   └────┬─────┘   └────┬─────┘
             └──────────────┼──────────────┘
                            │
            ┌───────────────┼───────────────┐
            ▼               ▼               ▼
     ┌──────────┐   ┌──────────┐   ┌──────────┐
     │PostgreSQL│   │  Redis   │   │ RocketMQ │
     │  主从    │   │ Cluster  │   │  集群    │
     └──────────┘   └──────────┘   └──────────┘
```

### 5.2 高可用设计

- **qooauth**：无状态，水平扩展，K8s HPA 自动伸缩
- **Nacos 集群**：3 节点部署，AP 模式
- **数据库**：PostgreSQL 主从复制 + 自动故障切换
- **Redis**：Cluster 模式，跨 AZ 部署
- **HSM**：双活部署，自动切换
- **灾难恢复**：跨区域异步复制，RPO < 5min，RTO < 30min

### 5.3 容器化

```dockerfile
FROM eclipse-temurin:21-jre-alpine

RUN addgroup -S qooauth && adduser -S qooauth -G qooauth
USER qooauth

COPY target/qooauth-*.jar /app/app.jar

ENTRYPOINT ["java", \
    "--enable-preview", \
    "-XX:+UseG1GC", \
    "-XX:MaxGCPauseMillis=200", \
    "-XX:+UseContainerSupport", \
    "-XX:MaxRAMPercentage=75.0", \
    "-jar", "/app/app.jar"]
```
