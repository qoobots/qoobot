## 2. 模块职责

| Maven 模块 | 职责 | 依赖模块 | 关键框架 |
|------------|------|---------|---------|
| `qooauth-common` | 公共 DTO/VO/异常/工具类 | 无 | Lombok, Jackson, Jakarta Validation |
| `qooauth-gateway` | API 网关（限流/路由/JWT 预验签） | qooauth-common | Spring Cloud Gateway, Sentinel |
| `qooauth-auth` | 注册/登录/登出/JWT 签发/TOTP/Passkey | qooauth-common | Spring Security, jjwt, Nimbus JOSE, Bouncy Castle |
| `qooauth-user` | 用户 CRUD/资料/家庭/组织/账户恢复 | qooauth-common | Spring Data JPA, MyBatis-Plus |
| `qooauth-device` | 设备注册/激活/绑定/X.509 证书/锁死/定位 | qooauth-common | Bouncy Castle, Spring Data JPA |
| `qooauth-oauth` | OAuth 2.0 / OIDC Provider | qooauth-common | Spring Security OAuth2 Authorization Server |
| `qooauth-session` | 会话生命周期/Redis 共享会话/SSO | qooauth-common | Spring Session Data Redis |
| `qooauth-audit` | 审计日志采集/查询/合规报告 | qooauth-common | Spring Data JPA, MyBatis-Plus, RocketMQ |
| `qooauth-threat` | 异常登录检测/设备指纹/撞库防护 | qooauth-common | Sentinel, Redis |
| `api/` | API 规范定义（Protobuf + OpenAPI） | 无 | — |
| `web/` | Web 控制台前端 | api/openapi/ | React 18, TypeScript, Vite, shadcn/ui |
| `tests/` | 集成测试与 E2E 测试 | 全部业务模块 | Spring Boot Test, Testcontainers, Playwright |
| `deploy/` | K8s/Terraform 部署配置 | 无 | — |

## 5. Java 代码规范

### 5.1 编码规范

- 遵循 [Google Java Style Guide](https://google.github.io/styleguide/javaguide.html)（适配 QooBot 项目）
- 使用 **Checkstyle** + **PMD** + **SpotBugs** 进行静态分析
- 包名使用小写，多单词用点分隔：`com.qoobot.qooauth.auth`
- 类名使用 `UpperCamelCase`：`AuthController`, `JwtTokenProvider`
- 方法名使用 `lowerCamelCase`：`validateToken()`, `createSession()`
- 常量使用 `UPPER_SNAKE_CASE`：`MAX_LOGIN_ATTEMPTS`, `TOKEN_EXPIRE_SECONDS`
- 所有公共 API 必须有 Javadoc 注释
- 使用 Lombok 减少样板代码（`@Data`, `@Builder`, `@Slf4j`）
- 单元测试覆盖率 > 80%（JaCoCo 强制执行）

### 5.2 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| Maven 模块 | kebab-case | `qooauth-auth`, `qooauth-common` |
| 包名 | 全小写 | `com.qoobot.qooauth.auth.service` |
| 类名 | UpperCamelCase | `AuthController`, `JwtTokenProvider` |
| 接口名 | UpperCamelCase（不加 I 前缀） | `AuthService`, `UserRepository` |
| 实现类 | 接口名 + Impl | `AuthServiceImpl` |
| 方法名 | lowerCamelCase | `validateToken()`, `findByEmail()` |
| 常量 | UPPER_SNAKE_CASE | `MAX_LOGIN_ATTEMPTS` |
| 变量 | lowerCamelCase | `accessToken`, `refreshToken` |
| DTO 类 | 名词 + Request/Response | `LoginRequest`, `LoginResponse` |
| 数据库表 | snake_case 复数 | `api_keys`, `audit_logs` |
| REST 端点 | kebab-case 复数 | `/api/v1/auth/refresh-token` |
| SQL 迁移文件 | V{序号}__{描述}.sql | `V1__create_users.sql` |
| 配置属性 | kebab-case | `qooauth.jwt.access-token-expire` |

### 5.3 静态分析配置

### 5.4 异常处理规范

```java
// ✅ 正确：使用项目统一异常体系
throw new InvalidCredentialsException("邮箱或密码不正确");

// ✅ 正确：Controller 层统一异常处理（@ControllerAdvice）
@RestControllerAdvice
public class GlobalExceptionHandler {
    @ExceptionHandler(InvalidCredentialsException.class)
    public ResponseEntity<ErrorResponse> handleInvalidCredentials(
            InvalidCredentialsException ex) {
        return ResponseEntity.status(401)
            .body(ErrorResponse.of(ErrorCode.INVALID_CREDENTIALS, ex.getMessage()));
    }
}

// ❌ 错误：直接抛出 RuntimeException 或返回 null
throw new RuntimeException("login failed"); // 禁止
return null; // 禁止
```

### 5.5 日志规范

```java
// 使用 Lombok @Slf4j + SLF4J
@Slf4j
public class AuthService {

    public LoginResponse login(LoginRequest request) {
        // ✅ 正确：结构化日志，不记录敏感信息
        log.info("User login attempt: email={}, ip={}",
            maskEmail(request.getEmail()), request.getIp());

        // ❌ 错误：记录密码明文
        // log.info("Login: email={}, password={}", email, password); // 禁止！
    }

    // 邮箱脱敏：user@example.com → u***@example.com
    private String maskEmail(String email) {
        int at = email.indexOf('@');
        if (at <= 1) return email;
        return email.charAt(0) + "***" + email.substring(at);
    }
}
```
