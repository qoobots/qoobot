# 03 — 交互设计

> 版本：v0.3 | 最后更新：2026-06-27 | 状态：Draft
> 子项目：qooauth（账号与安全）| 架构：单一 Spring Boot 微服务

---

## 1. 多入口交互矩阵

qooauth 作为基础设施服务，面向 4 类用户提供不同的交互入口：

| 入口 | 目标用户 | 交互形式 | 技术栈 |
|------|---------|---------|--------|
| REST API | 第三方应用/服务/Web 控制台 | HTTP/JSON | SpringDoc OpenAPI 3.0 + Spring MVC |
| gRPC API | 内部微服务 (qoocloud/qooeco/qoosvc) | Protobuf | grpc-java + Spring Boot |
| OAuth 2.0 / OIDC | Web/移动端登录 | 标准授权流程 | Spring Security OAuth2 Authorization Server |
| Web 控制台 | 终端用户/管理员 | Web UI | Vue 3 + Element Plus |

## 2. REST API 设计

### 2.1 API 概览

```
Base URL: https://auth.qoobot.com/api/v1

认证端点:
  POST   /auth/register              用户注册
  POST   /auth/login                  密码登录
  POST   /auth/login/otp              验证码登录
  POST   /auth/login/passkey          Passkey 登录
  POST   /auth/refresh                Token 刷新
  POST   /auth/logout                 登出
  POST   /auth/revoke                 吊销所有会话

用户端点:
  GET    /users/me                    当前用户信息
  PUT    /users/me                    更新用户资料
  DELETE /users/me                    注销账户
  POST   /users/me/recovery           账户恢复

设备端点:
  GET    /devices                     设备列表
  POST   /devices/activate            激活设备
  DELETE /devices/{id}                移除设备
  POST   /devices/{id}/lock           远程锁死
  POST   /devices/{id}/locate         设备定位

OAuth 端点:
  GET    /oauth/authorize             授权页面
  POST   /oauth/token                 获取 Token
  POST   /oauth/revoke                吊销 Token
  GET    /oauth/.well-known/openid-configuration   OIDC 发现

会话端点:
  GET    /sessions                    会话列表
  DELETE /sessions/{id}               踢出会话

API Key 端点:
  POST   /api-keys                    创建 API Key
  GET    /api-keys                    列出 API Key
  DELETE /api-keys/{id}               吊销 API Key

家庭/团队端点:
  POST   /families                    创建家庭
  POST   /families/{id}/members       邀请成员
  DELETE /families/{id}/members/{uid} 移除成员
  POST   /organizations               创建组织
  PUT    /organizations/{id}/roles    角色管理

审计端点:
  GET    /audit/logs                  操作日志
  GET    /audit/report                合规报告
```

### 2.3 错误码体系

| HTTP 状态码 | 错误码 | 含义 |
|------------|--------|------|
| 400 | INVALID_REQUEST | 请求参数错误 |
| 401 | INVALID_CREDENTIALS | 凭证无效 |
| 401 | TOKEN_EXPIRED | Token 已过期 |
| 403 | INSUFFICIENT_PERMISSIONS | 权限不足 |
| 403 | ACCOUNT_LOCKED | 账户已锁定 |
| 403 | DEVICE_BANNED | 设备已被封禁 |
| 404 | USER_NOT_FOUND | 用户不存在 |
| 409 | EMAIL_TAKEN | 邮箱已被注册 |
| 429 | RATE_LIMITED | 请求频率过高（Sentinel 限流） |
| 500 | INTERNAL_ERROR | 服务器内部错误 |

### 2.4 SpringDoc OpenAPI 配置

```java
@Configuration
public class OpenApiConfig {

    @Bean
    public OpenAPI qooauthOpenAPI() {
        return new OpenAPI()
            .info(new Info()
                .title("QooAuth API")
                .version("v1")
                .description("QooBot 统一身份认证服务 — 单一微服务"))
            .addSecurityItem(new SecurityRequirement().addList("BearerAuth"))
            .components(new Components()
                .addSecuritySchemes("BearerAuth",
                    new SecurityScheme()
                        .type(SecurityScheme.Type.HTTP)
                        .scheme("bearer")
                        .bearerFormat("JWT")));
    }
}
```

## 3. OAuth 2.0 / OIDC 交互

### 3.1 Authorization Code Flow (Web 应用)

```
┌─────────┐     ┌──────────┐     ┌────────────┐     ┌─────────┐
│ 用户浏览器│     │ 第三方应用 │     │   qooauth   │     │ 资源服务器│
└────┬────┘     └────┬─────┘     └─────┬──────┘     └────┬────┘
     │               │                │                  │
     │ 1. 访问应用    │                │                  │
     │──────────────>│                │                  │
     │               │                │                  │
     │ 2. 重定向授权  │                │                  │
     │<──────────────│  GET /oauth/authorize?            │
     │               │  client_id=...&redirect_uri=...   │
     │               │  &scope=profile+device.read       │
     │               │  &state=random_state              │
     │               │                │                  │
     │ 3. 登录/授权   │                │                  │
     │──────────────────────────────>│                  │
     │               │  Spring Security OAuth2 AS 处理    │
     │               │                │                  │
     │ 4. 授权码回调  │                │                  │
     │               │<───────────────│                  │
     │               │  ?code=...&state=...              │
     │               │                │                  │
     │               │ 5. 换取 Token  │                  │
     │               │──────────────>│ POST /oauth/token │
     │               │                │  code=...        │
     │               │<──────────────│  access_token     │
     │               │                │                  │
     │               │ 6. 使用 Token  │                  │
     │               │──────────────────────────────────>│
```

### 3.2 OAuth Scopes 设计

| Scope | 描述 | 适用对象 |
|-------|------|---------|
| `openid` | OIDC 基础 scope，返回 id_token | 所有应用 |
| `profile` | 用户基本信息（昵称、头像） | Web/移动应用 |
| `email` | 用户邮箱 | 需要通知的应用 |
| `device.read` | 读取设备列表 | 设备管理应用 |
| `device.control` | 控制设备（锁死、定位） | 安全应用 |
| `skill.install` | 安装技能 | qooeco |
| `skill.publish` | 发布技能 | 开发者 |
| `audit.read` | 读取审计日志 | 企业管理员 |

## 4. gRPC 内部 API

### 4.1 服务定义

```protobuf
syntax = "proto3";
package qooauth.v1;

option java_multiple_files = true;
option java_package = "com.qoobot.qooauth.grpc";
option java_outer_classname = "QooAuthProto";

service AuthService {
  rpc VerifyToken(VerifyTokenRequest) returns (VerifyTokenResponse);
  rpc GetUser(GetUserRequest) returns (User);
  rpc CheckPermission(CheckPermissionRequest) returns (CheckPermissionResponse);
}

service DeviceService {
  rpc GetDevice(GetDeviceRequest) returns (Device);
  rpc VerifyDeviceCertificate(VerifyDeviceCertRequest) returns (VerifyDeviceCertResponse);
  rpc ListUserDevices(ListUserDevicesRequest) returns (ListUserDevicesResponse);
}

service SessionService {
  rpc CreateSession(CreateSessionRequest) returns (Session);
  rpc ValidateSession(ValidateSessionRequest) returns (ValidateSessionResponse);
  rpc RevokeSession(RevokeSessionRequest) returns (google.protobuf.Empty);
}

service AuditService {
  rpc RecordEvent(RecordEventRequest) returns (google.protobuf.Empty);
  rpc QueryEvents(QueryEventsRequest) returns (QueryEventsResponse);
}
```

## 6. CLI 交互（开发者工具）

### 6.1 命令体系

```bash
# 开发者认证
qoo auth login              # 登录 QooBot ID
qoo auth logout             # 登出
qoo auth whoami             # 查看当前身份

# 开发者证书
qoo auth cert create        # 创建开发者证书
qoo auth cert list          # 列出证书
qoo auth cert revoke <id>   # 吊销证书

# API Key 管理
qoo auth apikey create      # 创建 API Key
qoo auth apikey list        # 列出 API Key
qoo auth apikey revoke <id> # 吊销 API Key

# 技能签名
qoo auth sign <skill_path>  # 签名技能包
qoo auth verify <skill_path># 验证技能签名
```
