# QooBot 产业链 — ① 设计（Design）

> 最后更新：2026-05-27
>
> 本文档定义 QooBot 双足仿生人产业链第一要素：设计。

---

## 1. 设计理念

QooBot 采用**开源协同设计**模式，区别于传统封闭厂商的内部设计流程：

- **开放协作**：全球开发者可参与机械结构、电路、软件架构的评审与贡献
- **模块化架构**：头部/躯干/手臂/腿足/灵巧手各自独立设计，可自由组合
- **版本迭代**：设计图纸通过 GitHub 进行版本管理，支持 Fork/Branch/Merge/PR

---

## 2. 设计范畴

| 设计域 | 内容 | 工具链 | 负责人/社区 |
|--------|------|--------|-------------|
| **机械设计** | 骨架结构、关节传动、外壳造型、热管理 | FreeCAD / SolidWorks → STEP | qoobody/mechanical |
| **电子设计** | 主控板、驱动板、传感器板、电源板 | KiCad / Altium → Gerber | qoobody/electronics |
| **软件架构** | qoobrain OS、中间件、驱动层、应用层 | C++17/Python/Java/TypeScript | 各子项目团队 |
| **工业设计** | 外观造型、人机工程、CMF（颜色/材质/表面处理） | Blender / Rhino | qoobody/industrial |
| **仿真验证** | 运动学/动力学仿真、热仿真、电磁兼容仿真 | Isaac Sim / MuJoCo / ANSYS | qoodev/simulation |
| **合规设计** | 安全认证（ISO 10218/13482）、EMC、无线认证 | — | qoocompliance |

## 5. 设计协作流程

```
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│ 需求收集   │──▶│ 方案设计   │──▶│ 社区评审   │──▶│ 原型验证   │──▶│ 发布归档   │
│ Issues/   │   │ 机械/电子 │   │ PR Review│   │ 仿真/实物 │   │ GitHub    │
│ RFC       │   │ /软件架构 │   │ SIG 评审  │   │ 测试报告  │   │ Release   │
└──────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘
```

### 5.1 设计评审标准

| 评审维度 | 检查项 | 通过标准 |
|----------|--------|----------|
| **功能性** | 是否满足需求规格 | 全部需求点覆盖 |
| **可制造性** | 加工工艺可行性 | DFM/DFA 检查通过 |
| **可采购性** | 物料可获得性 | 至少 2 个供应商可选 |
| **可测试性** | 测试点与验证方案 | 测试覆盖率 > 90% |
| **合规性** | 安全/EMC/认证 | 满足目标市场法规 |
| **成本** | BOM 成本估算 | 不超过目标成本 120% |

## 6. 设计变更管理

| 变更类型 | 流程 | 审批 |
|----------|------|------|
| **小改动** (Minor) | PR → 1 位 Reviewer 批准 | 模块负责人 |
| **功能变更** (Major) | RFC → 设计文档 → PR → 2 位 Reviewer | SIG 组长 |
| **架构变更** (Breaking) | RFC → 社区讨论 → 技术委员会投票 → PR | 技术委员会 |
