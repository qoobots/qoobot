## 🏢 商业公司竞品

### 1. Tesla — Optimus

> **定位**：通用人形机器人，汽车产线内部应用先行，远期面向消费者。
> **阶段**：Gen 2 原型，工厂内部测试（2025-2026）。

| 维度 | Tesla Optimus | QooBot | 差异分析 |
|------|---------------|--------|----------|
| **硬件** | 自研执行器/传感器，闭源 | 开放参考设计（qoobody） | QooBot 硬件透明，第三方可制造 |
| **AI 大脑** | FSD 复用 + Dojo 训练，完全闭源 | 开源感知/认知/规划全栈（qoobrain） | QooBot 可被社区审查与改进 |
| **芯片** | 自研 Dojo D1 + FSD Chip | 开放推理引擎 + 14 款芯片支持（qoocore） | QooBot 不绑定特定芯片 |
| **开发者生态** | 不对外开放 | IDE 插件/CLI/SDK/仿真器（qoodev） | 根本性差异：闭源 vs 开放 |
| **技能扩展** | 内部定义，不可扩展 | 技能市场 .qooskills（qoostore） | QooBot 有第三方技能生态 |
| **商业模式** | 卖机器人硬件 | 开源免费，生态服务增值 | QooBot 无需硬件销售即可生存 |
| **社区** | 无 | 开源社区（qoocommunity） | QooBot 社区驱动创新 |

**QooBot 优势**：开放生态 vs 封闭系统；多芯片支持 vs 绑定 Dojo；社区贡献 vs 单一团队迭代。

**QooBot 劣势**：没有汽车产线验证场景；资金规模不在同一量级；品牌知名度远不及 Tesla。

**应对策略**：不与 Tesla 在制造规模和消费级硬件上正面竞争，而是成为"机器人世界的 Linux"——任何想造机器人的团队都可以基于 QooBot 起步。

### 2. Figure AI

> **定位**：通用人形机器人，聚焦仓储物流与制造业。
> **阶段**：Figure 02 原型，与 OpenAI 合作，BMW 产线试点（2025-2026）。

| 维度 | Figure AI | QooBot | 差异分析 |
|------|-----------|--------|----------|
| **硬件** | 全自研执行器，闭源 | 开放参考设计（qoobody） | QooBot 降低硬件准入门槛 |
| **AI** | OpenAI 大模型驱动，闭源 | 自研 LLM 集成 + 端侧推理（qoobrain + qoocore） | QooBot 可离线运行，不依赖 API |
| **端侧推理** | 依赖云端 OpenAI API | 端侧芯片加速推理（qoocore） | QooBot 隐私性更强 |
| **技能** | 单一场景训练 | 技能市场 + 行为树编排（qoobrain + qoostore） | QooBot 更易多场景复用 |
| **合规** | 未披露 | 全栈合规体系（qoocompliance） | QooBot 有系统化的合规方案 |

**QooBot 优势**：离线推理不依赖 API；开放硬件标准可被其他公司采用；合规体系完整。

**QooBot 劣势**：没有 OpenAI 级别的模型支持；缺少真实工厂验证场景。

**应对策略**：qooauth + qoocloud 提供企业级私有部署方案，满足对数据主权敏感的客户；强调端侧推理的低延迟和隐私优势。

### 3. Boston Dynamics — Atlas

> **定位**：最先进的人形机器人运动控制研究平台。
> **阶段**：Atlas 电动版，面向研究机构（2025+）。

| 维度 | Boston Dynamics Atlas | QooBot | 差异分析 |
|------|----------------------|--------|----------|
| **运动控制** | 世界顶尖，闭源 | OpenLoong 开源 + 全身控制（qoobody） | QooBot 运动控制社区可参与 |
| **硬件** | 全定制，天价 | 开放参考设计，可降本 | QooBot 硬件成本可控 |
| **软件栈** | 不对外 | 全开源（qoobrain） | 根本性差异 |
| **开发者** | 仅限合作研究机构 | 面向全球开发者（qoodev） | QooBot 受众广几个数量级 |
| **供应链** | 内部制造 | 开放 BOM + 产线设计（qoochain） | QooBot 支持第三方量产 |

**QooBot 优势**：开源软件栈；可制造的硬件设计；社区生态。

**QooBot 劣势**：运动控制性能短期内难以匹敌 Atlas 数十年积累。

**应对策略**：通过 MuJoCo 仿真 + qoodev 遥操作录制，实现 Sim-to-Real 持续改进；社区众包数据反哺运动控制模型。

> **阶段**：H1 已量产（春晚表演），G1 量产中（2025-2026），$16,000 起。

#### 宇树核心能力

| 能力 | 详情 |
|------|------|
| **关节电机** | 自研低惯量高速内转子 PMSM，峰值扭矩密度 189 N·m/kg，世界领先 |
| **灵巧手** | Dex3-1/Dex5-1 系列，力位混合控制 + 触觉传感器，已商品化 |
| **4D 激光雷达** | L2/L1 自研，不依赖 Velodyne 等第三方 |
| **运动控制** | H1 移动速度 3.3m/s 世界纪录；集群舞蹈（冬奥/春晚）|
| **机器人大模型** | UnifoLM（统一机器人大模型），模仿学习 + 强化学习驱动 |
| **软件平台** | UniFolm（遥控）+ UniStore（应用商店）+ UniPlatform（开发者平台）|
| **OTA** | 持续软件升级 |
| **行业方案** | 电力巡检 + 消防救援已验证落地 |
| **品牌** | 春晚/冬奥/亚运会背书，全球知名度极高 |

#### 战略定位对比：闭源硬件 vs 开源全栈

```
宇树 = 闭源版 Apple（硬件+软件全自研，不对外开放设计）
       │
       ├── 自研电机 → 闭源，不公开设计图纸/BOM
       ├── 自研灵巧手 → 闭源，不公开设计
       ├── 自研激光雷达 → 闭源，不公开设计
       ├── 自研整机 → 闭源，仅宇树可制造
       └── 提供 SDK → 软件层有限开放

QooBot = 开源版 Apple（硬件+软件全自研，设计全部公开）
         │
         ├── 自研电机 → 开源设计图纸/BOM/工艺，任何人可制造
         ├── 自研灵巧手 → 开源设计，任何人可制造
         ├── 自研激光雷达 → 开源设计，任何人可制造
         ├── 自研整机(qoobody) → 开放参考设计 + qoochain 量产指南
         └── 全栈开源 → QOS + qoocore + qoodev + qoocloud 全部开源
```

| 维度 | Unitree | QooBot | 根本差异 |
|------|---------|--------|----------|
| **电机** | 自研闭源，仅宇树可制造 | 自研开源，设计图纸/BOM/工艺全部公开 | 宇树卖电机，QooBot 让全世界造电机 |
| **灵巧手** | Dex3-1/5-1 自研闭源 | 自研开源，MFQ 认证体系 | 宇树卖灵巧手，QooBot 认证第三方灵巧手 |
| **激光雷达** | 自研 4D LiDAR L2 闭源 | 自研开源，传感器 HAL 抽象层 | 宇树卖雷达，QooBot 任何雷达即插即用 |
| **整机** | H1/G1/R1 量产销售 | qoobody 参考设计 + qoochain 量产指南 | 宇树卖机器人，QooBot 让任何人造机器人 |
| **软件深度** | 基础运动 API + UnifoLM | QOS 全栈（感知→认知→决策→规划→控制→安全） | QooBot 软件栈更深更完整 |
| **AI 推理** | 依赖外部集成 | qoobrain 端侧 LLM + qoocore 14 款芯片加速 | QooBot 端侧推理更强 |
| **芯片** | 绑定 Intel/NVIDIA | qoocore 芯片中立，14 款芯片支持 | QooBot 不锁定芯片厂商 |
| **技能生态** | UniStore（自有 App） | qoostore 技能市场（第三方技能） | QooBot 更像 App Store |
| **开发者工具** | 基础开发手册 | qoodev IDE 插件 + CLI + 仿真 + 调试 | QooBot 开发者体验更好 |
| **云端** | 无 | qoocloud 设备管理/OTA/远程推理/数字孪生 | QooBot 云端能力完整 |
| **身份安全** | 无 | qooauth QooIDaaS 统一身份 | QooBot 有账号体系 |
| **合规** | 页面安全提示 | qoocompliance 系统化全球法规合规 | QooBot 合规更专业 |
| **供应链** | 封闭自有产线 | qoochain 开放 BOM + 产线设计 + 标定指南 | QooBot 赋能第三方量产 |
| **配件生态** | 自有机械臂 Z1/D1-T | qoogear MFQ 认证，第三方配件即插即用 | QooBot 开放配件市场 |
| **社区** | 无 | qoocommunity 开源社区 | QooBot 社区驱动创新 |

**核心差异**：宇树是"闭源版 Apple"——什么都自己做，但设计不公开，只有宇树能造。QooBot 是"开源版 Apple"——同样什么都自己做，但设计图纸、BOM、工艺全部公开，**全世界任何人都能基于 QooBot 设计制造自己的人形机器人**。

**QooBot 劣势**：
- 硬件自研尚在早期，电机/灵巧手/激光雷达暂无实物
- 运动控制未在物理机器人上验证
- 品牌知名度远不及宇树（春晚/冬奥背书）
- 宇树已有量产能力和行业落地案例

**应对策略**：
- qoobody 所有硬件设计以开源为第一原则——设计图纸、BOM、装配工艺、标定流程全部公开
- qoochain 提供完整量产指南，降低第三方制造门槛
- qoocore 芯片中立 + qoobody HAL 统一接口，让任何电机/传感器厂商都能接入 QOS 生态
- 不与宇树比量产规模，而是比生态开放度——宇树只能卖自己的机器人，QooBot 能让 100 家公司造出 100 种机器人

## 📚 开源项目竞品

### 1. ROS / ROS 2

> **定位**：机器人中间件事实标准。
> **关系**：基础依赖而非竞品。

| 维度 | ROS 2 | QooBot（qoobrain） | 关系 |
|------|-------|---------------------|------|
| **定位** | 通用机器人中间件 | 人形机器人全栈 OS | ROS 2 是 qoobrain 的基础设施 |
| **AI 能力** | 无 | 感知/认知/决策/LLM 集成 | QooBot 在 ROS 2 之上构建 AI 层 |
| **芯片加速** | 无 | qoocore 端侧推理 | QooBot 补齐推理能力 |
| **行为树** | 无内置 | BehaviorTree.CPP 集成 | QooBot 提供任务编排 |

> **策略**：QooBot 构建在 ROS 2 之上，与其是增强关系而非竞争。qoobrain 可以理解为"ROS 2 + AI + 人形机器人专用能力"。

---

### 2. NVIDIA Isaac

> **定位**：机器人 AI 平台（仿真 + 感知 + 操作）。
> **关系**：既合作又竞争。

| 维度 | NVIDIA Isaac | QooBot | 关系 |
|------|-------------|--------|------|
| **仿真** | Isaac Sim（领先） | MuJoCo + Isaac Sim 适配（qoodev） | 互补：QooBot 支持多仿真器 |
| **感知** | cuOpt + FoundationPose | YOLO + SAM + ORB-SLAM3（qoobrain） | 竞争：QooBot 不绑定 NVIDIA |
| **芯片** | 锁定 NVIDIA GPU/Jetson | 14 款芯片支持（qoocore） | 竞争：QooBot 芯片中立 |
| **开放度** | 部分开源 | 全开源 | QooBot 更开放 |
| **云端** | Omniverse | 自建 qoocloud | 竞争 |

**QooBot 优势**：芯片中立（不锁定 NVIDIA）；全开源；社区驱动。

**QooBot 劣势**：仿真能力不如 Isaac Sim；没有 NVIDIA 的硬件生态。

**应对策略**：qoodev 同时支持 Isaac Sim 和 MuJoCo，用户可按需选择；qoocore 支持 CUDA 后端但不依赖。

### 3. Hugging Face LeRobot

> **定位**：AI 社区驱动的机器人学习平台，专注模仿学习与策略训练。
> **阶段**：活跃开发中，社区增长快（2025-2026）。

| 维度 | LeRobot | QooBot | 关系 |
|------|---------|--------|------|
| **聚焦** | 模仿学习 + 数据共享 | 全栈机器人 OS | 互补：QooBot 可集成 LeRobot 训练 |
| **硬件** | 低成本机械臂（Koch v1） | 全尺寸人形机器人 | 不同定位 |
| **社区** | HuggingFace 社区 | QooBot 自建社区 | 合作空间大 |
| **模型** | ACT / Diffusion Policy | LLM + 多模态感知（qoobrain） | 可互补集成 |
| **芯片加速** | 无 | qoocore 端侧推理 | QooBot 有推理加速 |

> **策略**：LeRobot 的模仿学习能力可作为 qoodev 示教模块的补充；QooBot 的端侧推理芯片可为 LeRobot 模型提供高效部署。

## 🎯 QooBot 差异化优势总结

### 1. 全栈开源（Unique Selling Point）

没有人形机器人项目同时做到：
- 硬件参考设计开放（qoobody）
- 软件全栈开源（qoobrain）
- 芯片推理引擎开源（qoocore）
- 开发者工具开源（qoodev）
- 云端服务开源（qoocloud）

### 2. 芯片中立

qoocore 支持 14 款芯片，不锁定任何一家芯片厂商。这在整个行业中独一无二。

### 3. 统一身份基础设施

qooauth 对标 Apple ID，提供端到端的设备认证、用户授权、数据加密，是机器人联网安全的基石。

### 4. 技能市场生态

qoostore 是机器人领域的 App Store，让第三方开发者可以创造和分发机器人应用。

### 5. 合规护航

qoocompliance 系统化覆盖全球机器人法规标准，帮助企业降低合规风险。

### 6. 社区驱动

qoocommunity 构建开源社区的完整基础设施——从文档到贡献流程到开发者大会。
