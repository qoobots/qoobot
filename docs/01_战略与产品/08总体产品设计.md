## 🎯 产品定位

### 一句话定位

**QooBot = 机器人领域的 Android —— 开源、开放、为所有机器人而生。**

### 产品愿景

打造全球最开放、最强大的仿生人开源生态。让仿生人不再是少数公司的专利，而是像智能手机一样普及的通用平台。

### 核心价值主张

| 价值 | 描述 |
|------|------|
| **开源全栈** | 从芯片到云端、从硬件到软件，全部 Apache 2.0 开源 |
| **硬件中立** | 不绑定特定硬件，任何厂商可基于 QooBot 制造机器人 |
| **技能生态** | 机器人 App Store，第三方技能分发与商业化 |
| **开发者友好** | 完善的 SDK、仿真器、IDE 插件、调试工具 |
| **端云协同** | 端侧自主运行 + 云端增强推理，无缝伸缩 |
| **全球合规** | 内置完整合规框架 (ISO 10218/13482/GDPR) |

## 🖥️ QooBotOS (QOS) 产品设计

QOS 是双足仿生人的操作系统，定位为**开源版 Android**：

> Android = Linux Kernel + HAL + ART + Framework + System Apps + Play Store + Google Services
>
> QooBot  = QOS Kernel + QOS HAL + qoobrain Runtime + QOS Framework + 出厂技能 + QooStore + qoocloud

### QOS 分层对标

| Android 层 | QOS 层 | QOS 组件 | 说明 |
|-------------|--------|----------|------|
| **Kernel** | QOS Kernel | Linux + RT-Preempt + ROS 2 DDS | 硬实时内核 |
| **HAL** | QOS HAL | qoobody HAL (电机/灵巧手/雷达/IMU/摄像头) | 硬件厂商接入标准 |
| **Native Libs** | QOS Native | qoocore 推理引擎 + MuJoCo 仿真 + 运动控制库 | 端侧 AI 加速 |
| **ART Runtime** | QOS Runtime | qoobrain Runtime (Python AI + C++ 实时引擎) | 双引擎架构 |
| **Framework** | QOS Framework | QOS System Server + 技能 SDK API | 机器人系统服务 |
| **System Apps** | 出厂技能 | 视觉/导航/语音/安全/遥控 | 开机即用 |
| **Google Play** | QooStore | qoostore 技能市场 | 第三方技能分发 |
| **GMS** | qoocloud | 远程推理/OTA/设备管理/数字孪生 | 云端能力 |
| **Google Account** | QooIDaaS | qooauth 统一身份 | 设备认证/用户授权 |
| **Android Studio** | qoodev | CLI + IDE 插件 + 仿真 + 调试 | 开发者工具链 |

### QOS 两大核心子系统

```
QOS (QooBot OS)
│
├── 🧠 qoobrain — 自主意识大脑
│   │  感知 → 认知 → 决策 → 规划 → 控制 → 安全
│   │  端侧自主运行，无需网络连接
│   │  技术栈: Python 3.11 + C++17 + TypeScript + ROS 2
│   │
└── 🎮 qooremote — 远程人工遥控系统
    │  人在回路：当自主大脑无法应对的长尾场景
    │  人类操作员可远程接管机器人
    │  动捕/VR/手柄/视频回传/语音对讲/示教录制
```

## 📦 技能生态模型

### .qooskills 技能包

```
my-skill.qooskills              ← 技能包 (ZIP 压缩)
├── qooskills.yaml              ← 技能清单 (对标 AndroidManifest.xml)
├── <entry_script>.py           ← 技能逻辑
├── <behavior_tree>.xml         ← 行为树定义
├── models/                     ← AI 模型 (ONNX)
├── resources/                  ← 资源文件
├── libs/                       ← 依赖库
└── signatures/                 ← 数字签名
```

### 技能生命周期

```
INSTALLED → LAUNCHING → RUNNING ⇄ PAUSED → STOPPED/REMOVED
                         │
                         └── CRASHED → 自动重启 (最多 3 次)
```

### 技能权限模型

| 级别 | 示例权限 | 授权方式 |
|------|----------|----------|
| **normal** | CAMERA_RGB, TTS, LOGGER.WRITE | 安装时自动授予 |
| **dangerous** | CAMERA_DEPTH, LIDAR, MICROPHONE, WALK, ARM, GRASP | 运行时用户确认 |
| **signature** | EMERGENCY_STOP, TAKEOVER, DEVICE_CREDENTIAL | 仅系统签名可用 |

### 出厂技能清单

| 优先级 | 技能 | 数量 |
|:--:|------|:--:|
| **P0 开机即用** | 视觉感知、空间导航、语音交互、人物识别、自检诊断、系统设置、身份认证、安全监护、远程遥控 | 9 |
| **P1 开机可用** | 表情交互、场景记忆、任务记录、事件提醒、自主充电、行为编排、多机协同、QooStore、状态面板 | 9 |
| **P2 后续 OTA** | 人物跟随、物品查找、家庭联动、远程查看、多语言、运动报告、数据管理、环境感知、远程推理 | 9 |

## 🎯 产品策略

### 开源策略

| 策略 | 说明 |
|------|------|
| **全栈开源** | 从芯片到云端全部 Apache 2.0，无 Open Core 模式 |
| **开放治理** | RFC 流程 + TSC 技术委员会 + 社区共识决策 |
| **商业友好** | Apache 2.0 允许闭源商用，降低企业采用门槛 |
| **硬件图纸开放** | 机械结构/电路/PCB 全开源，任何人都可自行制造 |

### 生态策略

| 策略 | 说明 |
|------|------|
| **Android 对标** | 借鉴 Android 生态成功经验：HAL 硬件适配、技能市场分发、开发者工具链 |
| **硬件中立** | 不绑定特定硬件，任何厂商可基于 QooBot 制造机器人 |
| **MFQ 认证** | Made for QooBot 配件认证体系，确保兼容性 |
| **技能经济** | QooStore 技能市场，开发者通过技能分发获得收益 |

### 竞争策略

| 维度 | QooBot 策略 | 行业现状 |
|------|------------|----------|
| **开源深度** | 全栈 Apache 2.0 | 多数仅开放 SDK/API |
| **硬件中立** | 适配任意硬件 | 软件与自有硬件绑定 |
| **生态开放** | 技能市场 + 配件认证 | 封闭生态 |
| **开发者体验** | 完整 IDE 工具链 | 缺少专业工具链 |
| **资金门槛** | 零成本启动 | 硬件采购成本高昂 |

---

> **QooBot 的产品哲学**：不是做一个机器人，而是做一套让机器人"活"起来的操作系统。Android 让手机从功能机变成智能机——不是因为硬件更强，而是因为有了应用生态。QOS 同样如此——不是因为电机更快或雷达更准，而是因为机器人可以安装技能、扩展能力、持续进化。
