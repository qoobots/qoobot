# QooBot 总体需求分析

> 最后更新：2026-06-29
>
> 本文档定义 QooBot 项目的总体需求，涵盖功能需求、非功能需求、用户角色、约束条件与需求优先级。

---

## 📋 需求概述

QooBot 是一个面向仿生人的**开源全栈生态**，核心需求是构建驱动下一代物理 AI 的通用操作系统——以仿生人为核心，覆盖工业、服务、家庭等全场景应用。

> 一个大脑，任意身体，无处不在。

---

## 👥 用户角色与场景

### 用户角色矩阵

| 角色 | 描述 | 核心诉求 | 优先级 |
|------|------|----------|:--:|
| **机器人终端用户** | 家庭/企业中使用机器人的个人或组织 | 开箱即用、安全可靠、自然交互 | P0 |
| **机器人厂商 (OEM)** | 基于 QooBot 制造机器人本体的企业 | 硬件适配、品牌定制、量产方案 | P0 |
| **技能开发者** | 为机器人开发第三方应用的开发者 | 完善的 SDK、仿真环境、分发渠道 | P1 |
| **算法研究者** | 研究机器人感知/规划/控制算法的人员 | 开放接口、基准测试、数据集 | P1 |
| **配件厂商** | 制造传感器、执行器等外设的企业 | HAL 接口标准、认证体系 | P2 |
| **系统集成商** | 部署多机器人方案的服务商 | 多机编排、云端管理、运维监控 | P1 |
| **社区贡献者** | 开源社区参与代码/文档贡献的开发者 | 清晰的贡献流程、文档、导师 | P2 |
| **合规审计者** | 安全认证/法规合规的审查机构 | 完整技术文档、合规报告、可追溯 | P2 |

### 核心用户旅程

```
终端用户旅程：
  购买机器人 → 开机自检 → 账号绑定 → 语音/视觉初始化 → 日常使用
                                                              │
                                              ┌─────────────────┘
                                              ▼
                              日常服务 → 自主充电 → OTA 升级 → 持续进化

技能开发者旅程：
  Fork 仓库 → 搭建环境 → 学习 SDK → 编写技能 → 仿真验证 → 真机测试
                                                              │
                                                              ▼
                                          QooStore 发布 → 获取收益

机器人厂商旅程：
  获取参考设计 → 选型替换 → HAL 适配 → 产线搭建 → 出厂标定 → 量产上市
```

## ⚡ 非功能需求 (Non-Functional Requirements)

### NFR1: 性能

| 指标 | 目标值 | 度量方式 |
|------|--------|----------|
| **感知延迟** | < 50ms (端到端) | 从传感器采集到感知结果输出 |
| **控制闭环频率** | ≥ 1kHz | 运动控制器运行频率 |
| **推理延迟 (视觉)** | < 30ms (YOLO 640×640) | 端侧 NPU 推理 |
| **推理延迟 (LLM)** | < 2s (7B 首 Token) | 端侧 GPU 推理 |
| **技能冷启动** | < 1s | 从技能安装到首次运行 |
| **系统启动** | < 30s | 从上电到待命状态 |
| **紧急停止响应** | < 10ms | 从检测到危险到执行停止 |
| **API 响应 (P99)** | < 200ms | 云端微服务 API |
| **视频流端到端延迟** | < 100ms | 遥控场景视频回传 |

### NFR2: 可靠性

| 指标 | 目标值 | 说明 |
|------|--------|------|
| **系统可用性 (云端)** | 99.99% | 年度宕机时间 < 53 分钟 |
| **平均无故障时间 (MTBF)** | > 1,000 小时 | 机器人端连续运行 |
| **平均修复时间 (MTTR)** | < 4 小时 | 故障检测 + 修复 + 恢复 |
| **数据持久性** | 99.999999999% (11个9) | 对象存储持久性 |
| **故障自恢复** | 自动故障转移 < 30s | K8s 自愈 + 主备切换 |
| **安全完整性等级** | SIL 2 (IEC 61508) | 功能安全认证目标 |

### NFR3: 安全性

| 指标 | 目标值 | 说明 |
|------|--------|------|
| **认证强度** | 多因素认证 (TOTP + FIDO2) | 关键操作强制 MFA |
| **加密算法** | AES-256-GCM / Ed25519 / TLS 1.3 | 数据加密与通信安全 |
| **密钥管理** | HSM FIPS 140-2 Level 3 | 硬件安全模块 |
| **渗透测试** | 季度渗透测试 | 安全漏洞持续检测 |
| **漏洞响应** | 严重漏洞 24h 内修复 | 安全响应 SLA |
| **访问控制** | RBAC + 最小权限原则 | 细粒度权限控制 |

### NFR4: 可扩展性

| 指标 | 目标值 | 说明 |
|------|--------|------|
| **水平扩展** | 支持 10,000+ 机器人同时在线 | 云端微服务弹性扩缩 |
| **技能并发** | 单机器人同时运行 ≥ 8 个技能 | 多技能并行执行 |
| **插件架构** | HAL 热插拔、技能动态加载 | 无需重启系统 |
| **多机器人协作** | 支持 ≥ 100 台机器人协同 | 集群任务编排 |
| **数据规模** | 支持 PB 级数据存储 | 云端数据湖 |

### NFR5: 兼容性

| 指标 | 目标值 | 说明 |
|------|--------|------|
| **硬件平台** | ≥ 3 种 SoC (Jetson/RK3588/Snapdragon) | 芯片兼容矩阵 |
| **传感器** | ≥ 20 种主流传感器适配 | HAL 驱动覆盖 |
| **执行器** | ≥ 10 种电机/灵巧手适配 | 电机驱动覆盖 |
| **ROS 版本** | ROS 2 Humble + Iron | 最新两个 LTS 版本 |
| **Python 版本** | 3.10 - 3.12 | 主流 Python 版本 |
| **浏览器兼容** | Chrome/Firefox/Safari/Edge 最新两个大版本 | Web 前端 |

### NFR6: 可维护性

| 指标 | 目标值 | 说明 |
|------|--------|------|
| **代码覆盖率** | ≥ 80% | 单元测试覆盖率 |
| **文档完整性** | 所有公开 API 100% 文档覆盖 | API 文档自动生成 |
| **日志规范** | 结构化日志 + 分布式追踪 | OpenTelemetry 标准 |
| **健康检查** | 每个服务实现 /health + /readiness | K8s 就绪探针 |
| **依赖管理** | 第三方依赖已知漏洞 0 | 定期依赖扫描 |
| **技术债务** | < 5% 代码标记 TODO/FIXME | 技术债务量化管理 |

## 🔒 约束条件

### 技术约束

| 约束 | 描述 |
|------|------|
| **语言锁定** | 仅使用 C/C++、Python、Java、TypeScript 四种语言，禁止引入新语言 |
| **通信协议统一** | 跨项目通信统一使用 gRPC + Protobuf |
| **数据库统一** | 关系型数据统一 PostgreSQL 16，缓存统一 Redis 7 |
| **容器化** | 所有子项目必须提供 Dockerfile + Docker Compose/K8s 部署方案 |
| **构建系统** | CMake (C++)、Maven/Gradle (Java)、setuptools/pyproject.toml (Python)、npm/pnpm (TS) |
| **开源协议** | Apache 2.0 |

### 业务约束

| 约束 | 描述 |
|------|------|
| **端侧优先** | 核心计算在端侧完成，云端作为补充和协同，支持离线运行 |
| **隐私优先** | 默认本地处理，数据最小化采集，用户可控 |
| **安全基线** | 满足 ISO 13482/10218 机器人安全标准 |
| **全球化** | 中英双语优先，支持全球合规 (GDPR/CCPA/PIPL) |
| **社区驱动** | RFC 流程决策，开放式治理 |

### 资源约束

| 约束 | 描述 |
|------|------|
| **端侧算力** | 典型配置：Jetson Orin (275 TOPS) 或同等算力 |
| **端侧内存** | 最低 16GB RAM + 8GB 显存 |
| **端侧存储** | 最低 256GB SSD |
| **网络带宽** | 端云通信 ≥ 10Mbps（视频流 5Mbps+） |
| **功耗预算** | 整机 (含计算平台) < 500W |
