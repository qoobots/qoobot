# QooBot 总体交互设计

> 最后更新：2026-06-29
>
> 本文档定义 QooBot 项目的总体交互设计，涵盖用户交互模式、界面架构、交互规范与多平台体验一致性策略。

---

## 🎯 交互设计目标

| 目标 | 描述 |
|------|------|
| **自然交互** | 语音 + 视觉 + 触控多模态交互，降低机器人使用门槛 |
| **跨平台一致** | Web/Mobile/机器人端保持统一的交互语言和视觉风格 |
| **安全可控** | 关键操作需明确确认，危险操作需多重验证 |
| **开发者友好** | CLI + IDE + Web 三层开发者体验，覆盖全开发流程 |
| **无障碍优先** | 支持屏幕阅读、高对比度、键盘导航等无障碍特性 |

### 语音交互规范

| 场景 | 唤醒方式 | 示例 | 反馈 |
|------|----------|------|------|
| **唤醒** | "小Q小Q" / "Hey Qoo" | 唤醒词 | 灯光呼吸效果 + 语音 "我在" |
| **指令** | 自然语言 | "去客厅把灯打开" | 语音确认 + 状态指示灯 |
| **询问** | 疑问句 | "今天天气怎么样" | 语音回答 + 屏幕展示 |
| **确认** | 是/否 | "需要我继续吗？" | 语音 + 屏幕按钮 |
| **警告** | 主动播报 | "前方有障碍物" | 语音 + 红色灯光闪烁 |
| **紧急** | 强制指令 | "停止！" | 立即停止 + 语音 "已停止" |

## ♿ 无障碍设计

| 要求 | 标准 | 实现 |
|------|------|------|
| **屏幕阅读器** | WCAG 2.1 AA | 语义化 HTML、ARIA 标签 |
| **键盘导航** | 全部功能可键盘操作 | Tab 键导航、快捷键 |
| **色彩对比度** | ≥ 4.5:1 (正文) | 高对比度主题 |
| **文字缩放** | 200% 无溢出 | rem 单位、弹性布局 |
| **语音反馈** | 关键操作有语音反馈 | TTS 播报 |
| **减少动画** | `prefers-reduced-motion` | 尊崇系统偏好 |

## 📐 交互设计原则

| # | 原则 | 说明 |
|---|------|------|
| 1 | **渐进式披露** | 先展示核心信息，按需展开详情 |
| 2 | **容错设计** | 关键操作可撤销，提供确认步骤 |
| 3 | **反馈即视** | 每个操作都有即时反馈 (视觉/听觉/触觉) |
| 4 | **一致性** | 相同功能在端侧/Web/Mobile 使用相同交互模式 |
| 5 | **效率优先** | 高频操作提供快捷方式，减少操作步数 |
| 6 | **安全默认** | 危险操作默认阻止，需额外确认 |
| 7 | **多模态冗余** | 重要信息通过多通道同时传递 |
| 8 | **上下文感知** | 根据环境和用户状态调整交互方式 |
