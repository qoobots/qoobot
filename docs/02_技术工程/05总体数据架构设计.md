## 🎯 数据架构概述

QooBot 的数据架构遵循**端侧就近处理、云端集中管理**的原则，形成端-云-边三级数据体系：

```

│  │                      云端数据层                                 │   │
│  │  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐        │   │
│  │  │PostgreSQL │ │  Redis   │ │  MinIO   │ │ RocketMQ │        │   │
│  │  │ 关系数据  │ │ 缓存/会话 │ │ 对象存储  │ │ 消息队列  │        │   │
```

## 🗄️ 存储策略

### 存储矩阵

| 数据类别 | 端侧存储 | 云端存储 | 同步策略 |
|----------|----------|----------|----------|
| **用户账户** | Token 缓存 | PostgreSQL (主) | 登录时同步 |
| **设备信息** | 本地配置 | PostgreSQL (主) | 激活时同步 |
| **技能包** | 本地文件系统 | MinIO 对象存储 | OTA 下载/更新 |
| **AI 模型** | 本地 .qoomodel | MinIO 对象存储 | OTA 增量同步 |
| **传感器数据** | 时序缓冲 (最近 24h) | 数据湖 (按需) | 按需上传 |
| **推理结果** | 本地 SQLite | PostgreSQL (聚合) | 批量异步上传 |
| **审计日志** | 本地环形缓冲 | PostgreSQL + Kafka | 实时流式上传 |
| **诊断数据** | 本地 SQLite | PostgreSQL (汇总) | 定期批量上传 |
| **配置数据** | 本地 YAML | Nacos Config | 实时推送 |
| **会话数据** | — | Redis (集中) | 实时 |
| **知识库** | 本地向量索引 | PostgreSQL + pgvector | 增量同步 |

### 存储选型理由

| 存储系统 | 版本 | 用途 | 选型理由 |
|----------|------|------|----------|
| **PostgreSQL** | 16 | 主数据库 (全部云端服务) | 功能丰富、扩展性强、开源成熟 |
| **Redis** | 7 | 缓存/会话/速率限制/分布式锁 | 高性能、数据结构丰富 |
| **MinIO** | Latest | 对象存储 (技能包/模型/日志) | S3 兼容、开源、高性能 |
| **RocketMQ** | 5.x | 异步消息/事件驱动 | 高吞吐、低延迟、事务消息 |
| **SQLite** | 3.x | 端侧本地存储 | 零配置、嵌入式、轻量 |
| **TimescaleDB** | 2.x | 时序数据 (qoochain 质检) | PostgreSQL 扩展、专业时序 |
| **pgvector** | 0.7+ | 向量存储 (知识库/语义搜索) | PostgreSQL 扩展、无需额外服务 |

## 📐 数据治理

### 数据分类与生命周期

| 数据分类 | 示例 | 端侧保留 | 云端保留 | 归档策略 |
|----------|------|:--:|:--:|----------|
| **热数据** | 会话 Token、设备状态 | 实时 | 7 天 | 过期自动删除 |
| **温数据** | 推理统计、诊断报告 | 30 天 | 90 天 | 压缩归档到 MinIO |
| **冷数据** | 传感器回放、历史日志 | 24h | 1 年 | 按需取回 |
| **归档数据** | 审计日志、合规报告 | — | 7 年 | 对象存储 + WORM |
| **永久数据** | 用户账户、设备证书 | — | 永久 | 主数据库 |

### 数据隐私层级

| 层级 | 数据示例 | 处理位置 | 加密 |
|:--:|------|----------|:--:|
| **L0 本地** | 摄像头原始帧、麦克风音频 | 端侧 | — |
| **L1 处理中** | 人脸特征向量、语音文本 | 端侧 | — |
| **L2 去标识化** | 匿名化推理结果 | 端→云 | TLS 传输 |
| **L3 聚合** | 统计数据、联邦梯度 | 云端 | 差分隐私 (ε=1) |
| **L4 加密存储** | 用户 PII、设备密钥 | 云端 | AES-256-GCM |

### 数据质量维度

| 维度 | 度量 | 目标 |
|------|------|:--:|
| **完整性** | 必填字段覆盖率 | 100% |
| **准确性** | 数据偏差 (与真值) | < 1% |
| **一致性** | 跨服务数据一致率 | > 99.9% |
| **及时性** | 数据端到端延迟 | < 5s (遥测) |
| **唯一性** | 重复数据率 | < 0.1% |
| **有效性** | 格式/范围合规率 | > 99.9% |

## 🔒 数据安全

### 数据安全矩阵

| 安全措施 | 静态数据 | 传输中 | 使用中 |
|----------|:--:|:--:|:--:|
| **加密** | AES-256-GCM | TLS 1.3 / mTLS | TEE / 安全飞地 |
| **密钥管理** | HSM / Keychain | 自动轮换 | 会话密钥 |
| **访问控制** | RBAC + 行级安全 | JWT + mTLS | 进程隔离 |
| **审计** | PostgreSQL 审计日志 | 网关日志 | — |
| **脱敏** | 动态数据脱敏 | — | 差分隐私 |

### 备份与灾备

| 数据类型 | 备份频率 | RPO | RTO | 存储 |
|----------|----------|-----|-----|------|
| PostgreSQL | 持续 (WAL) + 每日全量 | < 1 分钟 | < 30 分钟 | 异地副本 |
| Redis | RDB 每小时 | 1 小时 | < 5 分钟 | 主从 + 哨兵 |
| MinIO | 实时复制 | 0 | < 30 分钟 | 多 AZ 纠删码 |
| 端侧 SQLite | 按需同步 | 24 小时 | — | 云端备份 |
