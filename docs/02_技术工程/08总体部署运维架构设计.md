# QooBot 总体部署运维架构设计

> 最后更新：2026-06-29
>
> 本文档定义 QooBot 项目的总体部署与运维架构，涵盖部署拓扑、容器化策略、CI/CD 流水线、监控告警、日志管理及灾备方案。

---

## 🎯 运维设计目标

| 目标 | 指标 | 说明 |
|------|------|------|
| **高可用** | 99.99% (云端) | 年度宕机时间 < 53 分钟 |
| **弹性伸缩** | 自动扩缩容 | HPA + VPA 自动调节 |
| **故障自愈** | 自动恢复 < 30s | K8s 自愈 + 主备切换 |
| **零停机部署** | 滚动更新 / 蓝绿部署 | 用户无感知升级 |
| **全链路可观测** | Metrics + Tracing + Logging | 三位一体 |
| **GitOps** | 声明式 + 版本控制 | 基础设施即代码 |

## 💾 备份与灾备

### 备份策略

| 数据类型 | 方法 | RPO | RTO | 存储 |
|----------|------|:--:|:--:|------|
| **PostgreSQL** | 持续归档 (WAL) + 每日全量 | < 1 min | < 30 min | S3/MinIO 异地 |
| **Redis** | RDB 每小时 + AOF | 1 hour | < 5 min | 主从 + 哨兵 |
| **MinIO** | 异地实时复制 | 0 | < 30 min | 多 Region 纠删码 |
| **K8s 配置** | Git + ArgoCD | < 5 min | < 10 min | Git 仓库 |
| **Docker 镜像** | ghcr.io (内置复制) | 0 | 即时 | Registry |

### 灾备演练

| 演练场景 | 频率 | 验证指标 |
|----------|:--:|----------|
| **单 AZ 故障** | 季度 | 流量自动切换到其他 AZ，服务中断 < 5 min |
| **Region 故障** | 半年 | DNS 切换至备用 Region，RTO < 30 min |
| **数据库主库故障** | 季度 | 自动 Failover，RTO < 1 min |
| **配置错误回滚** | 按月 | ArgoCD 回滚，RTO < 5 min |
| **勒索软件恢复** | 年度 | 从异地备份恢复，RTO < 4 h |

## 🚀 发布策略

### 发布类型

| 类型 | 频率 | 审批 | 回滚 |
|------|:--:|------|------|
| **Hotfix** | 按需 | 1 人审批 | 立即回滚 |
| **Patch** | 每周 | 自动 (CI 通过) | ArgoCD 一键回滚 |
| **Minor** | 每月 | 技术负责人审批 | ArgoCD 一键回滚 |
| **Major** | 按路线图 | RFC 审批 | 蓝绿部署回滚 |

### 灰度发布流程

```
v1.2.0 发布流程:

  1. Staging 验证 (全部流量)  ──▶  ✓ 通过
  2. Prod 灰度 10% (金丝雀)  ──▶  监控 30min
  3. Prod 灰度 50%            ──▶  监控 1h
  4. Prod 100%                ──▶  监控 24h
  5. 清理旧版本

  回滚条件 (任一触发):
  · 错误率 > 基准 × 2
  · P99 延迟 > 基准 × 1.5
  · 用户投诉 ≥ 3
  · 安全漏洞发现
```

### OTA 升级策略 (端侧)

```
OTA 升级流程:

  固件/模型/技能新版本
        │
        ▼
  ┌──────────────┐
  │ 增量包生成     │  (bsdiff / xdelta3)
  │ Ed25519 签名  │
  └──────┬───────┘
         │
         ▼
  ┌──────────────┐
  │ 灰度发布       │  1% → 5% → 25% → 100%
  │ 按设备批次     │  每个阶段监控 ≥ 24h
  └──────┬───────┘
         │
         ▼
  ┌──────────────────────────────┐
  │  设备端 A/B 分区升级          │
  │  下载增量包 → 写入 B 分区     │
  │  → 校验签名 → 重启到 B 分区   │
  │  → 失败自动回退到 A 分区      │
  └──────────────────────────────┘
```
