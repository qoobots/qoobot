## 🧩 系统分层详解

### 四层架构

| 层 | 职责 | 关键组件 | 平台 |
|----|------|----------|------|
| **生态应用层** | 面向终端用户、开发者、厂商的应用入口 | qooweb、qoocommunity、qoogear、qoochain | 🌐📱☁️ |
| **平台服务层** | 云端核心业务能力，连接端侧与用户 | qoocloud、qoostore、qoodev、qoocompliance | ☁️🖱️ |
| **核心能力层** | 机器人端核心智能，自主感知/决策/控制 | qoobrain、qoocore、qoosvc、qooauth | 🖥️ |
| **基础设施层** | 硬件抽象、通信总线、数据存储 | qoobody HAL、Linux RT、ROS 2 DDS、PostgreSQL | 🖥️☁️ |

## 🔗 服务间通信拓扑

### 通信矩阵

| 源 → 目标 | 协议 | 说明 |
|-----------|------|------|
| **端侧内部** | ROS 2 DDS | 实时控制总线，1kHz |
| **端侧 AI ↔ 控制** | gRPC (Protobuf) | 感知/决策结果传递 |
| **端侧 → 云端** | gRPC + MQTT | 设备上报、远程推理请求 |
| **云端 → 端侧** | gRPC + MQTT | OTA 下发、远程指令 |
| **云端内部** | gRPC (Protobuf) + REST | 微服务间通信 |
| **Web 前端 ↔ 云端** | gRPC-Web + WebSocket + REST | 用户界面 |
| **机器人间** | mTLS gRPC | 多机器人协作 |
| **第三方集成** | OAuth 2.0/OIDC + REST API | 外部系统对接 |

### 微服务治理组件

| 组件 | 技术 | 用途 |
|------|------|------|
| **服务注册/发现** | Nacos 2.x | 服务注册、健康检查、元数据管理 |
| **配置中心** | Nacos Config | 动态配置管理、灰度发布 |
| **API 网关** | Spring Cloud Gateway | 路由、限流、鉴权、跨域 |
| **熔断降级** | Sentinel | 流量控制、熔断降级、系统保护 |
| **分布式事务** | Seata 2.x | AT/TCC/Saga 模式 |
| **链路追踪** | SkyWalking | 分布式追踪、拓扑图、性能分析 |
| **指标监控** | Prometheus + Grafana | 指标采集、告警、仪表盘 |
| **日志聚合** | ELK (Elasticsearch + Logstash + Kibana) | 日志搜索与分析 |

## 📐 架构设计原则

| # | 原则 | 说明 |
|---|------|------|
| 1 | **端侧优先** | 核心计算在端侧完成，云端作为补充和协同 |
| 2 | **接口标准化** | 跨项目通信统一 gRPC + Protobuf |
| 3 | **关注点分离** | 每层职责清晰，通过标准接口交互 |
| 4 | **无状态设计** | 云端服务无状态，会话外置到 Redis |
| 5 | **弹性伸缩** | 基于 HPA/VPA 的自动扩缩容 |
| 6 | **故障隔离** | 熔断器 + 舱壁模式，防止级联故障 |
| 7 | **安全内建** | 认证/加密/审计贯穿所有服务 |
| 8 | **可观测性优先** | 每个服务内置 metrics/health/logging/tracing |
