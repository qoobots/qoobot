## 📱 QooBotOS (简称 QOS)

**QOS** 是双足人形机器人的操作系统，定位为**开源版 iPhone**：

> iPhone = 硬件 + iOS + Apple ID + App Store + iCloud + Xcode
>
> QooBot = 硬件 + QOS + QooIDaaS + QooStore + qoocloud + qoodev

| 维度 | iPhone | QooBot |
|------|--------|--------|
| **硬件** | iPhone 设备 | qoobody 参考设计（自研开源） |
| **关节电机** | — | 自研开源 — 设计图纸/BOM/工艺全部公开 |
| **灵巧手** | — | 自研开源 — 设计图纸/BOM/工艺全部公开 |
| **激光雷达** | — | 自研开源 — 设计图纸/BOM/工艺全部公开 |
| **操作系统** | iOS | QOS (QooBot OS) |
| **大脑** | A 系列芯片 + Neural Engine | qoobrain 自主意识大脑 |
| **芯片加速** | ANE / GPU | qoocore 推理芯片 |
| **账号体系** | Apple ID | QooIDaaS |
| **应用商店** | App Store | QooStore |
| **开发者工具** | Xcode + Swift | qoodev + Python/C++/TypeScript |
| **默认 App** | 电话/短信/相机/地图/健康... | 见出厂技能清单 |
| **云端服务** | iCloud | qoocloud |
| **配件生态** | MFi 认证 | MFQ 认证 (qoogear) |
| **供应链** | 苹果自有工厂 | qoochain 开放 BOM + 产线设计，第三方可制造 |
| **隐私安全** | 隐私标签 + ATT | qooregs 合规 + 隐私保护 |
| **社区** | Apple Developer | qoocommunity |

## 🧠 QOS 核心架构

QOS 是完整的人形机器人操作系统，由两大核心子系统构成：

```
QOS (QooBot OS)
│
├── 🧠 qoobrain — 自主意识大脑
│   │
│   │  感知 → 认知 → 决策 → 规划 → 控制 → 安全
│   │
│   │  端侧自主运行：机器人独立完成环境感知、任务决策、
│   │  运动规划、实时控制、安全监护，无需网络连接。
│   │
│   └── 技术栈：Python 3.11 (AI 引擎) + C++17 (实时引擎)
│       + TypeScript (Web 前端) + ROS 2 (中间件)
│
└── 🎮 qooremote — 远程人工遥控系统
    │
    │  人在回路：当自主大脑无法应对的长尾场景，
    │  人类操作员可远程接管机器人的感知、决策与控制。
    │
    ├── 遥操作 (动作捕捉 / VR / 手柄实时操控)
    ├── 远程接管 (自主 → 人工无缝切换)
    ├── 远程查看 (视频/传感器画面实时回传)
    ├── 远程对讲 (双向语音通信)
    └── 示教录制 (操作数据采集 → 训练自主模型)
```

### 为什么需要双系统？

| 场景 | 自主大脑 (qoobrain) | 远程遥控 (qooremote) |
|------|---------------------|----------------------|
| 日常家庭服务 | ✅ 端侧自主完成 | — |
| 固定产线巡检 | ✅ 自主导航+检测 | — |
| 陌生复杂环境 | ⚠️ 可能失败 | ✅ 人工接管探索 |
| 精细操作任务 | ⚠️ 成功率不足 | ✅ 遥操作精准控制 |
| 紧急异常情况 | ⚠️ 自主恢复可能不足 | ✅ 人工紧急干预 |
| 数据采集/训练 | — | ✅ 操作数据录制 |
| 安全合规要求 | — | ✅ 人工接管通道 |

> 现代人形机器人无一例外采用"自主 + 人在回路"双模架构。Tesla Optimus、Figure AI、1X NEO、Sanctuary AI 都保留了远程遥控通道。这不是技术不足的妥协，而是**安全监管要求**与**长尾场景兜底**的必然设计。

## 🎮 qooremote — 远程人工遥控系统详细设计

### 系统定位

qooremote 不是独立子项目，而是跨现有子项目的**架构概念**——将分布在 qoodev、qoosvc、qoocloud 中的遥控能力统一为一个系统视角。

### 遥控能力矩阵

| 能力 | 实现位置 | 说明 |
|------|----------|------|
| **动作捕捉遥操作** | qoodev (遥操作录制) | 操作员穿戴动捕设备，实时控制机器人全身运动 |
| **VR 沉浸式遥控** | qoodev + qoosvc | VR 头显呈现机器人第一视角画面，手柄控制 |
| **手柄/键盘遥控** | qoosvc | 基础移动与操作，用于调试与紧急接管 |
| **远程视频回传** | qoosvc + qoocloud | 机器人摄像头画面实时回传至遥控端 |
| **远程语音对讲** | qoosvc (语音交互) | 操作员与机器人周围的人双向语音通信 |
| **远程接管** | qoobrain (安全监护) | 自主 → 人工模式无缝切换，保持运动连续性 |
| **示教录制与回放** | qoodev (数据录制) | 操作员示范一次，机器人学习并复现 |
| **远程诊断** | qoocloud (设备管理) | 操作员远程查看机器人健康状态、日志、传感器数据 |

### 自主 ↔ 遥控 切换机制

```
        ┌──────────────────────┐
        │    QOS 运行模式       │
        └──────────┬───────────┘
                   │
       ┌───────────┼───────────┐
       ▼           ▼           ▼
┌──────────┐ ┌──────────┐ ┌──────────┐
│ 完全自主  │ │ 混合模式  │ │ 完全遥控  │
│          │ │          │ │          │
│ 机器人独立│ │ 自主执行  │ │ 人类完全  │
│ 感知/决策 │ │ 人类监督  │ │ 控制机器人│
│ /执行    │ │ 可随时接管│ │ 运动与操作│
└──────────┘ └──────────┘ └──────────┘
    日常        复杂场景      紧急/训练
```

切换触发条件：
- **自动切换**：碰撞风险/未知物体/任务超时/置信度过低 → 自动请求人工接管
- **人工切换**：操作员通过 qooremote 主动接管/交还控制权
- **安全切换**：安全监护检测到危险 → 立即急停并通知操作员

## 🎯 产品哲学

```
iPhone = 硬件 + iOS + Apple ID + App Store + iCloud + Xcode

QooBot = 自研开源硬件(qoobody:电机/灵巧手/激光雷达/整机)
         + QOS(qoobrain+qooremote) + QooIDaaS(qooauth)
         + QooStore(qooeco) + 云(qoocloud) + 开发者(qoodev)
         + 开放供应链(qoochain)
```

> **QOS 不是做一个机器人，而是做一套让机器人"活"起来的操作系统。**
>
> 正如 iPhone 重新定义了手机，QOS 致力于重新定义人形机器人——从一个执行固定程序的机器，变成一个可安装技能、可扩展能力、可不断进化的智能体。
>
> **QooBot 与宇树的本质区别**：宇树是"闭源版 Apple"——电机/灵巧手/激光雷达全部自研但设计不公开，只有宇树能制造。QooBot 是"开源版 Apple"——同样全部自研，但设计图纸、BOM、工艺全部公开，**全世界任何人都能基于 QooBot 设计制造自己的人形机器人**。
>
> 自主大脑让机器人拥有"意识"，远程遥控让人类拥有"兜底"——两者缺一不可，共同构成 QOS 的完整操作系统体验。
