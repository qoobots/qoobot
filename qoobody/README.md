# qoobody — 具身机器人硬件本体

> 机器人的"iPhone 硬件"：传感器、执行器、计算平台、机械结构、能源系统。

## 定位

qoobody 是 QooBot 生态的硬件参考设计，定义机器人本体的标准接口与规范，
让不同形态的机器人都能运行 qoobrain 大脑操作系统。

## 模块

| 模块 | 说明 | 状态 |
|------|------|------|
| `docs/` | 硬件接口规范（传感器、执行器、通信总线等 9 份设计文档） | ✅ 已完成 |
| `hal/` | 硬件抽象层（HAL）— 9 个头文件定义全部 C 接口 | ✅ 已完成 |
| `drivers/` | 传感器/电机驱动参考实现（IMU/CANopen 电机） | ✅ 已完成 |
| `firmware/` | 嵌入式固件（安全 MCU 固件、电机 FOC 控制） | ✅ 已完成 |
| `compute/` | 计算平台方案（SoC 选型、NPU 推理、实时配置） | ✅ 已完成 |
| `mechanical/` | 机械结构参考设计（关节模组、材料、散热） | ✅ 已完成 |
| `power/` | 能源系统（电池组、电源树、充电底座） | ✅ 已完成 |
| `safety_hw/` | 硬件安全（安全控制器、急停回路、状态指示） | ✅ 已完成 |

## 支持形态

- 🦾 机械臂（协作/工业）
- 🤖 人形机器人（双足）
- 🚗 移动底盘（轮式/履带）
- 🖐️ 灵巧手

## 与 qoobrain 的关系

```
qoobody (硬件) ←→ qoobrain (大脑OS)
       │                │
       └── HAL 接口 ────┘
```

qoobody 通过硬件抽象层（HAL）向 qoobrain 暴露标准化的传感器数据和控制接口，
qoobrain 通过 HAL 向 qoobody 下发运动指令。
