# qoobody — 具身机器人硬件本体

> 机器人的"iPhone 硬件"：传感器、执行器、计算平台、机械结构、能源系统。

## 定位

qoobody 是 QooBot 生态的硬件参考设计，定义机器人本体的标准接口与规范，
让不同形态的机器人都能运行 qoobrain 大脑操作系统。

## 模块

| 模块 | 说明 | 状态 |
|------|------|------|
| `specs/` | 硬件接口规范（传感器、执行器、通信总线） | 📋 规划中 |
| `hal/` | 硬件抽象层（HAL）参考实现 | 📋 规划中 |
| `drivers/` | 传感器/电机驱动参考实现 | 📋 规划中 |
| `firmware/` | 嵌入式固件（电机控制、安全监控） | 📋 规划中 |
| `compute/` | 计算平台方案（边缘 GPU、嵌入式板卡） | 📋 规划中 |
| `mechanical/` | 机械结构参考设计 | 📋 规划中 |
| `power/` | 能源系统（电池、电源管理） | 📋 规划中 |
| `safety_hw/` | 硬件安全（急停、碰撞检测、冗余） | 📋 规划中 |

## 支持形态

- 🦾 机械臂（协作/工业）
- 🤖 人形机器人（双足）
- 🚗 移动底盘（轮式/履带）
- 🖐️ 灵巧手

## 与 qoobrain 的关系

```
qoobody (硬件) ←→ qoobrain (大脑OS)
       │                │
       └── HAL 接口 ────┘
```

qoobody 通过硬件抽象层（HAL）向 qoobrain 暴露标准化的传感器数据和控制接口，
qoobrain 通过 HAL 向 qoobody 下发运动指令。
