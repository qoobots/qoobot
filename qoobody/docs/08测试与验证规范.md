# 08 — 测试与验证规范

> 版本：v0.1 | 最后更新：2026-02-09 | 状态：Draft

---

## 1. 概述

测试与验证是确保 qoobody 硬件规范可落地的关键环节。本规范定义硬件在环（HIL）测试框架、可靠性测试标准、性能基准套件和互操作性验证方法。

### 2.2 传感器信号模拟

| 传感器类型 | 模拟方式 | 注入接口 | 关键参数 |
|-----------|---------|---------|---------|
| RGB-D 相机 | 虚拟场景渲染 + 深度图 | MIPI CSI 回环 / USB3 虚拟设备 | 分辨率、帧率、噪声模型 |
| LiDAR | 点云射线投射 | UDP 回放 / SPI 注入 | 点数、扫描频率、距离噪声 |
| IMU | 6轴/9轴运动数据 | SPI 回放 | 采样率、偏置、随机游走 |
| 麦克风阵列 | 空间音频合成 | I2S/TDM 回放 | 通道数、采样率、SNR |
| 关节编码器 | 运动学正解算 | CAN FD / EtherCAT 虚拟从站 | 分辨率、更新率 |
| 力矩传感器 | 动力学计算 | CAN FD 虚拟从站 | 量程、噪声密度 |

### 2.3 故障注入矩阵

| 故障类型 | 注入方式 | 预期响应 | 验收标准 |
|---------|---------|---------|---------|
| 传感器数据中断 | 停止数据流 | 500ms 内检测，切换到降级模式 | 检测延迟 < 500ms |
| 传感器数据异常 | 注入噪声/偏移 | 异常检测触发，数据被丢弃 | 误检率 < 1% |
| 电机通信中断 | 断开 CAN/EtherCAT | 安全 MCU 触发抱闸 | 响应时间 < 100ms |
| 电机过流 | 注入过流信号 | 安全 MCU 切断驱动使能 | 响应时间 < 50ms |
| 电源电压跌落 | 降低供电电压 | BMS 触发低功耗模式 / 安全关机 | 过渡无数据丢失 |
| 计算板过热 | 模拟温度传感器超限 | 降频 → 安全停止 | 分级响应 |
| 安全回路断开 | 断开急停回路 | 立即切断动力，抱闸制动 | 响应时间 < 20ms |
| 时钟同步丢失 | 断开 gPTP 同步 | 降级为本地时间戳，告警上报 | 切换时间 < 1s |

## 4. 性能基准套件

### 4.1 基准测试指标

| 指标 | 定义 | 基准值 | 测量方法 |
|------|------|--------|---------|
| 感知延迟 | 传感器曝光 → qoobrain 收到数据 | < 30ms | 硬件触发 + 时间戳对比 |
| 控制带宽 | 控制指令下发频率 | ≥ 1kHz (关节) | 示波器测量 CAN/EtherCAT 帧间隔 |
| 通信吞吐量 | 各总线实际数据速率 | CAN FD ≥ 4Mbps, EtherCAT ≥ 80Mbps | 总线分析仪 |
| 端到端延迟 | 感知 → 规划 → 控制 → 执行 | < 100ms | 端到端时间戳追踪 |
| 启动时间 | 上电 → qoobrain 就绪 | < 30s | 上电时序测量 |
| 安全停止时间 | 急停触发 → 关节静止 | < 200ms | 高速相机 + 编码器 |
| 姿态估计漂移 | 1h 静态 IMU 漂移 | < 1° (偏航) | 高精度转台 |

### 4.2 基准测试自动化

```python
# 性能基准测试框架
class PerformanceBenchmark:
    """qoobody 性能基准测试套件"""

    def run_all(self):
        results = {}
        results["perception_latency"] = self.measure_perception_latency()
        results["control_bandwidth"] = self.measure_control_bandwidth()
        results["comm_throughput"] = self.measure_comm_throughput()
        results["e2e_latency"] = self.measure_e2e_latency()
        results["boot_time"] = self.measure_boot_time()
        results["safety_stop_time"] = self.measure_safety_stop()
        results["imu_drift"] = self.measure_imu_drift()
        return self.generate_report(results)

    def generate_report(self, results):
        """生成基准测试报告，与规范值对比"""
        report = []
        for name, value in results.items():
            baseline = self.baselines[name]
            status = "PASS" if value <= baseline else "FAIL"
            report.append(f"{name}: {value} (baseline: {baseline}) [{status}]")
        return "\n".join(report)
```

## 5. 互操作性测试

### 5.1 测试矩阵

| 被测接口 | 兼容组件 A | 兼容组件 B | 验证项 |
|---------|-----------|-----------|--------|
| 关节电机 CANopen | 供应商 X 电机 | 供应商 Y 电机 | 控制模式切换、SDO/PDO 通信、紧急报文 |
| 关节电机 EtherCAT | 供应商 Z 伺服 | 供应商 W 伺服 | CoE 协议、DC 时钟同步、过程数据映射 |
| RGB-D 相机 | Intel RealSense | Orbbec | 深度图格式、时间戳、外参标定 |
| LiDAR | 速腾聚创 | 禾赛科技 | 点云格式、UDP 协议、PTP 同步 |
| IMU | Bosch BMI270 | TDK ICM-42688 | SPI 协议、数据速率、偏置稳定性 |
| BMS | 供应商 A | 供应商 B | SMBus 协议、SOC 精度、保护阈值 |
| Wi-Fi/BLE 模组 | 供应商 M | 供应商 N | 吞吐量、漫游切换、功耗 |

### 5.2 即插即用验证流程

```
1. 物理连接验证
   ├── 连接器型号匹配检查
   ├── 引脚定义一致性
   └── 供电电压/电流兼容

2. 协议层验证
   ├── 设备识别 (VID/PID, 设备描述符)
   ├── 协议握手
   └── 默认参数协商

3. 功能验证
   ├── 基础数据收发
   ├── 性能基准达标
   └── 故障模式响应

4. 互操作评分
   ├── A 级：完全兼容，无需修改
   ├── B 级：需要配置参数调整
   └── C 级：需要驱动适配
```

## 6. 验证阶段与门禁

| 阶段 | 验证内容 | 门禁标准 | 输出 |
|------|---------|---------|------|
| EVT (工程验证) | 功能验证、HIL 测试 | 所有 P0 功能通过 | HIL 报告、功能测试报告 |
| DVT (设计验证) | 可靠性、EMC、性能基准 | 全部测试通过，无 P0/P1 缺陷 | 可靠性报告、EMC 报告、基准报告 |
| PVT (生产验证) | 互操作性、产线测试 | 互操作 A/B 级 ≥ 90% | 互操作报告、产线测试方案 |
| MP (量产) | 抽检、持续监控 | AQL 抽检合格率 ≥ 99% | 出货检验报告 |
