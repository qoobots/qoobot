# qoobody — 硬件参考设计 功能清单与开发进度

> 最后更新：2026-06-29 | 🎉 工程基础设施全面就绪！
>
> **定位**：人形机器人硬件规范。定义传感器接口、执行器驱动、计算平台标准——qoobrain 需要运行在符合规范的硬件上。
> **依赖**：qoobrain（需了解软件对硬件的能力要求）
> **被依赖**：qoogear、qoochain
>
> **设计理念**：硬件参考设计不是一款具体产品，而是一套"乐高积木"式的模块化规范。它定义各子系统间的接口协议、机械/电气/热学约束、以及性能基准，让任何符合规范的第三方组件都能无缝集成到 QooBot 生态中。
>
> **状态说明**：本清单追踪设计文档与参考实现的完成度。✅ 完整 = 设计文档 + 参考实现均已交付；📄 设计完成 = 设计文档已完成、参考实现待补充；🚧 进行中 = 正在开发。
>
> **工程基础设施**：✅ CMake 多平台构建系统(x86_64/aarch64/armv7-r) | ✅ 交叉编译工具链(aarch64-linux-gnu/arm-none-eabi) | ✅ GitHub Actions CI/CD(5并行任务) | ✅ Yocto 构建配方(5个bb/bbappend) | ✅ Docker 多阶段构建(dev→build→test→deploy) | ✅ .gitignore | ✅ test/测试参考指南 | ✅ docs/00文件结构.md 刷新至实际文件数
