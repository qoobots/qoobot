> 版本：v0.1 | 最后更新：2026-02-09 | 状态：Draft

---

## 1. 设计目标

QooBot 硬件参考设计（qoobody）不是一款具体产品，而是一套模块化的硬件规范。它定义各子系统间的接口协议、机械/电气/热学约束、以及性能基准。任何符合规范的第三方组件都能无缝集成到 QooBot 生态中。

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| **模块化** | 计算/感知/执行/能源/通信/结构六大子系统独立可替换 |
| **接口标准化** | 每对子系统之间定义明确的机械、电气、协议接口 |
| **安全内建** | 安全控制器独立于主控，硬实时安全回路 |
| **可制造性** | 从设计之初考虑 DFM/DFA，支持从 EVT 到 MP 的完整流程 |
| **开放生态** | 接口规范公开，第三方可开发兼容组件 |

## 4. 关键性能指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| **感知延迟** | < 5ms | 传感器曝光到数据进入 NPU |
| **控制周期** | 1kHz (关节) / 100Hz (全身) | 实时核 MCU 硬实时 |
| **推理延迟** | < 30ms (视觉) / < 100ms (LLM) | NPU 推理 + 云端 fallback |
| **通信延迟** | < 1ms (EtherCAT) / < 100μs (gPTP 同步精度) | 总线级 |
| **系统功耗** | 待机 < 20W / 典型 < 150W / 峰值 < 500W | 移动平台场景 |
| **防护等级** | 躯干 IP54 / 关节 IP67 | 家庭/轻工业 |
| **工作温度** | 0°C ~ 45°C (室内) / -10°C ~ 50°C (工业) | 分级定义 |

## 6. 开发阶段

| 阶段 | 名称 | 目标 | 产出 |
|------|------|------|------|
| EVT | 工程验证 | 验证核心架构可行性 | 开发板 + 核心传感器 + 单关节 |
| DVT | 设计验证 | 验证完整功能和接口 | 完整原型机、标定完成 |
| PVT | 生产验证 | 验证可制造性 | 小批量试产、DFM 优化 |
| MP | 量产 | 稳定批量生产 | 量产 BOM、SOP、质检标准 |
