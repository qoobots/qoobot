# 05 — 通信总线规范

> 版本：v0.1 | 最后更新：2026-02-12 | 状态：Draft

---

## 1. 概述

通信总线是 QooBot 的"神经系统"，连接计算平台、传感器、执行器和安全系统。本规范定义内部总线分配、高速数据链路、无线通信方案和时间同步网络。

---

## 2. 内部总线规范

### 2.1 总线矩阵

```
  ┌─────────────────────────────────────────────────────────┐
  │                    总线分配矩阵                           │
  │                                                         │
  │  主控 SoC                                                │
  │  ┌──────────────────────────────────────────────────┐  │
  │  │ CAN FD ×4 ─── 安全 MCU / 电机驱动器 / BMS / 工具 │  │
  │  │ EtherCAT ─── 大型伺服电机 (菊花链)               │  │
  │  │ SPI ×6  ─── IMU / ADC / 触觉传感器 / NOR Flash   │  │
  │  │ I²C ×6  ─── 温湿度 / 气压 / 环境光 / EEPROM      │  │
  │  │ UART ×4 ─── GPS / 2D LiDAR / 调试串口            │  │
  │  │ I²S/TDM ─── 麦克风阵列 / 音频 Codec              │  │
  │  │ GPIO ×N ─── 中断 / 状态指示 / 按键               │  │
  │  └──────────────────────────────────────────────────┘  │
  │                                                         │
  │  安全 MCU                                                │
  │  ┌──────────────────────────────────────────────────┐  │
  │  │ CAN FD ×2 ─── 电机驱动器 (实时控制) / 主控 SoC   │  │
  │  │ SPI ×2  ─── 编码器读取 / 安全传感器              │  │
  │  │ GPIO   ─── 急停回路 / 限位开关 / 制动器          │  │
  │  │ ADC    ─── 关节力矩传感器                        │  │
  │  └──────────────────────────────────────────────────┘  │
  └─────────────────────────────────────────────────────────┘
```

### 2.2 CAN FD 总线

| 参数 | 规格 |
|------|------|
| 物理层 | ISO 11898-2, 双绞线 120Ω 终端 |
| 速率 | 仲裁段 1Mbps, 数据段 5Mbps |
| 帧格式 | CAN FD (64 bytes payload) |
| 拓扑 | 线性总线，最多 16 节点 |
| 最大长度 | 40m @ 1Mbps / 10m @ 5Mbps |

**CAN ID 分配规则**：

| ID 范围 | 用途 | 优先级 |
|---------|------|--------|
| 0x000~0x0FF | 紧急/安全消息 | 最高 |
| 0x100~0x1FF | 实时控制指令 | 高 |
| 0x200~0x3FF | 状态反馈 | 中 |
| 0x400~0x5FF | 传感器数据 | 中 |
| 0x600~0x7FF | 配置/诊断 | 低 |

### 2.3 EtherCAT 总线

| 参数 | 规格 |
|------|------|
| 物理层 | 100BASE-TX, RJ45 / M8 连接器 |
| 拓扑 | 菊花链 (Daisy Chain) |
| 周期 | 1kHz (1ms) / 可降至 500μs |
| 同步精度 | < 1μs (分布式时钟) |
| 从站数 | 最多 65535 (实际 ≤ 32) |
| 协议 | EtherCAT CoE (CANopen over EtherCAT) |

## 3. 高速数据链路

### 3.1 高速通道分配

| 接口 | 速率 | 连接 | 用途 |
|------|------|------|------|
| PCIe 3.0 ×4 | 4 GB/s | NVMe SSD | 高速数据记录 |
| USB 3.2 Gen2 | 10 Gbps | USB Hub → 外设 | 外部传感器 / 调试 |
| MIPI CSI-2 | 2.5 Gbps/lane ×4 | 相机模组 | 视频流输入 |
| MIPI DSI | 1.5 Gbps/lane ×4 | 显示屏 | 机身显示屏 |
| GMSL2 | 6 Gbps | 远程相机 | 远距离相机连接 (>5m) |

### 3.2 高速信号完整性要求

| 要求 | 规范 |
|------|------|
| PCIe | 差分 85Ω ±10%, 等长 ±5mil |
| MIPI CSI/DSI | 差分 100Ω ±10%, 对内等长 ±2mil, 对间等长 ±10mil |
| USB 3.2 | 差分 90Ω ±7%, 等长 ±5mil |
| 参考层 | 连续地平面, 无跨分割 |

## 4. 无线通信

### 4.1 无线方案

| 技术 | 标准 | 频段 | 速率 | 距离 | 用途 |
|------|------|------|------|------|------|
| Wi-Fi | 802.11ax (Wi-Fi 6E) | 2.4/5/6 GHz | 1.2 Gbps | 50m | 高速数据/OTA |
| BLE | 5.3 | 2.4 GHz | 2 Mbps | 100m | 配件/手机连接 |
| UWB | IEEE 802.15.4z | 3.5~6.5 GHz | 6.8 Mbps | 50m | 室内定位 ±10cm |
| 5G | 3GPP Rel-17 | Sub-6 + mmWave | 1 Gbps | 蜂窝 | 广域通信 |
| NFC | ISO 14443 | 13.56 MHz | 424 kbps | 10cm | 快速配对 |

### 4.2 天线布局建议

```
  机器人顶部视图
  ┌─────────────────────────────────┐
  │    Wi-Fi ANT0        Wi-Fi ANT1 │ ← 顶部两端 (MIMO)
  │    (2.4/5/6G)        (2.4/5/6G) │
  │                                 │
  │          BLE ANT               │ ← 中部
  │                                 │
  │    UWB ANT0          UWB ANT1  │ ← 底部两端 (AOA 定位)
  │    5G ANT                       │ ← 侧面
  └─────────────────────────────────┘
  
  要求：
  - 天线间距 > λ/2 (最低工作频率)
  - 远离金属结构 > 10mm
  - 远离电机/电源 > 30mm
```

## 5. 时间同步网络

### 5.1 gPTP (IEEE 802.1AS) 架构

```
  ┌──────────────────────────────────────────────┐
  │              gPTP 时间同步网络                 │
  │                                              │
  │  ┌──────────────────────────────────────┐   │
  │  │  Grandmaster (GM)                     │   │
  │  │  主控 SoC / 以太网交换机              │   │
  │  │  高精度时钟源 (TCXO/OCXO)             │   │
  │  └──────────┬───────────────────────────┘   │
  │             │                                │
  │    ┌────────┼────────┬──────────┐           │
  │    │        │        │          │           │
  │  ┌─▼──┐ ┌──▼──┐ ┌──▼──┐  ┌───▼───┐       │
  │  │相机│ │LiDAR│ │IMU │  │音频   │       │
  │  │    │ │     │ │    │  │Codec  │       │
  │  │Slave│ │Slave│ │Slave│  │Slave  │       │
  │  └────┘ └─────┘ └─────┘  └───────┘       │
  │                                              │
  │  同步精度：< 1μs (PTP 硬件时间戳)             │
  │  同步周期：125ms (8Hz)                        │
  │  时钟等级：GM ClockClass 6 (锁定 GNSS 时)     │
  └──────────────────────────────────────────────┘
```

### 5.2 传感器时间戳同步流程

```
  1. gPTP 同步各节点时钟到 < 1μs
  2. 安全 MCU 产生 PWM 触发信号 (30/60Hz)
  3. 触发信号同时到达所有相机
  4. 每个传感器记录：
     - 采集时刻 (gPTP 时钟)
     - 到达计算平台时刻
     - 序列号
  5. 数据融合时按采集时刻对齐
```

## 6. 外部扩展接口

### 6.1 对外接口

| 接口 | 连接器 | 防护 | 用途 |
|------|--------|------|------|
| USB-C (调试) | USB-C 24pin | ESD 保护 | 调试、固件烧录 |
| Ethernet | RJ45 带变压器 | ESD + 浪涌 | 有线网络、EtherCAT |
| GPIO 扩展 | JST GH 1.25mm | 3.3V 电平 | 自定义外设 |
| UART 扩展 | JST SH 1.0mm | 3.3V TTL | 传感器扩展 |
| microSD | Push-Push 卡座 | ESD | 存储扩展 |
| HDMI | Mini HDMI | ESD | 外接显示 |

### 6.2 USB-C 调试口

```
  USB-C 引脚分配 (调试模式)：
  A5 CC1 ── 5.1kΩ 下拉 (识别为 UFP)
  A6 DP  ── UART TX (调试串口)
  A7 DN  ── UART RX (调试串口)
  A1 GND ── 地
  A4 VBUS ── 5V 输入 (可选供电)
  B5 CC2 ── NC
  B6 DP  ── SWDIO (MCU 调试)
  B7 DN  ── SWCLK (MCU 调试)
```
