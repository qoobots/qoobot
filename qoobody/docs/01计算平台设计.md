# 01 — 计算平台设计

> 版本：v0.1 | 最后更新：2026-02-08 | 状态：Draft

---

## 1. 概述

计算平台是 QooBot 的"大脑"硬件载体，承载 qoobrain 操作系统的全部计算任务。采用 **CPU + GPU + NPU + 安全 MCU** 异构架构，平衡算力、功耗与实时性。

---

## 2. SoC 选型指南

### 2.1 选型维度

| 维度 | 要求 | 权重 |
|------|------|------|
| NPU 算力 | ≥ 20 TOPS (INT8) | P0 |
| GPU 算力 | ≥ 1 TFLOPS (FP16) | P0 |
| CPU 性能 | ≥ 4×Cortex-A78 + 4×Cortex-A55 级别 | P0 |
| 功耗 | 典型 < 15W，峰值 < 30W | P0 |
| 内存带宽 | ≥ 50 GB/s | P0 |
| 视频输入 | ≥ 4×MIPI CSI-2 4-lane | P0 |
| 工业级温度 | -20°C ~ 85°C (可选) | P1 |
| 硬件安全 | TEE/TrustZone、安全启动 | P1 |

### 2.2 推荐方案

| 方案 | SoC | NPU | GPU | 适用场景 |
|------|-----|-----|-----|---------|
| **方案 A (旗舰)** | NVIDIA Jetson AGX Orin | 275 TOPS | 2048-core Ampere | 高端研发、工业场景 |
| **方案 B (主流)** | Qualcomm QCS8550 | 48 TOPS | Adreno 740 | 消费级、移动平台 |
| **方案 C (轻量)** | Rockchip RK3588 | 6 TOPS | Mali-G610 | 入门级、教育场景 |
| **方案 D (国产)** | 地平线 J6 / 华为 Ascend 310B | 96/20 TOPS | — | 国产化需求 |

### 2.3 主控 SoC 接口需求

                    ┌─────────────────────────┐

  MIPI CSI ×4 ──────┤ ISP/VI                   │

## 3. 异构计算拓扑

### 3.1 计算单元职责分配

```
┌─────────────────────────────────────────────────────────┐
│                    异构计算拓扑                           │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  ┌──────────────────────────────────────────────┐      │
│  │  CPU Cluster (A78×4 + A55×4)                 │      │
│  │  • 任务调度、行为树引擎                        │      │
│  │  • 通信管理 (DDS/ROS2)                        │      │
│  │  • 系统服务、状态管理                          │      │
│  │  • 非实时推理 (LLM 推理编排)                   │      │
│  └──────────────────┬───────────────────────────┘      │
│                     │                                   │
│         ┌───────────┼───────────┐                      │
│         │           │           │                      │
│  ┌──────▼─────┐ ┌───▼────┐ ┌───▼────────┐             │
│  │    GPU     │ │  NPU   │ │ 安全 MCU    │             │
│  │            │ │        │ │ (Cortex-R)  │             │
│  │ • 视觉渲染  │ │• 视觉  │ │ • 运动控制  │             │
│  │ • SLAM 加速 │ │  推理  │ │ • 力控回路  │             │
│  │ • 点云处理  │ │• 语音  │ │ • 急停监护  │             │
│  │ • 仿真渲染  │ │  识别  │ │ • 编码器读取│             │
│  │ • 深度估计  │ │• 多模态│ │ • 安全状态机│             │
│  └────────────┘ └────────┘ └─────────────┘             │
│                                                         │
│  数据流：                                                │
│  传感器 → NPU(推理) → CPU(决策) → MCU(执行) → 执行器     │
│  关键路径延迟：< 5ms (传感器到 NPU) + < 1ms (MCU 控制周期)│
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 3.2 计算负载分配

| 任务 | 运行位置 | 周期/延迟要求 | 说明 |
|------|---------|-------------|------|
| 目标检测 (YOLO) | NPU | 30fps, < 33ms | INT8 量化模型 |
| 语义分割 | NPU | 15fps, < 66ms | 场景理解 |
| 姿态估计 | NPU | 30fps, < 33ms | 人体/物体姿态 |
| 语音识别 (ASR) | NPU | 流式, < 200ms | 端侧小模型 |
| SLAM | GPU+CPU | 30fps, < 33ms | 视觉/激光里程计 |
| 点云配准 | GPU | 10fps, < 100ms | ICP/NDT 算法 |
| 运动规划 | CPU | 100Hz, < 10ms | 全身轨迹优化 |
| 力控伺服 | MCU | 1kHz, < 1ms | 硬实时 |
| 安全监护 | MCU | 1kHz, < 1ms | 独立于主控 |
| LLM 推理 | NPU/Cloud | < 2s (端) / < 500ms (云) | 端云混合 |

## 4. 内存/存储规范

### 4.1 内存层次

| 层级 | 类型 | 容量 | 带宽 | 用途 |
|------|------|------|------|------|
| L1/L2 Cache | SoC 内置 | — | — | CPU/GPU 缓存 |
| 系统内存 | LPDDR5/LPDDR5X | 16~32 GB | ≥ 68 GB/s | 模型/数据/帧缓冲 |
| NPU 专用内存 | 片上 SRAM | 8~32 MB | ≥ 200 GB/s | NPU 推理权重 |
| GPU 显存 | 统一内存 | 与系统共享 | — | GPU 计算 |

### 4.2 存储分级

| 层级 | 类型 | 容量 | 速度 | 用途 |
|------|------|------|------|------|
| 系统存储 | eMMC 5.1 / UFS 3.1 | 128~512 GB | ≥ 400 MB/s (UFS) | OS、系统服务 |
| 高速存储 | NVMe SSD (M.2 2280) | 256 GB~2 TB | ≥ 2 GB/s | 数据记录、模型库 |
| 可移动存储 | microSD | 256 GB~1 TB | ≥ 100 MB/s | 日志导出、技能导入 |
| 安全存储 | eMMC RPMB / eFuse | 512 KB~16 MB | — | 密钥、设备证书 |

## 5. 实时计算单元

### 5.1 安全 MCU 要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 架构 | Cortex-R52/R82 或双核锁步 Cortex-R5 | 功能安全认证 |
| 主频 | ≥ 400 MHz | 满足 1kHz 控制周期 |
| 内存 | ≥ 2 MB 片上 SRAM (ECC) | 无外部 DRAM，提高可靠性 |
| 安全等级 | SIL 2 / ASIL B 以上 | IEC 61508 / ISO 26262 |
| 通信接口 | CAN FD ×2, EtherCAT Slave, SPI, GPIO | 执行器/传感器直连 |
| 冗余 | 双核锁步 (Lockstep) | 故障检测 |

### 5.2 主控 ↔ 安全 MCU 通信

```
  主控 SoC                         安全 MCU
  ┌────────┐                    ┌──────────────┐
  │qoobrain│ ◄──── SPI ──────► │ 安全状态机    │
  │        │     (状态/指令)     │              │
  │        │                    │ 运动控制      │
  │        │ ◄──── GPIO ──────► │ (1kHz)       │
  │        │     (中断/心跳)     │              │
  │        │                    │ 力控伺服      │
  │        │ ◄── 独立看门狗 ──  │              │
  └────────┘                    └──────┬───────┘
                                       │ CAN FD / EtherCAT
                                  ┌────┴────┐
                                  │ 执行器   │
                                  │ 传感器   │
                                  └─────────┘
```

## 6. 硬件安全模块

### 6.1 安全架构

| 功能 | 实现方式 | 说明 |
|------|---------|------|
| 安全启动 | SoC BootROM → 签名校验链 | 每一级固件验签 |
| 信任根 | eFuse 烧录根公钥哈希 | 不可篡改 |
| TEE | ARM TrustZone / Qualcomm QTEE | 敏感计算隔离 |
| 密钥存储 | eMMC RPMB / 独立安全芯片(SE) | 设备私钥、证书 |
| 硬件唯一 ID | SoC 芯片 UID + 安全芯片序列号 | 设备身份绑定 |
| 防回滚 | 固件版本号单调递增 + eFuse 计数 | 防止降级攻击 |

## 7. 散热设计参考

### 7.1 热设计功耗 (TDP)

| 场景 | 计算平台功耗 | 散热方案 |
|------|------------|---------|
| 待机 | < 5W | 被动散热（散热片） |
| 典型工作 | 10~15W | 被动 + 低速风扇 |
| 峰值推理 | 20~30W | 主动散热（风扇全速） |

### 7.2 散热方案分级

| 等级 | 方案 | 适用场景 |
|------|------|---------|
| 被动散热 | 铝合金散热片 + 导热硅脂 + 机身自然对流 | 轻量/教育方案 |
| 主动散热 | 热管 + 鳍片 + PWM 调速风扇 | 主流/消费方案 |
| 液冷 | 均温板 + 微型泵 + 散热排 | 旗舰/工业方案 |
