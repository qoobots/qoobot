# qoocompliance — 技术架构设计

> 版本：v0.4 | 最后更新：2026-06-29 | 状态：Stable | 子项目：qoocompliance（法规合规）
> 技术栈：Java 17 + Spring Boot 3 + PostgreSQL + Vue 3 + Element Plus

## 2. 关键技术决策

### 2.1 法规知识结构化

```json
// 法规 → 条款 → 需求 → 检查项 的四层结构
{
  "regulation": "GDPR",
  "articles": [
    {
      "article": "Art. 17 Right to erasure",
      "requirements": [
        {
          "id": "GDPR-17-01",
          "description": "用户可请求删除其个人数据",
          "applicable_to": ["qoocloud", "qoostore", "qooauth"],
          "implementation": "提供用户数据删除 API 和 UI 入口"
        }
      ]
    }
  ]
}
```

## 4. API 设计要点

### 4.1 检查清单 API

```
POST   /api/v1/checklists/generate  — 根据市场+产品生成检查清单
GET    /api/v1/checklists/{id}       — 获取检查清单详情
PUT    /api/v1/checklists/{id}/items/{itemId} — 更新检查项状态
GET    /api/v1/checklists/progress   — 按子项目查看进度
```

### 4.2 法规监控 API

```
GET    /api/v1/regulations            — 法规列表（筛选/搜索）
GET    /api/v1/regulations/{id}       — 法规详情
GET    /api/v1/regulations/changes    — 法规变更历史
POST   /api/v1/regulations/watch      — 订阅法规监控
```

## 6. 安全设计

| 层面 | 措施 |
|:-----|------|
| 认证 | 集成 qooauth OIDC，SSO 登录 |
| 授权 | RBAC：合规官/工程师/只读 |
| 审计 | 所有操作记录到 AuditLog，不可删除 |
| 数据 | 合规证据文件加密存储、访问日志 |
| 通信 | HTTPS + API 签名验证 |
