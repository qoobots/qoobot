## 定位

qoocompliance 是 QooBot 的**合规护航**模块。机器人在全球市场上市需要跨越大量法规门槛——从安全标准到无线认证、从隐私保护到伦理审查。qoocompliance 为所有子项目提供合规框架、检查清单和认证指导，贯穿全栈。

| 维度 | 说明 |
|------|------|
| **定位** | 合规护航：安全标准、无线认证、隐私法规——贯穿所有硬件/软件/云端环节 |
| **对标** | CE/FCC 认证体系 + GDPR 合规框架 + ISO 机器人安全标准族 |
| **依赖** | 无（独立合规评估，但需理解各子项目的技术方案） |
| **被依赖** | 所有其他子项目（合规约束贯穿全栈） |
| **设计文档** | ✅ 02架构设计 | ✅ 03交互设计 | ✅ 04数据设计 | ✅ 05项目目录结构 |
| **基础设施** | ✅ Spring Boot 启动类 (`ComplianceApplication`) | ✅ Maven 构建系统 (`pom.xml`) |
| **后端 API** | ✅ 合规检查清单 6 个端点 | ✅ 法规监控 3 个端点 | ✅ OpenAPI/Swagger 文档 |

---

## 设计理念

1. **合规即设计**：合规需求不是事后补丁，而是架构设计的一部分
2. **可审计性**：所有合规决策、测试结果、审查记录均可追溯
3. **区域适配**：按目标市场（中国/欧盟/北美/日本）自动生成合规检查清单
4. **持续合规**：法规变化时自动触发差距分析和整改建议
