# 02 — 架构设计

> 版本：v0.1 | 最后更新：2026-03-15 | 状态：Draft
> 子项目：qoocompliance（法规合规）| 架构：文档驱动的合规知识库 + 自动化检查工具
> 对标：CE/FCC 认证体系 + GDPR 合规框架 + ISO 机器人安全标准族

---

## 1. 设计目标与原则

### 1.1 核心设计目标

| 目标 | P0 指标 | P1 目标 | 说明 |
|------|---------|---------|------|
| 市场准入覆盖 | 中国 + 欧盟 + 北美 | + 日本 + 韩国 + 东南亚 | 目标市场合规检查清单覆盖 |
| 法规更新响应 | < 7 天 | < 24 小时 | 法规变化→差距分析→整改提醒 |
| 合规文档生成 | 自动化 80% | 自动化 95% | 技术文档/测试报告/合规声明 |
| 认证追踪 | 实时仪表盘 | 自动延期提醒 | 各市场认证状态与有效期管理 |
| 安全标准覆盖 | ISO 13482/10218/13849 | + ISO 3691-4/TS 15066 | 机器人安全标准族 |

### 1.2 设计原则

- **合规即设计**：合规需求不是事后补丁，而是架构设计的一部分
- **可审计性**：所有合规决策、测试结果、审查记录均可追溯
- **区域适配**：按目标市场自动生成合规检查清单和差距分析
- **持续合规**：法规变化时自动触发差距分析和整改建议
- **模板驱动**：标准化的技术文档模板，减少重复劳动
- **知识库优先**：法规解读、最佳实践、案例库持续积累

### 2.2 模块职责

| 模块 | 职责 | 输入 | 输出 |
|------|------|------|------|
| 合规知识库 | 法规原文、解读、检查项、案例 | 官方法规文本 | 结构化合规要求 |
| 检查清单生成器 | 按市场/产品/阶段生成检查清单 | 目标市场 + 产品配置 | 逐项检查清单 |
| 差距分析引擎 | 对比现状与法规要求 | 产品规格 + 法规要求 | 差距报告 + 整改建议 |
| 文档生成器 | 自动生成合规文档 | 产品数据 + 模板 | CE技术文件/FCC报告等 |
| 法规监控器 | 追踪全球法规变化 | RSS/官网/订阅 | 变更通知 + 影响分析 |
| 认证追踪器 | 管理各市场认证状态 | 认证申请/结果 | 仪表盘 + 预警 |
| 风险评估器 | HAZOP/FMEA 辅助分析 | 系统设计信息 | 风险矩阵 + 缓解措施 |

## 3. 技术选型

| 组件 | 技术选型 | 理由 |
|------|---------|------|
| 核心语言 | Python 3.11 | 文档处理、数据分析、CLI 工具生态丰富 |
| 文档格式 | Markdown + YAML | 人类可读、Git 版本控制友好 |
| 知识库存储 | SQLite + JSON 文件 | 轻量、离线可用、无需服务端 |
| 文档生成 | Jinja2 模板引擎 | 灵活模板、支持条件/循环 |
| CLI 工具 | Typer + Rich | 命令行友好、美观输出 |
| 法规爬取 | httpx + BeautifulSoup4 | 轻量 HTTP + HTML 解析 |
| 报告生成 | WeasyPrint / Pandoc | Markdown→PDF、DOCX 转换 |
| 版本控制 | Git | 所有合规文档版本化 |
| 检查清单格式 | YAML | 结构化、可程序化处理 |
| 风险评估 | 自研 FMEA/HAZOP 模板 | 机器人领域专用模板 |

## 4. 合规工作流

### 4.1 新产品合规流程

```
[新产品立项] 
     │
     ▼
[确定目标市场] → 检查清单生成器 → 按市场生成合规检查清单
     │
     ▼
[产品设计阶段] → 合规即设计
     │
     ├── 安全标准评估 (HAZOP/FMEA)
     ├── 隐私设计审查 (DPIA)
     ├── 无线方案预审 (频谱/EMC)
     └── AI 伦理边界定义
     │
     ▼
[原型验证阶段] → 差距分析引擎
     │
     ├── 逐项检查 vs 法规要求
     ├── 生成差距报告
     └── 整改建议优先级排序
     │
     ▼
[认证测试阶段] → 认证追踪器
     │
     ├── 送检实验室管理
     ├── 测试报告收集
     └── 认证进度仪表盘
     │
     ▼
[市场准入] → 文档生成器
     │
     ├── CE 技术文件
     ├── FCC 测试报告
     ├── 合规声明 (DoC)
     └── 用户手册安全章节
     │
     ▼
[上市后监控] → 法规监控器
     │
     ├── 法规变化追踪
     ├── 事故报告机制
     └── 认证续期管理
```

### 4.2 法规变更响应流程

```
[法规变更检测] → 法规监控器 (每日爬取)
     │
     ▼
[影响分析] → 差距分析引擎
     │
     ├── 影响哪些产品/市场？
     ├── 影响哪些设计决策？
     └── 合规截止日期？
     │
     ▼
[整改提醒] → 通知相关团队
     │
     ├── 生成整改任务
     ├── 分配责任人
     └── 设置截止日期
     │
     ▼
[整改跟踪] → 认证追踪器
     │
     └── 整改完成 → 更新合规状态
```
