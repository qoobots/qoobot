## 2. 当前 v0.3 DTO 定义（内存存储）

### 2.1 ComplianceChecklistService DTOs

```java
// 合规项目
public record ComplianceProject(
    String projectId,           // UUID
    String name,                // 项目名称
    List<String> targetMarkets, // 目标市场列表 ["CN","EU","US","JP"]
    String status,              // DRAFT / ACTIVE / COMPLETED
    Instant createdAt
) {}

// 合规检查项
public static class ComplianceItem {
    String itemId;              // "CN-SAF-001"
    String market;              // CN / EU / US / JP
    String category;            // ROBOT_SAFETY / WIRELESS_EMC / PRIVACY_DATA / AI_ETHICS / ENVIRONMENTAL
    String priority;            // P0 / P1 / P2 / P3
    String title;               // 检查项标题
    String description;         // 检查项描述
    String status;              // NOT_STARTED / IN_PROGRESS / COMPLIANT / NON_COMPLIANT
    String evidence;            // 证据文件路径
    String notes;               // 备注
    Instant lastUpdated;
}

// 项目进度
public record ProjectProgress(
    String projectId,
    long total, long compliant, long inProgress,
    long nonCompliant, long notStarted,
    double progressPercent,
    Map<String, CategoryProgress> byCategory
) {}

// 合规报告
public record ComplianceReport(
    ComplianceProject project,
    ProjectProgress progress,
    List<ComplianceItem> criticalGaps,
    Instant generatedAt
) {}
```

## 3. v0.4 JPA 实体与 PostgreSQL 表设计

> 以下为 v0.4 已实现的 JPA Entity 与 Flyway V1 迁移 SQL（对应 `src/main/resources/db/migration/V1__init_compliance.sql`）。Entity 位于 `com.qoobot.qoocompliance.domain`，Repository 位于 `com.qoobot.qoocompliance.repository`。

### 3.1 法规表 (`compliance_regulation`)

Entity: `domain/Regulation.java` → Repository: `repository/RegulationRepository.java`

```sql
CREATE TABLE IF NOT EXISTS compliance_regulation (
    id              BIGSERIAL       PRIMARY KEY,
    regulation_id   VARCHAR(64)     NOT NULL,
    title           VARCHAR(255)    NOT NULL,
    short_name      VARCHAR(128),
    category        VARCHAR(64),
    market          VARCHAR(8),
    authority       VARCHAR(128),
    summary         TEXT,
    status          VARCHAR(32),
    effective_date  DATE,
    created_at      TIMESTAMP       NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMP       NOT NULL DEFAULT NOW()
);
CREATE UNIQUE INDEX idx_regulation_regulation_id ON compliance_regulation (regulation_id);
```

## 5. v0.3 存储现状

当前 v0.3 版本所有数据存储在 Service 类的 `HashMap` 中：

- `ComplianceChecklistService`: `Map<String, ComplianceProject> projects` + `Map<String, List<ComplianceItem>> projectItems`
- `RegulationMonitorService`: `Map<String, List<Regulation>> regulationsByMarket`
- 各评估 Service 的评估结果按需生成，不持久化

**限制**：服务重启后数据丢失，仅适合开发和原型验证阶段。

## 7. 数据安全

| 数据类别 | 敏感级别 | 访问控制 | 加密 |
|---------|---------|---------|------|
| 法规知识库 | 公开 | 全员可读 | 否 |
| 产品技术规格 | 机密 | 项目团队 | Git 权限控制 |
| 认证证书 | 机密 | 合规团队 | 文件加密存储 |
| 测试报告 | 机密 | 项目+合规团队 | 文件加密存储 |
| 差距分析报告 | 内部 | 项目+管理层 | Git 权限控制 |
| 审计记录 | 内部 | 合规团队 | 数据库加密 |
