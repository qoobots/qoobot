> 版本：v0.1 | 最后更新：2026-06-29 | 状态：Draft | 子项目：qoocompliance（法规合规）

---

## 1. 架构概览

qoocompliance 采用**合规知识库 + 管理平台**两层架构。知识库存储法规解读与检查规则，管理平台提供检查清单、文档生成、认证追踪等工具能力。

## 2. 模块职责

### 2.1 Compliance Web Portal

| 页面 | 功能 |
|:-----|------|
| Dashboard | 合规总览、认证进度、风险热图、即将到期提醒 |
| Checklist | 按市场/子项目筛选检查清单、逐项填写、进度追踪 |
| Documents | 合规文档生成（DoC/技术文件/测试报告）、版本管理 |
| Regulatory | 法规库浏览、变更通知、影响分析 |
| Audit | 审计日志、合规决策追溯、证据链管理 |
| Settings | 市场配置、用户权限、通知设置 |

### 2.2 Compliance Engine

| 引擎 | 输入 | 输出 | 核心逻辑 |
|:-----|------|------|---------|
| Checklist Generator | 目标市场 + 产品类型 | 检查清单 JSON | 法规→项目→需求条目映射 |
| Document Generator | 模板 + 产品数据 | PDF/DOCX 文档 | Jinja2 模板渲染 |
| Regulatory Monitor | 法规源订阅 | 变更告警 | diff 分析 + 影响评估 |

### 2.3 Knowledge Base

| 存储 | 内容 | 格式 |
|:-----|------|------|
| Regulation Store | 法规原文 + 解读 + 标签 | PostgreSQL JSONB |
| Checklist Store | 预定义检查模板 | YAML/JSON |
| Template Store | DOCX/HTML 模板 | 文件存储 |
| Audit Log | 操作日志 + 合规决策 | PostgreSQL |

## 4. 检查清单生成流程

```
用户选择: 目标市场=EU, 产品类型=个人服务机器人
    │
    ▼
查询法规图谱
    │
    ├──▶ 欧盟法规集: [CE MD, RED, RoHS, WEEE, GDPR, EU AI Act, ...]
    │
    ▼
按子项目分解
    │
    ├──▶ qoobody: 机械安全(15项) + EMC(8项) + RoHS(12项)
    ├──▶ qoobrain: 功能安全(10项) + AI透明度(6项)
    ├──▶ qoosvc: 导航安全(5项)
    ├──▶ qoocloud: GDPR合规(20项) + 数据安全(8项)
    └──▶ qoostore: 消费者保护(5项)
    │
    ▼
生成检查清单 (总计 ~100项)
    │
    ├── 状态: pending / in_progress / passed / failed / n/a
    ├── 责任人: 工程师A
    ├── 截止日期: 2026-09-30
    └── 证据: 测试报告链接 / 设计文档引用
```
