## 2. 安装方式

```bash
# 完整安装 (推荐)
pip install qoodev

# 仅 CLI
npm install -g qoodev-cli

# VS Code 插件
# 在 VS Code Extensions 搜索 "qoodev"

# Docker 仿真环境
docker pull qoobot/qoodev-sim:latest
docker run -it --gpus all qoodev-sim
```

## 5. 运维监控

| 监控维度 | 工具 | 指标 |
|:---------|:-----|:-----|
| **CLI 使用统计** | 匿名遥测 (opt-in) | 命令频率/错误率/耗时 |
| **插件崩溃** | VS Code Telemetry | 崩溃率/激活失败 |
| **仿真环境** | Docker stats | 资源使用/启动时间 |
| **用户反馈** | GitHub Issues | Bug报告/功能请求 |

## 6. 配置管理

```yaml
# ~/.qoodev/config.yaml
qoodev:
  sim:
    default_backend: mujoco
    scene_path: ~/.qoodev/scenes/
    
  debug:
    default_port: 9090
    auto_connect: true
    
  telemetry:
    enabled: true
    anonymous_id: "uuid"
    
  paths:
    skills: ~/.qoodev/skills/
    models: ~/.qoodev/models/
    data: ~/.qoodev/data/
```

## 7. 升级策略

| 组件 | 升级方式 | 向后兼容 |
|:-----|:---------|:---------|
| VS Code 插件 | Marketplace 自动更新 | 保证 2 个大版本兼容 |
| CLI | `pip install --upgrade qoodev` | SemVer 约束 |
| 仿真 Docker | `docker pull` 新 tag | tag 固定版本 |
| 项目模板 | `qoo init` 自动拉取最新 | 已有项目不受影响 |
