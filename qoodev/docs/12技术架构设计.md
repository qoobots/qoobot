## 1. 技术栈

| 层级 | 技术选型 | 理由 |
|:-----|:---------|:-----|
| **VS Code 插件** | TypeScript + VS Code Extension API | 生态首选 |
| **Web Dashboard** | React 18 + TypeScript + Vite | 组件化开发 |
| **3D 渲染** | Three.js + @react-three/fiber | 高性能WebGL |
| **CLI** | Python Click + Rich | 终端美化 |
| **核心服务** | Python 3.11 asyncio | 异步IO性能 |
| **仿真接口** | MuJoCo Python bindings / Isaac Sim Python API | 统一仿真接口 |
| **行为树** | BehaviorTree.CPP v4 + py_trees | C++引擎 + Python绑定 |
| **通信协议** | Protobuf + WebSocket | 高效序列化 |
| **LSP** | pygls (Language Server Protocol) | 代码智能 |
| **打包** | setuptools + PyInstaller | 跨平台分发 |

## 2. 关键技术决策

### 2.1 VS Code Extension vs 独立 IDE

| 维度 | VS Code Extension | 独立 IDE (Electron) |
|:-----|:------------------|:---------------------|
| **开发成本** | 低 | 高 |
| **用户获取** | VS Code Marketplace 零门槛 | 需独立分发 |
| **生态集成** | 与现有编辑器无缝集成 | 独立体验 |
| **维护成本** | 跟随 VS Code API | 需维护完整应用 |

**决策**：**VS Code Extension 为主 + JetBrains 插件为辅**。降低分发和维护成本。

### 2.2 仿真后端：MuJoCo + Isaac Sim 双后端

```
           ┌─────────────────┐
           │   Sim Interface  │  ← 统一抽象
           └────────┬────────┘
                    │
        ┌───────────┴───────────┐
        │                       │
  ┌─────┴─────┐          ┌─────┴─────┐
  │  MuJoCo   │          │ Isaac Sim │
  │  快速原型  │          │  高保真   │
  │  轻量级    │          │  ROS2集成 │
  │  MIT开源   │          │  NVIDIA   │
  └───────────┘          └───────────┘
```

**决策**：**双后端统一接口**。MuJoCo 用于快速迭代（MIT开源、轻量），Isaac Sim 用于高保真仿真和 ROS2 集成。

### 2.3 调试协议：WebSocket + Protobuf

| 维度 | WebSocket | TCP 原始套接字 | HTTP 轮询 |
|:-----|:----------|:--------------|:----------|
| **实时性** | ✅ 双向推送 | ✅ 双向 | ❌ 轮询延迟 |
| **Web 兼容** | ✅ 原生支持 | ❌ 不可用 | ✅ 但低效 |
| **序列化** | ✅ Protobuf | ⚠️ 自定义 | ⚠️ JSON大 |

**决策**：**WebSocket + Protobuf**。Web 原生支持，Protobuf 高效序列化适合高频传感器数据。

### 3.2 WebSocket 调试协议

```protobuf
// debug.proto
syntax = "proto3";

message DebugMessage {
    oneof payload {
        SensorFrame sensor_frame = 1;
        BehaviorTreeState bt_state = 2;
        VariableValue var_value = 3;
        LogEntry log_entry = 4;
        DebugCommand command = 5;
        DebugResponse response = 6;
    }
}

message SensorFrame {
    int64 timestamp_ns = 1;
    string sensor_name = 2;
    oneof data {
        ImageFrame image = 3;
        PointCloud lidar = 4;
        ImuData imu = 5;
        AudioChunk audio = 6;
    }
}

message BehaviorTreeState {
    repeated NodeState nodes = 1;
    string active_path = 2;
    double tick_rate_hz = 3;
}

message DebugCommand {
    enum Type {
        SET_BREAKPOINT = 0;
        CONTINUE = 1;
        STEP_OVER = 2;
        STEP_INTO = 3;
        GET_VARIABLES = 4;
        EVALUATE = 5;
    }
    Type type = 1;
    string target = 2;
}
```

## 4. 性能设计

| 优化 | 策略 | 目标 |
|:-----|:-----|:-----|
| **传感器流渲染** | Web Worker + OffscreenCanvas | 60fps |
| **点云渲染** | LOD + WebGL instancing | 100K+ 点 |
| **行为树编辑器** | 虚拟滚动 + requestAnimationFrame | 1000+ 节点流畅 |
| **仿真帧率** | GPU加速物理 + async I/O | >30fps |
| **远程调试延迟** | Protobuf + WebSocket 二进制帧 | <100ms RTT |
