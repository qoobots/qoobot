## 1. 需求概述

qoodev 是面向机器人技能开发的完整工具链，对标 Xcode + Instruments + TestFlight。机器人开发有本质特殊性：需要物理仿真、远程调试、多传感器可视化、示教到自主的渐进式流程。

## 2. 功能性需求 (FR)

### FR-100: IDE 集成

| 需求编号 | 功能 | 描述 | 优先级 |
|:----------|:-----|:-----|:-------|
| FR-101 | VS Code 插件 | 语法高亮、智能补全、代码片段、项目模板、行为树编辑器 | P0 |
| FR-102 | JetBrains 插件 | PyCharm/CLion集成、C++/Python混合调试 | P1 |
| FR-103 | 项目脚手架 | `qoo init` 一键创建技能/服务/模型项目 | P0 |
| FR-104 | 行为树编辑器 | 可视化拖拽编排行为树，节点参数配置，实时预览 | P0 |
| FR-105 | 技能清单编辑器 | 技能元数据+权限声明+隐私标签编辑 | P1 |
| FR-106 | 代码生成器 | 传感器配置→感知管线+行为树→技能代码自动生成 | P1 |

### FR-200: 仿真环境

| 需求编号 | 功能 | 描述 | 优先级 |
|:----------|:-----|:-----|:-------|
| FR-201 | 物理仿真集成 | Isaac Sim / MuJoCo 统一接口，一键启动 | P0 |
| FR-202 | 传感器仿真 | RGB-D 相机、LiDAR、IMU、麦克风阵列虚拟传感器 | P0 |
| FR-203 | 场景库 | 家居/工厂/仓库/医院预置场景，自定义场景导入 | P0 |
| FR-204 | 数字孪生 | 导入真实环境3D扫描生成仿真场景 | P2 |
| FR-205 | 域随机化 | 光照/纹理/物理/传感器/场景/相机6大类参数随机化 | P1 |
| FR-206 | 多机器人仿真 | 同一场景多机器人协作仿真 | P1 |
| FR-207 | 硬件在环 | 仿真环境连接真实硬件，虚实混合测试 | P2 |
| FR-208 | 加速仿真 | 超实时仿真（>1x实时），批量并行训练 | P2 |

### FR-300: 调试工具

| 需求编号 | 功能 | 描述 | 优先级 |
|:----------|:-----|:-----|:-------|
| FR-301 | 实时日志 | 分级日志流、过滤/搜索、时间线视图 | P0 |
| FR-302 | 行为树调试 | 节点状态高亮+转移历史+断点+步进+回放 | P1 |
| FR-303 | 变量监控 | 运行时变量可视化、张量形状/数值检查 | P0 |
| FR-304 | 断点调试 | 远程断点、条件断点、Python/C++混合栈追踪 | P1 |
| FR-305 | 传感器可视化 | 相机图像流、LiDAR点云、IMU波形实时渲染 | P0 |
| FR-306 | 3D可视化 | 机器人位姿、关节角度、力矢量3D实时显示 | P0 |
| FR-307 | 回放调试 | 记录/回放传感器+控制数据，离线复现问题 | P1 |

### FR-400: 性能剖析

| 需求编号 | 功能 | 描述 | 优先级 |
|:----------|:-----|:-----|:-------|
| FR-401 | 端到端延迟 | 感知→认知→决策→控制全链路延迟分析 | P0 |
| FR-402 | 资源监控 | CPU/GPU/NPU/内存/功耗实时监控 | P0 |
| FR-403 | 火焰图 | 调用栈火焰图，热点函数识别 | P1 |
| FR-404 | 通信剖析 | ROS2 Topic/Service 延迟和频率统计 | P1 |
| FR-405 | 推理剖析 | qoocore 推理延迟、逐层统计 | P2 |

### FR-500: CLI 工具

| 需求编号 | 功能 | 描述 | 优先级 |
|:----------|:-----|:-----|:-------|
| FR-501 | qoo init | 项目脚手架，Jinja2模板引擎 | P0 |
| FR-502 | qoo build | 技能/服务构建编译 | P0 |
| FR-503 | qoo sim | 仿真环境启动控制 | P0 |
| FR-504 | qoo debug | 调试会话管理 | P1 |
| FR-505 | qoo profile | 性能剖析 | P1 |
| FR-506 | qoo package | 技能打包（.qooskills格式） | P0 |

## 3. 非功能性需求 (NFR)

### 性能

| 指标 | 目标值 |
|:-----|:-------|
| VS Code 插件启动 | < 2s |
| 行为树编辑器渲染 | 60fps (100+节点) |
| 传感器数据流延迟 | < 100ms (WebSocket) |
| 仿真帧率 | > 30fps (物理仿真) |
| 断点调试响应 | < 200ms |
| 3D 场景渲染 | > 30fps (Three.js Web) |

### 兼容性

| 指标 | 目标值 |
|:-----|:-------|
| 操作系统 | Windows 10+, macOS 12+, Ubuntu 20.04+ |
| IDE | VS Code 1.80+, JetBrains 2023.2+ |
| Python | 3.10 - 3.12 |
| 仿真引擎 | Isaac Sim 2023.1+, MuJoCo 3.1+ |

## 4. 用户角色

| 角色 | 描述 | 核心需求 |
|:-----|:-----|:---------|
| **技能开发者** | 为QooBot开发技能应用 | VS Code插件、仿真测试、调试工具 |
| **模型训练者** | 训练AI模型 | 仿真环境、数据标注、域随机化 |
| **测试工程师** | 质量验证 | 硬件在环、回放调试、性能剖析 |
| **初学者** | 学习机器人开发 | 项目脚手架、入门教程、场景库 |
