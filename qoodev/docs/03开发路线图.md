# qoodev 开发路线图

> 最后更新：2026-06-27
>
> **总体策略**：MVP 优先 → 核心闭环 → 体验优化 → 生态扩展。先让开发者能"写出技能、在仿真中调试、打包发布"，再逐步丰富调试/分析/协作能力。

---

## 版本规划总览

```
v0.1 ──► v0.3 ──► v0.5 ──► v0.7 ──► v1.0 ──► v1.5+
 |        |        |        |        |        |
 CLI      IDE      Sim      Full     GA       Advanced
 脚手架   基础      Debug    闭环     正式版    生态
```

| 版本 | 代号 | 目标 | 预计周期 |
|------|------|------|---------|
| **v0.1** | Scaffold | CLI 脚手架 + Python SDK 基础 | 4 周 |
| **v0.3** | IDE Base | VS Code 插件基础体验 | 6 周 |
| **v0.5** | Simulate | 仿真环境集成 + 可视化调试 | 8 周 |
| **v0.7** | Full Loop | 打包签名 + CI/CD + 示教录制 | 8 周 |
| **v1.0** | GA | 正式发布，完整开发闭环 | 4 周 |
| **v1.5+** | Ecosystem | JetBrains 插件、高级分析、数字孪生 | 持续 |

## v0.1 — Scaffold（脚手架）`预计 4 周`

> **目标**：开发者能用 CLI 创建项目、编写代码、本地运行测试

### 交付物

| 模块 | 功能 | 优先级 |
|------|------|--------|
| **CLI** | `qoo init` — 项目脚手架（技能/服务/模型模板） | P0 |
| **CLI** | `qoo build` — 本地编译构建 | P0 |
| **CLI** | `qoo run` — 本地运行（连接 qoobrain 运行时） | P0 |
| **CLI** | `qoo test` — 运行单元测试 | P0 |
| **Python SDK** | `qoobot-sdk` 基础包 — 感知 API、控制 API、通信 API | P0 |
| **Python SDK** | 技能基类 `QooSkill`，生命周期管理 | P0 |
| **文档** | API 参考文档（自动生成） | P0 |
| **文档** | 快速入门教程 + 3 个示例项目 | P0 |

### 技术里程碑

- [ ] CLI 框架搭建（Python Click/Typer）
- [ ] 项目模板系统（Jinja2 模板引擎）
- [ ] `qoobot-sdk` 包结构设计与发布到 PyPI
- [ ] 与 qoobrain API 的基础对接（感知/控制接口）
- [ ] Sphinx/MkDocs 自动文档生成

## v0.5 — Simulate（仿真调试）`预计 8 周`

| 模块 | 功能 | 优先级 |
|------|------|--------|

| **仿真引擎** | 传感器仿真 — RGB-D / LiDAR / IMU 数据流 | P0 |
| **调试器** | 实时日志流 — 分级、过滤、时间线视图 | P0 |
| **调试器** | 传感器数据可视化 — 相机图像 / LiDAR 点云 / IMU 波形 | P0 |
| **调试器** | 3D 场景可视化 — 机器人位姿、关节角度实时显示 | P0 |
| **调试器** | 变量监控 — 运行时变量可视化 | P0 |
| **性能剖析** | 端到端延迟分析 — 感知→规划→控制链路 | P0 |
| **Web Dashboard** | 仿真监控面板（集成上述可视化） | P0 |
| **CLI** | `qoo sim` — 仿真环境管理命令 | P0 |

## 风险与依赖

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| qoobrain API 变更 | 所有模块需适配 | 与 qoobrain 团队保持 API 版本对齐，定义稳定 API 版本 |
| 仿真器兼容性 | 仿真环境集成受阻 | Sim Bridge 抽象层隔离差异，优先支持 MuJoCo（开源可控） |
| 3D 可视化性能 | Web Dashboard 卡顿 | Three.js + Web Worker 渲染，大场景 LOD 优化 |
| qooauth 未就绪 | 代码签名功能阻塞 | 签名接口预留，v0.7 前 qooauth 需提供基础证书服务 |
| qoocore 未就绪 | 模型编译集成阻塞 | 预留编译工具链接口，v1.5+ 阶段再集成 |
