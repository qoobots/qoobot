# qoodev 技术选型

> 最后更新：2026-06-27
>
> **选型原则**：
> 1. 优先开源、社区活跃的成熟项目
> 2. 与 qoobrain 技术栈保持一致（Python + C++）
> 3. 跨平台支持（Linux 主力开发，Windows/macOS 客户端兼容）
> 4. 低耦合，各模块可独立演进

---

## 1. CLI 工具链

| 组件 | 选型 | 版本 | 理由 |
|------|------|------|------|
| CLI 框架 | **Typer** | ≥0.9 | 基于类型提示，自动生成帮助，与 FastAPI 同源 |
| 项目模板 | **Jinja2** | ≥3.1 | 灵活模板引擎，支持继承/宏 |
| 构建系统 | **CMake** (C++) + **setuptools**/**poetry** (Python) | — | 行业标准，与 qoobrain 一致 |
| 包管理 | **Poetry** | ≥1.5 | 依赖锁定、构建发布一体化 |
| 终端美化 | **Rich** | ≥13.0 | 表格/进度条/语法高亮终端输出 |
| 配置管理 | **TOML** (pyproject.toml + qoo.toml) | — | Python 生态标准格式 |

### 备选方案

| 组件 | 备选 | 放弃理由 |
|------|------|---------|
| CLI 框架 | Click | Typer 更现代，类型提示驱动开发效率更高 |
| 包管理 | pip + requirements.txt | 缺少依赖锁定，Poetry 更专业 |

## 5. 核心服务

| 组件 | 选型 | 理由 |
|------|------|------|
| 主语言 | **Python 3.11+** | 与 qoobrain 一致，开发效率高 |
| 性能关键路径 | **C++17** + **pybind11** | 图像处理、点云处理等计算密集型 |
| RPC 框架 | **gRPC** | 高性能、强类型、多语言支持 |
| 服务定义 | **Protocol Buffers** (proto3) | gRPC 原生 IDL |
| 进程管理 | **supervisor** / **systemd** | 服务保活 |

## 6. 仿真环境

| 组件 | 选型 | 理由 |
|------|------|------|
| 主力仿真器 | **Isaac Sim** (NVIDIA) | 物理精度最高，GPU 加速，机器人开发首选 |
| 轻量仿真器 | **MuJoCo** (Google DeepMind) | 开源、轻量、快速，适合 CI/CD 和快速迭代 |
| 兼容仿真器 | **Gazebo** (Ignition) | ROS 社区标准，生态丰富 |
| 抽象层 | 自定义 **Sim Bridge** (Python ABC) | 统一接口，切换仿真器无需改代码 |
| 场景格式 | **USD** (Universal Scene Description) | Isaac Sim 原生格式，工业标准 |
| 传感器插件 | 各仿真器原生传感器 API | 通过 Sim Bridge 统一 |

### 仿真器对比

| 特性 | Isaac Sim | MuJoCo | Gazebo |
|------|-----------|--------|--------|
| 物理精度 | ★★★★★ | ★★★★☆ | ★★★☆☆ |
| GPU 加速 | ✅ 原生 | ❌ | ❌ |
| 传感器仿真 | ★★★★★ | ★★★☆☆ | ★★★★☆ |
| ROS 2 集成 | ✅ | 需自行桥接 | ✅ 原生 |
| 许可证 | 免费（NVIDIA 生态） | Apache 2.0 | Apache 2.0 |
| 学习曲线 | 陡峭 | 平缓 | 中等 |
| 推荐场景 | 正式开发 | 快速原型/CI | ROS 项目迁移 |

## 7. 调试与诊断

| 组件 | 选型 | 理由 |
|------|------|------|
| 远程调试协议 | **Debug Adapter Protocol** (DAP) | VS Code 调试标准协议 |
| Python 调试 | **debugpy** | VS Code Python 调试器后端 |
| C++ 调试 | **lldb** / **gdb** + DAP wrapper | 标准 C++ 调试器 |
| 日志收集 | **structlog** (结构化日志) | 机器可解析，便于过滤和分析 |
| 日志传输 | **WebSocket** + JSON | 实时推送 |
| 数据录制 | 自定义 `.qoodata` 格式 (Protobuf + 时间戳索引) | 高效序列化，随机访问回放 |

## 8. 性能剖析

| 组件 | 选型 | 理由 |
|------|------|------|
| Python 剖析 | **py-spy** (采样) + **cProfile** (插桩) | 无侵入采样 + 精确插桩 |
| C++ 剖析 | **perf** (Linux) + **Tracy** | 低开销实时剖析 |
| 内存分析 | **memray** (Python) + **heaptrack** (C++) | 内存分配火焰图 |
| GPU 剖析 | **Nsight Systems** (NVIDIA) | GPU 利用率、显存分析 |
| 端到端延迟 | 自定义 tracing (基于 DDS 时间戳) | 全链路延迟拆解 |

## 9. 构建与打包

| 组件 | 选型 | 理由 |
|------|------|------|
| 打包格式 | `.qooskills` = ZIP + `manifest.json` + 签名 | 简单、可扩展 |
| 清单格式 | JSON Schema (manifest.schema.json) | 可校验、IDE 支持 |
| 依赖管理 | 自定义依赖解析器 (类 pip 算法) | 版本锁定、冲突检测 |
| 代码签名 | 对接 qooauth 证书服务 API | ECDSA / Ed25519 签名 |
| 多架构 | Docker 多架构构建 + QEMU | x86 仿真 + ARM 端侧 |
