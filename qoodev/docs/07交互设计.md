# qoodev 交互设计

> 最后更新：2026-06-27
>
> **设计原则**：开发者优先、高效操作、反馈即时、渐进式披露。qoodev 面向专业机器人开发者，交互设计需兼顾 CLI 的高效性与 IDE 的直观性。

---

### 2.3 交互原则

| 原则 | 说明 |
|------|------|
| **即时反馈** | 每个命令 100ms 内显示初始状态，长任务显示进度条 |
| **渐进披露** | 默认简洁输出，`--verbose` 显示详细日志，`--json` 输出机器可读 |
| **可恢复** | 操作失败给出明确的修复建议和命令 |
| **幂等性** | 相同输入多次执行结果一致（`qoo init` 除外） |
| **干运行** | 关键操作支持 `--dry-run` 预览效果 |

| Desktop | ≥ 1440px | 三栏：侧栏 + 3D 视图 + 面板 |
| Laptop | 1024-1439px | 两栏：侧栏折叠 + 3D 视图 + 面板 |
| Tablet | 768-1023px | 单栏：Tab 切换 |
