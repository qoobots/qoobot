# qoodev 数据设计

> 最后更新：2026-03-15
>
> **设计原则**：格式标准化、高效序列化、版本可追溯、隐私安全。qoodev 涉及多种自定义数据格式——技能包、录制数据、标注数据、性能数据——需要统一的设计规范。

## 2. .qooskills — 技能打包格式

### 2.1 格式规范

```
.qooskills = ZIP 压缩包
├── manifest.json          # 技能清单（必需）
├── signature.sig          # 数字签名（可选）
├── cert.pem               # 开发者证书（可选）
├── sbom.json              # 软件物料清单（可选）
├── code/                  # 代码目录
│   ├── __init__.py
│   ├── skill.py           # 技能主入口
│   └── ...                # 其他源代码
├── models/                # 模型文件（可选）
│   └── *.qoo.model
├── resources/             # 资源文件（可选）
│   ├── meshes/            # 3D 模型
│   ├── textures/          # 纹理
│   └── audio/             # 音频
├── config/                # 配置文件（可选）
│   └── default.yaml
└── docs/                  # 文档（可选）
    └── README.md
```

### 2.3 签名格式

```
签名算法: Ed25519 (首选) / ECDSA P-256
签名输入: manifest.json 的 SHA-256
签名输出: signature.sig (DER 编码)

验证流程:
1. 解压 .qooskills → 读取 manifest.json
2. 使用 cert.pem 中的公钥验证 signature.sig
3. 验证证书链 (→ qooauth 根证书)
4. 验证证书有效期和吊销状态
```

## 3. .qoodata — 录制数据格式

### 3.1 格式规范

```
.qoodata = 自定义二进制格式

文件结构:
┌──────────────────────────────────────────────┐
│ Header (64 bytes)                            │
│  ├── magic: "QOODATA" (8 bytes)              │
│  ├── version: uint16 (2 bytes)               │
│  ├── flags: uint16 (2 bytes)                 │
│  ├── stream_count: uint32 (4 bytes)          │
│  ├── index_offset: uint64 (8 bytes)          │
│  ├── index_size: uint64 (8 bytes)            │
│  ├── metadata_offset: uint64 (8 bytes)       │
│  ├── metadata_size: uint64 (8 bytes)         │
│  └── checksum: uint64 (8 bytes)              │
├──────────────────────────────────────────────┤
│ Stream Data Blocks (交错排列)                  │
│  ├── Block 1 (timestamp + stream_id + data)  │
│  ├── Block 2                                │
│  └── ...                                     │
├──────────────────────────────────────────────┤
│ Time Index                                   │
│  ├── Stream 0: [(t0, offset), (t1, offset)]  │
│  ├── Stream 1: [(t0, offset), (t1, offset)]  │
│  └── ...                                     │
├──────────────────────────────────────────────┤
│ Metadata (JSON)                              │
│  ├── recording_info                         │
│  ├── stream_definitions                     │
│  └── user_tags                              │
└──────────────────────────────────────────────┘
```

### 3.3 Protobuf 消息定义

```protobuf
syntax = "proto3";

package qoodev.data;

// 数据块头
message DataFrame {
  uint64 timestamp_ns = 1;      // 纳秒时间戳
  uint32 stream_id = 2;          // 数据流 ID
  bytes payload = 3;             // 序列化数据
}

// 关节状态
message JointState {
  repeated string name = 1;
  repeated double position = 2;
  repeated double velocity = 3;
  repeated double effort = 4;
}

// 控制指令
message ControlCommand {
  enum CommandType {
    JOINT_POSITION = 0;
    JOINT_VELOCITY = 1;
    JOINT_TORQUE = 2;
    CARTESIAN_POSE = 3;
    CARTESIAN_VELOCITY = 4;
    GRIPPER = 5;
    MOBILE_BASE = 6;
  }
  CommandType type = 1;
  repeated double values = 2;
  map<string, double> params = 3;
}

// 事件
message Event {
  enum EventType {
    SKILL_START = 0;
    SKILL_END = 1;
    SKILL_ERROR = 2;
    USER_INTERVENTION = 3;
    COLLISION = 4;
    MILESTONE = 5;
    CUSTOM = 6;
  }
  EventType type = 1;
  string description = 2;
  map<string, string> metadata = 3;
}

// IMU 数据
message IMUData {
  double ax = 1;  // 加速度 X
  double ay = 2;  // 加速度 Y
  double az = 3;  // 加速度 Z
  double gx = 4;  // 角速度 X
  double gy = 5;  // 角速度 Y
  double gz = 6;  // 角速度 Z
}
```

### 4.5 导出格式映射

| .qooannot 字段 | COCO JSON | YOLO TXT |
|---------------|-----------|----------|
| `bbox` | `bbox: [x, y, w, h]` | `class_id x_center y_center w h` (归一化) |
| `class_id` | `category_id` | `class_id` (0-based) |
| `keypoints` | `keypoints: [x1,y1,v1,...]` | 不支持 |
| `segmentation` | `segmentation: [[x1,y1,...]]` | `x1 y1 x2 y2 ...` (归一化) |
| `image_size` | `images[].width/height` | — |

## 7. 数据库设计

### 7.1 本地 SQLite 数据库

qoodev CLI 使用 SQLite 存储项目元数据、构建缓存、依赖信息：

```sql
-- 项目表
CREATE TABLE projects (
    id          TEXT PRIMARY KEY,        -- 项目 UUID
    name        TEXT NOT NULL,
    path        TEXT NOT NULL UNIQUE,    -- 项目绝对路径
    type        TEXT NOT NULL,           -- skill | service | model
    created_at  TEXT NOT NULL DEFAULT (datetime('now')),
    updated_at  TEXT NOT NULL DEFAULT (datetime('now'))
);

-- 构建记录表
CREATE TABLE builds (
    id          TEXT PRIMARY KEY,
    project_id  TEXT NOT NULL REFERENCES projects(id),
    version     TEXT NOT NULL,
    target_arch TEXT NOT NULL,
    status      TEXT NOT NULL,           -- success | failed | in_progress
    started_at  TEXT NOT NULL,
    finished_at TEXT,
    artifact_path TEXT,                  -- .qooskills 文件路径
    artifact_size INTEGER,              -- 字节
    checksum    TEXT,
    error_log   TEXT,
    compiler_version TEXT
);

-- 依赖缓存表
CREATE TABLE dependency_cache (
    project_id  TEXT NOT NULL,
    package     TEXT NOT NULL,
    version     TEXT NOT NULL,
    resolved_at TEXT NOT NULL DEFAULT (datetime('now')),
    PRIMARY KEY (project_id, package)
);

-- 录制数据索引表
CREATE TABLE recordings (
    id          TEXT PRIMARY KEY,
    project_id  TEXT NOT NULL REFERENCES projects(id),
    name        TEXT NOT NULL,
    path        TEXT NOT NULL,
    duration_s  REAL,
    size_bytes  INTEGER,
    stream_count INTEGER,
    tags        TEXT,                   -- JSON 数组
    created_at  TEXT NOT NULL
);

-- 标注项目表
CREATE TABLE annotation_projects (
    id          TEXT PRIMARY KEY,
    name        TEXT NOT NULL,
    path        TEXT NOT NULL,
    type        TEXT NOT NULL,          -- 2d | 3d | trajectory | mixed
    total_frames INTEGER,
    labeled_frames INTEGER DEFAULT 0,
    created_at  TEXT NOT NULL,
    updated_at  TEXT NOT NULL
);

-- 崩溃报告表
CREATE TABLE crash_reports (
    id          TEXT PRIMARY KEY,
    project_id  TEXT,
    timestamp   TEXT NOT NULL,
    exception_type TEXT NOT NULL,
    message     TEXT NOT NULL,
    stack_trace TEXT,
    qoodev_version TEXT NOT NULL,
    os_info     TEXT,
    python_version TEXT,
    reported    INTEGER DEFAULT 0       -- 是否已上报
);

-- 索引
CREATE INDEX idx_builds_project ON builds(project_id, started_at);
CREATE INDEX idx_recordings_project ON recordings(project_id);
CREATE INDEX idx_crash_reports_time ON crash_reports(timestamp);
```

## 8. 数据生命周期

### 8.1 技能开发数据流

```
开发者编写代码
     │
     ▼
qoo.toml (项目配置)
     │
     ▼
qoo build
     │
     ├──► 编译产物 (build/)
     │
     ▼
qoo package
     │
     ├──► manifest.json (元数据)
     ├──► signature.sig (签名)
     └──► .qooskills (最终产物)
              │
              ▼
         qoo eco submit (发布到市场)
              │
              ▼
         云端存储 + CDN 分发
```

### 8.2 录制数据生命周期

```
机器人/仿真器
     │
     ▼
qoo record start
     │
     ▼
.qoodata 文件 (本地存储)
     │
     ├──► qoo record replay (回放调试)
     ├──► qoo record export (导出为 rosbag/mp4)
     └──► 导入标注工具
              │
              ▼
         .qooannot 项目
              │
              ├──► qoo annotate export (导出 COCO/YOLO)
              └──► 训练数据集
```

### 8.3 数据保留策略

| 数据类型 | 默认保留 | 清理策略 |
|----------|---------|---------|
| 构建产物 | 最近 10 次构建 | `qoo clean` 清理旧构建 |
| 录制数据 | 手动管理 | 用户自行删除 |
| 标注数据 | 永久 | 用户自行删除 |
| 性能剖析 | 最近 30 天 | 自动轮转 |
| 崩溃报告 | 上报后 7 天 | 自动清理 |
| 依赖缓存 | 永久 | `qoo doctor --clean-cache` |

## 9. 数据安全

### 9.1 加密策略

| 数据类型 | 加密方式 | 密钥管理 |
|----------|---------|---------|
| 录制数据 (本地) | AES-256-GCM (可选) | 用户口令派生 |
| 录制数据 (上传) | TLS 1.3 传输加密 | 云端密钥管理 |
| 崩溃报告 | 传输加密，不含敏感数据 | — |
| 签名私钥 | 本地加密存储 (PKCS#8) | 系统密钥链 |
| 配置文件 | 明文 (不含密钥) | — |

### 9.2 隐私设计

- 录制数据默认仅本地存储，不上传云端
- 传感器数据标注时自动模糊人脸/车牌
- 崩溃报告自动脱敏（去除路径、环境变量、参数值）
- 技能隐私标签在 manifest.json 中声明，用户安装前可见
