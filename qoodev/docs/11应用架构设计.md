## 2. 模块职责详解

### 2.1 L1 — UI 展示层

| 模块 | 技术 | 职责 |
|:-----|:-----|:-----|
| **VS Code Extension** | TypeScript + VS Code API | 编辑器、行为树编辑器、技能清单编辑器、调试面板 |
| **Web Dashboard** | React + TypeScript + Three.js | 3D场景可视化、传感器数据面板、性能仪表盘 |
| **CLI** | Python Click | 命令行工具入口、项目脚手架、构建打包 |

### 2.2 L2 — 核心服务层

| 模块 | 职责 |
|:-----|:-----|
| **Project Manager** | 项目创建/模板管理/依赖解析/配置管理 |
| **Sim Manager** | 仿真后端管理(Isaac Sim/MuJoCo)、场景加载、仿真控制 |
| **Debug Server** | WebSocket调试服务、断点管理、变量查询 |
| **Profile Engine** | 性能数据收集/聚合/分析 |
| **Data Manager** | 数据录制/回放/标注/版本管理 |
| **Package Builder** | .qooskills打包、签名、依赖打包 |
| **LSP Server** | 代码补全/跳转/诊断 |

### 2.3 L3 — 工具引擎层

| 模块 | 技术 | 职责 |
|:-----|:-----|:-----|
| **Code Generator** | Jinja2 模板 | 传感器配置→代码、行为树→技能骨架 |
| **BT Engine** | BehaviorTree.CPP | 行为树解析/验证/序列化 |
| **Scene Loader** | MuJoCo/Isaac Sim API | 场景文件加载/解析/预处理 |
| **Sensor Sim** | 仿真后端 API | 虚拟传感器配置/数据生成 |

### 2.4 L4 — 运行时桥接层

| 模块 | 协议 | 职责 |
|:-----|:-----|:-----|
| **WebSocket Server** | WebSocket + Protobuf | 调试指令/传感器数据/状态同步 |
| **gRPC Client** | gRPC + TLS | 调用 qoobrain/qoocore API |

## 5. 安全边界

| 边界 | 措施 |
|:-----|:-----|
| **调试通信** | WebSocket TLS + 认证 Token |
| **仿真隔离** | 容器化(MuJoCo)/独立进程(Isaac Sim) |
| **技能沙箱** | Python subprocess隔离 + 资源限制 |
| **代码签名** | 技能包 Ed25519 签名 |
| **遥测数据** | 匿名化 + 用户可选择退出 |
