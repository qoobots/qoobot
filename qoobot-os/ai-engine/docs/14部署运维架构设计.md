## 2. 构建与交叉编译

### 2.1 CMake 构建配置

```cmake
# CMakeLists.txt (简化)
cmake_minimum_required(VERSION 3.20)
project(qoocore VERSION 1.0.0 LANGUAGES CXX)

set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

# 交叉编译工具链
if(CMAKE_SYSTEM_PROCESSOR MATCHES "aarch64")
    set(TARGET_ARCH "aarch64")
elseif(CMAKE_SYSTEM_PROCESSOR MATCHES "x86_64")
    set(TARGET_ARCH "x86_64")
endif()

# 编译选项
add_compile_options(
    -Wall -Wextra -Werror
    -fstack-protector-strong
    -D_FORTIFY_SOURCE=2
    -fPIE -fPIC
)

# 目标
add_library(qoocore SHARED src/core/*.cpp)
add_library(qoocore_compiler SHARED src/compiler/*.cpp)
add_library(qoocore_hal_npu SHARED src/hal/npu/*.cpp)
add_library(qoocore_hal_gpu SHARED src/hal/gpu/*.cpp)
add_library(qoocore_hal_cpu SHARED src/hal/cpu/*.cpp)
```

### 2.2 构建目标矩阵

| 目标平台 | 工具链 | 编译器 | NPU 后端 |
|:---------|:-------|:-------|:---------|
| aarch64-linux-gnu | GCC 12+ / Clang 16+ | ARM GCC | QNN/BPU/RKNN |
| x86_64-linux-gnu | GCC 12+ / Clang 16+ | GCC/Clang | Mock NPU |
| armv7l-linux-gnueabihf | GCC 12+ | ARM GCC | — |

      - arch: x86_64

## 4. OTA 模型更新

```
云端模型仓库                     qoocore (机器人端)
     │                                 │
     │  1. 查询可用更新                 │
     │ <────────────────────────────────│
     │                                 │
     │  2. 返回版本列表 + diff大小      │
     │ ────────────────────────────────>│
     │                                 │
     │  3. 下载增量包 (.qoomodel.diff) │
     │ ────────────────────────────────>│
     │                                 │
     │                          4. SHA256 + Ed25519 校验
     │                                 │
     │                          5. 写入 B 分区
     │                                 │
     │                          6. 原子切换 A↔B
     │                                 │
     │                          7. 验证推理结果一致性
     │                                 │
     │                          8. 上报更新状态
     │ ────────────────────────────────>│
```

### 4.1 A/B 分区策略

| 分区 | 用途 | 更新流程 |
|:-----|:-----|:---------|
| **A 分区** | 当前运行版本 | 运行中只读 |
| **B 分区** | 新版本写入 | 下载→校验→写入 |
| **切换** | 符号链接原子更新 | 验证通过→ln -sf→重载 |

## 6. 故障恢复

| 场景 | 恢复策略 |
|:-----|:---------|
| **NPU 驱动崩溃** | → GPU fallback → CPU fallback |
| **模型加载失败** | → 使用 A 分区备份 → 上报错误 |
| **推理超时** | → 取消推理 → 降级到轻量模型 |
| **OOM** | → LRU 淘汰 → 卸载非关键模型 |
| **OTA 更新失败** | → 回滚到 A 分区 → 保留旧版本 |
