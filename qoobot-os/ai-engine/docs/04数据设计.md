## 2. .qoomodel 文件格式设计

### 2.1 设计目标

- 硬件无关的描述 + 硬件相关的代码：同一 .qoomodel 可跨 NPU 厂商加载
- 单文件自包含：模型权重 + 编译代码 + 配置 + 元数据
- 快速加载：mmap 友好，加载时零拷贝
- 可验证：内嵌 SHA-256 校验和
- 可扩展：FlatBuffer 格式，向前/向后兼容

### 2.2 文件二进制布局

```
[0] Magic Number (4 bytes): "QOO\\x01" (0x514F4F01)

[1] Header (64 bytes)
    version          : uint32
    flags            : uint32  (bit0=加密, bit1=量化, bit2=零拷贝友好)
    model_name       : char[32]
    input_count      : uint32
    output_count     : uint32
    target_backend    : uint32
    compiled_size     : uint64
    weight_size       : uint64
    config_size       : uint64
    metadata_size     : uint64

[2] Compiled Model (变长)
    NPU: QNN/BPU/RKNN graph
    GPU: CUDA PTX / OpenCL SPIR-V
    CPU: 优化机器码

[3] Model Weights (变长)
    量化后权重数据（紧密打包）

[4] Model Config (变长)
    YAML 格式，zstd 压缩

[5] Metadata (变长)
    JSON 格式，zstd 压缩

[6] Checksum (32 bytes)
    SHA-256 校验和
```

### 2.3 FlatBuffer Schema

```flatbuffers
namespace qoocore;

enum BackendType : uint8 {
    NPU_QNN=0, NPU_BPU=1, NPU_RKNN=2,
    GPU_CUDA=10, GPU_OPENCL=11, GPU_VULKAN=12,
    CPU_NEON=20, CPU_AVX512=21, AUTO=255,
}

enum DType : uint8 {
    FLOAT32=0, FLOAT16=1, INT32=2, INT8=3,
    INT4=4, UINT8=5, BOOL=6,
}

enum TensorLayout : uint8 { NCHW=0, NHWC=1, NC=2, NCDHW=3 }

table QuantizationInfo {
    dtype:DType=FLOAT32; scales:[float]; zero_points:[int32];
    per_channel:bool=false;
}
table TensorInfo {
    name:string; dtype:DType; layout:TensorLayout=NCHW;
    shape:[uint32]; quant:QuantizationInfo;
}
table BackendCompiledInfo {
    backend_type:BackendType; compiled_data:[ubyte]; backend_config:string;
}
table ModelWeights { weights_data:[ubyte]; weight_size:ulong; }
table QooModel {
    version:uint32=1; magic:[ubyte](fixed:4); flags:uint32;
    model_name:string; inputs:[TensorInfo]; outputs:[TensorInfo];
    compiled_infos:[BackendCompiledInfo]; target_backend:BackendType;
    weights:ModelWeights; config_yaml:string; metadata_json:string;
    benchmark_latency_ms:float; benchmark_memory_mb:float;
}
root_type QooModel;
file_identifier "QOO1";
file_extension "qoomodel";
```

## 3. Tensor 数据设计

### 3.1 内存布局

| 布局 | 形状 | 适用 |
|------|------|------|
| NCHW | [N,C,H,W] | PyTorch 默认 |
| NHWC | [N,H,W,C] | TensorFlow/GPU |
| NC | [N,C] | 全连接 |
| NCDHW | [N,C,D,H,W] | 3D 卷积 |

内部统一 NCHW，编译时自动插入 layout 转换。

### 3.2 量化格式

INT8: 每元素 1 字节，直接排列
INT4: 每字节存 2 个元素（低 4 位=偶数索引，高 4 位=奇数索引）

### 3.3 C++ 元数据结构

```cpp
struct QuantParams {
    DType target_dtype;
    std::vector<float> scales;
    std::vector<int32_t> zero_points;
    bool per_channel;
};
struct TensorMetadata {
    std::string name;
    DType dtype; TensorLayout layout;
    std::vector<int64_t> shape;
    size_t size_bytes;
    void* data; // 可能指向 ION 内存
    std::optional<QuantParams> quant_params;
};
```

## 4. 零拷贝内存设计（ION/DMA-BUF）

### 4.1 问题

传统流水线：相机→[memcpy]→主存→[memcpy]→NPU→[memcpy]→GPU （4次拷贝）
零拷贝流水线：相机──ION──►NPU──ION──►GPU （0次拷贝）

### 4.2 ION 分配器 C++ 接口

```cpp
namespace qoocore::memory {

struct IonBuffer { int fd=-1; void* cpu_ptr=nullptr; size_t size=0; };

class IonAllocator {
public:
    static Result<IonBuffer> alloc(size_t size, size_t align=4096);
    static Result<int> share_fd(const IonBuffer& buf);
    static Result<IonBuffer> import_fd(int fd);
    static Result<void*> map(const IonBuffer& buf);
    static Result<void> cache_flush(const IonBuffer& buf);
    static Result<void> cache_invalidate(const IonBuffer& buf);
    static void free(IonBuffer& buf);
};

}
```

### 4.3 使用示例

```cpp
// 相机→NPU→GPU 零拷贝
auto ion_buf = IonAllocator::alloc(640*640*3);
camera_capture(ion_buf.fd);  // 相机直接写 ION

Tensor input = Tensor::from_ion_fd(
    ion_buf.fd, {1,3,640,640}, DType::UINT8, 640*640*3);
auto output = engine.infer(model, input);  // NPU 零拷贝读取

int out_fd = engine.get_output_ion_fd(model, 0);
gpu_postprocess(out_fd);  // GPU 零拷贝读取 NPU 输出

IonAllocator::free(ion_buf);
```

## 8. 数据完整性设计

### 8.1 加载校验步骤

1. Magic Number 检查
2. 版本号检查（向前兼容）
3. SHA-256 校验
4. 后端兼容性检查
5. 权重 CRC32 检查（快速）

### 8.2 模型加密（可选）

加密区：Header + Compiled Model + Weights（AES-256-GCM）
明文区：Config (YAML) + Metadata (JSON)
签名：HMAC-SHA256（开发者密钥）

密钥：开发阶段本地 ~/.qoocore/key.pem；生产阶段 TEE 内存储。
