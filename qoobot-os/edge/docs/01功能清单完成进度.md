# qooedge — 边缘计算层 功能清单与开发进度

> 最后更新：2026-06-30 | **🎉 v0.1 — 项目骨架搭建完成，4 个核心模块已实现**
>
> **定位**：QooBot 边缘计算中间件，位于端侧 AI 引擎 (QooCore) 和云端服务 (QooCloud) 之间。管理端-云推理卸载、数据同步和多机器人 Mesh 网络。
> **依赖**：ai-engine/ (QooCore)、qoobot-service/cloud (QooCloud)
> **被依赖**：qoostore（技能运行时的高计算任务可卸载到边缘运行时）
>
> **设计理念**：对标 AWS Greengrass + Azure IoT Edge + ROS 2 DDS。qooedge 不是单一服务，而是一个三层边缘计算框架：推理层（EdgeRuntime）、决策层（EdgeOffload）、通信层（EdgeSync + EdgeMesh）。核心理念：算力不浪费、网络不拥塞、机器人不孤岛。
>
> **基础设施**：✅ CMake 构建系统 | ✅ 公共类型系统(edge_types.h) | ✅ 4 个子模块 | ✅ 主入口(daemon+CLI) | ✅ 测试框架(22 用例)
