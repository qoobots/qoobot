# qoostore-edge — 技能商店边缘运行时 功能清单与开发进度

> 最后更新：2026-06-30 | **✅ v0.2 — 核心模块 stub 替换完成，依赖解析实现，测试就绪**
>
> **定位**：QooBot 机器人端技能运行环境。管理第三方/第一方技能的安装、沙箱隔离、权限管控、运行时监控和技能间通信。
> **依赖**：qoobrain（技能通过 brain 的认知/行为树 API 与系统交互）
> **被依赖**：qoocloud（云端技能商店通过 edge runtime 分发技能）
>
> **设计理念**：对标 Android PackageManager + iOS 沙箱 + Docker 容器隔离。qoostore-edge 不只是包管理器——它是机器人端技能的完整生命周期管理框架。核心理念：技能可插拔、运行可隔离、权限可管控、依赖可解析。
>
> **设计文档**：✅ 02架构设计 | ✅ 03交互设计 | ✅ 04数据设计 | ✅ 05项目目录结构（待创建）
>
> **基础设施**：✅ CMake 构建系统 | ✅ 公共类型系统(skill_types.h) | ✅ 7个子模块骨架 | ✅ 主入口(daemon+CLI) | ✅ 测试框架 | ✅ 公共库(json_utils/sha256)
>
> **v0.1 → v0.2 变更 (2026-06-30)**：
> - ✅ JSON 工具库：替换全项目手动字符串拼接，实现 json::Value/Parser/dump
> - ✅ SHA-256 哈希：纯 C++17 实现，用于包校验 + 证书指纹
> - ✅ SkillManager：installed.json 读写使用 json_utils，安装/卸载/更新/启用全流程完善
> - ✅ SignatureVerifier：集成 SHA-256 指纹计算，JSON 格式签名文件支持
> - ✅ SkillDownloader：SHA-256 包校验、进度回退、增量更新判断
> - ✅ RuntimeMonitor：Linux /proc 资源读取支持（非 Linux 回退模拟值）
> - ✅ SkillResolver：全新模块 — 拓扑排序安装、循环依赖检测、版本约束检查
> - ✅ 主入口：qoostore-daemon（守护进程）+ qoostore-cli（命令行工具）
> - ✅ 测试套件：test_json_utils(15) + test_sha256(10) + test_skill_manager(9) + test_dependency_resolver(11)
