## 1. 架构总览

qoosvc 采用**分层微服务**架构，以 ROS2 Node 为运行单元，通过共享内存和 Topic/Service 高效通信。每个服务引擎独立运行、独立升级，但共享公共基础设施。

## 2. 模块职责

### 2.1 Speech Engine (语音引擎)

| 模块 | 职责 | 输入 | 输出 |
|:-----|------|------|------|
| Wake Word | 唤醒词检测，误唤醒抑制 | 麦克风音频流 (16kHz/16bit) | 唤醒事件 |
| ASR | 语音→文本转换 | 音频流 | 文本（含置信度） |
| NLU | 意图识别+槽位填充 | 文本 | Intent + Slots JSON |
| TTS | 文本→语音合成 | 文本+情感标签 | 音频流 (48kHz) |
| SpeakerID | 声纹识别 | 语音片段 | 说话人 ID |
| DOA | 声源方向估计 | 多通道音频 | 角度 (θ, φ) |
| Translator | 语音翻译 | 源语言语音 | 目标语言文本+语音 |

### 2.2 Navigation Engine (导航引擎)

| 模块 | 职责 | 输入 | 输出 |
|:-----|------|------|------|
| GlobalPlanner | 全局路径搜索 | 地图+起点+终点 | 全局路径 waypoints |
| LocalPlanner | 局部轨迹优化 | 全局路径+传感器 | 速度指令 v, ω |
| Exploration | 未知区域探索 | 地图+边界 | 探索目标点 |
| CollisionAvoid | 动态避障 | 传感器+轨迹 | 修正轨迹 |
| Recovery | 故障恢复 | 故障状态 | 恢复行为序列 |
| ZoneManager | 区域约束管理 | 区域定义 | 约束条件 |

### 2.3 Spatial Engine (空间引擎)

| 模块 | 职责 | 输入 | 输出 |
|:-----|------|------|------|
| SLAM | 同时定位与建图 | LiDAR+相机+IMU | 全局一致地图+位姿 |
| SemanticMap | 语义理解与标注 | 地图+qoobrain 感知 | 语义地图 |
| SceneRecon | 3D 场景重建 | 多视角 RGB-D | 稠密点云/网格 |
| PoseEstimation | 6D 位姿估计 | RGB-D+物体类型 | 物体位姿 |
| ObjectRetrieval | 物体位置查询 | 物体名称 | 位置+导航路径 |

### 2.4 Diagnostic Engine (诊断引擎)

| 模块 | 职责 | 输入 | 输出 |
|:-----|------|------|------|
| POST | 开机自检 | 设备列表 | 检测报告 |
| Predictor | 故障预测 | 传感器历史数据 | 预测告警 |
| HealthReport | 健康报告生成 | 各组件状态 | 综合健康评分 |
| Healer | 自愈执行 | 故障信息 | 恢复操作 |

## 6. 关键设计决策

| 决策 | 选择 | 理由 |
|:-----|------|------|
| 服务间通信 | ROS2 Topic/Service | 与 qoobrain 统一通信范式，零拷贝共享内存 |
| SDK 语言 | Python 优先 + C++ 绑定 | Python 降低技能开发门槛，C++ 性能关键路径 |
| 模型部署 | 端侧为主，云端 fallback | 隐私/延迟优先，复杂场景再上云 |
| 配置管理 | YAML 配置文件 + 动态更新 | 无需重启即可调整参数 |
| 日志系统 | 结构化 JSON 日志 | 便于诊断分析和云端聚合 |
