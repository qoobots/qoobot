### 2.2 进程看门狗

```yaml
# qoosvc_watchdog.yaml
watchdog:
  nodes:
    speech_node:
      heartbeat_interval: 1s
      restart_on_failure: true
      max_restarts: 3
      restart_delay: 5s
      fallback: "cloud_asr"  # 降级到云端
    
    nav_node:
      heartbeat_interval: 500ms
      restart_on_failure: true
      max_restarts: 5
      safety_stop: true       # 崩溃时立即停止移动
    
    spatial_node:
      heartbeat_interval: 1s
      restart_on_failure: true
```

```

  ▼

  └──▶ 冷重启参数（模型路径、硬件绑定等）
```

## 4. 监控与可观测性

### 4.1 关键指标

| 类别 | 指标 | 采集方式 | 告警阈值 |
|:-----|------|---------|:------:|
| 🎙️ 语音 | 唤醒延迟、ASR 实时率 | 内嵌计时器 | 延迟>200ms |
| 🗺️ 导航 | 规划耗时、避障频率 | Nav2 内置 metrics | 规划>200ms |
| 🏠 空间 | 定位协方差、回环检测率 | SLAM diagnostics | 协方差>0.1 |
| 🩺 诊断 | CPU温度、电池电压 | 传感器读取 | 温度>85°C |
| 💾 资源 | CPU/内存/磁盘使用率 | cgroup/prometheus | CPU>80% |

## 5. OTA 更新

### 5.1 更新流程

```
云端发布新版本
      │
      ▼
设备检查更新 (定时/推送)
      │
      ▼
下载增量包 (.delta)
      │
      ▼
校验签名 (Ed25519)
      │
      ▼
预加载到备用分区
      │
      ▼
切换软链接 → 重启节点
```

### 5.2 A/B 分区策略

```
/opt/qoo/qoosvc/
├── v1.0.0/              # A 分区（当前）
│   ├── speech_engine
│   ├── nav_engine
│   └── ...
├── v1.0.1/              # B 分区（待切换）
│   ├── speech_engine
│   ├── nav_engine
│   └── ...
└── current -> v1.0.0/   # 符号链接
```

## 6. 灾备策略

| 场景 | RTO | RPO | 措施 |
|:-----|:---:|:---:|------|
| 单个节点崩溃 | <10s | 0 | 看门狗自动重启 |
| 全部节点崩溃 | <30s | 0 | systemd 服务重启 |
| 存储故障 | <5min | <1min | 配置从云端恢复 |
| 系统盘损坏 | <30min | <1h | 镜像烧录 + 云端恢复 |
| 硬件损坏 | N/A | <24h | 送修/更换，云端恢复配置 |
