# qoosvc — 需求分析

> 版本：v0.1 | 最后更新：2026-06-29 | 状态：Draft | 子项目：qoosvc（系统服务）
> 对标：iOS Core Services / Android System Services | 技术栈：Python 3.11 + C++17 + ROS2 + Protobuf

---

## 1. 需求概述

qoosvc 是 QooBot 的"系统服务层"，为所有机器人应用提供共享基础能力。它不是面向最终用户的技能，而是语音交互、自主导航、空间理解、设备诊断等系统级能力的统一聚合与暴露层。

---

## 2. 功能需求 (FR)

### FR-100：语音交互子系统
| 编号 | 功能需求 | 描述 | 优先级 |
|:-----|---------|------|:------:|
| FR-101 | 语音唤醒 | 支持自定义唤醒词"Hey QooBot"，误唤醒率<1次/24h，支持中/英/日多语言 | P0 |
| FR-102 | 自动语音识别 (ASR) | 端侧流式 ASR，实时率<0.3，中文普通话+英语+日语，-5dB SNR 下字准确率>95% | P0 |
| FR-103 | 自然语言理解 (NLU) | 意图识别（50+领域）、槽位填充、多轮对话上下文管理，意图准确率>92% | P0 |
| FR-104 | 语音合成 (TTS) | 自然语音合成、情感语调（高兴/平静/关切）、多音色切换（男/女/童声） | P0 |
| FR-105 | 声源定位 | 4-7麦阵列 DOA 估计，角度精度<10°，支持说话人方向跟踪 | P1 |
| FR-106 | 说话人识别 | 声纹识别区分家庭成员，注册3s即可，等误率<2% | P1 |
| FR-107 | 多语言翻译 | 中↔英↔日实时语音翻译，延迟<2s，字幕+语音双通道输出 | P2 |
| FR-108 | 离线语音 | 无网络环境下基础指令集（停止/过来/充电/巡逻等50+条），离线唤醒+ASR+NLU | P1 |

| FR-204 | 多楼层导航 | 跨楼层路径（含电梯/楼梯），楼层切换自动注册地图 | P2 |

| FR-206 | 窄道通行 | 宽度<1.2x机器人宽度狭窄空间通过策略，成功率>95% | P2 |
| FR-207 | 区域管理 | 定义禁区/限速区/优先区，支持多边形/语义两种定义方式 | P1 |

## 5. 约束条件

| 约束 | 说明 |
|:-----|------|
| 硬件约束 | 依赖麦克风阵列（4-7麦）、激光雷达、IMU、RGB-D 相机 |
| 平台约束 | 运行在 qoobrain OS 之上，通过 ROS2 通信 |
| 依赖约束 | 依赖 qoobrain（感知/认知输出）、qoobody（硬件接口） |
| 合规约束 | 语音数据需满足 GDPR/PIPL，导航需满足 ISO 3691-4 安全标准 |
| 离线能力 | 核心功能（唤醒/基础ASR/基础导航）需离线可用 |

|:-----|---------|

| 导航 | 100次室内导航成功率>99%，平均完成时间在理论最优150%内 |
| 健康自检 | POST 30s内完成全部传感器/电机/电池检测，误报率<1% |
