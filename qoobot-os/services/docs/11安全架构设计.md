# qoosvc — 安全架构设计

> 版本：v0.1 | 最后更新：2026-06-29 | 状态：Draft | 子项目：qoosvc（系统服务）

---

## 1. 安全设计原则

1. **隐私优先**：语音、地图等敏感数据默认端侧处理，不上传云端
2. **最小权限**：每个服务模块仅获得所需的最小权限集
3. **纵深防御**：网络→服务→数据→硬件多层安全防护
4. **安全默认**：不安全的行为需要显式 opt-in，不能是默认配置

## 2. 威胁模型

### 2.1 威胁表面

```
┌──────────────────────────────────────────────────────────┐
│                      威胁表面                              │
├──────────────┬───────────────────────────────────────────┤
│ 🎤 语音通道   │ 恶意唤醒词注入、录音窃听、语音指令重放       │
│ 📡 通信通道   │ ROS2 消息窃听/篡改、gRPC MITM               │
│ 🗺️ 空间数据   │ 地图数据泄露（家庭布局）、定位欺骗            │
│ 🩺 诊断通道   │ 诊断数据泄露、恶意固件注入                    │
│ 📱 外部接口   │ 第三方技能越权、SDK 接口滥用                  │
│ 💾 存储       │ 敏感数据明文存储、模型文件篡改                │
└──────────────┴───────────────────────────────────────────┘
```

### 2.2 威胁缓解矩阵

| 威胁 | 严重性 | 可能性 | 缓解措施 |
|:-----|:-----:|:-----:|---------|
| 语音窃听 | 🔴 高 | 🟡 中 | 端侧处理、不存储原始音频、TLS 传输 |
| ROS2 消息篡改 | 🟠 中 | 🟢 低 | DDS Security (Auth+Encryption) |
| 地图数据泄露 | 🔴 高 | 🟡 中 | 端侧存储、加密、云端仅上传匿名化特征 |
| 定位欺骗 | 🟠 中 | 🟢 低 | 多传感器融合校验、IMU 一致性检查 |
| 第三方技能越权 | 🔴 高 | 🟡 中 | 沙箱隔离、权限声明+动态授权 |
| 模型文件篡改 | 🔴 高 | 🟢 低 | Ed25519 签名验证、安全启动链 |
| 指令重放攻击 | 🟡 低 | 🟡 中 | 时间戳+nonce、声纹二次确认 |

## 3. 语音安全

### 3.1 语音数据保护

```
麦克风 ──▶ 音频缓冲（环形，最多10s）
                │
    ┌───────────┼───────────┐
    ▼           ▼           ▼
 唤醒检测     ASR 推理   丢弃（不存储）
 (本地)       (本地)
    │           │
    ▼           ▼
  唤醒事件   识别文本（不关联原始音频）
```

### 3.2 唤醒词安全

| 防护 | 机制 |
|:-----|------|
| 防误唤醒 | 声纹绑定：仅注册用户的声音可唤醒 |
| 防注入 | 声学指纹校验，拒绝电子设备播放的唤醒词 |
| 防重放 | 音频信号 time-of-flight 校验（物理距离验证） |

### 3.3 关键指令二次确认

```
普通指令: "打开灯光" → 直接执行
                      ↓
敏感指令: "开门" → 声纹确认 → [通过] → 执行
                             → [未通过] → "请确认身份"
                      ↓
危险指令: "解锁所有门" → 声纹 + 密码 → 执行
```

## 4. 通信安全

### 4.1 ROS2 DDS Security

```
┌──────────────────────────────────────┐
│         ROS2 节点通信安全             │
├──────────────────────────────────────┤
│ · Authentication：基于证书的节点身份  │
│ · Access Control：Topic 级权限控制    │
│ · Encryption：AES-256-GCM 消息加密   │
│ · Integrity：HMAC-SHA256 消息完整性   │
└──────────────────────────────────────┘
```

### 4.2 gRPC 安全

| 层面 | 措施 |
|:-----|------|
| 传输 | mTLS 双向认证 |
| 认证 | JWT Bearer Token（由 qooauth 签发） |
| 授权 | 基于 Scope 的权限检查 (speech:read, nav:write) |
| 审计 | 所有 API 调用记录到审计日志 |

## 6. 数据安全

### 6.1 敏感数据分级

| 级别 | 数据类型 | 加密策略 | 存储位置 |
|:----:|---------|---------|---------|
| L3 (公开) | TTS 合成音色名称、导航状态 | 无 | 本地/云端 |
| L2 (内部) | 系统日志、导航路径历史 | 传输加密 | 本地加密存储 |
| L1 (敏感) | 地图数据、声纹特征、NLU 模型 | 传输+存储加密 | 仅本地 |
| L0 (机密) | 原始音频、家庭成员信息、密码 | 端到端加密 | 仅本地内存 |

### 6.2 密钥管理

```
qooauth 密钥服务
      │
      ├── 设备密钥 (TPM 存储，不可导出)
      ├── 服务密钥 (定期轮换，HSM 管理)
      └── 会话密钥 (内存，进程退出即销毁)
```

## 8. 安全事件响应

| 事件类型 | 响应时间 | 措施 |
|:---------|:------:|------|
| 未授权访问检测 | <1min | 锁定服务、通知管理员 |
| 语音重放攻击 | <100ms | 忽略指令、记录事件 |
| ROS2 异常消息 | <10ms | 丢弃消息、告警 |
| 模型文件完整性失败 | 启动时 | 拒绝加载、还原备份 |
