# 09 — 操作系统配置规范

> 版本：v0.1 | 最后更新：2026-03-16 | 状态：Draft

---

## 1. 概述

QooBot 计算平台运行定制版 Linux 操作系统，基于 Ubuntu 22.04 LTS 或 Yocto 构建。本规范定义内核配置、实时性补丁、设备树定制和安全基线，确保系统满足机器人控制的硬实时要求。

---

## 2. 内核配置

### 2.1 基线内核版本

| 项目 | 规格 |
|------|------|
| 主线内核 | Linux 5.15 LTS / 6.1 LTS |
| 实时补丁 | PREEMPT_RT 对应版本 |
| 板级支持 | SoC BSP 提供的设备树和驱动 |
| 编译器 | GCC 11.4+ / LLVM 14+ |

### 2.2 关键内核配置项

参见 `os/qoo_kernel_defconfig`。

**必须启用的选项：**

| 配置项 | 说明 |
|--------|------|
| `CONFIG_PREEMPT_RT=y` | 实时补丁（硬实时） |
| `CONFIG_HIGH_RES_TIMERS=y` | 高精度定时器 |
| `CONFIG_NO_HZ_FULL=y` | 全动态 tick（降低延迟） |
| `CONFIG_CPU_FREQ_GOV_PERFORMANCE=y` | 性能调速器（禁用变频） |
| `CONFIG_PCIEASPM=y` | PCIe 电源管理 |
| `CONFIG_CAN=y` | CAN 总线子系统 |
| `CONFIG_CAN_FD=y` | CAN FD 支持 |
| `CONFIG_ETHTOOL_SUPPORT=y` | EtherCAT 依赖 |
| `CONFIG_GPIO_CDEV=y` | 新版 GPIO 字符设备 ABI |
| `CONFIG_SPI_SPIDEV=y` | SPI 用户态接口 |
| `CONFIG_I2C_CHARDEV=y` | I²C 用户态接口 |
| `CONFIG_VIDEO_V4L2=y` | V4L2 视频采集框架 |
| `CONFIG_VIDEO_MIPI_CSI2=y` | MIPI CSI-2 支持 |
| `CONFIG_SND_SOC=y` | ALSA SoC 音频框架 |
| `CONFIG_USB_XHCI_HCD=y` | USB 3.x 主机控制器 |
| `CONFIG_NET_PTP_CLASSIFY=y` | PTP 硬件时间戳 |
| `CONFIG_MMC=y` | MMC/eMMC 驱动 |
| `CONFIG_USB_STORAGE=y` | USB 存储（U 盘/移动硬盘） |

**必须禁用的选项（降低延迟）：**

| 配置项 | 说明 |
|--------|------|
| `CONFIG_CPU_FREQ=n` | 禁用 CPU 频率调节（或固定性能模式）|
| `CONFIG_SMP=n`（可选）| 单核启动模式下禁用 SMP（降低延迟）|
| `CONFIG_HOTPLUG_CPU=n` | 禁用 CPU 热插拔 |
| `CONFIG_NUMA=n` | 禁用 NUMA（非均匀内存访问）|
| `CONFIG_ACPI=n`（嵌入式）| 嵌入式中通常不需要 ACPI |
| `CONFIG_PM_SLEEP=n` | 禁用系统挂起（机器人不需睡眠）|

### 2.3 实时内核参数调优

```
# /etc/sysctl.d/99-qoobot-rt.conf

# 禁用 swap（避免页面换入换出引入延迟）
vm.swappiness = 0

# 提高实时进程优先级上限
kernel.sched_rt_runtime_us = 950000
kernel.sched_rt_period_us = 1000000

# 禁用透明大页（避免页面压缩导致延迟尖峰）
vm.nr_overcommit_hugepages = 0

# 允许 mlockall() 锁定所有内存
vm.overcommit_memory = 2
```

### 2.4 CPU 隔离（CPU Isolation）

将某些 CPU 核心从内核调度器中隔离出来，专用于实时控制任务：

```
# 内核启动参数（GRUB / U-Boot）
isolcpus=2,3 nohz_full=2,3 rcu_nocbs=2,3

# 解释：
#   isolcpus=2,3    → CPU 2,3 不从 CFS 调度器分配任务
#   nohz_full=2,3    → CPU 2,3 停止周期性 tick
#   rcu_nocbs=2,3    → CPU 2,3 的 RCU 回调在其它 CPU 执行
```

用户态实时线程通过 `sched_setaffinity()` 绑定到隔离核心。

## 3. PREEMPT_RT 实时补丁

### 3.1 补丁适配流程

参见 `os/qoo_patch_rt.md`。

**步骤概览：**

```
1. 获取主线内核源码（与 SoC BSP 版本匹配）
2. 下载对应版本的 PREEMPT_RT 补丁
   - 内核 5.15  → RT 补丁 5.15-rt
   - 内核 6.1   → RT 补丁 6.1-rt
3. 将 RT 补丁打到 BSP 内核源码上
   $ patch -p1 < patch-5.15-rt.patch
4. 配置内核：启用 CONFIG_PREEMPT_RT
5. 编译并烧录到目标板
6. 验证实时性：cyclictest
   $ cyclictest -t 5 -p 80 -n -i 1000 -D 1h
   → 期望：最大延迟 < 50μs（x86/ARM 高性能平台）
```

### 3.2 实时性验证基准

| 平台 | cyclictest 平均延迟 | 最大延迟（合格线） |
|------|-------------------|-------------------|
| NVIDIA Jetson AGX Orin | < 10μs | < 50μs |
| Rockchip RK3588 | < 15μs | < 100μs |
| 树莓派 CM4 | < 20μs | < 150μs |

## 4. 设备树定制

### 4.1 设备树概述

设备树（Device Tree）描述硬件平台上的外设连接关系，由 Bootloader 在启动时传递给内核。本规范定义 QooBot 参考平台的设备树模板。

参见 `os/qoo_device_tree.dts`。

### 4.2 关键节点定义

| 节点 | 说明 |
|------|------|
| `/soc/i2c@...` | I²C 总线，挂载 IMU、温湿度传感器 |
| `/soc/spi@...` | SPI 总线，挂载 ADC、触觉传感器 |
| `/soc/can@...` | CAN FD 控制器 |
| `/soc/ethernet@...` | 以太网控制器（EtherCAT 使用）|
| `/soc/mipi-csi@...` | MIPI CSI-2 相机接口 |
| `/soc/i2s@...` | I²S/TDM 音频接口 |
| `/soc/gpio@...` | GPIO 控制器（急停、制动器）|
| `/soc/pwm@...` | PWM 控制器（相机触发）|
| `/soc/timer@...` | 高精度定时器（实时控制）|

### 4.3 设备树编译与部署

```bash
# 编译设备树
$ dtc -I dts -O dtb -o qoo_device_tree.dtb qoo_device_tree.dts

# 部署到 Boot 分区（JOOT 规范）
$ cp qoo_device_tree.dtb /boot/dtbs/custom/qoo_body_v1.dtb

# U-Boot 启动时指定
=> setenv fdt_file qoo_body_v1.dtb
=> saveenv
```

## 5. 操作系统安全基线

### 5.1 安全启动（Secure Boot）

| 阶段 | 验签内容 | 密钥 |
|------|---------|------|
| Boot ROM | SPL / TEE | SoC 根密钥（eFuse）|
| SPL | ARM Trusted Firmware (ATF) | OEM 密钥 |
| ATF | U-Boot / GRUB | OEM 密钥 |
| GRUB | Linux 内核 + Initramfs | OEM 密钥 |
| 内核 | 内核模块 | 模块签名密钥 |

### 5.2 可信执行环境（TEE）

使用 ARM TrustZone 或 RISC-V MultiZone 提供隔离执行环境：

```
Normal World (Linux)                Secure World (OP-TEE)
┌──────────────────┐               ┌──────────────────┐
│  qoobrain       │               │ 密钥管理         │
│  （行为决策）     │   SMC 调用    │ 设备证书         │
│                  ├──────────────►│ 安全存储         │
│                  │               │ 加密引擎         │
└──────────────────┘               └──────────────────┘
```

### 5.3 文件系统保护

| 分区 | 文件系统 | 挂载选项 |
|------|---------|---------|
| `/boot` | vfat | `ro,nodev,nosuid,noexec` |
| `/` | ext4 | `ro,nodev,nosuid`（只读根文件系统）|
| `/var` | ext4 | `rw,nodev,nosuid,noexec` |
| `/tmp` | tmpfs | `rw,nodev,nosuid,noexec,size=256M` |
| `/home` | ext4 | `rw,nodev,nosuid` |

## 6. 用户态实时编程指南

### 6.1 实时线程设置（C 代码示例）

```c
/* os/sample_rt_thread.c */
#include <sched.h>
#include <pthread.h>
#include <sys/mman.h>

void setup_rt_thread(pthread_t *thread)
{
    struct sched_param param;
    pthread_attr_t attr;

    /* 1. 锁定内存，避免 page fault 引入延迟 */
    mlockall(MCL_CURRENT | MCL_FUTURE);

    /* 2. 设置实时调度策略 */
    pthread_attr_init(&attr);
    pthread_attr_setschedpolicy(&attr, SCHED_FIFO);
    param.sched_priority = 80; /* 范围 1~99，99 为最高 */
    pthread_attr_setschedparam(&attr, &param);

    /* 3. 创建实时线程 */
    pthread_create(thread, &attr, rt_control_loop, NULL);
}

void *rt_control_loop(void *arg)
{
    struct timespec next_period;
    clock_gettime(CLOCK_MONOTONIC, &next_period);

    while (1) {
        /* 周期性控制逻辑（1kHz = 1ms）*/
        motor_control_tick();
        safety_monitor_tick();

        /* 等待下一个周期 */
        next_period.tv_nsec += 1000000; /* +1ms */
        if (next_period.tv_nsec >= 1000000000) {
            next_period.tv_nsec -= 1000000000;
            next_period.tv_sec++;
        }
        clock_nanosleep(CLOCK_MONOTONIC, TIMER_ABSTIME,
                        &next_period, NULL);
    }
    return NULL;
}
```

### 6.2 内存锁定

实时任务必须调用 `mlockall()` 锁定所有虚拟内存，防止分页导致不可预测的延迟。

## 7. 构建系统

### 7.1 Yocto 构建（推荐）

```
qoobody-image.bb
├── qoo-kernel-rt.bbappend     # 实时内核配方
├── qoo-drivers.bbappend       # 自定义驱动配方
├── qoo-systemd-conf.bbappend  # 系统服务配置
└── qoo-security.bbappend       # 安全基线配方
```

### 7.2 Docker 容器化（应用层）

非实时应用（LLM 推理、Web 服务）运行在 Docker 容器中，与实时控制平面隔离：

```
┌─────────────────────────────────┐
│  Docker Container              │ ← LLM、Web 前端
│  （非实时，best-effort）        │
├─────────────────────────────────┤
│  qoobrain 主进程               │ ← 行为决策（SCHED_NORMAL）
├─────────────────────────────────┤
│  实时控制线程                   │ ← 运动控制（SCHED_FIFO, P80）
├─────────────────────────────────┤
│  安全 MCU 固件                 │ ← 独立硬件，双核锁步
└─────────────────────────────────┘
```
