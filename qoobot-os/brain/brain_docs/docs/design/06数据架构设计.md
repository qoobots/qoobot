# 06 数据架构设计

QooBot Brain 数据模型、存储策略与数据流设计。

---

## 核心数据模型

### SceneGraph (场景图)

```
SceneGraph
├── Entities[]
│   ├── entity_id: uuid
│   ├── type: "person"|"object"|"room"|"robot"
│   ├── class_label: str
│   ├── pose: Pose3D (position + quaternion)
│   ├── bbox: BoundingBox3D
│   ├── properties: Dict (color/size/material)
│   └── confidence: float
├── Relations[]
│   ├── subject_id: uuid
│   ├── predicate: "on"|"in"|"near"|"holding"|"contains"
│   ├── object_id: uuid
│   └── confidence: float
└── SpatialTopology
    ├── rooms: Room[]
    ├── connectivity: Graph
    └── coordinate_frame: str
```

### TaskPlan (任务计划)

```
TaskPlan
├── goal: str ("制作一杯咖啡")
├── subtasks[]
│   ├── id: uuid
│   ├── action: "navigate"|"grasp"|"place"|"pour"
│   ├── target: EntityRef
│   ├── parameters: Dict
│   ├── dependencies: [uuid]
│   ├── priority: int
│   └── estimated_duration: float
└── status: "pending"|"executing"|"completed"|"failed"
```

### BehaviorTree (行为树)

```
BehaviorTree
├── root: ControlNode (Sequence/Fallback/Parallel)
├── nodes[]
│   ├── id: str
│   ├── type: "action"|"condition"|"control"|"decorator"
│   ├── name: str
│   ├── parameters: Dict
│   ├── blackboard_inputs: [str]
│   └── blackboard_outputs: [str]
├── blackboard: Dict[str, Any]
└── tick_rate_hz: float
```

### SensorFrame (传感器帧)

```
SensorFrame
├── timestamp_ns: int64
├── sensor_id: str
├── sensor_type: "camera"|"lidar"|"imu"|"microphone"
├── data: bytes (Protobuf encoded)
├── calibration: CalibrationParams
└── quality: QualityMetrics
```

### SkillManifest (技能清单)

```
SkillManifest
├── name: str
├── version: semver
├── permissions: [str]
├── dependencies: Dict[str, semver]
├── resources: ResourceRequirements
├── privacy_labels: [PrivacyLabel]
├── entry_point: str
└── signature: bytes (Ed25519)
```

## 存储策略

| 数据 | 存储位置 | 格式 | 保留策略 |
|:-----|:---------|:-----|:---------|
| 场景图 | 内存 (最新) + SQLite (历史) | 二进制 | 会话期间 |
| 任务计划 | 内存 | JSON | 会话期间 |
| 行为树 | 磁盘 (.xml) | XML | 永久 |
| 传感器帧 | 环形缓冲 (最新N帧) | Protobuf | 5分钟 |
| 技能清单 | 磁盘 (.yaml) | YAML | 永久 |
| 世界模型 | SQLite + 云端PostgreSQL | 表 | 30天本地 |
| 日志 | 磁盘 + 云端ELK | 结构化JSON | 7天本地 |
| 健康数据 | Prometheus TSDB | 时序 | 90天 |

### 端云数据同步

| 数据 | 同步方向 | 频率 |
|:-----|:---------|:-----|
| 经验回放数据 | 端→云 | 每小时/按事件 |
| 联邦学习梯度 | 端→云 | 每轮训练 |
| 模型更新 | 云→端 | OTA推送 |
| 技能更新 | 云→端 | 按需/后台 |
| 世界模型 | 端↔云 | 实时增量 |

## 隐私分级

| 级别 | 数据 | 处理方式 |
|:-----|:-----|:---------|
| **L0 本地** | 原始传感器数据 | 不离开设备 |
| **L1 脱敏** | 检测结果（无图像） | 可上传云端 |
| **L2 聚合** | 统计信息 | 可上传云端 |
| **L3 开放** | 使用统计数据(匿名) | 可公开 |
