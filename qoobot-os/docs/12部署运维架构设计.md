  qoobot-dev:
    image: qoobot-os:dev
    build:
      context: .
      dockerfile: Dockerfile.dev
    volumes:
      - .:/workspace/qoobot-os
      - models:/workspace/models
    privileged: false    # 开发环境不需要特权
    network_mode: host
    environment:
      - DISPLAY=${DISPLAY}

## 3. 配置管理

### 3.1 配置分层

```
配置优先级 (从低到高):
1. 默认配置 (/opt/qoobot/config/defaults/*.yaml)     ← 出厂默认
2. 系统配置 (/opt/qoobot/config/*.yaml)               ← 管理员配置
3. 用户配置 (/var/lib/qoobot/config/*.yaml)           ← 运行时可写
4. 环境变量 (QOOBOT_*)                                ← 调试用
5. 命令行参数 (--config key=value)                    ← 临时覆盖
```

### 3.2 配置热加载

```python
from qoobot_brain import ConfigManager

config = ConfigManager()

# 监听配置变更
@config.on_change("navigation.max_speed")
def on_speed_change(new_value):
    navigator.set_max_speed(new_value)

# 启动配置监听
config.watch()
```

## 4. OTA 更新

### 4.1 更新策略

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| 全量更新 | 完整系统镜像替换 | 大版本升级 |
| 增量更新 | 仅变更文件 (bsdiff) | 常规补丁 |
| 模型更新 | 仅更新 .qoomodel | AI 模型迭代 |
| 配置更新 | 仅更新 YAML/JSON | 参数调节 |
| 技能更新 | 仅更新技能包 | 技能市场 |

### 4.2 更新流程

```
云端 OTA Server
    │
    │ 1. 查询可用更新
    ▼
QooBot (检查更新)
    │
    │ 2. 下载 + 签名验证 (Ed25519)
    ▼
QooBot (预安装)
    │ 安装到备用分区 (A/B 系统)
    │
    │ 3. 校验 (SHA-256)
    ▼
QooBot (待重启)
    │ 通知用户 / 等待空闲
    │
    │ 4. 原子切换
    ▼
QooBot (重启进入新版本)
    │
    │ 5. POST 自检
    ▼
    成功 → 标记新版为 active
    失败 → 自动回滚到旧版
```

### 4.3 回滚机制

- **A/B 分区**：双系统分区，更新安装到非活跃分区
- **自动回滚**：启动后 POST 自检失败 → 自动切回旧分区
- **保留旧版**：至少保留 1 个可回滚版本
- **远程回滚**：云端可强制回滚到指定安全版本

## 5. 监控与告警

### 5.1 监控指标体系

| 类别 | 指标 | 采集频率 | 告警阈值 |
|------|------|:--------:|----------|
| CPU | 使用率、温度 | 1 Hz | > 90%, > 80°C |
| 内存 | RSS、可用 | 1 Hz | < 10% 可用 |
| GPU/NPU | 使用率、温度、功耗 | 1 Hz | > 85°C |
| 推理 | 延迟 P95/P99、吞吐量 | 持续 | P95 > 100ms |
| 关节 | 位置、速度、力矩、温度 | 1 kHz | 力矩 > 120% |
| 电池 | 电压、电流、SOC、SOH | 10 Hz | SOC < 20% |
| 网络 | Wi-Fi RSSI、带宽 | 1 Hz | RSSI < -75 dBm |
| 安全 | 安全状态、碰撞事件 | 100 Hz | 任何安全事件 |

### 5.2 日志采集

```yaml
# 日志架构
collectors:
  - type: journald          # 系统日志
  - type: file              # 应用日志 (/var/log/qoobot/)
  - type: ros2_bag          # ROS 2 bag (按需)

outputs:
  - type: local_disk        # 本地存储 (30天旋转)
    path: /var/log/qoobot/
    max_size_gb: 10

  - type: cloud_forward     # 云端转发 (可选)
    endpoint: logs.qoobot.io
    auth: device_cert
    filter: WARNING+ only   # 仅上传 WARNING 及以上
```

### 6.1 systemd 服务定义

```ini
# /etc/systemd/system/qoobot-os.service
[Unit]
Description=QooBot Humanoid OS
After=network.target

[Service]
Type=simple
User=qoobot
ExecStart=/usr/bin/qoobot-os-launcher
Restart=always
RestartSec=5
LimitNOFILE=65536
LimitMEMLOCK=infinity
CPUSchedulingPolicy=fifo
CPUSchedulingPriority=80

[Install]
WantedBy=multi-user.target
```

## 7. 灾难恢复

| 场景 | 恢复方案 | RTO |
|------|----------|:---:|
| 系统崩溃 | Watchdog 自动重启 | < 5s |
| 文件系统损坏 | A/B 分区切换到备分区 | < 30s |
| 存储硬件故障 | 替换存储 + 云端恢复关键配置 | < 1h |
| 安全芯片失效 | 退化到软件安全模式 (功能受限) | < 1s |
| 云端不可达 | 离线模式运行 (核心功能不受影响) | < 100ms |
