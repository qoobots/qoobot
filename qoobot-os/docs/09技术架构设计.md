## 1. 技术选型

### 1.1 技术栈全景

| 层次 | 技术 | 版本 | 选择理由 |
|------|------|:----:|----------|
| **系统语言** | C++ | 17 | 零开销抽象、实时性、芯片 SDK 均为 C/C++ |
| **AI/算法** | Python | 3.11+ | PyTorch/Transformers 生态、快速迭代 |
| **前端可视化** | TypeScript + React + Three.js | — | 3D 可视化生态成熟 |
| **构建系统** | CMake + colcon + setuptools | 3.25+ | 业界标准，ROS 2 原生支持 |
| **序列化** | FlatBuffers + Protobuf | — | FlatBuffers 零拷贝、Protobuf gRPC |
| **中间件** | ROS 2 (Humble) | 2023 | DDS 通信、TF2、rviz、Gazebo 集成 |
| **RPC 框架** | gRPC | 1.59+ | 多语言、流式、TLS、高性能 |
| **物理仿真** | MuJoCo | 3.0+ | 高性能接触动力学，Google DeepMind 维护 |
| **数学库** | Eigen 3.4 | 3.4+ | 线性代数，header-only，ROS 2 依赖 |
| **AI 推理** | ONNX Runtime + QNN/BPU/RKNN SDK | — | 多芯片适配 |
| **测试** | pytest + Google Test + Vitest | — | 语言原生测试框架 |
| **CI/CD** | GitHub Actions | — | 免费、深度集成 |
| **容器** | Docker | — | 可复现开发/测试环境 |

### 1.2 为什么不选...

| 候选方案 | 为什么不用 |
|----------|-----------|
| Rust 替代 C++ | ROS 2 主要 C++ API、芯片 SDK 均为 C/C++，迁移成本极高 |
| gRPC 替代 ROS 2 | ROS 2 提供 TF2/RViz/Gazebo/MoveIt2 完整生态，不是纯 RPC 问题 |
| TensorFlow Lite | ONNX 更开放，支持更多芯片后端 |
| NVIDIA Isaac Sim (闭源) | 需要开源可定制的仿真，MuJoCo 免费且高性能 |
| Bazel 构建 | CMake 是 ROS 2 + 嵌入式的事实标准，团队更熟悉 |
| Linux RT 内核 | PREEMPT_RT 已合入主线，足够满足 1kHz 控制回路 |

## 2. 关键模块技术方案

### 2.1 推理引擎 (ai-engine/)

```
ONNX Model
    │
    ▼
┌──────────────┐
│ Importer      │  ONNX Runtime C++ API 解析图
└──────┬───────┘
       ▼
┌──────────────┐
│ IR Builder    │  构建设备无关中间表示 (IrGraph)
└──────┬───────┘
       ▼
┌──────────────┐
│ Optimizer     │  常量折叠 → 死代码消除 → 算子融合 → 内存复用
└──────┬───────┘
       ▼
┌──────────────┐
│ Quantizer     │  PTQ 校准 → scale/zero_point → INT8/FP16 量化
└──────┬───────┘
       ▼
┌──────────────┐
│ Codegen       │  NPU 指令 → GPU kernel → CPU 向量化 → DSP 固件
└──────┬───────┘
       ▼
  .qoomodel (FlatBuffers 打包)
       │
       ▼
┌──────────────┐
│ Runtime       │  load → warmup → infer → profile
│               │  NPU/GPU/DSP/CPU 多后端动态分派
└──────────────┘
```

**后端分派策略**：
1. 检查模型精度 (INT8 → NPU, FP16 → GPU, FP32 → CPU)
2. 检查算子支持矩阵 (硬件是否支持该模型所有算子)
3. 检查当前负载 (温度/功耗/内存)
4. 自动选择最优后端 + fallback 链

### 2.2 运动控制 (brain_core + HAL mechanical)

```
┌──────────────────────────────────────────────────────┐
│                  Motion Planning Pipeline              │
│                                                       │
│  Task Goal ──► IK Solver ──► Trajectory Generator    │
│                              │                        │
│                              ▼                        │
│                         Collision Check (FCL)         │
│                              │                        │
│                              ▼                        │
│  Joint Cmd ◄── WBC (qpOASES) ◄── MPC (qpOASES)      │
│     │                                                  │
│     ▼                                                  │
│  CAN FD / EtherCAT → Motor Drivers                    │
└──────────────────────────────────────────────────────┘
```

**WBC (Whole-Body Control)**：
- 任务空间优先级：平衡 > 姿态 > 手部位置 > 避障
- 零空间投影：低优先级任务不干扰高优先级
- QP 求解：qpOASES (活动集方法，< 1ms 求解)

**MPC (Model Predictive Control)**：
- 预测步长：N=10, dt=50ms (0.5s 预测窗口)
- 状态：质心位置/速度/角速度 + 双足步态相位
- 控制：地面反作用力 (GRF) → 关节力矩 (WBC 映射)

### 2.3 安全架构

```
┌──────────────────────────────────────────────┐
│              主控 SoC (Linux)                 │
│                                              │
│  brain/ ── safety_monitor ──► Watchdog     │
│      │                              │        │
│      │                    ┌─────────▼──────┐ │
│      │                    │ heartbeat      │ │
│      │                    │ (CAN FD, 100Hz)│ │
│      │                    └─────────┬──────┘ │
│      │                              │        │
└──────┼──────────────────────────────┼────────┘
       │                              │
       │              ┌───────────────▼──────────────┐
       │              │      安全 MCU (SIL2)          │
       │              │                               │
       │              │  · 双通道急停输入              │
       └── disable ───►  · 关节力矩独立监控            │
                      │  · 碰撞检测 (电流阈值)         │
                      │  · 电源切断继电器              │
                      │  · 独立于主 SoC 运行           │
                      └───────────────────────────────┘
```

**安全设计要点**：
1. 安全 MCU 独立于主控 SoC，即使 SoC 完全崩溃也能执行紧急制动
2. 双通道冗余急停 (硬件按钮 + CAN FD 指令)
3. 关节力矩独立监控 (电流传感器直连安全 MCU)
4. 看门狗超时自动切断动力电源
5. 安全状态机：NORMAL → DEGRADED → SAFE_STOP → EMERGENCY_STOP

## 3. 性能优化策略

| 优化方向 | 策略 | 预期收益 |
|----------|------|:--------:|
| 内存 | DMA-BUF 零拷贝管道 | 减少 50% 内存带宽 |
| 推理 | INT8 量化 + Winograd 卷积 | 3x 吞吐提升 |
| 控制 | qpOASES 热启动 (warm start) | 50% 求解加速 |
| 并行 | 感知/规划流水线 Overlap | 30% 端到端延迟降低 |
| 编译 | 模型编译缓存 + 增量编译 | 10x 重编译加速 |
| 启动 | 延迟加载非关键服务 | 50% 冷启动时间缩减 |
