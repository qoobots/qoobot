## 2. 技能架构

### 2.1 技能生命周期

```
                     ┌──────────┐
                     │  已安装   │
                     └────┬─────┘
                          │ activate()
                          ▼
                     ┌──────────┐
               ┌────►│  运行中   │◄────┐
               │     └────┬─────┘     │
               │          │           │
               │ pause()  │  error()  │ resume()
               │          ▼           │
               │     ┌──────────┐     │
               │     │  错误态   │     │
               │     └────┬─────┘     │
               │          │ recover() │
               │          ▼           │
               │     ┌──────────┐     │
               └─────│  已暂停   │─────┘
                     └────┬─────┘
                          │ deactivate()
                          ▼
                     ┌──────────┐
                     │  已卸载   │
                     └──────────┘
```

### 2.2 技能隔离模型

| 隔离维度 | 实现方式 | 说明 |
|----------|----------|------|
| 进程隔离 | 每个技能独立进程 | 崩溃不互相影响 |
| 内存限制 | cgroup memory.limit | 防止 OOM |
| CPU 限制 | cgroup cpu.shares | 防止抢占核心任务 |
| 文件系统 | chroot/overlayfs | 只能访问技能自身目录 |
| 网络 | seccomp filter | 限制网络 syscall |
| 权限 | 细粒度 capability 控制 | 最小权限原则 |

### 2.3 技能间通信

```
技能 A (清洁)                    技能 B (安防)
     │                                │
     │  [需要安防解除警戒]              │
     │──► brain_core/event_bus ───────►│
     │         (领域事件)              │
     │                                │
     │         [共享地图数据]           │
     │◄── QooSvc/navigation/map ─────►│
             (共享数据服务)
```

## 4. 子系统交互模式

### 4.1 请求-响应模式

```
Skill ──gRPC unary──► brain_core ──► QooSvc ──► HAL
                        │              │
                        ◄── response ──┘
```

适用：配置查询、状态获取、一次性指令

### 4.2 发布-订阅模式

```
HAL/sensor ──ROS2 topic──► brain_core/event_bus
                              │
                    ┌─────────┼─────────┐
                    ▼         ▼         ▼
                 Skill A   QooSvc    monitor
```

适用：传感器数据流、事件广播、状态变更

### 4.3 流水线模式

```
Camera ─► HAL ─► QooCore(infer) ─► brain_ai(fusion) ─► brain_core(plan) ─► HAL(actuator)
  │          DMA-BUF            shared memory         gRPC                CAN FD
  └─────────────────── 端到端延迟 < 100ms ───────────────────────────────────┘
```

适用：感知-规划-控制闭环

### 4.4 主从模式

```
           brain_core (Master)
                │
    ┌───────────┼───────────┐
    ▼           ▼           ▼
 QooSvc      QooStore     QooEdge
(设备/OTA   (技能运行时)  (边缘计算)
 导航/语音)
```

适用：系统服务编排、资源分配

## 5. 扩展点设计

| 扩展点 | 机制 | 说明 |
|--------|------|------|
| 新 AI 后端 | NpuHal 接口实现 + dlopen 加载 | 新芯片只需实现 HAL |
| 新传感器 | QooBody HAL sensor interface | 标准化驱动接口 |
| 新执行器 | QooBody HAL actuator interface | CANopen/EtherCAT 协议栈 |
| 新技能类型 | Brain SDK Scenario API 实现 | Python 类继承 |
| 新通信协议 | Proto 定义 + gRPC service 注册 | gRPC 反射支持 |
| 新可视化面板 | brain_web React 组件 | 插件化 Dashboard |
