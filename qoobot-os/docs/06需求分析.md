> 版本：v0.1 | 最后更新：2026-06-30 | 状态：Draft

## 1. 需求全景

qoobot-os 作为仿生人操作系统的核心运行时，承载六大需求域：

```
                    ┌─────────────────────────┐
                    │   实时性 (Real-time)     │
                    │   1kHz 控制 / <1ms 响应   │
                    └─────────────┬───────────┘
                                  │
    ┌─────────────────┐  ┌───────┴───────┐  ┌─────────────────┐
    │ 鲁棒性 (Robust)  │  │ 智能性 (AI)   │  │ 安全性 (Safety)  │
    │ 99.9% 可用性     │  │ 端侧多模态    │  │ SIL2/SIL3 认证   │
    └────────┬────────┘  └───────┬───────┘  └────────┬────────┘
             │                   │                   │
             └───────────────────┼───────────────────┘
                                 │
              ┌──────────────────┴──────────────────┐
              │                                     │
    ┌─────────┴─────────┐               ┌───────────┴───────────┐
    │ 可扩展 (Extensible)│               │ 低功耗 (Low Power)    │
    │ 技能市场/插件化    │               │ > 2h 续航 / DVFS     │
    └───────────────────┘               └───────────────────────┘
```

## 2. 功能需求

| 编号 | 需求 | 描述 | 优先级 |

| FR-CORE-01 | 多后端推理 | 支持 NPU/GPU/DSP/CPU 四种后端统一 API | P0 |

| FR-CORE-05 | 混合精度 | INT8/FP16/FP32 混合精度推理 | P0 |
| FR-CORE-06 | 实时调度 | 固定优先级抢占+时间片轮转+流水线并行 | P0 |
| FR-CORE-07 | 内存池管理 | Arena 分配器+权重共享+LRU 淘汰 | P0 |

## 3. 非功能需求

### 3.1 性能需求

| 编号 | 需求 | 指标 | 测量方法 |
|:----:|------|------|----------|
| NFR-PERF-01 | 推理延迟 | 视觉 < 10ms, VLA < 100ms | NPU INT8 推理端到端 |
| NFR-PERF-02 | 控制频率 | 关节 > 1kHz, 全身 > 100Hz | RT 任务周期抖动 |
| NFR-PERF-03 | 启动时间 | 冷启动 < 30s | POST + 服务初始化 |
| NFR-PERF-04 | 内存占用 | 全运行时 < 8GB | /proc/meminfo RSS |
| NFR-PERF-05 | 帧率 | 多模型并发 > 30 FPS | 感知→控制流水线 |
| NFR-PERF-06 | 导航成功率 | > 99% 室内 | 100 次导航测试 |

### 3.2 可靠性需求

| 编号 | 需求 | 指标 | 说明 |
|:----:|------|------|------|
| NFR-REL-01 | 系统可用性 | 99.9% | 故障恢复 < 30s |
| NFR-REL-02 | 单关节故障 | 降级运行，不影响其他关节 | 孤立故障点 |
| NFR-REL-03 | 安全回路 | 独立于主控 SoC | 硬件冗余 |
| NFR-REL-04 | 数据持久化 | 日志 ≥ 30 天保留 | 旋转归档 |

### 3.3 安全需求

| 编号 | 需求 | 指标 | 说明 |
|:----:|------|------|------|
| NFR-SEC-01 | 功能安全等级 | SIL2 (IEC 61508) | 安全 MCU 认证 |
| NFR-SEC-02 | 碰撞检测延迟 | < 10ms | 关节电流+接近传感 |
| NFR-SEC-03 | 紧急制动 | < 100ms | 双通道冗余 |
| NFR-SEC-04 | 数据加密 | AES-256 (at rest), TLS 1.3 (in transit) | 模型/日志/通信 |
| NFR-SEC-05 | 安全启动 | 信任链 (ROM→Bootloader→OS→App) | 防篡改 |

### 3.4 可维护性需求

| 编号 | 需求 | 指标 | 说明 |
|:----:|------|------|------|
| NFR-MAINT-01 | 代码覆盖率 | > 80% 核心模块 | pytest + gtest |
| NFR-MAINT-02 | 文档覆盖率 | 100% 公共 API | 13 文档标准 |
| NFR-MAINT-03 | CI 流水线 | 每次 PR 自动运行 | GitHub Actions |
| NFR-MAINT-04 | 日志规范 | JSON 结构化，可查询 | ELK 兼容 |

## 4. 约束条件

| 约束 | 说明 |
|------|------|
| 硬件平台 | Qualcomm 8295 及以上 / NVIDIA Orin / Horizon J6 / Rockchip RK3588 |
| 操作系统 | Linux 5.15+ (PREEMPT_RT for control loop) |
| 编程语言 | C++17 (核心运行时), Python 3.11+ (AI/SDK), TypeScript (Web 可视化) |
| 构建系统 | CMake 3.25+ (C++), colcon (ROS 2), setuptools (Python) |
| 内存限制 | 2~16GB LPDDR5 (取决于硬件平台) |
| 功耗限制 | < 50W (SoC + AI 推理全负载), < 150W (整机含电机) |
| 散热约束 | 无风扇 ≤ 85°C 结温, 主动散热 ≤ 65°C |
| 实时约束 | 控制回路硬实时, AI 推理软实时, 交互服务普通优先级 |
