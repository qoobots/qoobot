## 2. 模型文件格式 (.qoomodel)

### 2.1 文件结构

QooCore 使用 FlatBuffers 定义的 `.qoomodel` 格式，是 ONNX → IR 编译后的产物：

```
┌──────────────────────────────────────────────────────┐
│                   .qoomodel 文件                      │
├──────────────────────────────────────────────────────┤
│  Header (128 bytes)                                   │
│  ├─ magic:       "QMOD"  (4 bytes)                   │
│  ├─ version:     0x00010000 (4 bytes)                │
│  ├─ ir_size:     uint64   (8 bytes)                  │
│  ├─ weights_size: uint64  (8 bytes)                  │
│  ├─ code_size:   uint64   (8 bytes)                  │
│  ├─ checksum:    SHA-256  (32 bytes)                  │
│  ├─ signature:   Ed25519  (64 bytes)                  │
│  └─ reserved:    32 bytes                             │
├──────────────────────────────────────────────────────┤
│  IR Section (FlatBuffers 序列化)                      │
│  ├─ model_name, target_backend, precision             │
│  ├─ input_tensors[], output_tensors[]                 │
│  ├─ nodes[] (operator type + params)                  │
│  └─ quantization_params (scale/zero_point)            │
├──────────────────────────────────────────────────────┤
│  Weights Section (uint8[])                             │
│  ├─ layer1_weights  (quantized INT8/FP16)             │
│  ├─ layer1_bias                                      │
│  └─ ...                                               │
├──────────────────────────────────────────────────────┤
│  Code Section (backend-specific native code)          │
│  └─ NPU opcode / GPU kernel / DSP firmware            │
└──────────────────────────────────────────────────────┘
```

### 2.2 Tensor 数据类型

| 类型 | 字节 | 范围 | 使用场景 |
|------|:----:|------|----------|
| `FLOAT32` | 4 | ±3.4e38 | 训练/高精度推理 |
| `FLOAT16` | 2 | ±65504 | GPU 推理 |
| `INT8` | 1 | -128~127 | NPU 量化推理 |
| `INT4` | 0.5 | -8~7 | 极致压缩 |
| `UINT8` | 1 | 0~255 | 图像输入 |
| `INT32` | 4 | ±2.1e9 | 索引/计数 |

### 2.3 推理结果类型

```cpp
// 检测结果
struct Detection {
    int32_t class_id;
    float confidence;
    Rect bbox;        // x, y, w, h (归一化 0~1)
};

// 分割结果
struct Segmentation {
    int32_t height, width, channels;
    float* mask;       // H × W × C 概率图
};

// 位姿估计
struct Pose6D {
    float translation[3];
    float rotation[4]; // 四元数 (w, x, y, z)
    float confidence;
};
```

## 3. 传感器数据模型

### 3.1 视觉数据

| 字段 | 类型 | 说明 |
|------|------|------|
| `timestamp` | uint64 (ns) | gPTP 同步时间戳 |
| `width` / `height` | uint32 | 图像分辨率 |
| `format` | enum | RGB8/BGR8/BAYER/YUV420 |
| `data` | uint8[] 或 ION fd | 像素数据 (DMA-BUF 零拷贝) |
| `camera_matrix` | float[9] | 内参矩阵 |
| `distortion` | float[5] | 畸变系数 |

### 3.2 LiDAR 数据

| 字段 | 类型 | 说明 |
|------|------|------|
| `timestamp` | uint64 (ns) | gPTP 同步时间戳 |
| `points` | Point[] | 点云数据 |
| `point.x/y/z` | float32 | 3D 坐标 (m) |
| `point.intensity` | float32 | 反射强度 (0~255) |
| `point.ring` | uint8 | 激光线号 |
| `ring_angles` | float32[] | 各线仰角 |

### 3.3 IMU 数据

| 字段 | 类型 | 频率 | 说明 |
|------|------|:----:|------|
| `accel_x/y/z` | float32 | 200Hz | 加速度 (m/s²) |
| `gyro_x/y/z` | float32 | 200Hz | 角速度 (rad/s) |
| `mag_x/y/z` | float32 | 100Hz | 磁力计 (μT) |
| `temperature` | float32 | 1Hz | 芯片温度 (°C) |
| `covariance` | float32[9] | — | 协方差矩阵 |

## 4. 控制数据模型

### 4.1 关节状态

```protobuf
message JointState {
    string name = 1;              // 关节名称 "left_shoulder_pitch"
    double position = 2;          // 位置 (rad)
    double velocity = 3;          // 速度 (rad/s)
    double effort = 4;            // 力矩 (Nm)
    double temperature = 5;       // 温度 (°C)
    uint64 timestamp = 6;         // 时间戳 (ns)
}
```

### 4.2 运动规划路径

```protobuf
message MotionTrajectory {
    repeated JointWaypoint waypoints = 1;
    double total_time = 2;        // 总执行时间 (s)
    TrajectoryType type = 3;      // B_SPLINE / CATMULL_ROM / MIN_JERK
}

message JointWaypoint {
    repeated double positions = 1; // N 个关节位置
    repeated double velocities = 2;
    double time_from_start = 3;   // 偏移时间 (s)
}
```

### 4.3 行为树定义 (XML)

```xml
<BehaviorTree ID="PickAndPlace">
  <Sequence>
    <Action ID="MoveTo" target="table" />
    <Action ID="DetectObject" class="cup" />
    <Action ID="PlanGrasp" />
    <Fallback>
      <Action ID="ExecuteGrasp" />
      <Action ID="RetryGrasp" />
    </Fallback>
    <Action ID="MoveTo" target="target_location" />
    <Action ID="Release" />
  </Sequence>
</BehaviorTree>
```

## 7. 数据流与管道

### 7.1 感知管道数据流

```
Camera/LiDAR/IMU
    │ (DMA-BUF 零拷贝)
    ▼
QooBody HAL (sensor)
    │ 时间戳同步 (gPTP) + 格式转换
    ▼
QooCore (推理引擎)
    │ 多模型并发: 检测 → 分割 → BEV → 点云
    ▼
QooBrain (brain_ai)
    │ 多模态融合 → 场景理解 → 对象追踪
    ▼
QooBrain (brain_core)
    │ 行为决策 → 运动规划 → 控制指令
    ▼
QooBody HAL (actuator)
    │ CAN FD / EtherCAT
    ▼
Motor Drivers
```

### 7.2 数据带宽估算

| 数据流 | 带宽 | 量化 |
|--------|:----:|------|
| 4× RGB Camera @ 1080p 30fps → HAL | ~2.4 Gbps | RAW → RGB 可减半 |
| 1× LiDAR @ 10Hz → HAL | ~100 Mbps | UDP 包 |
| IMU @ 200Hz → HAL | ~1 Mbps | — |
| HAL → QooCore (Inference Input) | ~5 Gbps | DMA-BUF shared memory |
| QooCore → QooBrain (Results) | ~1 Gbps | gRPC over UDS |
| QooBrain → HAL (Control) | ~10 Mbps | CAN FD / EtherCAT |
| Log/Diagnostic → Disk | ~1 Mbps | JSON structured |
