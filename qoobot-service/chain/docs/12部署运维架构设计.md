## 2. 产线环境

### 2.1 标定工作站配置

| 组件 | 规格 |
|:-----|------|
| 工控机 | x86_64, 8核, 32GB RAM, SSD 512GB |
| GPU | NVIDIA RTX 3060 (视觉算法加速) |
| USB 3.0 | 4x (高速相机) |
| 以太网 | 2x Gigabit Ethernet |
| 操作系统 | Ubuntu 22.04 LTS |
| 标定板 | CharuCo 标定板 (相机) + 特征板 (LiDAR) |

### 2.2 标定软件安装

```bash
# 安装标定工具链
sudo apt install python3-opencv libceres-dev

# 克隆安装
git clone https://github.com/qoobot/qoochain.git
cd qoochain/calibration
pip install -e .

# 运行标定
qoochain-calibrate --serial QS-2026-00001 --mode full
```

## 4. 监控与运维

### 4.1 产线监控

| 指标 | 监控方式 | 告警 |
|:-----|---------|------|
| 标定成功率 | 标定软件上报 | <99% |
| 一次合格率 (FPY) | MES 统计 | <95% |
| 产线节拍 | MES 节拍监控 | >标准120% |
| 标定工作站状态 | 心跳检测 | 离线 > 5min |
| SPC 超限 | 实时 SPC 监控 | 任何超限 |

### 4.2 备份策略

| 对象 | 策略 | 位置 |
|:-----|------|------|
| BOM 数据 | PostgreSQL 每日备份 | 云端 |
| 标定参数 | 每次标定即时上传 | 云端 MinIO |
| 检验记录 | MES → 云端实时同步 | 云端 |
| 产线配置 | Git 版本管理 | GitHub |

## 5. 运维手册

### 5.1 产线启动检查

- [ ] 所有标定工作站开机自检
- [ ] 标定板清洁/无损伤
- [ ] 传感器连接正常
- [ ] 云端连接正常
- [ ] BOM/SOP 版本最新

### 5.2 常见故障

| 故障 | 排查 |
|:-----|------|
| 标定精度不达标 | 标定板清洁度、光照条件、相机对焦 |
| MES 扫码失败 | 条码质量、扫描枪校准、网络连接 |
| 云端上传失败 | 网络检查、VPN、API Token 过期 |
| 标定工作站死机 | 强制重启、检查系统日志 |
