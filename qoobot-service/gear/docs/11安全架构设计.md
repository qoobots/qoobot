# qoogear — 安全架构设计

> 版本：v0.1 | 最后更新：2026-06-29 | 状态：Draft | 子项目：qoogear（配件生态）
> 对标：Apple MFi 安全体系

---

## 1. 安全设计原则

1. **安全优先**：任何安全风险优先于功能
2. **认证强制**：未经 MFQ 认证的配件受限运行
3. **故障隔离**：配件异常不影响机器人本体安全
4. **防伪防篡改**：配件身份不可伪造

---

## 2. 配件认证体系

### 2.1 认证芯片

```
每个 MFQ 认证配件内置认证芯片:
  · 存储: MFQ 证书 + 配件私钥
  · 安全: 芯片防篡改 (Secure Element)
  · 算法: Ed25519 签名
  · 接口: I2C / 1-Wire
```

### 2.2 认证挑战-响应

```
机器人端 (Verifier)              配件 (Prover)
      │                              │
      │── challenge (随机数) ───────▶│
      │                              │
      │                      ECDSA_Sign(challenge)
      │                              │
      │◀── signature ───────────────│
      │                              │
      │ 验证签名 (MFQ 公钥)           │
      │                              │
      ├── 有效 → 配件可信 → 完全功能   │
      └── 无效 → 配件未认证 → 受限/拒绝 │
```

### 2.3 未认证配件策略

| 配件状态 | 行为 | 用户体验 |
|:---------|------|---------|
| MFQ Premium/Pro | 完全功能 | 即插即用 |
| MFQ Basic | 基础功能 | 通知用户已认证 |
| 未认证但安全 | 受限功能（只读/慢速） | "此配件未经 MFQ 认证" |
| 危险配件 | 拒绝连接 | "此配件可能不安全" |

## 4. 功能安全

### 4.1 配件异常处理

| 异常 | 检测 | 响应 |
|:-----|------|------|
| 通信超时 | 心跳超时 >200ms | 优雅停止配件 |
| 过流 | 电流 > 额定 150% | 立即断电 |
| 过温 | 温度 >85°C | 降功率/断电 |
| 碰撞检测 | 电流/力矩突变 | 立即松开/停止 |
| 协议错误 | CRC/校验失败 | 重试3次后报警 |

### 4.2 安全等级

```
末端执行器安全等级:
  SIL 1: 基础夹爪 (无危险)
  SIL 2: 电动工具 (低风险)
  SIL 3: 协作末端 (人体接触风险) ← MFQ Pro 要求
```
