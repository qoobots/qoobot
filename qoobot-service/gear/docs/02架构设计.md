# 02 — 架构设计

> 版本：v0.1 | 最后更新：2026-04-21 | 状态：Draft
>
> 子项目：qoogear — 配件与外设生态
>
> 架构概述：Made for QooBot (MFQ) 认证体系云端管理平台，后端 Java 微服务 + 前端 Vue3 Web 门户
>
> 对标系统：Apple MFi (Made for iPhone) + USB-IF 认证 + Intel Evo 平台认证

---

## 1. 设计目标与原则

### 1.1 核心设计目标

| 目标 | P0 指标 | P1 目标 | 说明 |
|------|---------|---------|------|
| 认证即服务 | MFQ 三级认证在线申请/审核/颁发 | 授权测试实验室网络 | 对标 MFi Portal |
| 接口标准化 | 7 类配件接口标准公开 | 第三方 SDK 自助测试 | 机械/电气/通信/安全全覆盖 |
| 即插即用 | 认证配件自动识别与配置 | 配件能力自动注册到 qoobrain | 免用户干预 |
| 安全隔离 | 第三方外设故障隔离域 | 配件安全风险评估自动化 | 不影响核心系统 |
| 生态开放 | 开放接口规范 + 参考设计 | 配件市场联动 qoostore | 降低第三方接入门槛 |

### 1.2 设计原则

1. **标准先行**：接口标准文档即代码，机器可读、可验证
2. **认证分层**：MFQ Basic / Premium / Pro 三级，适配不同复杂度配件
3. **安全内建**：配件安全评估、故障模式分析内建于认证流程
4. **开发者友好**：SDK + 参考设计 + 模拟器，降低开发门槛
5. **生态闭环**：认证 → 量产 → 上架 qoostore → 用户即插即用

## 3. 核心子系统

### 3.1 MFQ 认证服务 (qoogear-cert)

**职责**：管理 MFQ 三级认证的全生命周期——申请、审核、测试、颁发、续期、吊销。

```
第三方厂商              qoogear-cert                  审核员/实验室
    │                       │                              │
    ├─ 注册开发者账号 ──────►│                              │
    ├─ 提交认证申请 ──────►│                              │
    │                       ├─ 自动合规检查                 │
    │                       ├─ 分配测试实验室 ─────────────►│
    │                       │                              ├─ 机械配合测试
    │                       │                              ├─ 电气兼容测试
    │                       │                              ├─ 通信协议测试
    │                       │                              ├─ 功能互操作
    │                       │◄── 测试报告 ─────────────────┤
    │                       ├─ 安全审查                     │
    │                       ├─ 质量审核                     │
    │◄── 颁发MFQ证书 ──────┤                              │
    │                       │                              │
    ├─ 量产 ───────────────►├─ 授权芯片烧录                 │
    │                       │                              │
    ├─ 上架 qoostore ──────►├─ 认证状态同步                 │
```

### 3.2 标准管理服务 (qoogear-standard)

**职责**：管理 7 类配件接口标准的定义、版本、发布。

| 标准类别 | 内容 | 版本控制 |
|----------|------|----------|
| 机械接口 | 法兰尺寸/螺栓孔位/定位销/负载等级 | SemVer + 兼容性矩阵 |
| 电气接口 | 供电规格/信号引脚/连接器型号 | 向前兼容 |
| 通信协议 | CAN/EtherCAT/RS-485/I2C/SPI 规范 | 协议版本号 |
| 传感器数据 | 视觉/深度/触觉/环境数据格式 | 向后兼容 |
| 快换系统 | 工具快换机械/电气/识别接口 | 强制更新 |
| 安全互锁 | 急停联动/脱落检测/紧急释放 | 不可降级 |
| 电源配件 | 充电触点/无线充电/电池通信 | 向前兼容 |

### 3.3 开发者服务 (qoogear-developer)

**职责**：为第三方配件开发者提供 SDK、参考设计、测试治具、模拟器。

```
开发者门户功能：
├── SDK 下载 (Python/C++ 驱动框架)
├── 参考设计库 (原理图/PCB/3D 模型)
├── 在线模拟器 (免硬件开发调试)
├── 认证自查工具 (预测试套件)
├── 接口规范文档 (交互式 API 浏览器)
└── 开发者社区 (论坛/问答/案例库)
```

### 3.4 实验室管理服务 (qoogear-lab)

**职责**：管理全球授权测试实验室网络、设备校准、审核流程。

### 3.5 安全审计服务 (qoogear-security)

**职责**：配件安全风险评估、故障模式分析 (FMEA)、安全认证要求。

## 4. 数据流

### 4.1 认证申请流程

```
开发者 → 提交申请 → 自动合规检查 → 人工审核 → 分配实验室 → 
测试执行 → 测试报告 → 安全审查 → 质量审核 → 颁发证书 → 
授权芯片 → 量产监督 → 上架 qoostore → 定期复审
```

### 4.2 配件即插即用流程

```
配件插入 → 识别芯片读取 → MFQ 证书验证 → 驱动自动加载 → 
能力注册到 qoobrain → 安全域分配 → 就绪通知
```
