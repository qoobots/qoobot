## 2. 配件管理核心流程

### 2.1 配件生命周期 (QooBot 端)

```
┌─────────────────────────────────────────────────────────┐
│                 配件生命周期管理                           │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  [插入] → [识别] → [认证] → [加载] → [运行] → [拔出]    │
│                                                          │
│  插入: 电气连接检测 (IO/ADC/CAN 总线检测)                 │
│  识别: 读取配件 EEPROM (VID/PID/SN/Capability)           │
│  认证: 验证 MFQ 证书 + 数字签名                           │
│  加载: 查找/下载驱动 → 实例化 ROS2 Node                   │
│  运行: 用户技能通过统一 API 调用配件                       │
│  拔出: 优雅卸载驱动 + 资源释放                            │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 2.2 配件能力发现

```
配件插入
    │
    ▼
读取能力清单 (二进制 Protobuf / JSON)
    │
    ▼
解析能力类型: end_effector / sensor / battery / ...
    │
    ▼
加载对应的驱动模板
    │
    ▼
注册 ROS2 Node:
  /qoo/peripherals/gripper_001/
    ├── /status       (Topic: 配件状态)
    ├── /command      (Topic: 控制指令)
    └── /health       (Service: 健康状态)
    │
    ▼
通知 qoobrain: 新配件可用
```

## 3. 配件通信协议栈

```
┌────────────────────────────────────────┐
│            应用层                        │
│  Protobuf Messages (配件能力/控制/状态) │
├────────────────────────────────────────┤
│            传输层                        │
│  CAN-FD / RS485 / USB / Ethernet       │
├────────────────────────────────────────┤
│            物理层                        │
│  接插件规范 · 引脚定义 · 电气特性       │
└────────────────────────────────────────┘
```

### 3.1 统一 Protobuf 消息

```protobuf
// 配件能力声明
message AccessoryCapability {
  string name = 1;
  string vendor = 2;
  string model = 3;
  string mfq_cert_id = 4;
  AccessoryType type = 5;
  repeated Capability capabilities = 6;
}

// 配件状态
message AccessoryStatus {
  enum State { CONNECTED = 0; READY = 1; ACTIVE = 2; ERROR = 3; }
  State state = 1;
  map<string, float> metrics = 2;  // 温度/电流/位置等
  repeated string errors = 3;
}

// 配件控制指令
message AccessoryCommand {
  string capability_id = 1;
  map<string, float> parameters = 2;
}
```
