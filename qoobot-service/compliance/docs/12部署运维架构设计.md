## 1. 部署架构

```

│             qoocompliance 部署选项                    │
├─────────────────────────────────────────────────────┤
│                                                      │
│  开发环境 (Docker Compose)                            │
│  ┌────────────────────────────────────────────┐    │
│  │  compliance-web :8080                       │    │
│  │  compliance-api :8081                       │    │
│  │  postgres :5432                             │    │
│  │  redis :6379                                │    │

│  生产环境 (K8s, 复用 qoocloud 集群)                  │
│  ┌────────────────────────────────────────────┐    │

│  │  ┌──────────┐ ┌──────────┐                 │    │

## 2. Docker Compose（v0.4 已实现）

**项目根目录** `qoocompliance/docker-compose.yml`:

```yaml
version: "3.9"
services:
  postgres:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: qoocompliance
      POSTGRES_USER: qoobot
      POSTGRES_PASSWORD: qoobot
    ports:
      - "5432:5432"
    volumes:
      - pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U qoobot -d qoocompliance"]

  app:
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - "8086:8086"
    environment:
      SPRING_DATASOURCE_URL: jdbc:postgresql://postgres:5432/qoocompliance
    depends_on:
      postgres:
        condition: service_healthy
```

**Dockerfile**（多阶段构建）:

```dockerfile
# Stage 1: Build (eclipse-temurin:17-jdk-alpine)
# Stage 2: Runtime (eclipse-temurin:17-jre-alpine)
# JVM: G1GC, MaxRAMPercentage=75%, HeapDumpOnOutOfMemoryError
```

**启动命令**:

```bash
# 构建并启动
docker compose up -d

# 查看日志
docker compose logs -f app

# 停止
docker compose down

## 4. 监控

| 监控项 | 工具 | 告警 |
|:-------|------|------|
| API 可用性 | Prometheus Blackbox | 连续 3 次失败 |
| 数据库连接 | Spring Actuator | 连接池耗尽 |
| 法规源可达性 | 自定义 HealthIndicator | 源不可达 >24h |
| 磁盘使用 | Node Exporter | >85% |
| JVM 内存 | Micrometer | Heap >80% |

## 5. 备份策略

| 对象 | 策略 | 保留 |
|:-----|------|:---:|
| PostgreSQL | pg_dump 每日 + WAL 归档 | 90 天 |
| 证据文件 (MinIO) | 版本化 + 异地复制 | 法规要求年限 |
| 配置文件 | Git 版本管理 | 永久 |
| 审计日志 | WAL 归档 + 定期导出 | 法规要求年限 |

## 6. 运维手册

### 6.1 日常检查

- 每日：检查法规变更告警
- 每周：审查认证进度、即将到期检查项
- 每月：备份验证、审计日志完整性检查
- 每季度：合规框架全面审查

### 6.2 故障处理

| 故障 | 处理 |
|:-----|------|
| API 不可用 | K8s 自动重启，检查 DB/Redis 连接 |
| 法规源不可达 | 使用缓存数据，人工检查源状态 |
| 数据库故障 | 从备份恢复，WAL 重放 |
| 证据文件丢失 | 利用 MinIO 版本化恢复 |
