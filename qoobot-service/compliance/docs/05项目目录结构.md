## 3. 模块职责一览

| 模块 | Package | 职责 | 类型 |
|------|---------|------|------|
| `ComplianceApplication` | 根包 | Spring Boot 启动入口 | 启动类 |
| `ComplianceController` | `checklist.controller` | 合规检查清单 REST API（约 45 端点） | Controller |
| `RegulationController` | `monitor.controller` | 法规监控 REST API（3 端点） | Controller |
| `ComplianceChecklistService` | `checklist.service` | 清单生成、检查项管理、进度统计、差距分析、报告生成 | Service |
| `RegulationMonitorService` | `monitor.service` | 法规查询、变更记录、即将生效筛选、变更历史 | Service |
| `RobotSafetyService` | `safety.service` | ISO 13482/10218/13849、SIL评估、HAZOP/FMEA、协作/移动机器人安全 | Service |
| `WirelessEmcService` | `wireless.service` | FCC/CE RED/SRRC/MIC认证、EMC测试(IEC 61000)、无线共存 | Service |
| `PrivacyDataService` | `privacy.service` | GDPR/CCPA/PIPL评估、DPIA、传感器隐私、跨境传输、DSR仪表盘 | Service |
| `AIEthicsService` | `aiethics.service` | EU AI Act合规、算法透明度、偏见检测、伦理审查 | Service |
| `ConsumerSafetyService` | `consumer.service` | CE机械指令/LVD、UL认证、儿童安全、产品责任 | Service |
| `ExportControlService` | `trade.service` | ECCN分类、加密出口、实体清单筛查、制裁合规 | Service |
| `EnvironmentalService` | `environmental.service` | RoHS/WEEE/REACH合规、碳足迹、能效评估 | Service |
| `ComplianceManagementService` | `management.service` | 文档模板、认证进度追踪、合规审查记录 | Service |
| `domain/*` | `domain` | JPA 实体层（7 Entity：Regulation/ComplianceChecklist/ComplianceItem/CertificationProgress/ComplianceReview/RegulationChange/AuditRecord） | Entity |
| `repository/*` | `repository` | Spring Data JPA 仓库（7 Repository） | Repository |

## 4. 构建与运行

### 4.1 Maven 构建

```bash
# 编译
mvn compile -f qoocompliance/pom.xml

# 打包
mvn package -f qoocompliance/pom.xml

# 跳过测试
mvn package -DskipTests -f qoocompliance/pom.xml
```

### 4.2 开发运行

```bash
# 需先准备 PostgreSQL 数据库
# createdb qoocompliance
# Flyway 会在启动时自动执行数据库迁移（V1 建表 + V2 补字段 + V3 种子数据）

# 启动服务
mvn spring-boot:run -f qoocompliance/pom.xml
```

### 4.3 API 文档

启动后访问：
- Swagger UI: http://localhost:8086/swagger-ui.html
- OpenAPI JSON: http://localhost:8086/v3/api-docs

### 4.4 健康检查

```bash
curl http://localhost:8086/actuator/health
curl http://localhost:8086/actuator/metrics
```

### 4.5 API 端点一览

#### 合规检查清单 (`/api/v1/compliance`)

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/checklist` | 按目标市场生成合规检查清单 |
| GET | `/projects/{projectId}/items` | 获取项目检查项 |
| PUT | `/projects/{projectId}/items/{itemId}` | 更新检查项状态 |
| GET | `/projects/{projectId}/progress` | 项目合规进度 |
| GET | `/projects/{projectId}/gaps` | 合规差距分析 |
| GET | `/projects/{projectId}/report` | 生成合规报告 |

#### 法规监控 (`/api/v1/regulations`)

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/{market}` | 获取市场法规列表 |
| GET | `/{market}/upcoming` | 即将生效的法规 |
| GET | `/{market}/changes` | 法规变更历史 |

#### 各域评估端点

覆盖 `safety/`、`wireless/`、`privacy/`、`ai/`、`consumer/`、`trade/`、`environmental/`、`management/` 共 8 个域，约 45+ REST 端点。
