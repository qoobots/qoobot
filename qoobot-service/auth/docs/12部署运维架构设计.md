## 2. 容器化策略

### 2.1 Dockerfile 最佳实践

```dockerfile
# 多阶段构建
FROM maven:3.9-eclipse-temurin-17 AS builder
WORKDIR /build
COPY pom.xml .
COPY qooauth-common/pom.xml qooauth-common/
COPY qooauth-auth/pom.xml qooauth-auth/
# ... 其他模块
RUN mvn dependency:go-offline -B
COPY . .
RUN mvn package -DskipTests -pl qooauth-auth -am

FROM eclipse-temurin:17-jre-alpine
RUN addgroup -S qooauth && adduser -S qooauth -G qooauth
USER qooauth
COPY --from=builder /build/qooauth-auth/target/*.jar app.jar
HEALTHCHECK --interval=30s --timeout=3s CMD wget -qO- http://localhost:8080/actuator/health || exit 1
ENTRYPOINT ["java", "-XX:+UseZGC", "-jar", "app.jar"]
```

### 2.2 Kubernetes 部署清单

| 资源 | 配置 |
|:-----|:-----|
| **Replicas** | 每个服务 ≥3 |
| **Resources** | Request: 512m CPU/512Mi RAM, Limit: 2 CPU/2Gi RAM |
| **HPA** | min=3, max=10, CPU 70% target |
| **PDB** | minAvailable: 2 |
| **Probes** | Liveness: /actuator/health, Readiness: /actuator/health/readiness |
| **NetworkPolicy** | 仅允许 Gateway → Service 流量 |
| **Service** | ClusterIP 内部, LoadBalancer 外部 |

## 4. 监控与可观测性

### 4.1 四金信号 (Four Golden Signals)

| 信号 | 指标 | 告警阈值 |
|:-----|:-----|:---------|
| **延迟** | P99 请求延迟 | > 500ms (API), > 50ms (gRPC) |
| **流量** | QPS / TPS | 偏离基线 ±30% |
| **错误** | 5xx 错误率 | > 1% |
| **饱和度** | CPU/内存/连接池 | > 80% |

### 4.2 Grafana 仪表盘

| 仪表盘 | 监控内容 |
|:-------|:---------|
| **服务概览** | QPS/延迟/错误率/可用性 (SLO) |
| **认证指标** | 登录成功率/2FA 完成率/Token 签发速率 |
| **安全监控** | 失败登录趋势/异常IP分布/WAF 拦截 |
| **基础设施** | PostgreSQL 连接/复制延迟, Redis 命中率/内存 |
| **业务指标** | 注册成功率/设备激活量/OAuth 授权量 |

### 4.3 告警规则

```yaml
# Prometheus Alert Rules 示例
groups:
  - name: qooauth-critical
    rules:
      - alert: HighErrorRate
        expr: rate(http_server_requests_seconds_count{status=~"5.."}[5m]) > 0.01
        for: 5m
        labels:
          severity: critical
        annotations:
          summary: "qooauth 错误率超过 1%"
          
      - alert: HighLatency
        expr: histogram_quantile(0.99, rate(http_request_duration_seconds_bucket[5m])) > 0.5
        for: 5m
        labels:
          severity: warning
          
      - alert: DatabaseReplicationLag
        expr: pg_replication_lag_bytes > 104857600  # 100MB
        for: 10m
        labels:
          severity: warning
```

## 5. 备份与灾备

### 5.1 备份策略

| 数据 | 方法 | 频率 | 保留 |
|:-----|:-----|:-----|:-----|
| PostgreSQL | pg_dump + WAL 归档 | 每日全量 + 持续 WAL | 30 天 |
| Redis | RDB 快照 + AOF | 每小时 RDB | 7 天 |
| 加密密钥 | HSM 备份卡 | 每次轮换 | 永久 |
| K8s 配置 | Git 仓库 | 每次变更 | 永久 |

### 5.2 灾难恢复

| 指标 | 目标 |
|:-----|:-----|
| **RTO** (恢复时间目标) | < 1 小时 |
| **RPO** (恢复点目标) | < 5 分钟 |
| **跨区域复制** | 主区域 → 灾备区域异步复制 |
| **故障转移** | Patroni 自动 PostgreSQL 主从切换 |
| **演练频率** | 每季度灾备演练 |
