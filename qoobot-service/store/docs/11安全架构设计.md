# qoostore — 安全架构设计

> 版本：v0.1 | 最后更新：2026-06-29 | 状态：Draft | 子项目：qoostore（技能市场）
> 对标：Apple App Store 安全体系

---

## 1. 安全设计原则

1. **信任但验证**：所有技能上传、下载、执行全程验证签名
2. **最小权限**：技能仅获得 manifest 声明的权限
3. **纵深防御**：云端审核 + 端侧沙箱 + 运行时监控
4. **可追溯**：所有操作记录可审计

---

## 2. 威胁模型

| 威胁 | 严重性 | 缓解措施 |
|:-----|:-----:|---------|
| 恶意技能上传 | 🔴 高 | 自动扫描 + 人工审核 + 签名验证 |
| 技能包篡改 | 🔴 高 | Ed25519 签名 + 哈希校验 |
| 技能越权访问 | 🔴 高 | 沙箱隔离 + 权限管控 + seccomp |
| 支付欺诈 | 🔴 高 | 支付平台回调验证 + 签名校验 |
| 虚假评价 | 🟠 中 | 购买验证 + 异常检测 |
| 审核绕过 | 🟠 中 | 多层审核 + 权限分离 |
| 数据泄露 | 🟠 中 | 端到端加密 + 租户隔离 |

## 4. 权限模型

### 4.1 权限声明 (manifest.yaml)

```yaml
permissions:
  # 传感器权限
  camera:
    - capture      # 拍照
    - stream       # 视频流
  microphone:
    - record       # 录音
  # 运动权限
  navigation:
    - move         # 基础移动
    - precise_move # 精确移动
  # 数据权限
  spatial:
    - read_map     # 读取地图
  user_data:
    - read_profile # 读取用户信息
  # 网络权限
  network:
    - internet     # 互联网访问
    - local        # 局域网访问
```

### 4.2 权限分级

| 级别 | 风险 | 授权方式 | 示例 |
|:----:|:---:|---------|------|
| L0 自动授予 | 无 | 安装时自动 | 读取技能配置 |
| L1 安装时确认 | 低 | 安装弹窗 | 读取地图、基础移动 |
| L2 运行时确认 | 中 | 首次使用时弹窗 | 摄像头、麦克风 |
| L3 每次确认 | 高 | 每次使用 | 支付、删除数据 |

## 5. 运行时安全

| 机制 | 实现 | 说明 |
|:-----|------|------|
| 进程隔离 | gVisor / seccomp cgroup | 技能独立运行空间 |
| 资源限制 | cgroup (cpu/mem/io) | 防止资源耗尽 |
| 网络控制 | netns + eBPF | 仅允许声明的网络访问 |
| 文件系统 | overlayfs (只读系统层) | 防止系统文件篡改 |
| API 管控 | gRPC interceptor | 权限检查 + 频率限制 |
| 异常终止 | Monitor 检测 + 强制 kill | 恶意行为自动终止 |

## 6. 支付安全

| 环节 | 措施 |
|:-----|------|
| 支付请求 | HTTPS + 签名 + 防重放 (nonce + timestamp) |
| 支付回调 | 签名验证 + IP 白名单 + 幂等处理 |
| 订单存储 | 加密存储 + 不可篡改审计 |
| 退款 | 多重审批 + 风控规则 |
| 结算 | 最小金额门槛 + 身份验证 |
