## 1. 产品定位

qoocloud 是连接每一台 QooBot 与云端智能的关键枢纽。它不是替代端侧智能，而是端侧能力的**补充与延伸**——在端侧无法胜任的重型推理、全局优化、数据沉淀等场景下提供云端支持。

## 4. 核心功能流程

### 4.1 设备激活流程

```
出厂 → 首次开机 → 连接WiFi → 生成设备密钥对
                │
                ▼
         云端注册请求 (设备SN+公钥)
                │
                ▼
         qoocloud 签发设备证书 (X.509)
                │
                ▼
         绑定用户账户 (qooauth)
                │
                ▼
         下发初始配置 → 激活完成
```

### 4.2 OTA 升级流程

```
开发者上传新版本 → 审核通过 → 灰度策略设置
                                  │
                    ┌─────────────┼─────────────┐
                    ▼             ▼             ▼
                 5%设备        20%设备       全量
                    │             │             │
            监控指标正常?   监控指标正常?   
            ├─ 是 → 扩大     ├─ 是 → 全量
            └─ 否 → 回滚     └─ 否 → 回滚
```
