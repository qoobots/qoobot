## 2. 关键技术决策

### 2.1 为什么选择 MQTT 而非 WebSocket 作为主设备协议

| 对比 | MQTT | WebSocket |
|:-----|:-----|:---------|
| 带宽效率 | 二进制协议，极小开销 | 文本协议，开销大 |
| QoS 等级 | QoS 0/1/2 三级保证 | 无原生 QoS |
| 离线消息 | 遗嘱消息、保留消息 | 需自建 |
| IoT 生态 | 事实标准 | 通用 Web 协议 |
| 双向通信 | 原生 Pub/Sub | 需自建通道 |

**结论**：MQTT 作为主设备协议，WebSocket 作为浏览器端 Web 控制台通道。

### 2.2 为什么 vLLM + Triton + TGI 同时使用

| 引擎 | 适用场景 | 优势 |
|:-----|---------|------|
| vLLM | 大语言模型推理 | PagedAttention、连续批处理 |
| Triton | 视觉/嵌入/传统 DL 模型 | 多框架支持、动态批处理 |
| TGI | HuggingFace 模型快速部署 | HF 生态集成、语法约束 |

**决策**：根据模型类型路由到最优推理引擎，统一推理 API 层屏蔽差异。

### 2.3 为什么需要 ClickHouse

- 设备遥测数据（每秒上千条）
- 推理调用日志（Token 级记录）
- 监控指标长期存储
- MySQL 不适合时序大规模写入
- ClickHouse 压缩比 10:1+，查询性能亚秒级

## 3. 性能设计

### 3.1 推理链路优化

```
请求 → API Gateway (Kong, 5ms)
     → 认证 (Redis, 2ms)
     → 限流检查 (Sentinel, 1ms)
     → 路由到推理服务 (内部 gRPC, 3ms)
     → GPU 推理 (模型 dependent, 主要耗时)
     → 流式返回 (SSE/gRPC stream)
     
API 层延迟：~11ms + 模型推理时间
```

### 3.2 缓存策略

| 层级 | 缓存内容 | TTL | 命中率目标 |
|:-----|---------|:---:|:--------:|
| API 网关 | 认证 Token | 5min | >95% |
| 推理服务 | 语义相似请求 | 1h | >30% |
| 推理服务 | KV Cache (Prefix Caching) | 会话期 | >40% |
| 设备管理 | 设备影子 | 实时 | N/A |
| 数据服务 | 热数据 | 1h | >80% |

### 3.3 容量规划

| 指标 | 1000 设备 | 10000 设备 | 100000 设备 |
|:-----|:--------:|:--------:|:---------:|
| MQTT 连接数 | 1000 | 10000 | 100000 |
| API QPS | 100 | 1000 | 10000 |
| 推理 TPS | 50 | 500 | 5000 |
| 日增数据量 | 10GB | 100GB | 1TB |
| 所需 GPU | 2 A10 | 8 A10 + 2 A100 | 集群 |

### 4.1 故障转移

| 故障类型 | 检测方式 | 恢复方式 | RTO | RPO |
|:---------|---------|---------|:---:|:---:|
| 单 Pod 故障 | K8s Liveness Probe | 自动重启 | <30s | 0 |
| 单节点故障 | K8s Node NotReady | Pod 漂移 | <2min | 0 |
| 单 AZ 故障 | 健康检查 | 流量切换到其他 AZ | <5min | <1min |
| 单 Region 故障 | GSLB 检测 | DNS 切换到备 Region | <15min | <5min |

## 5. 安全设计

### 5.1 数据安全

| 数据 | 传输加密 | 存储加密 | 访问控制 |
|:-----|:-------:|:-------:|:-------:|
| 设备凭据 | TLS 1.3 | AES-256-GCM | Vault HSM |
| 用户数据 | TLS 1.3 | AES-256-GCM | RBAC + 租户隔离 |
| 推理数据 | TLS 1.3 | 不存储（或立即删除） | 租户隔离 |
| 模型文件 | TLS 1.3 | AES-256 + 签名 | RBAC |
| 设备遥测 | TLS 1.3 | 可选加密 | 租户隔离 |

### 5.2 安全架构

```
外部请求
    │
    ▼
WAF (ModSecurity/Coraza)
    │
    ▼
DDoS 防护 (云厂商)
    │
    ▼
API Gateway (认证 + 限流)
    │
    ▼
微服务 (mTLS)
    │
    ▼
数据库 (TLS + 加密)
```
