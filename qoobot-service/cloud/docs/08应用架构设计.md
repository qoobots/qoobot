## 1. 架构总览

qoocloud 采用**微服务 + 事件驱动**架构，以 Spring Cloud 为微服务框架，Kafka 为事件总线，K8s 为容器编排平台。

## 2. 模块职责

### 2.1 qoocloud-gateway (API 网关)

| 功能 | 描述 |
|:-----|------|
| 统一认证 | JWT 校验 + qooauth 集成 |
| 限流熔断 | Sentinel 限流（用户级/API级） |
| 路由分发 | 根据 path/header 路由到对应微服务 |
| 协议转换 | HTTP ↔ gRPC ↔ WebSocket |

### 2.2 qoocloud-device (设备管理)

| 功能 | 描述 |
|:-----|------|
| 设备注册 | 设备凭证签发（X.509）、激活绑定 |
| 设备影子 | Desired/Reported 状态同步 |
| 心跳管理 | MQTT keepalive + 离线检测 |
| 命令下发 | 异步命令队列、离线缓存 |
| 配置管理 | 批量配置推送、版本管理 |

### 2.3 qoocloud-ota (OTA 升级)

| 功能 | 描述 |
|:-----|------|
| 包管理 | 固件/模型/技能包上传、版本化 |
| 差分生成 | bsdiff/zstd 增量包生成 |
| 灰度发布 | 策略配置、自动扩量/回滚 |
| 下载管理 | CDN 分发、断点续传、校验 |

### 2.4 qoocloud-inference (推理服务)

| 功能 | 描述 |
|:-----|------|
| 模型管理 | 模型注册、版本、热切换 |
| 推理调度 | GPU 资源分配、优先级队列 |
| LLM 服务 | vLLM/TGI 部署、流式推理 |
| 缓存层 | 语义缓存、KV 缓存复用 |

### 2.5 qoocloud-data (数据同步)

| 功能 | 描述 |
|:-----|------|
| 经验采集 | 操作轨迹、传感器数据流式接收 |
| 知识图谱 | 多设备共享地图/场景/物体数据 |
| 联邦学习 | 参数聚合、安全聚合 |
| 数据管道 | ETL、归档、过期策略 |

### 2.6 qoocloud-orchestra (多机器人编排)

| 功能 | 描述 |
|:-----|------|
| 集群管理 | 机器人注册、分组、拓扑 |
| 任务调度 | 基于能力/位置/负载的任务分配 |
| 冲突解决 | 路径冲突检测与规避 |
| 协作 API | 多机同步执行指令 |

### 2.7 qoocloud-twin (数字孪生)

| 功能 | 描述 |
|:-----|------|
| 状态同步 | 物理→虚拟状态实时映射 |
| 场景仿真 | MuJoCo/Isaac Sim 虚拟环境 |
| 异常推演 | What-If 分析、安全边界测试 |
| 回放分析 | 历史轨迹重放、事故复现 |

### 2.8 qoocloud-observability (可观测性)

| 功能 | 描述 |
|:-----|------|
| 链路追踪 | OpenTelemetry + Jaeger |
| 日志聚合 | ELK/Loki 集中日志 |
| 指标监控 | Prometheus + Grafana |
| 告警管理 | AlertManager 多级告警 |

## 3. 关键数据流

### 3.1 端云推理请求流

```
机器人端                      qoocloud                     GPU集群
   │                            │                            │
   │── gRPC InferenceRequest ──▶│                            │
   │                            │── 路由选择 ──────────────▶│
   │                            │   (vLLM / Triton / TGI)    │
   │                            │                            │
   │◀─ streaming tokens ───────│◀─ streaming tokens ────────│
   │   (SSE / gRPC stream)     │                            │
```

### 3.2 设备管理数据流

```
设备 MQTT 消息
    │
    ▼
Device Gateway ──▶ Kafka (device.events)
                      │
          ┌───────────┼───────────┐
          ▼           ▼           ▼
     Device Svc   Shadow Svc   Telemetry Svc
          │           │           │
          ▼           ▼           ▼
        MySQL       Redis     ClickHouse
```
