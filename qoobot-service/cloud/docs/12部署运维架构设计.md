## 2. 容器化

### 2.1 Dockerfile 规范

```dockerfile
# 多阶段构建
FROM eclipse-temurin:21-jre-alpine AS runtime
RUN addgroup -S qoo && adduser -S qoo -G qoo
USER qoo
COPY --from=builder /app/target/*.jar /app/app.jar
EXPOSE 8080
HEALTHCHECK --interval=30s --timeout=3s \
  CMD wget -q --spider http://localhost:8080/actuator/health || exit 1
ENTRYPOINT ["java", "-jar", "/app/app.jar"]
```

### 2.2 K8s 资源配置

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: qoocloud-device
spec:
  replicas: 3
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxUnavailable: 1
      maxSurge: 1
  template:
    spec:
      containers:
      - name: device-svc
        resources:
          requests:
            cpu: "500m"
            memory: "1Gi"
          limits:
            cpu: "2000m"
            memory: "2Gi"
        livenessProbe:
          httpGet:
            path: /actuator/health/liveness
            port: 8080
        readinessProbe:
          httpGet:
            path: /actuator/health/readiness
            port: 8080
```

## 7. 成本优化

| 优化策略 | 预期节省 | 实施计划 |
|:---------|:------:|---------|
| GPU 节点 Spot Instance | 40-60% | 非推理核心 GPU 任务 |
| K8s HPA + VPA 精细化 | 15-25% | 按负载自动调整 |
| ClickHouse TTL 自动归档 | 20-30% | 冷数据自动删除/归档至 S3 |
| CDN 加速 OTA 分发 | 30-50% | 减少源站带宽 |
| 推理缓存 | 10-20% | 减少重复推理 |
