## 1. 质量策略

qoocommunity 的质量保障不同于软件测试——它更多关注**社区健康度、文档质量、贡献者体验**。

---

## 2. 文档质量

### 2.1 文档检查清单

| 检查项 | 工具 | 频率 |
|:-------|------|:---:|
| 死链检查 | markdown-link-check / Lychee | 每次 PR |
| 拼写检查 | cspell / Vale | 每次 PR |
| 代码示例可运行 | pytest --doctest | 每次 PR |
| 格式一致性 | markdownlint | 每次 PR |
| API 文档完整性 | Sphinx coverage | 每次构建 |

### 2.2 文档质量指标

| 指标 | 目标 |
|:-----|:----:|
| API 文档覆盖率 | >90% |
| 文档死链 | 0 |
| 示例代码通过率 | 100% |
| 文档搜索满意度 | >80% |
| 文档 NPS | >60 |

### 2.3 自动文档检查

```yaml
# .github/workflows/docs-check.yml
jobs:
  docs-check:
    steps:
      - name: Check dead links
        run: npx markdown-link-check docs/**/*.md
        
      - name: Lint markdown
        run: npx markdownlint docs/**/*.md
        
      - name: Check spelling
        run: npx cspell docs/**/*.md
        
      - name: Test code examples
        run: pytest --doctest-modules docs/examples/
```

## 4. CI 质量门禁

| 检查 | 工具 | 阻断条件 |
|:-----|------|---------|
| Code Style (Python) | ruff | 任何错误 |
| Code Style (C++) | clang-format | 不匹配 |
| Unit Tests | pytest / GTest | 失败 |
| Doc Build | mkdocs build | 构建失败 |
| Link Check | lychee | 死链 |
| CLA Check | cla-assistant | CLA 未签署 |
| Commit Message | commitlint | 格式不符 |

## 5. 贡献者体验

### 5.1 新贡献者体验检查

```
新贡献者入门路径验证 (每月):
  [ ] 首页 → 快速入门 → 可 30min 内完成
  [ ] Good First Issue 标签有可用任务
  [ ] CONTRIBUTING.md 清晰易懂
  [ ] 本地开发环境 15min 内可搭建
  [ ] CLA 签署流程 < 2min
  [ ] 首次 PR 48h 内获得 Review
```

### 5.2 贡献者反馈收集

| 方式 | 频率 | 内容 |
|:-----|:---:|------|
| 贡献者调研 | 每季度 | NPS、痛点、建议 |
| 新贡献者采访 | 每月 | 入门体验、困难点 |
| 退出访谈 | 按需 | 离开原因 |
| 社区投票 | 持续 | 功能优先级 |
