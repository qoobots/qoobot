# qoocommunity — 安全架构设计

> 版本：v0.1 | 最后更新：2026-06-29 | 状态：Draft | 子项目：qoocommunity（开源社区）

---

## 1. 安全设计原则

1. **开放与安全平衡**：开源社区天然开放，但核心基础设施需保护
2. **最小权限**：GitHub Org 权限严格分级
3. **供应链安全**：依赖管理、CI 安全、发布签名
4. **社区安全**：反垃圾、反骚扰、行为准则执行

---

## 2. GitHub 组织安全

### 2.1 权限分级

| 角色 | 权限 |
|:-----|------|
| Outside Collaborator | 特定仓库读写 |
| Member | 所有公开仓库读写 |
| Maintainer | 特定仓库管理权 |
| Owner/Admin | 组织级管理、密钥访问 |

### 2.2 分支保护

```
main 分支保护规则:
  · 需要 PR (禁止直接 push)
  · 需要至少 1 个 Review
  · 需要 CI 通过
  · 需要 CLA 检查通过
  · 禁止 force push
  · 禁止删除分支
```

### 2.3 密钥管理

```
GitHub Secrets:
  · PYPI_TOKEN (发布 SDK)
  · DOCKER_PASSWORD (镜像发布)
  · CROWDIN_TOKEN (翻译同步)
  · DEPLOY_KEY (文档部署)

规则:
  · 仅 Owner 可管理 Secrets
  · Secrets 不可在 PR 中使用 (forks 安全)
  · 使用 OIDC 连接到云服务 (无长期密钥)
```

## 3. 供应链安全

### 3.1 依赖管理

| 措施 | 工具 | 频率 |
|:-----|------|:---:|
| 依赖更新 | Dependabot / Renovate | 每日 |
| 漏洞扫描 | GitHub Dependabot Alerts | 实时 |
| License 审计 | FOSSA / LicenseFinder | 每次 PR |

### 3.2 发布签名

```
SDK 发布流程:
  1. 构建 (CI)
  2. 测试 (CI)
  3. 签名 (GPG/HashiCorp Vault)
  4. 发布到 PyPI / npm / Conan
  5. SBOM 生成 (CycloneDX/SPDX)
  6. 发布通知
```

### 3.3 SBOM (软件物料清单)

```json
{
  "bomFormat": "CycloneDX",
  "specVersion": "1.5",
  "components": [
    {
      "name": "numpy",
      "version": "1.26.4",
      "purl": "pkg:pypi/numpy@1.26.4",
      "licenses": [{"license": {"id": "BSD-3-Clause"}}]
    }
  ]
}
```

## 4. 社区安全

### 4.1 内容安全

| 威胁 | 缓解 |
|:-----|------|
| 垃圾帖 | Akismet + 新用户限制 + 验证码 |
| 恶意链接 | 链接扫描 + 用户举报 |
| 骚扰 | 行为准则 + 举报 + 禁言/封禁 |
| 虚假账号 | GitHub 账号关联 + 行为检测 |

### 4.2 行为准则执行

```
违规举报 → 社区管理团队评审 (48h内)
    │
    ├── 轻微违规 → 警告
    ├── 中度违规 → 临时禁言 (7-30天)
    └── 严重违规 → 永久封禁 + 移除贡献
```

| 数据 | 保护措施 |
|:-----|---------|
