## 2. 文档站点 CI/CD

```yaml
# .github/workflows/docs.yml
name: Deploy Docs
on:
  push:
    branches: [main]
    paths: ['docs/**', 'python/**', 'cpp/**']

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      
      - name: Build API docs
        run: |
          pip install sphinx breathe mkdocs-material mkdocstrings
          cd docs && mkdocs build
      
      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./docs/site
```

## 3. 论坛部署

```yaml
# docker-compose.yml (Discourse)
version: '3'
services:
  discourse:
    image: discourse/discourse:stable
    ports:
      - "80:80"
    environment:
      DISCOURSE_HOSTNAME: community.qoobot.ai
      DISCOURSE_DEVELOPER_EMAILS: admin@qoobot.ai
      DISCOURSE_SMTP_ADDRESS: smtp.mailgun.org
    volumes:
      - discourse_data:/shared
  
  postgres:
    image: postgres:13
    volumes:
      - discourse_db:/var/lib/postgresql/data
```

## 4. 监控

| 指标 | 监控 | 告警 |
|:-----|------|------|
| 文档站点可用性 | UptimeRobot / Checkly | 不可达 > 5min |
| 文档搜索延迟 | Algolia Dashboard | P99 > 500ms |
| 论坛响应时间 | Discourse Admin | P95 > 2s |
| GitHub Actions 失败 | GitHub Notifications | 任何失败 |
| Discord Bot 在线 | 心跳检测 | 离线 > 5min |

## 6. 运维手册

### 6.1 日常运营

- 每日：检查 GitHub Issues/PRs 响应情况
- 每日：审核论坛新帖、处理举报
- 每周：发布社区周报 (Newsletter)
- 每月：更新贡献者统计数据、贡献者墙
- 每季度：社区健康报告

### 6.2 故障处理

| 故障 | 处理 |
|:-----|------|
| 文档站点不可达 | Vercel/GitHub Pages 状态页、DNS 检查 |
| 论坛故障 | Discourse 容器重启、DB 连接检查 |
| Discord Bot 离线 | Bot 重启、Token 检查 |
| 翻译未同步 | Crowdin 连接检查、手动触发同步 |
