## 1. 架构总览

qoochain 不同于其他子项目（不是纯软件），它是**制造流程体系 + 支撑工具链**的组合。核心是流程规范、标准模板、管理工具，辅以自动化标定软件和 MES 数据系统。

```
┌──────────────────────────────────────────────────────────────┐
│                     qoochain 体系架构                          │
├──────────────────────────────────────────────────────────────┤
│                                                               │
│  📋 流程规范层 (Process & Standards)                          │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  BOM 管理规范 │ 产线 SOP │ 标定规程 │ 检验标准 │ RMA  │   │
│  └──────────────────────────────────────────────────────┘   │
│                                                               │
│  ⚙️ 工具链层 (Toolchain)                                      │

│                                                               │
└──────────────────────────────────────────────────────────────┘
```

## 2. 核心工具

### 2.1 BOM 管理工具 (Python + SQLite/PostgreSQL)

```
功能:
  · EBOM → MBOM 自动转换
  · BOM 版本对比 (diff)
  · 成本自动核算
  · 替代料影响分析
  · ERP 系统对接 (API)
```

### 2.2 标定软件 (Python + C++)

```
功能:
  · 标定流程编排 (状态机)
  · 自动化标定算法 (相机/LiDAR/运动学/力觉)
  · 实时精度反馈
  · 标定参数打包存档
  · NG 自动重试
```

### 2.3 QMS 质量管理系统

```
功能:
  · 检验计划管理 (IQC/IPQC/OQC)
  · SPC 统计过程控制 (控制图、CPK)
  · 不合格品处理流程 (NCMR)
  · 质量报告自动生成
  · 纠正预防措施 (CAPA) 追踪
```

### 2.4 追溯系统

```
功能:
  · 器件批次→整机 SN 追溯链
  · 装配/测试/标定 操作记录
  · 数字护照生成 (PDF/JSON)
  · 召回范围精确定位
```

## 3. 标定数据流

```
标定工作站
  │
  ├── 相机图像采集 → 标定算法 → 内参/外参
  ├── LiDAR 扫描 → 点云处理 → 外参
  ├── 关节运动 → 编码器读取 → 运动学参数
  └── 力矩采集 → 零漂/灵敏度 → 力控参数
  │
  ▼
标定参数 JSON
  │
  ├── 写入机器人本地 /etc/qoo/calibration/
  └── 上传云端 (qoochain DB) 永久存档
```
