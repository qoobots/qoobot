# 04_系统数据设计

> 基于 DDD 战术设计，定义 Brain OS 的聚合、实体、值对象、数据存储策略和数据流管道。

---

## 1. DDD 聚合设计

### 1.1 聚合全景

```
限界上下文 → 聚合根 → 实体 → 值对象
```

| 限界上下文 | 聚合根 | 核心实体 | 值对象 |
|-----------|--------|---------|--------|
| **感知** | Scene | SlamPose, DetectedObject, OccupancyGrid | Point3D, Quaternion, BBox3D |
| **认知** | Task | Subtask, Constraint, Intent | SemanticLabel, TimeWindow |
| **决策** | Plan | BehaviorTree, Trajectory, Strategy | JointWaypoint, PoseWaypoint |
| **控制** | Motion | JointTrajectory, ForceCommand | JointState, Wrench |
| **知识** | Episode | Knowledge, ExperienceSample | Embedding, TimeRange |
| **交互** | UISession | TrajectoryView, HumanInput | RenderStyle, ColorRGBA |
| **安全** | SafetyIncident | SafetyRule, ConstraintZone | RiskLevel, VelocityLimit |

## 2. 核心聚合详细设计

### 2.1 感知上下文 — Scene 聚合

```
┌─────────────────────────────────────────────────────────────┐
│  Scene (聚合根)                                               │
│  ┌─────────────────────────────────────────────────────────┐│
│  │ scene_id: UUID                                          ││
│  │ timestamp: Timestamp                                    ││
│  │ robot_pose: Pose6D  (值对象)                             ││
│  │                                                         ││
│  │ 1 ─── * SlamPose (实体)                                  ││
│  │        ├── pose_id: UUID                                ││
│  │        ├── position: Point3D                            ││
│  │        ├── orientation: Quaternion                      ││
│  │        ├── covariance: Matrix6x6                        ││
│  │        └── confidence: float                            ││
│  │                                                         ││
│  │ 1 ─── * DetectedObject (实体)                            ││
│  │        ├── object_id: UUID                              ││
│  │        ├── track_id: int                                ││
│  │        ├── class_name: string                           ││
│  │        ├── bbox: BBox3D                                 ││
│  │        ├── mask: Mask (值对象)                           ││
│  │        ├── pose: Pose6D (值对象, 可选)                   ││
│  │        ├── confidence: float                            ││
│  │        ├── velocity: Velocity3D (值对象, 可选)           ││
│  │        └── attributes: dict (值对象)                     ││
│  │                                                         ││
│  │ 1 ─── 1 OccupancyGrid (实体)                             ││
│  │        ├── grid_id: UUID                                ││
│  │        ├── resolution: float                            ││
│  │        ├── dimensions: (int, int, int)                  ││
│  │        ├── origin: Point3D                              ││
│  │        ├── data: bytes (体素网格)                        ││
│  │        └── timestamp: Timestamp                         ││
│  │                                                         ││
│  │ 1 ─── 1 GSModel (实体)                                   ││
│  │        ├── model_id: UUID                               ││
│  │        ├── gaussian_count: int                          ││
│  │        ├── model_binary: bytes (或文件引用)              ││
│  │        └── render_quality: float                        ││
│  └─────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────┘
```

**不变量**：
- Scene 的 timestamp 必须单调递增
- SlamPose 必须在世界坐标系中
- OccupancyGrid 的 dimensions 必须与 resolution 一致
- DetectedObject 的 bbox 必须在场景范围内

### 2.2 认知上下文 — Task 聚合

```
┌─────────────────────────────────────────────────────────────┐
│  Task (聚合根)                                                │
│  ┌─────────────────────────────────────────────────────────┐│
│  │ task_id: UUID                                           ││
│  │ session_id: UUID                                        ││
│  │ raw_utterance: string (原始指令)                          ││
│  │ recognized_intent: Intent (值对象)                        ││
│  │   ├── action: string ("fetch"/"place"/"navigate"/...)   ││
│  │   ├── target: EntityRef (值对象)                         ││
│  │   ├── destination: EntityRef (值对象, 可选)              ││
│  │   └── constraints: Constraint[] (值对象列表)             ││
│  │ status: TaskStatus (enum)                                ││
│  │ created_at: Timestamp                                    ││
│  │                                                         ││
│  │ 1 ─── * Subtask (实体，有序列表)                          ││
│  │        ├── subtask_id: UUID                             ││
│  │        ├── order: int                                   ││
│  │        ├── skill_name: string                           ││
│  │        ├── parameters: dict                             ││
│  │        ├── preconditions: Condition[]                   ││
│  │        ├── expected_duration: float (秒)                 ││
│  │        ├── status: SubtaskStatus                        ││
│  │        └── dependencies: UUID[] (依赖的前置 subtask)     ││
│  │                                                         ││
│  │ 1 ─── 1 SemanticContext (实体)                            ││
│  │        ├── context_id: UUID                             ││
│  │        ├── scene_summary: string                        ││
│  │        ├── relevant_objects: EntityRef[]                ││
│  │        ├── safety_notes: string[]                       ││
│  │        └── confidence: float                            ││
│  └─────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────┘
```

**不变量**：
- Subtask 的 order 必须连续且唯一
- Subtask 的 dependencies 只能引用 order 更小的 subtask
- Task 状态转换：CREATED → PLANNING → EXECUTING → COMPLETED / FAILED / CANCELLED

### 2.3 决策上下文 — Plan 聚合

```
┌─────────────────────────────────────────────────────────────┐
│  Plan (聚合根)                                                │
│  ┌─────────────────────────────────────────────────────────┐│

│  │ task_id: UUID (关联 Task)                                ││
│  │ scene_id: UUID (关联 Scene)                              ││
│  │ status: PlanStatus (enum)                                ││
│  │ collaboration_mode: CollaborationMode (enum)             ││
│  │   ├── AUTONOMOUS / ADVISORY / CONFIRMATION              ││
│  │ created_at: Timestamp                                    ││
│  │                                                         ││
│  │ 1 ─── 1 BehaviorTree (实体)                               ││
│  │        ├── bt_id: UUID                                  ││
│  │        ├── bt_json: string (行为树 JSON 定义)            ││
│  │        ├── nodes_count: int                             ││
│  │        ├── root_node_id: string                         ││
│  │        └── version: string                              ││
│  │                                                         ││
│  │ 1 ─── * Trajectory (实体)                                 ││
│  │        ├── trajectory_id: UUID                          ││
│  │        ├── strategy: StrategyType (enum)                ││
│  │        │   OPTIMAL / CONSERVATIVE / AGGRESSIVE /        ││
│  │        │   EXPLORATORY / ADVERSARIAL                    ││
│  │        ├── score: float (0~1)                           ││
│  │        ├── scores_detail: ScoreDetail (值对象)           ││
│  │        │   ├── safety_score: float                      ││
│  │        │   ├── efficiency_score: float                  ││
│  │        │   ├── smoothness_score: float                  ││
│  │        │   └── success_probability: float               ││
│  │        ├── waypoints: Waypoint[] (值对象列表)             ││
│  │        │   ├── timestamp: float (相对时间,秒)            ││
│  │        │   ├── joint_positions: float[]                 ││
│  │        │   ├── ee_pose: Pose6D                          ││
│  │        │   └── gripper_state: float (0=合, 1=开)        ││
│  │        ├── duration: float (总时长,秒)                   ││
│  │        ├── key_events: KeyEvent[] (值对象列表)            ││
│  │        │   ├── event_time: float                        ││
│  │        │   ├── event_type: string ("grasp"/"release"/...) ││
│  │        │   └── description: string                      ││
│  │        └── is_selected: bool (是否被选定执行)             ││
│  │                                                         ││
│  │ 1 ─── 1 HITLDecision (实体, 可选: 人在回路决策记录)       ││
│  │        ├── decision_id: UUID                            ││
│  │        ├── selected_trajectory_id: UUID                 ││
│  │        ├── decision_source: string (human/auto/override)││
│  │        ├── decision_time_ms: int                        ││
│  │        ├── human_id: string (可选)                       ││
│  │        └── overridden_command: string (可选)             ││
│  └─────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────┘
```

**不变量**：
- 一个 Plan 至少包含 1 条 Trajectory（最优策略）
- 最终被执行的轨迹只有一条
- score 必须由安全、效率、平滑度、成功概率加权计算

## 3. 值对象汇总

| 值对象 | 属性 | 使用场景 |
|--------|------|----------|
| **Point3D** | x, y, z (float) | 空间位置 |
| **Quaternion** | x, y, z, w (float) | 姿态旋转 |
| **Pose6D** | position: Point3D, orientation: Quaternion | 6DoF 位姿 |
| **Velocity3D** | linear: Point3D, angular: Point3D | 速度 |
| **BBox3D** | center: Point3D, size: Point3D | 3D 包围盒 |
| **JointState** | name: str[], position: float[], velocity: float[], effort: float[] | 关节状态 |
| **Waypoint** | timestamp: float, joints: float[], ee_pose: Pose6D | 轨迹路径点 |
| **Intent** | action: str, target: EntityRef, constraints: Constraint[] | 意图结构 |
| **EntityRef** | entity_type: str, entity_id: str, attributes: dict | 实体引用 |
| **Constraint** | type: str, target: str, value: any, priority: int | 约束条件 |
| **ScoreDetail** | safety: float, efficiency: float, smoothness: float, success_prob: float | 轨迹评分 |
| **Timestamp** | seconds: int, nanoseconds: int | 时间戳 (ROS 2 兼容) |
| **Embedding** | vector: float[], dimension: int, model: str | 语义向量 |
| **ColorRGBA** | r, g, b, a (float 0~1) | 渲染颜色 |
| **RiskLevel** | S0/S1/S2/S3 | 安全风险等级 |
| **KeyEvent** | time: float, type: str, description: str | 轨迹关键事件 |

## 4. 数据存储设计

### 4.1 Phase 1：SQLite 单文件方案

#### 4.1.1 表结构设计

```sql
-- ============================================
-- 任务与计划
-- ============================================

CREATE TABLE tasks (
    task_id        TEXT PRIMARY KEY,
    session_id     TEXT NOT NULL,
    raw_utterance  TEXT NOT NULL,
    intent_json    TEXT NOT NULL,  -- JSON: Intent 值对象序列化
    status         TEXT NOT NULL DEFAULT 'CREATED',
    created_at     TEXT NOT NULL,  -- ISO 8601
    completed_at   TEXT
);

CREATE TABLE subtasks (
    subtask_id     TEXT PRIMARY KEY,
    task_id        TEXT NOT NULL REFERENCES tasks(task_id),
    order_idx      INTEGER NOT NULL,
    skill_name     TEXT NOT NULL,
    parameters_json TEXT,          -- JSON: dict
    dependencies_json TEXT,        -- JSON: UUID[]
    status         TEXT NOT NULL DEFAULT 'PENDING',
    UNIQUE(task_id, order_idx)
);

CREATE TABLE plans (
    plan_id        TEXT PRIMARY KEY,
    task_id        TEXT NOT NULL REFERENCES tasks(task_id),
    scene_id       TEXT NOT NULL,
    collaboration_mode TEXT NOT NULL,
    status         TEXT NOT NULL DEFAULT 'CREATED',
    created_at     TEXT NOT NULL
);

CREATE TABLE behavior_trees (
    bt_id          TEXT PRIMARY KEY,
    plan_id        TEXT NOT NULL REFERENCES plans(plan_id),
    bt_json        TEXT NOT NULL,  -- 行为树定义
    nodes_count    INTEGER,
    root_node_id   TEXT,
    version        TEXT
);

CREATE TABLE trajectories (
    trajectory_id  TEXT PRIMARY KEY,
    plan_id        TEXT NOT NULL REFERENCES plans(plan_id),
    strategy       TEXT NOT NULL,  -- OPTIMAL/CONSERVATIVE/...
    score          REAL NOT NULL,
    scores_detail_json TEXT,       -- JSON: ScoreDetail
    waypoints_json TEXT,           -- JSON: Waypoint[]
    duration       REAL,
    key_events_json TEXT,          -- JSON: KeyEvent[]
    is_selected    INTEGER DEFAULT 0
);

CREATE TABLE hitl_decisions (
    decision_id    TEXT PRIMARY KEY,
    plan_id        TEXT NOT NULL REFERENCES plans(plan_id),
    selected_trajectory_id TEXT NOT NULL REFERENCES trajectories(trajectory_id),
    decision_source TEXT NOT NULL,  -- human/auto/override
    decision_time_ms INTEGER,
    human_id       TEXT
);

-- ============================================
-- 感知与场景
-- ============================================

CREATE TABLE scenes (
    scene_id       TEXT PRIMARY KEY,
    timestamp      TEXT NOT NULL,
    robot_pose_json TEXT NOT NULL     -- JSON: Pose6D
);

CREATE TABLE detected_objects (
    object_id      TEXT PRIMARY KEY,
    scene_id       TEXT NOT NULL REFERENCES scenes(scene_id),
    track_id       INTEGER,
    class_name     TEXT NOT NULL,
    bbox_json      TEXT NOT NULL,     -- JSON: BBox3D
    pose_json      TEXT,              -- JSON: Pose6D (可选)
    confidence     REAL,
    attributes_json TEXT              -- JSON: dict (颜色/材质等)
);

-- ============================================
-- 运动与执行
-- ============================================

CREATE TABLE motions (
    motion_id      TEXT PRIMARY KEY,
    plan_id        TEXT NOT NULL REFERENCES plans(plan_id),
    trajectory_id  TEXT NOT NULL REFERENCES trajectories(trajectory_id),
    status         TEXT NOT NULL DEFAULT 'CREATED',
    started_at     TEXT,
    completed_at   TEXT
);

CREATE TABLE motion_events (
    event_id       TEXT PRIMARY KEY,
    motion_id      TEXT NOT NULL REFERENCES motions(motion_id),
    event_type     TEXT NOT NULL,
    timestamp      TEXT NOT NULL,
    joint_state_json TEXT,            -- JSON: JointState
    ee_pose_json   TEXT,              -- JSON: Pose6D
    deviation      REAL
);

-- ============================================
-- 知识存储
-- ============================================

CREATE TABLE episodes (
    episode_id     TEXT PRIMARY KEY,
    task_id        TEXT NOT NULL REFERENCES tasks(task_id),
    plan_id        TEXT NOT NULL REFERENCES plans(plan_id),
    outcome        TEXT NOT NULL,
    start_time     TEXT NOT NULL,
    end_time       TEXT NOT NULL,
    total_duration REAL,
    metrics_json   TEXT                -- JSON: EpisodeMetrics
);

CREATE TABLE knowledge_entries (
    knowledge_id   TEXT PRIMARY KEY,
    category       TEXT NOT NULL,      -- OBJECT/SCENE/SKILL/SAFETY
    content_json   TEXT NOT NULL,      -- JSONB
    embedding_blob BLOB,               -- 浮点数组的二进制存储
    source_episode_id TEXT REFERENCES episodes(episode_id),
    created_at     TEXT NOT NULL
);

-- ============================================
-- 安全事件
-- ============================================

CREATE TABLE safety_incidents (
    incident_id    TEXT PRIMARY KEY,
    level          TEXT NOT NULL,       -- S0/S1/S2/S3
    type           TEXT NOT NULL,
    timestamp      TEXT NOT NULL,
    source_context TEXT,
    description    TEXT,
    system_response TEXT,
    recovery_json  TEXT
);

-- ============================================
-- 索引
-- ============================================

CREATE INDEX idx_tasks_session ON tasks(session_id);
CREATE INDEX idx_tasks_status ON tasks(status);
CREATE INDEX idx_plans_task ON plans(task_id);
CREATE INDEX idx_trajectories_plan ON trajectories(plan_id);
CREATE INDEX idx_scenes_timestamp ON scenes(timestamp);
CREATE INDEX idx_objects_scene ON detected_objects(scene_id);
CREATE INDEX idx_objects_class ON detected_objects(class_name);
CREATE INDEX idx_episodes_outcome ON episodes(outcome);
CREATE INDEX idx_safety_incidents_level ON safety_incidents(level);
CREATE INDEX idx_safety_incidents_timestamp ON safety_incidents(timestamp);
```

### 4.2 Phase 2：PostgreSQL + pgvector 升级

```sql
-- 向量检索（pgvector 扩展）
CREATE EXTENSION vector;

-- 知识库升级为向量表
CREATE TABLE knowledge_entries (
    knowledge_id   UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    category       TEXT NOT NULL,
    content        JSONB NOT NULL,
    embedding      vector(1024),        -- pgvector 类型
    source_episode_id UUID REFERENCES episodes(episode_id),
    created_at     TIMESTAMPTZ NOT NULL DEFAULT now()
);

-- IVFFlat 索引（百万级以下）
CREATE INDEX idx_knowledge_embedding 
    ON knowledge_entries 
    USING ivfflat (embedding vector_cosine_ops) 
    WITH (lists = 100);

-- 语义检索查询示例
SELECT knowledge_id, category, content,
       1 - (embedding <=> $query_embedding) AS similarity
FROM knowledge_entries
ORDER BY embedding <=> $query_embedding
LIMIT 10;
```

| 配置文件 | `.yaml` / `.json` | `~/.brain_os/config/` | 持久保留 |

### 5.3 事件结构（Protobuf 定义示例）

```protobuf
// 轨迹规划完成事件
message TrajectoriesPlanned {
  string plan_id = 1;
  string task_id = 2;
  repeated TrajectoryInfo trajectories = 3;
  Timestamp timestamp = 4;
}

message TrajectoryInfo {
  string trajectory_id = 1;
  StrategyType strategy = 2;
  float score = 3;
  ScoreDetail score_detail = 4;
  float duration = 5;
  repeated KeyEvent key_events = 6;
}

// 人在回路选择事件
message TrajectorySelected {
  string plan_id = 1;
  string trajectory_id = 2;
  DecisionSource source = 3;  // HUMAN / AUTO / OVERRIDE
  int32 decision_time_ms = 4;
  string human_id = 5;
}

enum DecisionSource {
  AUTO = 0;
  HUMAN = 1;
  OVERRIDE = 2;
}
```

## 6. 数据流管道

### 6.1 感知数据流

```
传感器原始数据
    │
    ├── RGB 图像 ──→ YOLOv11 ──→ DetectedObject[]
    ├── RGB 图像 ──→ SAM 2 ────→ Mask
    ├── RGB-D ─────→ ORB-SLAM3 ─→ SlamPose
    ├── RGB-D ─────→ 3DGS ─────→ GSModel
    ├── RGB-D ─────→ FoundationPose → Pose6D
    │
    ├── 点云 + 检测 ──→ 占用网络 ──→ OccupancyGrid
    │
    └────────────────→ Scene (聚合) ─→ /brain_os/events/perception/*
```

### 6.2 认知→决策数据流

```
用户指令 (文本)
    │
    ▼
Qwen2.5-7B ──→ Intent ──→ Task(聚合)
    │                         │
    │                    Subtask[]
    │                         │
    ▼                         ▼
Scene(聚合) ──→ TaskDecomposer ──→ Plan(聚合)
                                       │
                         ┌─────────────┼─────────────┐
                         ▼             ▼             ▼
                    BehaviorTree  Trajectory[]  HITLDecision
                         │             │             │
                         ▼             ▼             ▼
                    行为树JSON     Waypoint[]    selected_id
```

### 6.3 执行数据流

```
Trajectory (selected)
    │
    ▼
Waypoint[] ──→ JointTrajectoryPoint[]
    │
    ▼
MoveIt 2 ──→ TRAC-IK ──→ 关节角度序列
    │
    ▼
FCL 实时碰撞检测（安全上下文旁路监控）
    │
    ▼
ROS 2 JointTrajectory action ──→ 硬件驱动
    │
    ▼
JointState 反馈 ──→ MotionEvent[] ──→ Motion(聚合)
```

### 6.4 知识积累数据流

```
MotionCompleted + TaskCompleted
    │
    ▼
EpisodeRecorder ──→ Episode(聚合) ──→ SQLite / PostgreSQL
    │                      │
    │                Knowledge (嵌入向量索引)
    │                      │
    ▼                      ▼
Lance 格式            pgvector 检索
(训练数据)            (语义匹配)
```

## 7. 数据生命周期

### 7.1 热数据（实时内存）

| 数据 | 保留策略 | 最大容量 |
|------|----------|----------|
| 当前 Scene | 覆盖更新 | 1 个 |
| 工作记忆上下文 | 当前 Task 期间 | 1 个 Task |
| 最近 100 条事件 | 环形缓冲区 | 内存 10MB 上限 |
| Ghost Trail 渲染数据 | 当前 Plan 期间 | 5 条轨迹 |

### 7.2 温数据（本地磁盘，定期访问）

| 数据 | 保留策略 | 存储位置 |
|------|----------|----------|
| 历史 Scenes | 最近 100 个 | SQLite |
| 历史 Tasks | 最近 1000 个 | SQLite |
| 3DGS 模型 | 每个场景快照 1 个 | 文件系统 |
| 模型权重 | 持久 | 文件系统 |

### 7.3 冷数据（归档/云端，低频访问）

| 数据 | 保留策略 | 存储位置 |
|------|----------|----------|
| 所有 Episodes | 永久 | PostgreSQL / 云端 |
| ROS 2 Bag 日志 | 30 天 | 文件系统（压缩） |
| 安全事件 | 永久 | PostgreSQL |
| 训练数据集 | 永久 | 云端对象存储 |
