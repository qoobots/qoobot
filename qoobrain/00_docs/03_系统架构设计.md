### 3.2 层间依赖规则

```
依赖方向：上层 → 下层（单向依赖）
           应用层
             │
             ▼
          接口层
             │
             ▼
         业务逻辑层 ← 核心，最稳定
             │
             ▼
         基础服务层
             │
             ▼
         基础模型层
             │
             ▼
         基础设施层

规则：
1. 上层可依赖下层，下层不可依赖上层
2. 同层间通过事件/消息松耦合
3. 业务逻辑层只依赖接口抽象（依赖倒置）
4. 基础服务层通过适配器模式对接基础模型层
```

### 5.2 线程模型（brain_core）

```
brain_core 主进程
│
├── Main Thread (ROS 2 spinning)
│   ├── 订阅 /perception/* topics
│   ├── 发布 /control/* topics
│   └── 处理 Service/Action 请求
│
├── Behavior Tree Thread (独立 tick 循环)
│   ├── Tick 频率: 100Hz
│   ├── 执行当前行为树
│   └── 通过回调通知 Python 层
│
├── Safety Monitor Thread (最高优先级)
│   ├── 频率: 1000Hz (1kHz)
│   ├── 碰撞检测 + 限位检测
│   └── 紧急停止触发（零拷贝通知）
│
└── gRPC Server Thread Pool (4 线程)
    ├── 接收 Python 层的 gRPC 调用
    └── 接收 Web 前端的 gRPC-Web 调用
```

|------|------|

### 8.3 安全架构决策

```
安全设计原则：
1. 纵深防御 (Defense in Depth)
   - 硬件急停 → 安全监控 → 行为树约束 → 轨迹安全验证 → 人类覆写

2. 最小权限 (Least Privilege)
   - brain_core (root/cap_sys_nice) > brain_ai (normal user) > brain_viz (web server user)

3. 故障安全 (Fail-Safe)
   - 任何组件故障 → 回退到安全状态（停止运动）
   - 通信中断 → 超时停止
   - GPU 故障 → CPU 推理降级

4. 审计追踪 (Audit Trail)
   - 所有安全事件 → 不可篡改日志
   - 所有人类操作 → 操作记录
```
