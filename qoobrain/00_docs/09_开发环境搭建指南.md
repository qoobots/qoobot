## 1. 环境要求与前置依赖

### 1.1 硬件最低要求（Phase 1 开发）

| 资源 | 最低配置 | 推荐配置 | 说明 |
|------|----------|----------|------|
| **CPU** | 8 核 x86_64 | 16 核+ | 需 AVX2 指令集 |
| **内存** | 32 GB | 64 GB+ | brain_ai (Qwen2.5) 需 ~16GB |
| **GPU** | NVIDIA 8GB VRAM | NVIDIA 24GB VRAM+ | Qwen2.5-7B 推理/训练 |
| **磁盘** | 200 GB SSD | 500 GB NVMe | 模型权重 + Docker 镜像 + 源码 |
| **网络** | 稳定互联网 | - | 拉取 Docker 镜像、PyPI/npm 包 |

### 1.2 无 GPU 场景说明

若开发机无 NVIDIA GPU，brain_ai 将使用 CPU 降级方案：

| 降级项 | GPU 模式 | CPU 模式 |
|--------|----------|----------|
| LLM 推理 | Qwen2.5-7B (TensorRT-LLM) | Qwen2.5-1.5B (llama.cpp) |
| 推理速度 | ~50 tokens/s | ~10 tokens/s |
| 3DGS 重建 | CUDA 加速 | 禁用/跳过 |

> CPU 模式仅用于前端/逻辑开发调试，不可用于性能评估。

### 1.3 软件前置条件

```bash
# 检查操作系统版本
lsb_release -a
# 预期: Ubuntu 24.04 LTS (Noble Numbat)

# 检查内核版本
uname -r
# 预期: >= 6.8 (建议 RT_PREEMPT 内核)

# 检查 Docker
docker --version
# 预期: >= 27.x

# 检查 NVIDIA 驱动 (GPU 场景)
nvidia-smi
# 预期: Driver >= 550, CUDA >= 12.4
```

### 2.2 安装 Docker 与 NVIDIA Container Toolkit

```bash
# === Docker Engine ===
# 卸载旧版本(如有)
for pkg in docker.io docker-doc docker-compose docker-compose-v2 \
           podman-docker containerd runc; do
  sudo apt remove -y $pkg 2>/dev/null || true
done

# 添加 Docker 官方 APT 源
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
  sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

echo "deb [arch=$(dpkg --print-architecture) \
  signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io \
  docker-buildx-plugin docker-compose-plugin

# 验证
sudo docker run --rm hello-world

# 免 sudo 运行 Docker
sudo usermod -aG docker $USER
newgrp docker
docker run --rm hello-world   # 应无需 sudo
```

```bash
# === NVIDIA Container Toolkit (GPU 场景) ===
curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey | \
```

### 3.2 docker-compose.yml 关键配置

```yaml
# docker-compose.yml (根目录，已创建)
version: "3.9"

services:
  brain_core:
    build:
      context: ./brain_core
      dockerfile: Dockerfile.dev
    image: brain-os/core:latest
    container_name: brain_core
    hostname: brain-core
    privileged: true           # ROS 2 需要
    ipc: host                  # 共享内存 (ROS 2 DDS)
    pid: "host"                # 进程命名空间
    network_mode: host         # ROS 2 节点发现需要 host 网络
    environment:
      - ROS_DOMAIN_ID=42
      - RMW_IMPLEMENTATION=rmw_cyclonedds_cpp
    volumes:
      - ./brain_core:/workspace/brain_core
      - ./brain_proto:/workspace/brain_proto
      - /dev:/dev               # 硬件访问 (仿真模式可省略)
    command: ["/bin/bash", "-c", "source /opt/ros/jazzy/setup.bash && tail -f /dev/null"]
    # GPU 场景添加:
    # deploy:
    #   resources:
    #     reservations:
    #       devices:
    #         - driver: nvidia
    #           count: 1
    #           capabilities: [gpu]

  brain_ai:
    build:
      context: ./brain_ai
      dockerfile: Dockerfile.dev
    image: brain-os/ai:latest
    container_name: brain_ai
    hostname: brain-ai
    network_mode: host
    platform: linux/amd64
    environment:
      - ROS_DOMAIN_ID=42
      - RMW_IMPLEMENTATION=rmw_cyclonedds_cpp
      - PYTHONUNBUFFERED=1
      - NVIDIA_VISIBLE_DEVICES=all  # GPU 场景
    volumes:
      - ./brain_ai:/workspace/brain_ai
      - ./brain_proto:/workspace/brain_proto
      - ~/.cache/huggingface:/root/.cache/huggingface  # 模型缓存复用
      - /dev:/dev
    command: ["/bin/bash", "-c", "tail -f /dev/null"]
    # GPU 场景添加:
    # deploy:
    #   resources:
    #     reservations:
    #       devices:
    #         - driver: nvidia
    #           count: 1
    #           capabilities: [gpu]

  brain_viz:
    build:
      context: ./brain_viz
      dockerfile: Dockerfile.dev
    image: brain-os/viz:latest
    container_name: brain_viz
    hostname: brain-viz
    network_mode: host
    environment:
      - NODE_ENV=development
      - NEXT_PUBLIC_GRPC_PROXY=http://localhost:50052
      - NEXT_PUBLIC_WS_URL=ws://localhost:8765
    volumes:
      - ./brain_viz:/workspace/brain_viz
      - ./brain_proto:/workspace/brain_proto
      - brain_viz_node_modules:/workspace/brain_viz/node_modules
    ports:
      - "3000:3000"
    command: ["npm", "run", "dev"]

volumes:
  brain_viz_node_modules:
```

### 3.3 构建与启动

```bash
cd ~/workspace/brain_os

# ============================================
# 方式 A: GPU 开发机（推荐）
# ============================================
docker compose build --parallel
docker compose up -d

# ============================================
# 方式 B: CPU-only 开发机
# ============================================
# 编辑 docker-compose.yml，注释掉 deploy.resources 中的 GPU 段落
# 或使用覆盖文件
docker compose -f docker-compose.yml -f docker-compose.cpu-only.yml up -d

# ============================================
# 查看状态
# ============================================
docker compose ps
# 预期输出:
# NAME          IMAGE                STATUS
# brain_core    brain-os/core:latest Up
# brain_ai      brain-os/ai:latest   Up
# brain_viz     brain-os/viz:latest  Up

docker compose logs -f          # 查看所有日志
docker compose logs brain_ai    # 查看 brain_ai 日志
```

### 3.4 进入容器开发

```bash
# 进入 C++ 容器
docker exec -it brain_core bash

# 进入 Python 容器
docker exec -it brain_ai bash

# 前端容器通常不需要进入，直接访问 localhost:3000
```

### 3.5 停止与清理

```bash
# 停止所有服务
docker compose stop

# 停止并删除容器
docker compose down

# 停止 + 删除容器 + 删除数据卷
docker compose down -v

# 重新构建（Dockerfile 变更后）
docker compose build --no-cache brain_ai
docker compose up -d brain_ai
```

## 4. 分项目本地开发配置（脱离 Docker）

> 适用于需要 IDE 原生调试、性能剖析、或频繁修改 Dockerfile 不便的场景。

### 4.1 brain_core（C++17 / ROS 2 / CMake）

```bash
# === 1. ROS 2 Jazzy 安装 ===
sudo apt install -y software-properties-common
sudo add-apt-repository universe
sudo apt update && sudo apt install curl -y
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key \
  -o /usr/share/keyrings/ros-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) \
  signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] \
  http://packages.ros.org/ros2/ubuntu $(lsb_release -cs) main" | \
  sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null

sudo apt update
sudo apt install -y ros-jazzy-desktop ros-jazzy-ament-cmake \
  ros-jazzy-rmw-cyclonedds-cpp python3-colcon-common-extensions

# === 2. 额外依赖 ===
sudo apt install -y \
  libeigen3-dev libfcl-dev libompl-dev \
  ros-jazzy-moveit2 \
  libgoogle-glog-dev libgflags-dev \
  libgrpc++-dev libprotobuf-dev protobuf-compiler-grpc

# === 3. 构建 ===
cd ~/workspace/brain_os
source /opt/ros/jazzy/setup.bash

# 方式 A: colcon（ROS 2 标准）
colcon build --packages-select brain_core_msgs brain_core \
  --cmake-args -DCMAKE_BUILD_TYPE=RelWithDebInfo \
               -DCMAKE_EXPORT_COMPILE_COMMANDS=ON
source install/setup.bash

# 方式 B: 纯 CMake（非 ROS 包场景）
cd brain_core
mkdir -p build && cd build
cmake .. -DCMAKE_BUILD_TYPE=RelWithDebInfo \
         -DCMAKE_EXPORT_COMPILE_COMMANDS=ON \
         -GNinja
ninja -j$(nproc)

# === 4. 运行测试 ===
cd ~/workspace/brain_os
colcon test --packages-select brain_core
colcon test-result --verbose
```

### 4.2 brain_ai（Python 3.12 / gRPC / PyTorch）

```bash
# === 1. Python 虚拟环境 ===
cd ~/workspace/brain_os/brain_ai
python3 -m venv .venv
source .venv/bin/activate

# === 2. PyTorch (GPU) ===
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu124

# 或 PyTorch (CPU-only)
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cpu

# === 3. 项目依赖 ===
pip install -e ".[dev]"   # 开发者模式安装

# 主要依赖清单（pyproject.toml 中定义）
# grpcio-tools >= 1.64
# openai >= 1.30        (LLM API 调用)
# transformers >= 4.44
# diffusers >= 0.30     (Diffusion Policy)
# langchain >= 0.3
# numpy >= 1.26
# opencv-python-headless >= 4.10

# === 4. LLM 模型下载 (GPU 场景) ===
huggingface-cli download Qwen/Qwen2.5-7B-Instruct \
  --local-dir ~/models/Qwen2.5-7B-Instruct
# 仅首次 ~15GB

# LLM 模型下载 (CPU 场景)
huggingface-cli download Qwen/Qwen2.5-1.5B-Instruct \
  --local-dir ~/models/Qwen2.5-1.5B-Instruct
# 约 ~3GB

# === 5. 编译 Protobuf ===
cd ~/workspace/brain_os/brain_proto
python -m grpc_tools.protoc \
  -I. --python_out=../brain_ai/brain_ai/generated \
  --grpc_python_out=../brain_ai/brain_ai/generated \
  *.proto

# === 6. 运行测试 ===
cd ~/workspace/brain_os/brain_ai
pytest tests/ -v --cov=brain_ai --cov-report=term-missing

# === 7. 启动 AI 服务 ===
cd ~/workspace/brain_os/brain_ai
python -m brain_ai.server --host 0.0.0.0 --port 50052
```

## 6. 日常开发工作流

### 6.1 一日开发流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                       开发者日常操作流                               │
│                                                                     │
│  ① git pull (拉最新代码)                                             │
│     └─ git lfs pull (如模型有更新)                                   │
│                                                                     │
│  ② 启动开发环境                                                      │
│     ├─ Docker 模式: docker compose up -d                            │
│     └─ 本地模式: 进入对应项目目录的 venv / colcon / npm dev           │
│                                                                     │
│  ③ 编码                                                             │
│     ├─ C++:  修改代码 → ninja -C build → 运行测试                    │
│     ├─ Python: 修改代码 → 热重载(uvicorn) → pytest                   │
│     └─ TS:     修改代码 → HMR 自动刷新 → npm test                    │
│                                                                     │
│  ④ 提交前检查                                                        │
│     ├─ pre-commit run --all-files                                   │
│     ├─ 所有测试通过                                                   │
│     └─ CI 本地模拟 (act, 可选)                                       │
│                                                                     │
│  ⑤ 提交 + 推送                                                      │
│     └─ git commit -m "feat(core): add trajectory scoring"           │
│                                                                     │
│  ⑥ 停止环境                                                         │
│     └─ docker compose stop                                          │
└─────────────────────────────────────────────────────────────────────┘
```

### 6.2 代码提交规范

```bash
# 提交前运行代码检查
# C++ 项目
cd brain_core && clang-format -i src/**/*.cpp include/**/*.h

# Python 项目
cd brain_ai && ruff check . && ruff format .
cd brain_sdk && ruff check . && ruff format .

# TypeScript 项目
cd brain_viz && npm run lint && npm run format

# 全项目 pre-commit 检查
pre-commit run --all-files
```


## 8. 常见问题与排错

### 8.1 Docker 相关

| 问题 | 原因 | 解决 |
|------|------|------|
| `docker compose up` 失败 | 镜像未构建 | 先运行 `docker compose build` |
| 容器启动后立即退出 | 入口命令错误 | `docker logs brain_core` 查看日志 |
| `nvidia-smi` 在容器内不可见 | nvidia-container-toolkit 未安装 | 参考 §2.2 重新配置 |
| ROS 2 节点互不可见 | 网络模式不一致 | 统一使用 `network_mode: host` |
| 端口冲突 (3000/50051 已占用) | 本地已有服务 | 修改 `docker-compose.yml` 中的端口映射 |
| Docker 磁盘空间不足 | 镜像/缓存堆积 | `docker system prune -a` |

### 8.2 ROS 2 相关

| 问题 | 原因 | 解决 |
|------|------|------|
| `ros2: command not found` | 未 source setup.bash | `source /opt/ros/jazzy/setup.bash` |
| 节点间通信失败 | RMW 实现不匹配 | 所有终端统一 export `RMW_IMPLEMENTATION=rmw_cyclonedds_cpp` |
| `ROS_DOMAIN_ID` 冲突 | 多人同机开发 | 每人设置不同的 `ROS_DOMAIN_ID` |
| 高 CPU 占用 | DDS 发现流量大 | 使用 `rmw_cyclonedds_cpp`（默认已配置） |

### 8.3 Python 相关

| 问题 | 原因 | 解决 |
|------|------|------|
| `ModuleNotFoundError: No module named 'brain_ai'` | 未安装为 editable | `pip install -e .` |
| CUDA out of memory | 显存不足 | 使用 CPU 降级方案或减小 batch size |
| HuggingFace 模型下载失败 | 网络限制 | 设置 `HF_ENDPOINT=https://hf-mirror.com` |
| gRPC import 错误 | proto 未编译 | 运行 §4.2 步骤 5 编译 proto |

### 8.4 Node.js / TypeScript 相关

| 问题 | 原因 | 解决 |
|------|------|------|
| `npm install` 报错 | node 版本不匹配 | 使用 `nvm use 22` |
| HMR 不生效 | node_modules 挂载问题 | 使用 docker volume 而非 bind mount |
| 3D 渲染空白 | WebGL 不可用 | 确保浏览器支持 WebGL 2.0 |
