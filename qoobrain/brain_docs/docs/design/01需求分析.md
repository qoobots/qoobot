# 01 需求分析

Brain OS (qoobrain) 作为人形机器人的大脑操作系统，承载从传感器数据到物理执行的完整智能闭环。本文档系统梳理功能需求(FR)、非功能需求(NFR)以及用户角色定义。

---

## 非功能需求 (NFR)

| 指标 | 目标值 | 测量方法 |
|:-----|:-------|:---------|
| 感知延迟 | <30ms(端到端) | 传感器帧→检测输出 |
| 决策频率 | >50Hz | 行为树Tick周期 |
| 控制回路 | <1ms(1kHz) | 关节指令发布间隔 |
| 内存占用 | <4GB(含3个并行模型) | RSS监控 |
| 可靠性(MTBF) | >720h | 连续运行监控 |
| 语音响应 | <2s(端到端) | 语音指令→执行开始 |

## 用户角色

| 角色 | 核心需求 |
|:-----|:---------|
| **机器人用户** | 语音交互、任务执行、状态查询 |
| **技能开发者** | SDK接入、行为树API、仿真测试 |
| **运维管理员** | 健康监控、OTA升级、日志诊断 |
| **HITL操作员** | 远程干预、轨迹选择、状态监控 |
