# qooweb — 架构设计

> **版本**：v1.0  
> **最后更新**：2026-06-29  
> **技术栈**：HTML5 + CSS3 + Vanilla JavaScript（零依赖纯静态网站）

---

## 一、总体架构

qooweb 采用 **纯静态多页面架构（MPA）**，无前端框架、无构建工具、无运行时依赖。

```
┌─────────────────────────────────────────────────────────────┐
│                      用户浏览器                              │
├─────────────────────────────────────────────────────────────┤
│  index.html  products.html  about.html  qoo*.html (×13)     │
│       │            │            │             │              │
│       └────────────┼────────────┼─────────────┘              │
│                    │            │                            │
│              ┌─────┴─────┐ ┌───┴────┐                       │
│              │ css/      │ │ js/    │                       │
│              │ style.css │ │main.js │                       │
│              └─────┬─────┘ └───┬────┘                       │
│                    │            │                            │
│              ┌─────┴────────────┴─────┐                      │
│              │     assets/            │                      │
│              │  SVG / MP4 / JPG      │                      │
│              └────────────────────────┘                      │
└─────────────────────────────────────────────────────────────┘
```

### 架构特点

1. **零构建**：HTML/CSS/JS 直接可部署，无需 webpack/vite 等构建工具
2. **零框架**：不依赖 Vue/React 等前端框架，使用原生 DOM API
3. **共享资源**：所有页面共用同一份 `css/style.css` 和 `js/main.js`
4. **国际化方案**：中文版在根目录，英文版在 `en/` 子目录，通过相对路径引用共享资源
5. **静态托管**：可直接部署到任何静态文件服务器（Nginx/CDN/GitHub Pages）

## 四、组件体系

由于采用纯静态 HTML，组件复用通过 **复制粘贴 HTML 片段** 实现，而非框架组件化。核心可复用"组件"包括：

### 4.1 全局导航栏（所有页面共享）

```html
<nav class="nav">
  <div class="nav-inner">
    <a href="index.html" class="nav-logo">...</a>
    <ul class="nav-links" id="nav-links">
      <!-- 导航链接，根据当前页面设置 .active -->
    </ul>
    <div class="nav-actions">
      <!-- 语言切换 + 主题切换 + 汉堡菜单 -->
    </div>
  </div>
</nav>
```

**状态变体**：
- 首页：`index.html` 标记为 `.active`
- 产品页：`products.html` 标记为 `.active`
- 生态子页：生态下拉链接标记为 `.active`
- 关于页：`about.html` 标记为 `.active`

### 4.2 全局页脚（所有页面共享）

```html
<footer class="footer">
  <div class="footer-inner">
    <!-- 5 列链接：产品 / 生态 / 开发者 / 社区 / 关于 -->
  </div>
  <div class="footer-bottom">
    <!-- 版权 + 许可证信息 -->
  </div>
</footer>
```

### 4.3 Section 布局容器

```html
<section class="section [section-dark]">
  <div class="section-inner">
    <div class="section-label">标签</div>
    <h2 class="section-title">标题</h2>
    <p class="section-desc">描述</p>
    <!-- 内容 -->
  </div>
</section>
```

### 4.4 淡入动画元素

```html
<div class="fade-up">...</div>
<!-- 通过 IntersectionObserver 自动触发 visible 类 -->
```

## 五、CSS 设计系统

### 5.1 设计令牌（CSS 自定义属性）

```
:root {
  --color-bg / --color-bg-secondary
  --color-text / --color-text-secondary
  --color-accent / --color-accent-hover
  --color-border / --color-card-bg / --color-footer-bg
  --font-sans / --font-display          # SF Pro 字体栈
  --max-width: 1200px                   # 内容最大宽度
  --nav-height: 48px                    # 导航栏高度
  --radius-sm: 12px / --radius-md: 18px / --radius-lg: 24px
  --shadow-card / --shadow-elevated
  --transition-fast / --transition-smooth
}
```

### 5.2 主题系统

- 浅色主题：`:root` 默认值
- 深色主题：`[data-theme="dark"]` 覆盖 CSS 变量
- 切换逻辑：`js/main.js` 通过 `document.documentElement.setAttribute("data-theme", "dark")` 切换
- 持久化：`localStorage.getItem("qoobot-theme")`

### 5.3 响应式策略

| 断点 | 适配策略 |
|------|----------|
| > 1024px | 桌面端完整布局（多列网格） |
| 768px - 1024px | 平板端（部分双列变单列） |
| < 768px | 移动端（导航折叠、全单列布局） |
| < 480px | 小屏手机（进一步简化） |

```
                    ┌──────────────┐
                    │   CDN / Nginx │
                    └──────┬───────┘
```

**部署方式**：
- GitHub Pages：直接推送即可
- Nginx：配置 `root /path/to/qooweb;`
- CDN：上传所有文件到对象存储
- 无需构建步骤、无需 Node.js 运行时

## 九、技术决策记录

| 决策 | 选择 | 原因 |
|------|------|------|
| 前端框架 | 无（Vanilla JS） | 品牌官网以内容展示为主，无需复杂状态管理 |
| CSS 方案 | 纯 CSS（CSS 变量 + 媒体查询） | 零依赖，主题切换通过 CSS 变量实现 |
| 构建工具 | 无 | 纯静态文件，无需编译 |
| 国际化 | 目录分离（en/） | 简单直接，无需 i18n 库 |
| 动画方案 | CSS Animation + Canvas | 粒子效果需要 Canvas，普通动画用 CSS |
| 图标方案 | Emoji | 零依赖，无需图标库 |
| 响应式 | CSS 媒体查询 | 覆盖主流断点 |
