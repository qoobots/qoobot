# qooweb — 数据设计

> **版本**：v1.0  
> **最后更新**：2026-06-29  
> **数据存储方式**：纯静态 HTML 硬编码 + `localStorage` 客户端持久化

---

## 一、数据架构总览

qooweb 作为纯静态网站，**没有后端数据库**，所有数据通过以下方式管理：

```
┌─────────────────────────────────────────────────────┐
│                    数据来源                           │
├─────────────────────────────────────────────────────┤
│  HTML 硬编码数据                                      │
│  ├── 页面内容（标题、描述、特性列表等）                  │
│  ├── 产品规格数据                                      │
│  ├── 生态子项目元数据                                   │
│  ├── 行业对比数据                                      │
│  └── 导航结构数据                                      │
├─────────────────────────────────────────────────────┤
│  localStorage 持久化数据                              │
│  └── qoobot-theme: "dark" | "light"                  │
├─────────────────────────────────────────────────────┤
│  静态资源元数据（文件系统）                              │
│  ├── 图片/视频文件                                     │
│  └── 图纸 SVG 文件                                    │
└─────────────────────────────────────────────────────┘
```

## 二、页面内容数据模型

### 2.1 首页（index.html）

| 数据区域 | 数据结构 | 字段 |
|----------|----------|------|
| Hero 统计 | 静态数组 | `{number, label}` |
| 特性卡片 | 静态数组 | `{icon, title, description}` |
| 行业对比 | 静态二维表 | `{dimension, qoobot, competitors[6]}` |
| 产品预览 | 静态对象 | `{name, tier, emoji, description, specs[], colors[], url}` |
| 大脑能力 | 静态数组 | `{number, icon, title, description}` |
| 架构层级 | 静态数组 | `{level, name, color, items[]}` |
| 愿景卡片 | 静态数组 | `{number, title, description}` |
| 路线图 | 静态数组 | `{year, label}` |
| 开源理由 | 静态数组 | `{icon, title, description}` |

### 2.2 产品与生态页（products.html）

| 数据区域 | 数据结构 | 字段 |
|----------|----------|------|
| 产品详情 | 静态对象数组 | `{id, name, emoji, tier, tagline, description, specs[{label, value}], colors[{hex, name}], drawing}` |
| 硬件图纸模块 | 静态对象数组 | `{id, name, icon, description, count, drawings[{code, name, svg}]}` |
| 架构层级 | 静态对象数组 | `{level, name, icon, description, color, items[{id, name, icon, description, tags[], status, url}]}` |
| 状态总览表 | 静态对象数组 | `{id, name, layer, description, techStack, status}` |

### 2.3 子项目详情页（qoo*.html）

| 数据区域 | 字段 |
|----------|------|
| 项目标识 | `{id, name, icon, layer, description}` |
| 核心能力 | `[{title, description}]` |
| 技术规格 | `[{label, value}]` |
| 功能模块 | `[{name, description, status}]` |
| 状态信息 | `{status, techStack, platform}` |

### 2.4 关于页（about.html）

| 数据区域 | 字段 |
|----------|------|
| 使命描述 | 纯文本段落 |
| 价值观 | `[{icon, title, description}]` |
| 联系方式 | `{email, discord, github}` |

## 十、SEO 元数据规范

每个页面需包含：

```html
<!-- 必需 -->
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<meta name="description" content="页面描述（120-160 字符）">
<title>页面标题 | QooBot</title>

<!-- 图标 -->
<link rel="icon" type="image/svg+xml" href="assets/qoobot-logo.svg">

<!-- 安全 -->
<!-- 外部链接使用 rel="noopener" -->
```
