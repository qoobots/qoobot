## 1. 安全架构总览

qoocore 作为机器人端侧推理引擎，安全关注点在模型完整性、运行时隔离和侧面信道防护。

```
┌─────────────────────────────────────────────────────────────────┐
│                    qoocore 安全架构                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  L4 — 模型安全                                                  │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │ 签名验证 | 加密权重 | 防篡改 | 供应链验证               │   │
│  └──────────────────────────────────────────────────────────┘   │
│                                                                 │
│  L3 — 运行时安全                                                │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │ 进程隔离 | 内存保护 | 资源限制 | 降级策略               │   │
│  └──────────────────────────────────────────────────────────┘   │
│                                                                 │
│  L2 — 模型推理安全                                              │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │ 对抗样本检测 | 输入校验 | 输出范围校验 | 异常检测       │   │

│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 2. 模型安全

### 2.1 模型签名验证

```
.qoomodel 文件
     │
     ▼
┌─────────────────────────────┐
│ 1. 提取 Header (0-95 字节)  │
│ 2. 提取 Signature (96-127)  │
│ 3. 使用公钥验证签名          │
│    Ed25519_Verify(pubkey,    │
│      header[0:96],          │
│      signature[96:128])     │
│ 4. 验证 SHA256(模型数据)     │
│    == Header.ModelID 前8字节 │
└─────────────────────────────┘
     │
     ▼
  验证通过 → 加载
  验证失败 → 拒绝 + 告警
```

### 2.2 权重加密

| 特性 | 实现 |
|:-----|:-----|
| **加密算法** | AES-256-GCM |
| **密钥来源** | TPM 密封的 Key Encryption Key (KEK) |
| **加密范围** | .qoomodel 权重段 |
| **解密时机** | LoadModel 时解密到受保护内存 |
| **密钥轮换** | OTA 更新时同步轮换 |

## 3. 运行时安全

### 3.1 进程隔离

```
┌──────────────────────────────────────────┐
│          qoobrain 进程 (Python)          │
│          ┌─────────────────────┐         │
│          │   qoocore C++ API   │         │
│          └──────────┬──────────┘         │
└─────────────────────┼────────────────────┘
                      │ 进程内直接调用 / IPC
                      ▼
┌──────────────────────────────────────────┐
│       qoocore 推理进程 (C++ 独立进程)    │
│  ┌────────────────────────────────────┐  │
│  │  seccomp-bpf 系统调用过滤          │  │
│  │  rlimit 资源限制 (CPU/内存/文件)   │  │
│  │  namespace 隔离 (可选)             │  │
│  │  W^X 内存策略 (不可同时写+执行)    │  │
│  └────────────────────────────────────┘  │
└──────────────────────────────────────────┘
```

### 3.2 内存保护

| 措施 | 描述 |
|:-----|:-----|
| **W^X Policy** | 内存页不可同时 Write + Execute |
| **ASLR** | 地址空间布局随机化 |
| **Stack Canary** | 编译选项 `-fstack-protector-strong` |
| **FORITFY_SOURCE** | 编译选项 `-D_FORTIFY_SOURCE=2` |
| **PIE** | 位置无关可执行文件 |
| **RELRO** | 只读重定位表 |

## 4. 侧面信道防护

| 攻击向量 | 缓解措施 |
|:---------|:---------|
| **时序攻击（模型提取）** | 恒定时间推理路径、随机化执行顺序、添加噪声延迟 |
| **功耗分析（模型窃取）** | 屏蔽层执行顺序、随机化算子调度 |
| **缓存侧信道** | 模型权重分段加载、Cache Flush |
| **电磁侧信道** | 数据屏蔽、随机化数据模式 |

## 5. 对抗样本防御

```
┌──────────────┐
│  传感器输入  │
└──────┬───────┘
       ▼
┌──────────────┐
│ 输入校验      │ ← 范围检查、统计异常检测
└──────┬───────┘
       ▼
┌──────────────┐
│ 预处理        │ ← JPEG 压缩防御、随机缩放
└──────┬───────┘
       ▼
┌──────────────┐
│ 模型推理      │
└──────┬───────┘
       ▼
┌──────────────┐
│ 输出校验      │ ← 置信度阈值、一致性检查
└──────────────┘
```

| 防御技术 | 说明 |
|:---------|:-----|
| **输入统计检测** | 像素分布异常、高频噪声检测 |
| **随机预处理** | 随机裁剪/缩放/JPEG压缩破坏对抗扰动 |
| **置信度阈值** | 低置信度结果标记为不可靠 |
| **多模型投票** | 多后端推理一致性交叉验证 |
| **输出范围校验** | 检测框/关键点范围合理性检查 |

## 6. 供应链安全

| 阶段 | 措施 |
|:-----|:-----|
| **ONNX 模型来源** | 模型签名验证（开发者证书链） |
| **编译工具链** | 可重现构建 (Reproducible Build) |
| **第三方依赖** | SBOM + 漏洞扫描 (Trivy) |
| **分发** | .qoomodel Ed25519 签名 + OTA TLS 下载 |
| **更新** | A/B 分区 + 回滚验证 |

## 7. 威胁模型

| 威胁 | 攻击向量 | 缓解 | 严重度 |
|:-----|:---------|:-----|:------:|
| **模型注入** | 恶意 .qoomodel 替换 | Ed25519 签名 + TPM 信任根 | 严重 |
| **模型窃取** | 侧信道提取权重 | 权重加密 + 恒定时间执行 | 高 |
| **对抗攻击** | 物理对抗样本 | 输入预处理 + 输出校验 | 中 |
| **DoS** | 超大规模模型/输入 | 资源限制 + 超时中断 | 中 |
| **提权** | 推理引擎漏洞 | 进程隔离 + seccomp | 高 |
| **数据泄露** | 中间结果窃取 | 受保护内存 + 进程隔离 | 中 |
