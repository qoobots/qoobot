## 2. 关键技术决策

### 2.1 IR 设计：自定义 IrGraph vs MLIR

| 维度 | 自定义 IrGraph | MLIR |
|:-----|:--------------|:-----|
| **复杂度** | 低（~2000 LOC） | 高（~500K LOC） |
| **依赖** | 零外部依赖 | LLVM 依赖链 |
| **端侧尺寸** | ~500KB | ~50MB+ |
| **可扩展性** | 17种算子可扩展 | 完整方言生态 |
| **性能** | 零拷贝遍历 | 多层抽象 |

**决策**：选用 **自定义 IrGraph**。端侧部署需要最小依赖和最小尺寸，17种机器人专用算子足够覆盖当前需求，按需扩展。

### 2.2 后端分派：编译时静态 vs 运行时动态

| 维度 | 编译时静态分派 | 运行时动态分派 |
|:-----|:--------------|:--------------|
| **灵活性** | 低（编译时绑定） | 高（运行时选择） |
| **性能** | 高（无虚函数开销） | 略低（间接调用） |
| **二进制大小** | 不含未使用后端 | 包含所有后端 |

**决策**：**运行时动态分派**。同一模型可能在不同芯片上运行，运行时选择最优后端。

### 2.3 内存管理：自定义 Arena vs 系统 malloc

| 维度 | 自定义 Arena | 系统 malloc |
|:-----|:-------------|:-----------|
| **分配延迟** | ~10ns (指针碰撞) | ~100ns+ |
| **碎片** | 零（整块释放） | 可能碎片化 |
| **内存利用率** | 可调优 | 通用策略 |
| **零拷贝** | DMA-BUF 直通 | 需额外拷贝 |

**决策**：**Arena + DMA-BUF 混合**。权重/激活使用 Arena，传感器数据使用 DMA-BUF 零拷贝。

## 3. 核心组件设计

### 3.1 推理引擎

```
┌─────────────────────────────────────────────────────────────────┐
│                        Engine                                   │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │ load_model(path) → ModelHandle                            │  │
│  │   ├── 解析 .qoomodel Header (128字节)                     │  │
│  │   ├── 验证 SHA256 + Ed25519 签名                           │  │
│  │   ├── 反序列化 IrGraph                                     │  │
│  │   └── 创建 Runtime (选择最优后端)                          │  │
│  ├───────────────────────────────────────────────────────────┤  │
│  │ infer(handle, inputs) → outputs                           │  │
│  │   ├── 输入校验 (shape/dtype)                              │  │
│  │   ├── 调度 (实时调度器)                                   │  │
│  │   ├── 分派 (Dispatcher → 后端)                            │  │
│  │   └── 统计 (延迟/吞吐量)                                  │  │
│  ├───────────────────────────────────────────────────────────┤  │
│  │ infer_async(handle, inputs, callback)                     │  │
│  │ infer_batch(handle, batch_inputs) → batch_outputs          │  │
│  │ infer_multi(models[], inputs[]) → outputs[]               │  │
│  ├───────────────────────────────────────────────────────────┤  │
│  │ unload_model(handle)                                       │  │
│  └───────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 实时调度器

```
优先级队列:
┌──────────────────────────────────┐
│ Q0: CRITICAL  (控制回路 <1ms)    │ ← 固定优先级抢占
│ Q1: HIGH      (感知推理 <5ms)    │ ← 抢占 Q2/Q3
│ Q2: NORMAL    (认知推理 <30ms)   │ ← 时间片轮转 (5ms)
│ Q3: LOW       (模型加载/诊断)    │ ← 后台执行
└──────────────────────────────────┘

调度策略:
  - CRITICAL 任务到达 → 立即抢占当前执行
  - 同优先级 → Round-Robin 时间片
  - 空闲时 → 执行 Q3 后台任务
  - 超时检测 → deadline miss 记录 + 降级
```

## 4. 性能设计

### 4.1 推理延迟优化

| 优化技术 | 应用层级 | 预期收益 |
|:---------|:---------|:---------|
| **算子融合** (Conv+BN+ReLU) | 编译时 | -30% 延迟 |
| **Winograd 卷积** | 算子层 | -40% 卷积延迟 |
| **INT8 量化** | 编译时 | -50% 延迟, -75% 带宽 |
| **FlashAttention** | 算子层 | -60% 注意力延迟 |
| **Arena 内存池** | 运行时 | -80% 分配延迟 |
| **流水线并行** | 调度层 | +50% 吞吐量 |
| **ION/DMA-BUF** | 运行时 | 零拷贝 |

### 4.2 内存优化

| 技术 | 描述 | 节省 |
|:-----|:-----|:-----|
| **权重共享** | 多模型共享相同权重 | -40% 内存 |
| **LRU 淘汰** | 不活跃模型权重卸载 | 按需释放 |
| **激活重用** | 流水线激活缓冲区复用 | -30% 内存 |
| **混合精度** | FP16 权重存储 | -50% 权重内存 |
